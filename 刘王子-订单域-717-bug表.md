### Bug 01: 依赖缺失 - 无法访问BPM模块API

| 属性 | 值 |
|------|------|
| **编号** | BUG-001 |
| **严重程度** | 高 |
| **状态** | ✅ 已修复 |
| **发现阶段** | 编译阶段 |

#### 问题描述

编译 `mitedtsm-module-trade` 模块时，出现以下错误：

```
[ERROR] package com.meession.etm.module.bpm.api.task does not exist
[ERROR] cannot find symbol: class BpmProcessInstanceApi
```

涉及文件：
- `TradeOrderApprovalServiceImpl.java` 第8行

#### 根因分析

Trade模块需要调用BPM模块的API发起审批流程，但Trade模块的pom.xml中未添加BPM API模块的依赖，导致编译时无法找到相关类。

#### 解决方案

在 `mitedtsm-module-trade` 的 pom.xml 中添加BPM API依赖：

```xml
<dependency>
    <groupId>com.meession</groupId>
    <artifactId>mitedtsm-module-bpm-api</artifactId>
    <version>${project.version}</version>
</dependency>
```

#### 修改文件

| 文件路径 | 修改内容 |
|---------|---------|
| `mitedtsm-module-trade/pom.xml` | 添加 mitedtsm-module-bpm-api 依赖 |

#### 验证结果

```
mvn compile -pl mitedtsm-module-mall/mitedtsm-module-trade -am -q
# BUILD SUCCESS
```

### Bug 02: SQL语法错误 - H2数据库不支持DATE()函数

| 属性 | 值 |
|------|------|
| **编号** | BUG-002 |
| **严重程度** | 高 |
| **状态** | ✅ 已修复 |
| **发现阶段** | 测试执行阶段 |

#### 问题描述

运行日报表测试时，出现SQL语法错误：

```
org.h2.jdbc.JdbcSQLSyntaxErrorException: Function "DATE" not found; SQL statement:
SELECT DATE(createTime) as date FROM trade_order
```

涉及文件：
- `TradeOrderReportMapper.java` 第23行

#### 根因分析

MySQL支持 `DATE()` 函数用于截取日期部分，但H2数据库不支持该函数。测试使用H2数据库，导致SQL执行失败。

#### 解决方案

方案一：使用H2兼容的语法

```sql
-- H2兼容语法
SELECT CAST(createTime AS DATE) as date
```

方案二：使用MyBatis动态SQL在Java层处理

```java
// 在Service层按日期分组
LocalDate date = order.getCreateTime().toLocalDate();
```

最终采用方案一，修改SQL为H2兼容语法：

```java
@Select("""
    SELECT
        CAST(createTime AS DATE) as date,
        COUNT(*) as orderCount,
        ...
    FROM trade_order
    GROUP BY CAST(createTime AS DATE)
    ORDER BY date DESC
    """)
List<TradeOrderDailyReportRespVO> selectDailyReport(@Param("reqVO") TradeOrderReportReqVO reqVO);
```

#### 修改文件

| 文件路径 | 修改内容 |
|---------|---------|
| `dal/mysql/report/TradeOrderReportMapper.java` | 将 `DATE(createTime)` 改为 `CAST(createTime AS DATE)` |

#### 验证结果

```
测试用例: testGetDailyReport 执行成功
```

### Bug 04: 事件监听失效 - 未过滤非订单审批流程

| 属性 | 值 |
|------|------|
| **编号** | BUG-004 |
| **严重程度** | 高 |
| **状态** | ✅ 已修复 |
| **发现阶段** | 测试阶段 |

#### 问题描述

BPM流程状态变更时，订单审批状态监听器错误地处理了非订单审批流程：

```
订单审批状态更新: processDefinitionKey=oa_leave, businessKey=123
错误: 订单123不存在
```

涉及文件：
- `TradeOrderApprovalStatusEventListener.java` 第21行

#### 根因分析

事件监听器未对 `processDefinitionKey` 进行过滤，导致所有BPM流程状态变更都触发订单审批状态更新：

```java
// 修改前
@EventListener
public void onProcessInstanceStatusChange(BpmProcessInstanceStatusEvent event) {
    Integer approvalStatus = convertStatus(event.getStatus());
    tradeOrderApprovalService.updateApprovalStatus(event.getBusinessKey(), approvalStatus, event.getReason());
}
```

#### 解决方案

添加流程定义KEY过滤：

```java
// 修改后
@EventListener
public void onProcessInstanceStatusChange(BpmProcessInstanceStatusEvent event) {
    if (!"trade_order".equals(event.getProcessDefinitionKey())) {
        return;
    }
    
    log.info("订单审批流程状态变更: processDefinitionKey={}, businessKey={}, status={}",
            event.getProcessDefinitionKey(), event.getBusinessKey(), event.getStatus());
    
    Integer approvalStatus = convertStatus(event.getStatus());
    tradeOrderApprovalService.updateApprovalStatus(event.getBusinessKey(), approvalStatus, event.getReason());
}
```

#### 修改文件

| 文件路径 | 修改内容 |
|---------|---------|
| `service/order/listener/TradeOrderApprovalStatusEventListener.java` | 添加 processDefinitionKey 过滤 |

#### 验证结果

```
测试用例: OA请假流程状态变更不影响订单审批状态
测试用例: 订单审批流程状态变更正确更新订单状态
```

### Bug 06: 枚举引用错误 - 状态转换映射错误

| 属性 | 值 |
|------|------|
| **编号** | BUG-006 |
| **严重程度** | 高 |
| **状态** | ✅ 已修复 |
| **发现阶段** | 测试阶段 |

#### 问题描述

审批通过后，订单状态未正确转换为待发货：

```
审批状态: APPROVED(1)
订单状态: 仍为PAID，未变更为DELIVERY
```

涉及文件：
- `TradeOrderApprovalServiceImpl.java` 第88行

#### 根因分析

状态转换逻辑中使用了错误的枚举值：

```java
// 修改前
if (TradeOrderApprovalStatusEnum.APPROVED.getStatus().equals(approvalStatus)) {
    updateObj.setStatus(TradeOrderStatusEnum.PAID.getStatus());  // 错误：应改为DELIVERY
}
```

#### 解决方案

修正状态转换逻辑：

```java
// 修改后
if (TradeOrderApprovalStatusEnum.APPROVED.getStatus().equals(approvalStatus)) {
    updateObj.setStatus(TradeOrderStatusEnum.DELIVERY.getStatus());  // 正确：审批通过→待发货
} else if (TradeOrderApprovalStatusEnum.REJECTED.getStatus().equals(approvalStatus)) {
    updateObj.setStatus(TradeOrderStatusEnum.CANCEL.getStatus());    // 审批不通过→已取消
}
```

#### 修改文件

| 文件路径 | 修改内容 |

- `TradeOrderApprovalServiceImpl.java` 第74行

```java
// 修改后
@Override
@Transactional(rollbackFor = Exception.class)

    Long orderId;
    try {
        orderId = Long.parseLong(businessKey);
    } catch (NumberFormatException e) {
        log.warn("无效的订单编号: businessKey={}", businessKey, e);
        return;
    }
    
    TradeOrderDO order = tradeOrderMapper.selectById(orderId);
    if (order == null) {
        log.warn("订单不存在: orderId={}", orderId);
        return;
    }
    // ...
}
```

### Bug 08: SQL注入风险 - 动态SQL未使用参数化

| 属性 | 值 |
|------|------|
| **编号** | BUG-008 |
| **严重程度** | 高 |
| **状态** | ✅ 已修复 |
| **发现阶段** | 代码审查阶段 |

#### 问题描述

代码审查发现报表查询SQL存在注入风险：

```java
// 原始实现（风险代码）
@Select("SELECT * FROM trade_order WHERE status = " + reqVO.getStatus())
TradeOrderReportRespVO selectReport(TradeOrderReportReqVO reqVO);
```

涉及文件：
- `TradeOrderReportMapper.java`

#### 根因分析

动态SQL直接拼接参数，存在SQL注入风险。应使用MyBatis动态SQL标签或参数绑定。

#### 解决方案

使用MyBatis动态SQL标签：

```java
// 修改后
@Select("""
    SELECT
        COUNT(*) as totalCount,
        SUM(CASE WHEN status = 1 THEN 1 ELSE 0 END) as paidCount,
        ...
    FROM trade_order
    WHERE 1=1
    <if test="reqVO.startTime != null">AND createTime >= #{reqVO.startTime}</if>
    <if test="reqVO.endTime != null">AND createTime <= #{reqVO.endTime}</if>
    <if test="reqVO.status != null">AND status = #{reqVO.status}</if>
    <if test="reqVO.type != null">AND type = #{reqVO.type}</if>
    <if test="reqVO.userId != null">AND userId = #{reqVO.userId}</if>
    """)
@Lang(XMLLanguageDriver.class)
TradeOrderReportRespVO selectReport(@Param("reqVO") TradeOrderReportReqVO reqVO);
```

#### 修改文件

| 文件路径 | 修改内容 |
|---------|---------|
| `dal/mysql/report/TradeOrderReportMapper.java` | 使用MyBatis动态SQL标签替代字符串拼接 |

#### 验证结果

```
SQL注入测试: 输入恶意参数不影响查询结果
功能测试: 报表查询功能正常
```
