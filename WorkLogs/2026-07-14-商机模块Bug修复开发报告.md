# 商机模块 Bug 修复开发报告

## 一、项目背景

根据项目系统 Bug 分析报告，商机模块存在多个问题需要修复。本次开发针对 CRM 商机模块进行全面修复和优化。

## 二、修复内容

### 2.1 数据类型不匹配问题

**问题描述**：`CrmBusinessSaveReqVO` 中 `count` 字段使用 `Integer` 类型，而 `CrmBusinessProductDO` 中使用 `BigDecimal` 类型，导致数据转换异常。

**修复方案**：将 `CrmBusinessSaveReqVO` 中 `count` 字段类型改为 `BigDecimal`。

**修改文件**：
- `Server/mitedtsm-module-crm/src/main/java/com/meession/etm/module/crm/controller/admin/business/vo/business/CrmBusinessSaveReqVO.java`

**代码变更**：
```java
// 修改前
private Integer count;

// 修改后
private BigDecimal count;
```

### 2.2 联系人关联字段缺失

**问题描述**：商机无法关联联系人，缺少 `contactId` 和 `contactName` 字段，导致商机列表和详情无法显示联系人信息。

**修复方案**：
1. 在 `CrmBusinessDO` 中添加 `contactId` 字段
2. 在 `CrmBusinessRespVO` 中添加 `contactId` 和 `contactName` 字段
3. 在 `CrmBusinessController.buildBusinessDetailList` 方法中添加联系人名称查询逻辑

**修改文件**：
- `Server/mitedtsm-module-crm/src/main/java/com/meession/etm/module/crm/dal/dataobject/business/CrmBusinessDO.java`
- `Server/mitedtsm-module-crm/src/main/java/com/meession/etm/module/crm/controller/admin/business/vo/business/CrmBusinessRespVO.java`
- `Server/mitedtsm-module-crm/src/main/java/com/meession/etm/module/crm/controller/admin/business/CrmBusinessController.java`

### 2.3 创建商机时 product_price 字段缺失

**问题描述**：创建商机时没有设置产品原始价格，导致数据库插入失败，错误信息为 `Field 'product_price' doesn't have a default value`。

**修复方案**：在 `CrmBusinessServiceImpl.validateBusinessProducts` 方法中添加获取产品原始价格的逻辑，从产品表中查询产品价格并设置到商机产品记录中。

**修改文件**：
- `Server/mitedtsm-module-crm/src/main/java/com/meession/etm/module/crm/service/business/CrmBusinessServiceImpl.java`

**代码变更**：
```java
private List<CrmBusinessProductDO> validateBusinessProducts(List<CrmBusinessSaveReqVO.BusinessProduct> list) {
    // 1. 校验产品存在
    Set<Long> productIds = convertSet(list, CrmBusinessSaveReqVO.BusinessProduct::getProductId);
    productService.validProductList(productIds);
    // 2. 获取产品原始价格
    Map<Long, CrmProductDO> productMap = productService.getProductMap(productIds);
    // 3. 转化为 CrmBusinessProductDO 列表
    return convertList(list, o -> BeanUtils.toBean(o, CrmBusinessProductDO.class,
            item -> {
                item.setTotalPrice(MoneyUtils.priceMultiply(item.getBusinessPrice(), item.getCount()));
                MapUtils.findAndThen(productMap, item.getProductId(), product -> item.setProductPrice(product.getPrice()));
            }));
}
```

**修改文件**：

## 六、问题与解决方案

### 6.1 编译错误：缺少 import 语句

**问题**：修改 `CrmBusinessServiceImpl` 时引入了 `Set`、`Map`、`CrmProductDO` 和 `MapUtils`，但未添加对应的 import 语句。

**解决方案**：添加缺失的 import 语句：
```java
import com.meession.etm.framework.common.util.collection.MapUtils;
import com.meession.etm.module.crm.dal.dataobject.product.CrmProductDO;
import java.util.Map;
import java.util.Set;
```

### 6.2 容器网络配置问题

**问题**：使用 `SPRING_PROFILES_ACTIVE=docker` 启动时，容器无法连接到 MySQL（主机名 `mysql` 无法解析）。

**解决方案**：在 Podman Pod 环境中，通过环境变量覆盖数据库连接地址，使用容器名称进行通信。
