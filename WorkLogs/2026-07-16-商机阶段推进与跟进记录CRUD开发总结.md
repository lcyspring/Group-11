# 商机阶段推进与跟进记录CRUD开发总结

## 一、工作日期

2026-07-16

## 二、工作内容

### 2.1 需求概述

根据业务需求，本次开发需要实现以下功能：

1. **商机阶段推进接口**：提供便捷的阶段推进功能，支持一键推进到下一阶段，同时支持直接推进到结束状态（赢单/输单/无效）
2. **跟进记录CRUD**：在现有创建、删除、查询基础上，新增更新功能，完善CRUD操作

### 2.2 技术方案

采用分层架构设计：
- Controller层：处理HTTP请求，参数校验
- Service层：业务逻辑实现，状态机流转校验
- Mapper层：数据访问

### 2.3 状态机设计

商机状态流转遵循以下规则：
- 状态按 sort 字段排序，只能推进到相邻的下一状态
- 支持直接推进到结束状态（赢单/输单/无效）
- 已结束的商机不能再次推进
- 推进时自动记录状态流转历史

## 五、遇到的问题及解决方案

### 5.1 问题1：状态机流转逻辑复用

**问题描述**：阶段推进需要复用现有的状态更新逻辑，但需要自动计算下一状态

**解决方案**：通过状态的 sort 字段计算下一状态，然后调用现有的 `updateBusinessStatus` 方法完成状态更新

### 5.2 问题2：跟进记录更新后关联数据同步

**问题描述**：更新跟进记录后，需要同步更新关联的客户、商机、线索、联系人、合同的跟进信息

**解决方案**：参考创建方法的逻辑，在更新方法中同样处理关联数据的同步更新

### 5.3 问题3：数据权限校验

**问题描述**：更新跟进记录时需要校验数据权限

**解决方案**：使用 `@CrmPermission` 注解进行声明式权限校验，基于 `bizType` 和 `bizId` 判断权限

## 六、接口文档

### 6.1 商机阶段推进接口

**接口地址**：`PUT /crm/business/advance-stage`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Long | 是 | 商机编号 |
| remark | String | 否 | 流转备注 |
| endStatus | Integer | 否 | 结束状态（1-赢单，2-输单，3-无效） |
| endRemark | String | 否 | 结束备注 |

**响应参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| code | Integer | 响应码 |
| msg | String | 响应消息 |
| data | Boolean | 操作结果 |

**说明**：
- 不传 `endStatus` 时，自动推进到下一阶段
- 传 `endStatus` 时，直接推进到结束状态

### 6.2 跟进记录更新接口

**接口地址**：`PUT /crm/follow-up-record/update`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Long | 是 | 跟进记录编号 |
| bizType | Integer | 是 | 数据类型（1-客户，2-商机，3-线索，4-联系人，5-合同） |
| bizId | Long | 是 | 数据编号 |
| type | Integer | 是 | 跟进类型 |
| content | String | 是 | 跟进内容 |
| nextTime | LocalDateTime | 是 | 下次联系时间 |
| businessIds | List\<Long\> | 否 | 关联的商机编号数组 |
| contactIds | List\<Long\> | 否 | 关联的联系人编号数组 |
| picUrls | List\<String\> | 否 | 图片 |
| fileUrls | List\<String\> | 否 | 附件 |

**响应参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| code | Integer | 响应码 |
| msg | String | 响应消息 |
| data | Boolean | 操作结果 |

### 6.3 跟进记录完整CRUD接口汇总

| 方法 | 接口地址 | 说明 |
|------|----------|------|
| POST | `/crm/follow-up-record/create` | 创建跟进记录 |
| PUT | `/crm/follow-up-record/update` | 更新跟进记录 |
| DELETE | `/crm/follow-up-record/delete?id={id}` | 删除跟进记录 |
| GET | `/crm/follow-up-record/get?id={id}` | 获取跟进记录 |
| GET | `/crm/follow-up-record/page` | 获取跟进记录分页 |
