## 二、工作内容

### 2.1 需求概述

根据需求分析文档，本次开发需要实现以下两个接口：

1. **产品报价计算接口**：根据产品列表和数量计算报价，支持整单折扣
2. **报价折扣计算接口**：根据订单金额、客户等级、产品数量计算综合折扣

### 2.2 技术方案

采用分层架构设计：
- Controller层：处理HTTP请求，参数校验
- Service层：业务逻辑实现，折扣规则计算
- 依赖产品服务（CrmProductService）进行产品数据校验和获取

### 2.3 折扣规则设计

| 规则类型 | 规则内容 |
|----------|----------|
| 金额阶梯折扣 | 满10000打85折（15%）、满5000打9折（10%）、满1000打95折（5%） |
| 客户等级折扣 | 钻石会员8%、金卡会员5%、银卡会员3%、普通会员0% |
| 数量折扣 | 购买50件以上5%、购买10件以上2% |
| 最大折扣限制 | 综合折扣最高不超过50% |

## 五、遇到的问题及解决方案

### 5.1 问题1：金额计算精度问题

**问题描述**：使用浮点数进行金额计算时出现精度丢失

**解决方案**：使用 `BigDecimal` 类型进行金额计算，统一使用 `MoneyUtils` 工具类处理金额运算

### 5.2 问题2：折扣规则可扩展性问题

**问题描述**：折扣规则硬编码在代码中，后续新增规则需要修改代码

**解决方案**：采用策略模式设计，将折扣规则封装为独立方法，后续新增规则只需添加新方法即可

### 5.3 问题3：客户等级与折扣对应关系维护问题

**问题描述**：客户等级与折扣的对应关系分散在多处

**解决方案**：将客户等级折扣规则集中在 `calculateCustomerLevelDiscount` 方法中，便于维护

## 六、接口文档

### 6.1 产品报价计算接口

**接口地址**：`POST /crm/quote/calculate`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| products | List\<Product\> | 是 | 产品列表 |
| products[].productId | Long | 是 | 产品编号 |
| products[].count | BigDecimal | 是 | 购买数量 |
| discountPercent | BigDecimal | 否 | 整单折扣百分比 |

**响应参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| totalProductPrice | BigDecimal | 产品总金额（未折扣） |
| discountPercent | BigDecimal | 整单折扣百分比 |
| discountAmount | BigDecimal | 折扣金额 |
| totalPrice | BigDecimal | 最终报价金额 |
| products | List\<Product\> | 产品明细列表 |

### 6.2 报价折扣计算接口

**接口地址**：`POST /crm/quote/discount-calculate`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| totalAmount | BigDecimal | 是 | 订单总金额 |
| customerLevel | Integer | 否 | 客户等级（1-普通，2-银卡，3-金卡，4-钻石） |
| productCount | Integer | 否 | 产品总数量 |
| businessTypeId | Long | 否 | 业务类型ID |

**响应参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| discountPercent | BigDecimal | 计算出的折扣百分比 |
| discountAmount | BigDecimal | 折扣金额 |
| finalAmount | BigDecimal | 折扣后的金额 |
| rules | List\<DiscountRule\> | 应用的折扣规则列表 |
