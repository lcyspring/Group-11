# 销售预测与商机赢单分析详细设计

## 1. 设计概述





基于CRM系统现有统计模块架构，新增销售预测和商机赢单分析两个核心功能，提供数据驱动的决策支持。

### 1.2 技术方案

| 模块 | 技术选型 | 说明 |



|------|----------|------|
| 框架 | Spring Boot 3 + MyBatis Plus | 沿用现有技术栈 |
| 数据库 | MySQL 8.0 | 沿用现有数据库 |
| 统计方式 | SQL聚合查询 + Java计算 | 复杂统计逻辑在Service层处理 |

### 4.2 核心查询逻辑

#### 4.2.1 销售预测查询

```sql
-- 按日期统计赢单金额
SELECT DATE_FORMAT(create_time, '%Y-%m') AS time,
       SUM(total_price) AS win_amount,
       COUNT(*) AS win_count
FROM crm_business
WHERE deleted = 0
  AND end_status = 1
  AND owner_user_id IN (...)
  AND create_time BETWEEN ... AND ...
GROUP BY time
ORDER BY time
```

#### 4.2.2 赢单分析查询

```sql
-- 商机赢单汇总
SELECT COUNT(*) AS total_count,
       SUM(CASE WHEN end_status = 1 THEN 1 ELSE 0 END) AS win_count,
       SUM(CASE WHEN end_status = 1 THEN total_price ELSE 0 END) AS win_amount
FROM crm_business
WHERE deleted = 0
  AND owner_user_id IN (...)
  AND create_time BETWEEN ... AND ...

-- 按行业统计赢单金额
SELECT c.industry,
       COUNT(*) AS win_count,
       SUM(b.total_price) AS win_amount
FROM crm_business b
LEFT JOIN crm_customer c ON b.customer_id = c.id
WHERE b.deleted = 0
  AND b.end_status = 1
  AND b.owner_user_id IN (...)
  AND b.create_time BETWEEN ... AND ...
GROUP BY c.industry
```

## 5. 业务逻辑设计

### 5.1 销售预测算法

1. 获取历史赢单数据，计算平均每月赢单金额和平均赢单率
2. 获取当前未结束商机数据，计算潜在商机金额
3. 根据历史趋势和当前商机情况，预测未来销售额

**预测公式**：
```
预测金额 = 历史平均赢单金额 × 预测周期 × 调整系数
调整系数 = （当前商机金额 / 历史平均商机金额）× 历史赢单率
```

### 5.2 赢单分析逻辑

1. 统计指定时间范围内的商机总数和赢单数量
2. 计算赢单率（赢单数/总商机数）
3. 按日期、行业、客户等级等维度进行分组统计

## 6. 安全设计

### 6.1 数据权限控制

- 使用已有的 `@CrmPermission` 注解进行权限校验
- 只能查看所属部门及子部门的数据

### 6.2 参数校验

- 使用 `@NotNull`、`@Size` 等注解进行参数校验
- 使用 `@InEnum` 注解校验枚举类型

## 7. 性能优化

### 7.1 查询优化

- 使用索引优化聚合查询
- 对大数据量统计使用分页或限制时间范围

### 7.2 缓存策略

- 对统计结果进行缓存，减少数据库查询
- 设置合理的缓存过期时间
