# 商城评论系统增强详细设计文档

## 1. 需求分析回顾

### 1.1 已实现功能

| 功能点 | 说明 |
|--------|------|
| 评论创建 | 支持会员端和管理后台创建 |
| 评论图片 | 支持上传多张图片（最多9张） |
| 评分系统 | 描述评分、服务评分（1-5星） |
| 商家回复 | 管理员可回复评论 |
| 可见性控制 | 管理员可隐藏/显示评论 |
| 评论筛选 | 会员端支持按类型筛选（全部/好评/中评/差评） |
| 匿名评论 | 支持匿名发表评论 |

### 1.2 待实现功能

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| 评论点赞 | 高 | 会员可点赞评论 |
| 用户回复 | 高 | 会员可回复其他用户的评论 |
| 评论统计 | 中 | 展示评论统计数据（好评率、平均评分等） |
| 评论排序 | 中 | 支持按时间、点赞数排序 |

### 1.3 业务规则

| 功能 | 规则 |
|------|------|
| 点赞 | 每个用户对同一条评论只能点赞一次，点赞后可取消 |
| 回复 | 支持多级回复，内容限制500字以内，需要登录 |
| 统计 | 展示总评论数、好评率、平均评分、有图评论数、星级分布 |
| 排序 | 支持最新、最热、有图三种排序方式 |

### 4.2 API 详细设计

#### 4.2.1 点赞/取消点赞

**请求路径**：`POST /app-api/comment/like`

**请求体**：
```json
{
    "commentId": 12345,
    "liked": true
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| commentId | Long | 是 | 评论ID |
| liked | Boolean | 是 | true-点赞，false-取消点赞 |

**成功响应**（200）：
```json
{
    "code": 0,
    "msg": "操作成功",
    "data": {
        "commentId": 12345,
        "liked": true,
        "likeCount": 15
    }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| commentId | Long | 评论ID |
| liked | Boolean | 当前点赞状态 |
| likeCount | Integer | 当前点赞数 |

**失败响应**（400）：
```json
{
    "code": 400,
    "msg": "评论不存在"
```

#### 5.2.4 排序 SQL 逻辑

**说明**：评论列表查询支持三种排序方式，通过 `sortType` 参数控制。

**排序方式映射**：

| sortType | SQL ORDER BY 子句 | 说明 |
|----------|-------------------|------|
| latest | `ORDER BY create_time DESC` | 按评论时间倒序（默认） |
| hot | `ORDER BY like_count DESC, create_time DESC` | 按点赞数倒序，点赞数相同按时间倒序 |
| hasPic | `ORDER BY pic_urls IS NOT NULL DESC, create_time DESC` | 优先显示带图片的评论，再按时间倒序 |

**Mapper XML 实现示例**：
```xml
<select id="selectPage" resultType="ProductCommentDO">
    SELECT * FROM product_comment
    WHERE spu_id = #{spuId} AND visible = #{visible}
    <if test="sortType == 'hot'">
        ORDER BY like_count DESC, create_time DESC
    </if>
    <if test="sortType == 'hasPic'">
        ORDER BY pic_urls IS NOT NULL DESC, create_time DESC
    </if>
    <if test="sortType == null or sortType == 'latest'">
        ORDER BY create_time DESC
    </if>
    LIMIT #{offset}, #{pageSize}
</select>
```

#### 5.3.4 回复树构建算法

**算法说明**：采用内存递归组装方式，限制最大嵌套深度为3层，避免性能问题。

**核心逻辑**：
```java
private List<AppCommentReplyTreeRespVO> buildReplyTree(List<ProductCommentReplyDO> replies, Long parentId, int depth) {
    // 最大深度限制为3层
    if (depth > 3) {
        return Collections.emptyList();
    }
    
    List<AppCommentReplyTreeRespVO> treeList = new ArrayList<>();
    for (ProductCommentReplyDO reply : replies) {
        if (Objects.equals(reply.getParentId(), parentId)) {
            AppCommentReplyTreeRespVO treeVO = convertToTree(reply);
            // 递归查找子回复
            treeVO.setChildren(buildReplyTree(replies, reply.getId(), depth + 1));
            treeList.add(treeVO);
        }
    }
    // 按创建时间排序
    treeList.sort(Comparator.comparing(AppCommentReplyTreeRespVO::getCreateTime));
    return treeList;
}
```

**调用方式**：
```java
public PageResult<AppCommentReplyTreeRespVO> getReplyPage(AppCommentReplyPageReqVO pageReqVO) {
    // 查询所有回复（不分页，因为通常回复数量较少）
    List<ProductCommentReplyDO> replies = productCommentReplyMapper.selectListByCommentId(pageReqVO.getCommentId());
    // 构建树形结构，从parentId=0开始
    List<AppCommentReplyTreeRespVO> treeList = buildReplyTree(replies, 0L, 1);
    // 手动分页
    int start = (pageReqVO.getPageNo() - 1) * pageReqVO.getPageSize();
    int end = Math.min(start + pageReqVO.getPageSize(), treeList.size());
    List<AppCommentReplyTreeRespVO> pageList = start >= treeList.size() ? 
        Collections.emptyList() : treeList.subList(start, end);
    return new PageResult<>(pageList, treeList.size(), pageReqVO);
}
```
