# 商机阶段流转状态机开发报告

## 一、开发概述

### 1.1 任务背景
本任务旨在开发商机阶段流转接口，实现商机阶段流转状态机，确保商机状态按预定顺序流转，并记录完整的流转历史。

### 1.2 开发目标
1. 创建商机状态流转历史记录表
2. 实现状态机校验逻辑，确保状态只能按顺序流转
3. 每次状态变更自动记录流转历史
4. 提供流转历史查询接口

---

## 二、需求分析

### 2.1 业务需求
- **状态流转约束**：商机状态必须按照状态类型定义的顺序依次流转，不允许跳过中间状态
- **流转历史记录**：每次状态变更都需要记录，包含旧状态、新状态、操作人、操作时间、备注等信息
- **流转历史查询**：支持根据商机ID查询完整的状态流转历史

### 2.2 状态流转规则
- 状态按照 `sort` 字段排序，只能向上或向下流转一个状态（相邻状态）
- 结束状态（赢单/无效）可以直接从任意状态设置
- 已结束的商机不允许再修改状态

### 3.3 状态机校验逻辑

在 `CrmBusinessServiceImpl.updateBusinessStatus()` 方法中增加：

1. **状态流转校验**：调用 `validateStatusTransition()` 方法
2. **保存旧状态**：使用 `BeanUtils.toBean()` 复制旧商机对象
3. **更新商机状态**：执行数据库更新
4. **创建流转记录**：调用 `statusRecordService.createRecord()`

**validateStatusTransition()** 方法逻辑：
```
1. 获取当前状态类型下的所有状态列表
2. 构建状态ID到sort值的映射
3. 计算新旧状态的sort差值
4. 如果差值绝对值大于1，抛出异常（不允许跳过状态）
```

## 六、遇到的问题及解决方案

### 6.1 问题列表

| 问题描述 | 原因 | 解决方案 |
|----------|------|----------|
| CRM API 返回 404 | 测试路径错误，应为 `/page` 而非根路径 | 使用正确的接口路径 |
| 状态更新返回"表结构未导入" | 缺少 `crm_business_status_record` 表 | 执行数据库迁移脚本创建表 |
| 创建记录时报错 `Unknown column 'update_time'` | DO 继承 BaseDO，表缺少 BaseDO 字段 | 添加 `update_time`、`creator`、`updater`、`deleted` 字段 |
| 创建记录时报错 `Field 'create_time' doesn't have a default value` | MyBatis Plus 自动填充未生效 | 给表字段添加默认值 `CURRENT_TIMESTAMP` |
| DO 类中重复定义 `createTime` 字段 | 误在子类中重复定义父类字段 | 删除子类中重复的 `createTime` 字段 |
| Lombok Builder 无法设置父类字段 | Builder 只生成子类字段的 setter | 使用 `record.setCreateTime()` 手动设置 |
