## 一、环境准备

### 1.1 前置操作

| 步骤 | 操作 | 说明 |
|------|------|------|
| 注册模块 | `Server/pom.xml` 添加 `<module>mitedtsm-module-marketing</module>` | 将营销模块纳入Maven聚合编译 |
| 添加依赖 | `Server/mitedtsm-server/pom.xml` 添加 marketing 依赖 | server引入marketing jar |
| 修复多租户 | `MarketingCampaignDO` 添加 `@TenantIgnore` 注解 | 表无 `tenant_id` 字段，需忽略租户自动过滤 |

### 1.2 建表

```bash
wsl -d Ubuntu-24.04 -- bash -c \
  "docker exec -i mitedtsm-mysql mysql -uroot -p1234 mitedtsm_database \
   < /home/panda/project/Group-11/Server/sql/mysql/marketing_campaign.sql"
```

验证：
```sql
DESC marketing_campaign;  -- 20 个字段，含 id, name, type, status, template_id 等
```

### 1.3 编译

```bash
cd Server && mvn install -pl mitedtsm-module-marketing -DskipTests -T 1C
```

结果：`BUILD SUCCESS`，24 个 Java 文件编译通过。

### 1.4 启动

```bash
cd Server && mvn spring-boot:run -pl mitedtsm-server -DskipTests
```

结果：`项目启动成功！` 端口 8080。

## 七、发现的问题与修复

| # | 问题 | 原因 | 修复 |
|---|------|------|------|
| 1 | 编译失败: `No plugin found for prefix 'spring-boot'` | 从子模块直接运行需安装到本地仓库 | `mvn install` 先安装全部模块 |
| 2 | 登录返回 `请求的租户标识未传递` | 多租户系统需传 `tenant-id` Header | `-H "tenant-id: 1"` |
| 3 | 接口返回 500: `Unknown column 'tenant_id'` | 多租户自动加 tenant_id 过滤，表无此字段 | `MarketingCampaignDO` 加 `@TenantIgnore` |
| 4 | 创建返回 500: `Invalid UTF-8 start byte 0xb4` | Windows Git Bash 中文 JSON 编码问题 | 使用 ASCII 英文测试，或 WSL 原生 curl |
