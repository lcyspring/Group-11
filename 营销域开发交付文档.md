## 六、关键设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 模板管理归属 | 营销域仅做**代理调用**，不复制数据 | 模板主数据仍在 `module-system`，避免数据分裂 |
| 批量发送方式 | 逐条调用现有 `sendSingleSms` / `sendSingleMail` | 复用现有底层能力，异常隔离 |
| 群发审核 | 预留 `bpmProcessInstanceId` 字段对接 Flowable | 待 BPM 流程定义后就绪 |
| 状态机 | `CampaignStatusEnum` 控制活动流转 | 草稿→待审→已审→进行→结束/终止，Service层校验 |
| 发送统计 | `sentCount / successCount / failCount` 三字段 | 支持后续发送分析（到达率/成功率） |
| 字段命名 | 遵循 CRM 模块的驼峰+注解风格 | 与项目保持一致 |
| 错误码段 | `1-030-xxx-xxx` | 营销域独立段，不冲突 |

## 七、活动状态流转

```
                    ┌──────────┐
                    │   草稿    │ (0) DRAFT
                    └────┬─────┘
                         │ submitApproval()
                    ┌────▼─────┐
                    │  待审核   │ (1) PENDING_APPROVAL
                    └────┬─────┘
                         │ BPM审批通过 (预留)
                    ┌────▼─────┐
                    │  已审核   │ (2) APPROVED
                    └────┬─────┘
                         │ startCampaign()
                    ┌────▼─────┐
                    │  进行中   │ (3) IN_PROGRESS
                    └────┬─────┘
                         │ 发送完成 / 手动
              ┌──────────┼──────────┐
         ┌────▼───┐                    ┌────▼───┐
         │ 已结束  │ (4) FINISHED      │ 已终止  │ (5) CANCELLED
         └────────┘                    └────────┘
```

**取消操作**: 草稿(0)、待审核(1)、已审核(2) 状态均可取消
