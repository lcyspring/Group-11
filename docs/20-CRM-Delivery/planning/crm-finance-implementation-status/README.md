# CRM 财务域实现状态核验

核验日期：2026-07-15。

## 结论

| 工作项 | 状态 | 仓库证据 | 尚缺 |
|---|---|---|---|
| 财务表结构：回款 | 已有并已完善 | `crm_receivable`、`crm_receivable_plan` | 会计入账/外部对账不在 CRM MVP |
| 财务表结构：发票 | 子闭环完成 | `crm_invoice`、`crm_invoice_action_record`、CHECK/唯一键/索引 | 税务平台与会计总账不在 CRM 台账边界 |
| 财务表结构：报销 | 未完成 | 无 CRM reimbursement 表 | ADR-017，避免与 OA 重复建模 |
| 财务表结构：退款 | 子闭环完成 | `crm_receivable_refund`、`crm_receivable_refund_action_record`、CHECK/唯一键/索引 | 真实 BPM 流程部署和银行/总账适配不在当前运行证据内 |
| 回款 CRUD 接口 | 子闭环完成 | 两个 Controller、Service、Mapper、前端页面和专项测试 | 银行到账、会计入账与核销不宣称完成 |
| 发票管理接口 | 子闭环完成 | Controller、Service、Provider、前端、迁移和专项测试 | 外部税务平台适配按需另行实施 |

## 回款已完成边界

- 回款与回款计划创建、查询、修改、删除、导出；
- 合同金额上限、计划金额守恒、正金额校验；
- 回款审批提交、驳回/取消后修订重提、旧回调幂等；
- 对象权限、计划并发期次/金额保护、已绑定计划保护；
- 审批通过口径的客户和统计汇总、逾期待办。

## 发票已完成边界

- 草稿创建、修改、删除和正式开具；
- 独立红票、部分/全部红冲、蓝票/红票作废和原票状态恢复；
- 合同锁内金额守恒、同租户税务号唯一、专票购方信息校验；
- CRM 对象权限、不可变生命周期轨迹、合同汇总和 Excel 导出；
- `local-ledger` 与未来税务平台的稳定幂等适配边界；
- 管理端完整操作入口和中文、英文、阿拉伯文文本。

## 退款/冲销已完成边界

- 客户退款、业务冲销两类 CRM 反向业务，不复用在线支付渠道退款；
- 草稿、审批中、通过、驳回、取消五态和修订重提；
- 原回款、客户、合同、负责人快照及不可变动作轨迹；
- 审批中与已通过金额共同占用额度，原回款锁内并发复核；
- 对象权限、字段篡改保护、幂等 BPM 回调和管理端完整入口；
- 单元、构建和创建/删除/超额拒绝运行验收已经通过；真实审批链等待流程定义部署。

## 不得混淆

- `pay_refund` 是在线支付渠道退款，不是 CRM 销售回款的业务冲销聚合；
- ERP 收付款不是 CRM 回款记录；
- 原测试文档中的“已覆盖”模板行不能替代 Controller、Service、表和可执行测试证据；
- 当前发票是 CRM 合同开票台账；不能把 `local-ledger` 描述成已完成税务申报或外部平台开票。
- 当前退款/冲销是 CRM 回款反向台账；不能描述成银行已退款或会计总账已冲销。
