# CRM 财务域实现状态核验

核验日期：2026-07-14。

## 结论

| 工作项 | 状态 | 仓库证据 | 尚缺 |
|---|---|---|---|
| 财务表结构：回款 | 已有并已完善 | `crm_receivable`、`crm_receivable_plan` | 会计入账/外部对账不在 CRM MVP |
| 财务表结构：发票 | 未完成 | 无 `crm_invoice` | ADR-009、实体/状态/表/权限/迁移 |
| 财务表结构：报销 | 未完成 | 无 CRM reimbursement 表 | ADR-017，避免与 OA 重复建模 |
| 财务表结构：退款 | 未完成 | 只有 Pay 渠道退款，不能等同 CRM 退款 | ADR-008 P1 的冲销、金额和审批规则 |
| 回款 CRUD 接口 | 子闭环完成 | 两个 Controller、Service、Mapper、前端页面和专项测试 | 银行到账、会计入账与核销不宣称完成 |
| 发票管理接口 | 未完成 | 无 CRM Invoice 代码 | ADR-009 后实施 |

## 回款已完成边界

- 回款与回款计划创建、查询、修改、删除、导出；
- 合同金额上限、计划金额守恒、正金额校验；
- 回款审批提交、驳回/取消后修订重提、旧回调幂等；
- 对象权限、计划并发期次/金额保护、已绑定计划保护；
- 审批通过口径的客户和统计汇总、逾期待办。

## 不得混淆

- `pay_refund` 是在线支付渠道退款，不是 CRM 销售回款的业务冲销聚合；
- ERP 收付款不是 CRM 回款记录；
- 原测试文档中的“已覆盖”模板行不能替代 Controller、Service、表和可执行测试证据；
- 发票开发前必须确认内部台账还是税务平台对接，以及作废、红冲、编号和金额边界。
