# CRM 剩余功能包台账

更新日期：2026-07-16。

## 统计口径

本台账对照 `docs/03-Gap-Analysis/`、`docs/Proj-Docs-v-6/03-Gap-Analysis/`、原型用户故事和
已交付功能索引，以“可以独立实现、测试和验收的功能包”为单位。57 个 GAP 包含 HR、OA、
安全与非功能条目，页面数也包含状态变体，因此二者都不能直接当作 CRM 剩余功能数。

完成综合经营统计血缘、CRM 回款退款/冲销、客户 360 净回款、资源权限加固和统一组织授权后，当前识别的无依赖功能包已清零。
公共线索完整状态机、线索活动迁移和报价版本已由 `CRM-FEATURE-042/043/044` 关闭。CRM 范围仍有 5 个功能包，
均必须先补决策或详细规则。退款与报销已从财务待办中移除；
该数字不包含明确属于 HR/OA 的完整新域，也不代表工期估算。

## 推进规则

- 每完成一个功能包，从本表删除或标记完成，并在 `features/`、`testing/` 和必要的 `bugs/logs/`
  建立独立目录；
- 受依赖项只允许做低耦合复用验证、安全修复和测试基线，不擅自固化业务阈值；
- 新功能继续使用 YAML 显式配置、Ubuntu 26.04 容器构建和 rootless Podman 运行验收。

合同正式补充协议的新版本审批命令仍属于合同模型后续增强，但不重复计为本轮已关闭功能包；进入实现前需先冻结补充协议编号、金额影响和原合同生效关系。
