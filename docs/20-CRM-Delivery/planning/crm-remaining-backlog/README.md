# CRM 剩余功能包台账

更新日期：2026-07-14。

## 统计口径

本台账对照 `docs/03-Gap-Analysis/`、`docs/Proj-Docs-v-6/03-Gap-Analysis/`、原型用户故事和
已交付功能索引，以“可以独立实现、测试和验收的功能包”为单位。57 个 GAP 包含 HR、OA、
安全与非功能条目，页面数也包含状态变体，因此二者都不能直接当作 CRM 剩余功能数。

本批合同发票生命周期完成后，CRM 范围剩余 17 个功能包：6 个可继续推进，11 个必须先
补决策或详细规则。该数字不包含明确属于 HR/OA 的完整新域，也不代表工期估算。

## 可继续推进：6 个

| 顺序 | 功能包 | 主要来源 | 当前边界 |
|---:|---|---|---|
| 1 | 合同签署、附件和变更历史 | GAP-CTR-001、ADR-002/004 | 沿用 CRM 合同，不新建重复订单聚合 |
| 2 | 客户四态生命周期基线 | CRM-FEATURE-007 后续、GAP-CUS-003 | 替代仅布尔成交的展示模型，迁移需兼容 |
| 3 | 客户 360 剩余只读聚合 | US-CUS-003、GAP-CUS-003 | 任务/附件/合同映射订单/财务摘要按已有对象逐项接入 |
| 4 | BPM 租户发布与统一待办运行治理 | GAP-APR-001/003 | 复用 Flowable，不复制流程引擎 |
| 5 | 综合经营统计与刷新/血缘展示 | US-RPT-010、GAP-RPT-001～003 | 只实现已有数据源和已明确公式 |
| 6 | 文件、附件和导出对象权限加固 | GAP-IAM-003、GAP-SEC-003 | 优先覆盖 CRM 已有对象和两租户负向测试 |

## 决策或规则依赖：11 个

| 功能包 | 依赖 |
|---|---|
| 默认角色—动作矩阵 | ADR-012 |
| 部门范围与 CRM 对象权限合并 | ADR-013 |
| 线索/客户公共池、垃圾池、保护期和回收规则 | ADR-014 |
| 线索转客户的任务、通话、短信活动迁移 | ADR-014 的迁移清单 |
| 报价版本、锁定、重开及高级价格快照 | 多币种、税率、SKU 版本规则 |
| CRM 合同到 ERP 履约订单 | 客户/产品映射、状态回传和失败补偿规则 |
| 工单抄送、下属/处理组分派 | ADR-010/012/013 的范围与合并算法 |
| 工单移动签到和地理围栏 | ADR-011 |
| 工单 SLA、节假日、暂停和自动升级 | ADR-011 |
| CRM 退款与费用聚合 | ADR-008 已列 P1，但状态、冲销和审批细则未冻结 |
| 营销活动、关怀及短信/邮件群发闭环 | ADR-016 |

## 推进规则

- 每完成一个功能包，从本表删除或标记完成，并在 `features/`、`testing/` 和必要的 `bugs/logs/`
  建立独立目录；
- 受依赖项只允许做低耦合复用验证、安全修复和测试基线，不擅自固化业务阈值；
- 新功能继续使用 YAML 显式配置、Ubuntu 26.04 容器构建和 rootless Podman 运行验收。
