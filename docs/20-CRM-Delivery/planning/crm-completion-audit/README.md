# CRM 功能完成审计

审计日期：2026-07-17。分支：`develop`。

## 结论与口径

以两套 Gap Analysis、原型业务链、`features/` 交付索引、自动化结果和真实运行验收交叉核对，
当前 CRM 业务范围没有未实现功能包。源 Gap 文档保留分析时点的历史判断，不直接改写；本文件和
`crm-remaining-backlog/` 是实现后的完成状态记录。

CRM 范围包括线索、客户、联系人、商机、产品/报价、合同、履约边界、审批、回款计划、回款、
退款/冲销、发票、报销、工单、经营统计和营销互动。完整 OA、HR、招聘、考勤、绩效、薪酬以及
外部电子签、银行/税控/真实短信邮件供应商不属于 CRM 内部功能完成口径，其边界均已通过 Provider
或 ADR 显式隔离，未用假实现冒充外部集成。

## 不阻断功能完成的已知治理项

以下事项仍需保留日志，但不代表存在未实现的 CRM 业务命令：

- 历史审批回款孤儿记录已经显式标记、保留原始 ID 并完成双口径对账；缺少权威新归属，因此仍不自动删除或重绑；
- 历史公开合同附件若仍存在，需要按来源可用性做一次性物理迁移；新附件已全部进入受保护目录；
- 全仓 `vue-tsc` 已解决内存退出问题，但仍报告 BPM、Mall、Pay 等多模块历史类型债务；本轮 CRM
  补充协议文件过滤后为 0 条新增错误；
- 外部电子签、银行到账、税控开票和真实消息供应商须在获得账号与协议后实现对应 Provider，当前
  本地记录或 record-only 模式不会伪造外部成功。

## 完成判定规则

后续如果新增原型范围、签署新的业务规则或选择外部供应商，应新增功能编号和独立测试目录，不能
倒推当前实现“从未完成”。当前已签署范围内，新增工作只应是缺陷修复、数据治理或外部适配项目。
