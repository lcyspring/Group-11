# 证据索引

## 核心实现与迁移

- CRM 服务端：`Server/mitedtsm-module-crm/src/main/`
- CRM 管理端：`Web/src/views/crm/`、`Web/src/api/crm/`
- OA/BPM 服务端：`Server/mitedtsm-module-bpm/src/main/`
- OA/BPM 管理端：`Web/src/views/bpm/`、`Web/src/api/bpm/`
- 增量迁移：`database/migrations/`
- 基线、可选数据集、维护和销毁：`database/bootstrap/`、`database/datasets/`、
  `database/maintenance/`、`database/teardown/`

财务域可直接核对 `new-crm-invoice.sql`、`new-crm-receivable-refund.sql`、
`new-crm-reimbursement.sql`、`new-crm-receivable-overdue-reminder.sql` 和
`new-crm-receivable-write-off.sql`。工单域可核对三个 `new-crm-work-order*.sql` 迁移；营销域可核对
`new-crm-marketing-foundation.sql` 与 `new-crm-marketing-delivery-analytics.sql`。

## 构建与运行

- Ubuntu 26.04 编译入口：`podman/compile.sh`
- rootless Podman 部署入口：`podman/deploy.sh`
- 中文操作指南：`podman/README_ZH.md`、`podman/OPERATIONS_ZH.md`、`podman/DEPLOY_GUIDE_ZH.md`
- YAML 字段说明：`podman/config/YAML_FIELDS_ZH.md`

本轮菜单合并使用 `podman/config/build-web-ubuntu-26.04.yaml` 构建管理端，并使用
`podman/config/runtime-local-rebuild-web.yaml` 替换 Web 容器。
