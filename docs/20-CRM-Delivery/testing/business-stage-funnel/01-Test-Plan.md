# 测试计划

## 后端

- 状态组字段为空时请求校验失败；
- SQL 以配置阶段为基表，零存量阶段仍返回；
- 活跃商机按状态组、负责人范围和创建时间过滤；
- 赢单单列，输单和无效不进入阶段漏斗；
- 空金额使用 `COALESCE` 返回零；
- Service 从后向前累加数量和金额，结果保持单调；
- 相邻阶段转化率按两位小数 `HALF_UP` 计算；
- 全量 CRM 单元测试及 JaCoCo。

## 前端

- 加载当前用户可用的商机状态组并默认选择第一项；
- 未取得状态组前不发送非法阶段漏斗请求；
- 切换状态组重新加载漏斗；
- 图表和表格共用服务端数量、金额和转化率；
- 赢单使用现有结束状态字典；
- 中、英、阿文案键完整；
- 严格 UTF-8 检查及 Ubuntu 26.04 Vite production build。
