# 线索活动迁移测试计划

## 自动化范围

- 任务创建、更新、状态转换、乐观并发、不可变动作轨迹和终态旧状态；
- 超时扫描的租户分页、Redis 集群锁、批量安全阈值和超时后继续处理；
- 通话时间顺序、YAML 最大时长、录音目录和短信字段校验；
- 任务通知模板、通知开关和失败隔离；
- 线索转换时客户、首联系人、跟进、任务、通话、短信和唯一审计的同事务迁移；
- `OWNER > WRITE > READ` 对象权限继承、公共线索只读和非公共对象失败关闭；
- Controller 功能权限、分页响应、人员名称装配和三语前端专项 ESLint。

## 真实运行验收

使用 `podman/tests/acceptance/verify-crm-clue-activity.sh`，唯一参数为显式 KDL 文件路径。脚本通过真实管理端 API：

1. 登录并创建临时线索；
2. 创建两个任务、一条通话和一条短信；
3. 开始并完成一个任务，校验动作序列；
4. 转客户并读取唯一转换审计；
5. 校验客户侧数量、`sourceClueId`、联系人绑定和 60 秒通话时长；
6. 校验原线索不再暴露活动，重复转换被拒绝；
7. 使用 trap 删除全部临时业务、权限、通知和轨迹数据并数据库对账。

## 构建与非回归门

- Ubuntu 26.04 容器 CRM 全量测试和 JaCoCo；
- 活动前端专项 ESLint 必须零警告；
- Ubuntu 26.04 Web production build；
- Ubuntu 26.04 Server 37 模块 production build；
- MySQL 迁移至少重复执行一次；
- 8080 健康、8081/8082 HTTP 200，Server 无新增 ERROR 或死锁。
