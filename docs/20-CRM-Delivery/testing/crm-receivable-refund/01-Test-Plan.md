# CRM 回款退款与冲销测试计划

## 后端自动化

- 创建时继承原回款客户、合同、负责人快照和 OWNER 权限；
- 未审批通过的原回款必须拒绝；
- 审批中和已通过金额共同占用额度，超额不得落库；
- 修改接口只能更新业务白名单字段；
- 提交必须按原回款加锁后复核金额；
- BPM 回调重复、旧实例和重复目标状态保持幂等；
- 已提交草稿不可删除；
- 可退金额摘要满足“原金额－占用金额＝可退金额”；
- 缺省 CRM 场景对普通用户不再产生无范围分页，管理员仍保留全量。

## 前端与容器构建

- 退款 API、列表、表单和详情目录执行独立零警告 ESLint；
- 表单远程检索负责人/参与人两个来源并去重，只保留审批通过回款，当前选中项不受分页截断；
- Server 和 Web production build 均使用 Ubuntu 26.04 容器；
- 构建命令只传 `podman/config/verify-crm-refund-ubuntu-26.04.yaml`。

## 运行验收

- 在租户 1 读取审批通过的原回款和可退金额；
- 创建 0.01 草稿，核对编号、五个快照、OWNER 权限和创建轨迹；
- 删除草稿并核对删除轨迹和清理结果；
- 以“可退金额 + 0.01”创建，验证业务错误且数据库无落行；
- 流程定义部署后补提交、通过、驳回、取消与旧回调的真实链路。
