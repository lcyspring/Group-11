# 客户查重测试计划

## 范围

- 请求条件校验与标准化。
- 名称/手机号 OR 匹配、排除自身、响应最小化。
- 创建表单候选展示、同名阻断、同手机确认继续。
- 用户确认的操作日志。
- CRM 对象权限和租户边界。

## 用例

| ID | 场景 | 层级 | 预期 |
|---|---|---|---|
| DUP-001 | 名称和手机均为空 | 单元/API | 不查库；API 400 |
| DUP-002 | 条件包含首尾空格 | 单元 | Mapper 收到已 trim 条件 |
| DUP-003 | 手机命中候选 | API | 返回 ID/名称/手机 |
| DUP-004 | 名称或手机任一命中 | API | OR 语义返回候选 |
| DUP-005 | excludeId 排除当前客户 | API | 当前客户不返回 |
| DUP-006 | 同名候选 | 前端/服务 | 前端阻断；后端名称唯一校验兜底 |
| DUP-007 | 同手机、不同名并确认 | API/日志 | 创建成功；日志记录确认选择 |
| DUP-008 | 普通用户无对象权限 | 集成 | 不返回无权候选 |
| DUP-009 | 跨租户同名/同手机 | 集成 | 不返回其他租户候选 |
