# CRM 群发送达与打开分析测试计划

1. 验证短信发送初始化、发送失败、回执成功和回执失败映射；
2. 验证邮件初始化、服务商接受和失败映射，不把接受称为送达；
3. 验证跟踪 URL 只接受无用户信息的 HTTP(S) 绝对地址并完成 HTML 转义；
4. 验证令牌格式、渠道绑定、无效令牌无查询、首次打开幂等；
5. 验证汇总中短信送达率和邮件打开率使用不同分母；
6. 验证创建人对象范围继续覆盖汇总与手工刷新；
7. 验证迁移连续执行两次，列和索引无重复；
8. 验证 Web API、三语言、手工刷新、逐收件人状态与 100% 专项覆盖率；
9. 在 Ubuntu 26.04 完成 CRM 全量、System 邮件专项、JaCoCo 和 production build；
10. 在真实 MySQL/Server/Web 运行态验证迁移、匿名像素和受权汇总接口。
