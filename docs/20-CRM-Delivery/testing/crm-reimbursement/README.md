# CRM 报销管理测试记录

- 分支：`develop`
- 状态：后端与数据库阶段通过

## 计划验证

1. SQL 连续执行两次且表、菜单、权限不重复；
2. 分类编码和名称租户内唯一，已使用分类不可删除；
3. 空明细、禁用分类、非正金额、合同客户不一致均被拒绝；
4. 服务端金额等于所有明细精确合计；
5. 审批状态迁移、重复回调和过期回调符合 ADR-017；
6. 不同用户无法越权读写报销对象；
7. Ubuntu 26.04 容器中执行专项测试、CRM 全量测试和 JaCoCo。
8. 外部 URL、其他报销单目录 URL 和非受管 URL 均不能保存为报销附件。

覆盖率将在专项与 CRM 全量测试完成后写入本目录，不用用例数替代覆盖率。

## Ubuntu 26.04 配置

编译检查使用 `podman/config/build-ubuntu-26.04-crm-reimbursement.kdl`，CRM 全量测试与覆盖率使用 `podman/config/test-ubuntu-26.04-crm-reimbursement.kdl`，命令行只指定配置文件路径。
