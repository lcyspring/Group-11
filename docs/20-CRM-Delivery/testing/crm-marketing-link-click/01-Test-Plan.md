# CRM 营销链接点击统计测试计划

## 服务端自动化

- 草稿链接编码、数量、协议和 Host 白名单校验；
- 逐收件人模板参数使用不同随机令牌，目标 URL 不进入匿名请求参数；
- 发送成功后令牌可用，发送失败的令牌不可用；
- 首次点击、重复点击及原子累计更新；
- 总计和逐链接的跟踪人数、独立点击、累计点击、独立点击率；
- `record-only` 不创建点击事实；
- 无效令牌与合法但不可用令牌均不泄漏目标地址。

## 管理端

- 草稿表单维护链接编码、名称和目标地址；
- 编辑时回显固化链接，详情展示点击总览和逐链接统计；
- 链接输入、指标展示和错误提示具备中、英、阿三语言资源；
- Node 专项契约 100% 覆盖、专项 ESLint、production build 和全仓 `vue-tsc --noEmit` 通过。

## 数据库与真实 HTTP

- bootstrap/compatibility manifest 均包含迁移，迁移连续执行两次成功；
- 通过管理 API 保存、回显并删除带链接的草稿，确认关联事实与链接完整清理；
- 已发送收件人的有效令牌返回 302，`Location` 等于数据库固化目标；
- 同一令牌连续点击后首次时间不变、最近时间更新、累计次数增加；
- 无效/畸形令牌返回统一 404；非白名单 URL 无法保存；
- 验收后清理临时群发、链接和逐收件人事实。
