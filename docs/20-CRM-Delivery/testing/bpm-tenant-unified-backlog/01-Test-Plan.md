# BPM 租户治理与统一待办测试计划

## 自动化范围

- 待办、已办和任务按 ID 查询必须附带当前 `taskTenantId`；
- 流程定义、部署、模型和 BPMN 原文读取必须先按当前租户查询；
- 模型 BPMN 原文更新不能绕过模型租户校验；
- CRM 待办组件通过专项 ESLint，Web 生产构建可生成；
- BPM 全部 `Bpm*Test` 在 Ubuntu 26.04 中执行并生成 JaCoCo。

## 运行验收

- 租户 1 登录后调用待办和已办分页，业务码应为 0；
- 8080 健康检查应为 `UP`，8081 应返回 200；
- 没有流程任务时，分页总数和列表均应为空，不视为失败。
