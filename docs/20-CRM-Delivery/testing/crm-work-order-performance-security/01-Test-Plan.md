# 测试计划：CRM 工单专项性能与安全

## 第一阶段：50 并发业务链

使用真实管理 API、当前 MySQL/Redis 和唯一运行前缀，创建 50 个临时工单，并分别以 50 并发执行：

1. 创建工单；
2. 读取工单 SLA；
3. 从待处理进入处理中；
4. 读取工单统计汇总；
5. 从处理中进入已完结。

每个场景记录应用错误、HTTP 状态、p95、p99 和容器资源采样。退出清理器删除工单、轨迹、SLA、
签到、抄送和通知，最终要求残留为 0。

## 第二阶段：安全负向

后续补充受限用户对象越权、跨租户访问、非法状态重复命令、超长批量参数、持久/反射 XSS 与 SQL
元字符输入。负向用例必须验证“拒绝且零写入”，不能只断言 HTTP 状态。

## 阈值

- 并发数不得低于 50；
- 各场景错误率不高于 YAML 阈值，当前为 1%；
- p95 不高于 1500 ms，p99 不高于 2500 ms；
- 运行结果只作为本地单节点基线，不等同生产容量承诺。
