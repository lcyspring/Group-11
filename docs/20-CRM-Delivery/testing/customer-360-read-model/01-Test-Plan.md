# 客户 360 只读聚合测试计划

## 自动化

- 聚合服务空值归一、财务差额和历史超额非负保护；
- SQL 契约中的审批状态、发票状态、软删除、工单参与人范围；
- 客户任务数从 `crm_task` 按客户业务类型真实聚合，不使用“任务域未接入”占位标记；
- 回款权限分页保留 `customerId` 过滤条件；
- 全量 CRM 单元测试和 JaCoCo 报告。

## 构建与运行

- Ubuntu 26.04 容器内 Server、Web 生产构建；
- 客户 360 专项 ESLint 零警告；
- MySQL 8 对真实客户执行聚合 SQL；
- 更新 Podman Server/Web 后调用真实摘要 API，并检查发票分页客户过滤。
