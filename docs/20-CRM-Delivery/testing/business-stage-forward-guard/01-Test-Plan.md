# 测试计划

## 请求约束

- 普通阶段推进缺少说明或只包含空白时拒绝；
- 推进说明最多 500 个字符；
- 阶段与结束状态继续保持互斥；
- 输单、无效和赢单的既有原因规则不回归。

## 服务行为

- 目标阶段排序大于当前阶段时更新成功；
- 目标阶段排序小于当前阶段时返回 `1020002005`；
- 目标等于当前阶段时继续返回原有重复状态错误；
- 不同状态组的阶段继续由既有校验拒绝；
- 并发更新影响 0 行时继续返回 `1020002004`；
- 推进说明去除首尾空白并进入状态操作日志上下文。

## 前端与回归

- 下拉框只显示排序在当前阶段之后的普通阶段；
- 选择普通阶段后动态显示必填推进说明；
- 选择赢单、输单或无效时不误用推进说明规则；
- 中、英、阿文案、Prettier、ESLint 和 production build；
- Ubuntu 26.04 CRM 全量测试和 JaCoCo。
