# 公共线索完整状态机测试计划

1. 未指定负责人创建后直接进入显式 PUBLIC 状态并写入归属事件；
2. 在管负责人填写原因手工退回，原因必填、去除首尾空白且最多 500 字；
3. 入池清空对象团队权限，领取或分配恢复目标用户 OWNER 权限；
4. 公共列表和详情只返回显式 PUBLIC、空负责人且未转换的线索；
5. 无对象权限用户可读公共详情，但不能读取授权范围外的在管线索；
6. 公共状态禁止编辑、删除、转移、转客户、跟进和团队维护；
7. 自助领取验证每日额度、重复领取冷却、批量上限和最大持有量；
8. 主管分配验证能力权限、批量上限、最大持有量且不占自助领取额度；
9. 同一线索并发领取只能一方成功，归属事件只能新增一次；
10. 不同线索并发分配给同一用户不能穿透最大持有量，也不能返回数据库死锁；
11. 普通更新不能绕过转移命令修改负责人；已转换线索不能进入公共池；
12. 自动迁移验证未跟进期限、负责人取得时间、已转换终态和锁后二次复核；
13. MySQL 迁移重复执行、Ubuntu 26.04 全量回归、前端 ESLint、生产构建和运行健康检查。
