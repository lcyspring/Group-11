# CRM 报销 BPM 闭环测试计划

- 分支：`develop`
- 环境：Ubuntu 26.04 rootless Podman + 本机 MySQL/Flowable

1. YAML 治理脚本语法和必填配置校验；
2. 首次创建审批角色、角色成员、流程分类、模型和定义；
3. 配置未变化时再次执行，不能新增流程版本；
4. 报销草稿提交后生成财务审批待办；
5. 审批通过回写 CRM 状态和不可变轨迹；
6. 审批驳回后允许修订，重提必须生成新的流程实例；
7. 发起人取消回写取消状态；
8. 旧流程和重复回调不能污染当前报销状态；
9. BPM 全量单元测试与 JaCoCo；
10. 完成后不能遗留运行中的测试实例。
