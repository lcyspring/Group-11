# 客户四态生命周期测试计划

环境：Ubuntu 26.04 构建镜像、OpenJDK 17、pnpm 11.3、MySQL 8.0、rootless Podman。

| 层次 | 检查项 |
|---|---|
| 创建 | 新客户默认潜在，兼容成交标记为 false，变更时间存在 |
| 命令 | 合法四态、相同状态拒绝、非法值拒绝、流失原因必填 |
| 一致性 | 行锁读取、主表与历史同事务、成交兼容字段同步 |
| 查询 | 分页四态条件、详情字段、历史按倒序且受对象读权限保护 |
| 统计 | SQL 按 `lifecycle_status` 分组和稳定排序，前端四态名称一致 |
| 迁移 | 旧数据回填、字段/索引、主表/历史表 CHECK、连续执行两次 |
| 前端 | 列表筛选、详情变更对话框、流失校验、历史页签、生产构建 |
