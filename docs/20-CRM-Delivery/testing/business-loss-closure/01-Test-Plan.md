# 测试计划

## 请求校验

- 输单无原因被拒绝；
- 无效状态的合法原因通过；
- 赢单不要求原因；
- 原因超过 500 字被拒绝；
- 原因不足 10 字被拒绝；
- 阶段与结束状态同时提交被拒绝。

## 服务行为

- 输单原因去除首尾空白并持久化；
- 赢单不持久化陈旧结束原因；
- 结束商机保留最后销售阶段；
- 条件更新影响 0 行时返回并发冲突错误。

## 前端与构建

- 输单/无效动态显示原因输入框；
- 原因执行 10 至 500 字校验；
- 详情页展示结束原因；
- Ubuntu 26.04 CRM 全量测试、JaCoCo 和 Web production build。
