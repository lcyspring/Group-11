# 工单 16 条 UAT 计划

## 公共前置条件

在隔离租户准备以下账号，密码只保存在本地未跟踪配置中：

| 角色代号 | 最小职责 |
|---|---|
| `WO_ADMIN` | 工单全局查询、建单、删除、跨组派单、处理组及 SLA 管理 |
| `GROUP_MANAGER_A` | A 组负责人，可管理本组工单 |
| `AGENT_A1/A2` | A 组成员，可领取、处理、签到和执行 SLA 动作 |
| `AGENT_B1` | B 组成员，用于跨组负向验证 |
| `CC_USER` | 仅作为抄送人，具有查询入口但没有处理权限 |

准备两个启用处理组 `UAT-A`、`UAT-B`，至少一个可访问客户、一个已赢单商机和一个已签署合同。
工单标题统一以批次前缀开头。每条用例新建独立工单，不复用前一条的最终状态。

## UAT-01 处理组维护

- 角色：`WO_ADMIN`。
- 前置：组编码 `uat-a` 不存在。
- 操作：进入“CRM → 客服处理组”，新增 `UAT-A`，配置负责人、成员、支持类型和启用状态；再次编辑名称和排序。
- 预期：保存后列表立即出现；编辑后无需刷新即可显示；编码稳定且负责人属于成员；重复编码被拒绝。
- 证据：新增/编辑后的列表截图、`/crm/work-order-group/list` 响应。
- 清理：全部工单用例结束后删除临时组；有引用时应先拒绝删除。

## UAT-02 客户来源建单

- 角色：`WO_ADMIN`。
- 前置：测试客户可访问，描述不少于 20 个字符。
- 操作：从客服工单新增，来源选客户，填写类型、优先级、描述和附件引用。
- 预期：生成唯一工单编号和创建轨迹；客户、来源、附件、创建人正确；同页列表立即出现。
- 证据：详情及创建轨迹截图。
- 清理：草稿态工单由管理员删除，或继续用于状态用例后统一清理。

## UAT-03 商机和合同来源约束

- 角色：`WO_ADMIN`。
- 前置：测试商机和合同均属于所选客户。
- 操作：分别以商机、合同为来源建单；再尝试提交不属于该客户的来源编号。
- 预期：合法来源创建成功；跨客户来源服务端拒绝且零写入，不依赖前端下拉框防护。
- 证据：两条成功详情、负向请求响应和列表总数对比。
- 清理：删除成功创建的临时工单。

## UAT-04 最小负载自动派单

- 角色：`WO_ADMIN`。
- 前置：`UAT-A` 支持当前工单类型，A1/A2 开放工单数可记录。
- 操作：选择处理组但不手工指定处理人，连续创建两条同类型工单。
- 预期：按开放工单负载、组排序、组 ID、用户 ID 稳定决胜；派单模式显示自动；记录分派时间和轨迹。
- 证据：创建前候选负载、两条详情和派单轨迹。
- 清理：完成或删除两条工单。

## UAT-05 无候选回退与领取

- 角色：`WO_ADMIN` 创建，`AGENT_A1` 领取。
- 前置：按测试 KDL 的回退策略使当前类型无自动候选，但保留 A 组成员。
- 操作：创建进入处理组未分配池的工单；A1 在“处理组未分配”视图领取。
- 预期：工单初始处理人为空；领取后处理人为 A1、派单模式为领取；列表立即移出未分配视图并进入“我处理的”。
- 证据：领取前后列表和详情轨迹。
- 清理：完成工单并恢复处理组配置。

## UAT-06 并发领取

- 角色：`AGENT_A1`、`AGENT_A2`。
- 前置：同一条 A 组未分配工单，记录当前版本。
- 操作：两个浏览器会话同时点击领取。
- 预期：仅一个请求成功；另一请求得到状态/版本冲突；最终只有一个处理人且只有一条有效领取轨迹。
- 证据：两侧响应、最终详情和轨迹。
- 清理：成功领取者完成工单。

## UAT-07 本组派单与跨组隔离

- 角色：`GROUP_MANAGER_A`、`AGENT_B1`、`WO_ADMIN`。
- 前置：A、B 两组各有成员和待处理工单。
- 操作：A 组负责人分派 A 组工单，再尝试分派 B 组工单；管理员执行跨组分派。
- 预期：本组分派成功；A 负责人跨组操作被服务端拒绝且处理人不变；管理员跨组分派成功并留轨迹。
- 证据：三个响应和最终轨迹。
- 清理：恢复各工单处理人或完成工单。

## UAT-08 抄送协作

- 角色：`WO_ADMIN` 建单，`CC_USER` 查看。
- 前置：抄送人数不超过 YAML 上限。
- 操作：创建时选择 `CC_USER`；抄送用户进入“抄送我的”查看详情并尝试开始处理。
- 预期：抄送视图立即显示；详情可读；开始处理被拒绝；重复抄送人被去重，超上限请求零写入。
- 证据：抄送视图、详情和处理负向响应。
- 清理：删除或完成工单及关联抄送。

## UAT-09 正常状态闭环

- 角色：已分派处理人。
- 前置：待处理工单。
- 操作：开始处理，填写不少于 20 个字符的解决方案并完结。
- 预期：状态依次为待处理、处理中、已完结；每次动作后列表和详情立即刷新；轨迹包含操作者、时间、旧状态和新状态。
- 证据：三态截图及完整轨迹。
- 清理：保留至统计用例结束后删除。

## UAT-10 退回与重新提交

- 角色：处理人退回，创建人重新提交。
- 前置：处理中工单。
- 操作：填写明确原因退回；创建人修正内容后重新提交。
- 预期：状态进入已退回并显示原因；重新提交回到待处理；空原因、非创建人重提和旧版本重复命令被拒绝。
- 证据：退回原因、重提轨迹和负向响应。
- 清理：完成或删除工单。

## UAT-11 移动签到围栏

- 角色：处理人。
- 前置：工单要求签到，配置服务经纬度和围栏半径；浏览器允许定位。
- 操作：在围栏内签到，再模拟围栏外坐标请求。
- 预期：服务端计算距离；围栏内记录坐标、精度、距离和操作者；围栏外拒绝且不新增签到记录。
- 证据：最近签到详情、围栏外响应和签到轨迹。
- 清理：删除工单时一并清理签到事实。

## UAT-12 SLA 初始化与工作日历

- 角色：`WO_ADMIN`。
- 前置：为当前优先级配置启用 SLA 策略，并配置一个休息日和一个补班日。
- 操作：创建工单，查看 SLA 截止时间。
- 预期：按优先级命中策略；截止时间跳过周末及休息日、计入补班日；重复保存同一日历日期保持唯一。
- 证据：策略、工作日历、工单 SLA 详情和人工计算表。
- 清理：删除临时日历覆盖项和工单。

## UAT-13 SLA 暂停、恢复和升级

- 角色：处理人暂停/恢复，调度器执行升级。
- 前置：处理中工单有 SLA 实例。
- 操作：暂停并等待可测时间后恢复；在隔离环境把另一实例调整到升级/逾期门槛并运行调度。
- 预期：暂停期间截止计时冻结；恢复后截止时间平移；升级/逾期只改变 SLA 事实和轨迹，不篡改工单业务状态。
- 证据：暂停前后截止时间、升级/逾期轨迹和业务状态。
- 清理：删除临时工单；恢复测试策略时间参数。

## UAT-14 待办与通知联动

- 角色：处理人、抄送人。
- 前置：新分派和新抄送工单。
- 操作：打开 CRM 待办和通知弹层，处理其中一条工单。
- 预期：新事项无需多次点击或手工刷新即可出现；详情路由携带有效工单 ID；处理后待办数量和列表自动更新；通知对象正确。
- 证据：处理前后待办、通知和详情。
- 清理：完成工单并清理临时通知。

## UAT-15 客户 360 与统计一致性

- 角色：有客户及统计权限的用户。
- 前置：同一客户至少有待处理、处理中、已完结各一条工单。
- 操作：查看客户详情工单页及“数据统计 → 工单统计”，按状态、类型、处理人和日期筛选。
- 预期：客户 360 数量及明细与授权范围一致；统计总量等于状态分项之和；创建/完结趋势按对应时间聚合；页面刷新动作即时取最新数据。
- 证据：列表、统计卡片、分组及趋势截图和 API 对账表。
- 清理：记录统计后删除批次工单并再次确认不计入结果。

## UAT-16 权限、非法输入与删除边界

- 角色：`AGENT_B1`、`CC_USER`、`WO_ADMIN`。
- 前置：A 组工单一条、已完结工单一条。
- 操作：B1 读取/处理 A 组工单；抄送人处理；待处理直接完结；提交脚本标签、SQL 元字符和 101 个抄送人；删除已进入业务流转的工单。
- 预期：对象越权、非法状态、XSS 和超限均拒绝且零写入；SQL 元字符仅按普通文本处理；有轨迹/关联事实的删除遵循服务端保护，不留下孤儿记录。
- 证据：各负向响应、操作前后表计数和孤儿检查。
- 清理：由 `WO_ADMIN` 按交接指南清理全部 `UAT-WO-` 数据。
