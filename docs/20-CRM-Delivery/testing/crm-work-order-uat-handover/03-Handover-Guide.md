# 工单知识转移与故障处置指南

## 操作入口

- 客服工单：`/crm/work-order`；
- 客服处理组：`/crm/work-order-group`；
- CRM 待办：`/crm/backlog` 的工单页；
- 客户 360：客户详情的工单页；
- 工单统计：`/crm/statistics/work-order`。

状态只允许 `待处理(10) → 处理中(20) → 已完结(30)`，或处理中退回为 `已退回(40)` 后重新提交。
抄送代表只读协作，不授予处理权；SLA 逾期是服务治理事实，不自动改写工单业务状态。

## 定位顺序

1. 先记录租户、用户、工单编号、页面路由、时间和业务提示；
2. 浏览器网络面板确认请求路径、HTTP 状态、业务码和请求中的工单 ID，禁止只截压缩 JS 堆栈；
3. 从工单详情核对处理组、处理人、状态、版本、派单模式、SLA 和轨迹；
4. 检查用户权限及组成员关系，再判断是菜单隐藏、对象范围还是状态机拒绝；
5. 核对应用日志中的工单 ID/租户 ID，不在文档或工单中粘贴 Token、密码和 Provider 秘钥；
6. 若是统计不一致，使用同一租户、时间范围和对象权限对比分页明细，避免直接查全租户表得出错误结论。

常见现象：

| 现象 | 优先检查 | 正确处置 |
|---|---|---|
| 新工单没有处理人 | 处理组启用、支持类型、成员和 YAML 回退模式 | 进入组未分配池领取，或由有权限人员派单 |
| 看不到派单按钮 | `assign`/`assign-all` 权限和工单状态 | 不直接改库授权；按角色矩阵调整权限 |
| 领取提示冲突 | 工单已被其他成员领取或版本已变化 | 刷新列表，以服务器最终处理人为准 |
| 签到失败 | 浏览器定位授权、经纬度、精度和围栏 | 修正地点/授权；不得绕过服务端距离校验 |
| SLA 时间不符 | 优先级策略、时区、周末和工作日历 | 核对日历覆盖项和 YAML，不手改实例截止时间 |
| 待办或统计滞后 | 请求是否成功、对象权限、页面重新取数 | 记录 Bug；不得要求用户反复刷新作为长期方案 |

## 数据清理

UAT 只在隔离租户执行。先按标题前缀导出待清理工单 ID，确认没有非 UAT 记录，再按外键依赖从属到主
清理签到、SLA、抄送、轨迹和工单；随后清理临时通知、工作日历、组成员及处理组。清理后必须验证：

- `UAT-WO-` 工单残留为 0；
- 工单轨迹、SLA、签到和抄送孤儿均为 0；
- 临时组和日历覆盖项残留为 0；
- 客户 360 和统计不再包含本批次。

禁止把清理 SQL 放入生产自动启动路径，也禁止关闭外键检查后无条件全表删除。

## 回滚

- 页面或服务回归：使用上一组已验证的 Server/Web 镜像摘要通过部署替换阶段回滚，数据库卷保持不变；
- YAML 策略错误：恢复上一份受控配置并重新执行部署前校验，再替换容器；
- 迁移问题：迁移采用前向修复，不对已有业务表直接执行破坏性降级；
- 数据误操作：停止写入，通过已验证备份在隔离库恢复、核对租户和对象后再制定回填方案。

任何回滚都应记录镜像摘要、配置校验和、数据库备份点、开始/结束时间和验证结果。

## 已知限制

- 浏览器定位受 HTTPS、设备权限和硬件精度约束；本地回环地址属于浏览器特殊安全上下文；
- SLA 调度触发时间受扫描周期影响，不承诺在截止毫秒点即时展示；
- UAT 基线是单节点本地环境，不替代生产容量评估；
- 外部短信/邮件送达取决于 Provider 账号，站内通知不等同供应商送达；
- 四方签署属于人员协作结果，仓库只能提供模板和工程证据。
