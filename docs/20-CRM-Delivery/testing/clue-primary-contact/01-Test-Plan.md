# 线索转客户创建首联系人测试计划

执行日期：2026-07-13

## 测试目标

- 验证转换请求必须显式提供合法的首联系人姓名和手机号。
- 验证客户、首联系人、线索关联和既有跟进复制位于同一事务。
- 验证重复请求和两个并发请求不会创建重复客户或联系人。
- 验证首联系人字段复制、客户详情回显、负责人和 OWNER 权限正确。
- 验证联系人写入失败时转换权、客户和关联信息完整回滚。
- 回归联系人唯一首联系人、手机号唯一和锁后当前读约束。

## 分层场景

| 层次 | 场景 | 预期 |
|---|---|---|
| VO 单元 | 缺少字段、非法手机号、完整请求 | 约束数量正确，完整请求通过 |
| Service 单元 | 原子抢占失败 | 返回 `1020001001`，不创建客户 |
| Service 单元 | 成功调用顺序及字段复制 | `claim → customer → contact → link` |
| 真实 API | 空姓名/手机号 | HTTP 业务码 400，不产生客户 |
| 真实 API/MySQL | 正常转换 | 线索已转换，客户只有一个首联系人，字段和权限正确 |
| 真实 API | 重复转换 | 返回 `1020001001`，客户/联系人仍各一条 |
| 真实并发 | 同一线索两个转换请求 | 仅一个成功，最终客户/联系人各一条 |
| 故障注入 | 联系人 INSERT 受控失败 | 线索未转换且无客户、联系人残留 |
| 回归 | 首联系人、手机号、锁等待脚本 | 9/9、7/7、2/2 |

## 环境与证据

- 运行环境：项目 rootless Podman Pod，MySQL 8.0，真实 Spring Boot API。
- 临时脚本：`/tmp/crm-clue-primary-contact-integration.sh`，不提交 Git。
- 覆盖率原始报告：`Server/mitedtsm-module-crm/target/site/jacoco/`，属于构建产物，不提交。
- 故障触发器只在单个回滚场景期间存在，脚本退出陷阱保证兜底移除。
