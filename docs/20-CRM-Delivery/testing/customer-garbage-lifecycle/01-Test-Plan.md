# 客户垃圾池测试计划

1. 普通用户不能查询、转入、恢复或永久删除垃圾池客户；
2. 只有显式 PUBLIC 客户可手工转入，原因必填且最多 500 字；
3. 转入后清除客户和所有联系人团队权限；
4. 锁定、成交、活跃商机和受保护合同阻止迁移；
5. 恢复后进入 PUBLIC，负责人保持为空并记录恢复轨迹；
6. 永久删除覆盖十一类下游引用保护和状态条件物理删除；
7. 自动迁移复核滞留时间、累计次数、池状态和业务保护；
8. YAML 批量关系、迁移幂等、菜单权限和数据库索引；
9. 普通 CRM 查询权限不能利用空团队权限直接读取垃圾客户详情；
10. 管理端筛选、详情动作、三语言、ESLint 和生产构建；
11. 真实 API 完成转入、查询、恢复、再次转入、永久删除和引用拒绝闭环。
