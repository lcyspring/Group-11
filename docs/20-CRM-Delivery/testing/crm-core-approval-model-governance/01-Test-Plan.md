# CRM 核心审批模型治理测试计划

1. Shell 语法与 YAML 必填项；
2. 四类角色、权限、分类、模型首次治理；
3. 权限和模型未变化时幂等复跑；
4. Flowable 每个 key 只能有一个有效定义；
5. 合同草稿提交、待办、通过及产品快照审计轨迹；
6. 回款草稿提交、待办、通过，合同累计回款与统计口径同步；
7. 退款草稿提交后产生匹配 key 的待办；
8. 驳回后回到可修订状态，修订后换新流程实例重提；
9. 通过后占用原回款可退额度并记录完整轨迹；
10. 四类模型不遗留运行测试实例；
11. Ubuntu 26.04 BPM 全量测试、JaCoCo、Web ESLint 和生产构建。
