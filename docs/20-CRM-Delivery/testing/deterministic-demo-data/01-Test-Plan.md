# 测试计划：固定种子 CRM 广覆盖演示数据

执行日期：2026-07-18。分支：`develop`。

1. 以 `check` 模式校验名称、固定种子、日期、规模上限、对象基数和输出目录，断言不写文件；
2. 以 `generate` 模式生成 cleanup、核心、关联域、加载后断言、manifest 和 SHA-256，断言不连接数据库；
3. 使用 `deploy.sh` 和临时 `mysql.dataset_mode: replace` 清理旧演示批次并插入新批次；
4. 由 manifest 的 `04-validate.sql` 在同一部署事务链中校验数量、引用、状态和统计真源；
5. 运行库二次核对商机阶段、四类结单结果、销售预测、客户画像、回款计划和财务状态；
6. 校验 Server 健康状态和 Web HTTP 状态；
7. 替换完成后恢复 `mysql.dataset_mode: preserve`，避免普通替换 Server 时重复导入。

重点失败条件：合同数量不得等于商机数量、回款数量不得等于计划数量、任一商机阶段不得为空、
开放商机必须有预计成交时间、赢单必须有实际结单时间，画像维度和财务状态不得退化为单值。
