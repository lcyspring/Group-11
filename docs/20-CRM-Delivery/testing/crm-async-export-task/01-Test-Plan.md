# CRM 异步导出任务测试计划

## 服务端自动化

- 提交时冻结筛选与授权对象 ID，且提交请求内不生成文件；
- 单用户活动任务容量和最大导出行数边界；
- `QUEUED → RUNNING → SUCCESS/FAILED` 原子状态流转；
- 生成前权限变化、对象删除和未知 Provider 转失败并保留诊断；
- 任务本人可见，跨用户和跨租户不可枚举；
- 令牌签发前与下载前重复校验对象权限；
- 数据库只存令牌摘要，原子单次消费，过期或错误令牌拒绝；
- 任务过期转态并删除受保护文件；调度器多实例锁与批次上限。

## 管理端与全仓类型

- 客户导出只提交后台任务，不回退到同步导出；
- 任务中心分页、状态筛选、活动轮询、失败原因和下载入口契约；
- 三语言资源、专项 ESLint、Node 契约和 production build；
- 标准 Ubuntu 26.04 入口执行全仓 `vue-tsc --noEmit`，脚本和模板必须同时为零诊断。

## 数据库与真实 HTTP

- compatibility manifest 可重复执行；
- 创建任务并观察五态流转；
- 首次令牌下载成功，同令牌第二次下载失败；
- 跨用户、权限变化和过期任务下载失败；
- 验收结束清理任务记录及受保护文件。
