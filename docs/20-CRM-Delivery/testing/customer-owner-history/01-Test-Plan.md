# CRM 客户归属记录测试计划

执行日期：2026-07-13

## 自动化与构建

1. Service 单测：初始分配、转移前后负责人、入公海字段、历史查询透传。
2. Controller 单测：三类用户名称批量拼接、空历史不调用用户服务。
3. CRM `Crm*Test` 全量回归并生成 JaCoCo 报告。
4. 相关 Vue/TS ESLint、Web 生产构建、Server 生产 JAR。

## 真实 Podman/API/MySQL

1. 迁移脚本连续执行两次，检查两个新字段各一列。
2. 创建客户后得到 `null -> owner` 的类型 3 记录。
3. 转移后得到 `oldOwner -> newOwner` 的类型 4 记录。
4. 入公海和领取分别得到 `owner -> null` 和 `null -> owner` 记录。
5. 历史按记录 ID 倒序，操作人与当前登录用户一致。
6. 为记录表临时建立失败触发器，强制转移最后的历史写入失败，检查客户负责人、CRM Owner 权限和历史数量均回滚。
7. 只有 `crm:customer:query` 但没有目标客户 CRM 数据权限的用户访问历史。
8. 租户 1 Token 配合租户 118 Header 访问。
9. 调用原公海按日期、按用户统计，验证新类型不破坏旧 SQL。
10. 清理临时角色权限、触发器、测试历史，检查无有效测试客户。
