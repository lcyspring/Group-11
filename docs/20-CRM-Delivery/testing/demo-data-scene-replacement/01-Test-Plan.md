# 演示数据分类与替换测试计划

## 目标

验证固定种子数据不只满足总量，还能覆盖基准管理员的负责人、参与、下属和公海场景；验证 `replace`
完整移除旧 CRM 数据集，同时保留管理员及策略配置。

## 用例

1. 在 Ubuntu 26.04 工具链环境生成 SQL，检查模板无未替换占位符。
2. 使用 `deploy.sh` 和 `mysql.dataset_mode: replace` 执行清理、插入、关联域插入和 SQL 断言。
3. 查询数据库总量、公海量、管理员 OWNER/READ 权限以及非当前批次业务键。
4. 使用超级管理员登录，调用客户各场景、联系人、线索、公共线索和 BPM 待办分页接口。
5. 编译并替换 Web，核对权限菜单只包含一个日程管理。
6. 将运行配置恢复为 `mysql.dataset_mode: preserve`。
