# 首联系人测试计划

## 范围

- 字段迁移、历史数据修复及幂等性。
- 创建、切换、取消、删除和跨客户移动规则。
- 同客户并发切换的唯一性。
- `master` 关键决策人与 `primaryContact` 首联系人语义独立。
- 生命周期操作日志。
- 客户列表/详情首联系人姓名与手机映射。
- 前端表单、列表、多语言和生产构建。

## 用例

| ID | 场景 | 层级 | 预期 |
|---|---|---|---|
| PRI-001 | 首个联系人请求 `false` | 单元/API | 强制保存为首联系人 |
| PRI-002 | 第二联系人请求 `false` | API | 原首联系人不变 |
| PRI-003 | 第二联系人请求 `true` | 单元/API | 原首联系人降级，新联系人升级 |
| PRI-004 | 当前首联系人直接改 `false` | 单元/API | 返回 `1020003004` |
| PRI-005 | 删除当前首联系人 | 单元/API | 返回 `1020003005`，不进入合同/删除副作用 |
| PRI-006 | 当前首联系人移动到另一客户 | 单元/API | 返回 `1020003006` |
| PRI-007 | 取消原首联系人更新数为 0 | 单元 | 返回 `1020003007` 并回滚 |
| PRI-008 | 两请求并发切换同一客户 | API/数据库 | 请求均成功，最终恰好一个首联系人 |
| PRI-009 | RR 事务已有普通查询快照 | API/日志 | 锁后使用当前读，第二请求取消最新首联系人 |
| PRI-010 | 关键决策人与首联系人 | API | 两字段互不覆盖 |
| PRI-011 | 客户响应映射 | API/UI | 返回并显示首联系人姓名和手机 |
| PRI-012 | 生命周期日志 | API/数据库 | 自动设置、切换均有可追溯文字 |
| PRI-013 | 历史迁移重复执行 | SQL | 不重复字段/索引，每客户一个首联系人 |
| PRI-014 | 中文前缀测试清理 | 测试设施 | utf8mb4 查询与清理后四类残留为 0 |
