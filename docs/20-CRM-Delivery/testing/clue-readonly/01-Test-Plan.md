# 已转换线索只读测试计划

执行日期：2026-07-13

## 目标

- 验证转换后的线索在前端不再展示写操作。
- 验证直接调用 API 也无法更新、删除、转移、修改跟进或团队权限。
- 验证跟进先插入、线索只读校验失败时事务会回滚插入。
- 验证权限 ID 不能借助伪造的 `bizType/bizId` 越权更新。
- 验证转换与更新争用线索行锁时遵循排队顺序且转换使用最新字段。

## 分层场景

| 层次 | 场景 | 预期 |
|---|---|---|
| Service 单元 | 已转换线索更新/跟进/删除/转移 | 均返回 `1020001002`，不继续写入 |
| Permission 单元 | 已转换线索添加成员 | 返回 `1020001002` |
| Permission 单元 | 权限 ID 不属于请求对象 | 返回 `1020007000` |
| 真实 API | 字段、负责人、删除 | 三类写均拒绝且数据不变 |
| 真实 API | 跟进新增/删除 | 新增失败无孤立记录，历史记录不可删 |
| 真实 API | 团队添加/编辑/删除 | 均拒绝且权限不变 |
| 真实 API | 伪造客户对象更新线索权限 ID | 返回 `1020007000`，权限不变 |
| MySQL 锁序 | 转换先排队、更新后排队 | 转换成功，更新返回只读错误 |
| MySQL 锁序 | 更新先排队、转换后排队 | 两者依次成功，客户使用更新后字段 |

## 环境

- Rootless Podman、MySQL 8.0、真实 Spring Boot API。
- 主验证脚本：`/tmp/crm-clue-readonly-integration.sh`。
- 锁序脚本：`/tmp/crm-clue-readonly-lock-integration.sh`。
- 两个临时脚本均不提交 Git，退出陷阱负责清理数据。
