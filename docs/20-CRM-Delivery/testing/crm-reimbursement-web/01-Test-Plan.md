# CRM 报销管理端测试计划

- 分支：`develop`
- 环境：Ubuntu 26.04 rootless Podman

## 自动化

1. 草稿、审批中、通过、驳回、取消五态的可编辑边界；
2. 仅从未提交的新草稿允许删除；
3. 六位小数页面合计；
4. 费用发生日期必须落在主单日期范围；
5. API、四个页面和三语文案 ESLint；
6. Web `prod` 模式生产构建。

## 运行验收

1. 菜单可进入报销列表；
2. 创建无附件草稿；
3. 创建带附件草稿并确认先创建、后上传、再回写；
4. 更新已有草稿并查看服务端总额；
5. 费用分类增改、启停和删除；
6. 详情展示明细、附件和不可变动作轨迹；
7. 流程发布后另行执行提交、待办、通过、驳回、取消、修订重提和旧回调验收。
