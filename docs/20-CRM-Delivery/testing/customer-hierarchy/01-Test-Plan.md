# 客户上下级关系测试计划

执行日期：2026-07-13

## 自动化范围

- Service：有效父关系、父客户持久化、自关联、父客户不存在、两节点环、深层环、删除保护、显式清空父关系。
- Mapper：`parentCustomerId` 直接下级分页条件。
- CRM 全量 `Crm*Test` 回归并生成 JaCoCo 报告。
- 前端相关文件 ESLint、生产构建；Server 生产 JAR 构建。

## 真实 API/MySQL 场景

1. 幂等迁移连续执行两次，检查字段与索引。
2. 创建根客户、两个直接下级和一个孙级客户。
3. 详情回显上级 ID/名称。
4. `parentCustomerId` 分页只返回两个直接下级。
5. 自关联返回 `1020006016`。
6. 跨租户父 ID 返回 `1020006017`。
7. 两节点和三节点循环返回 `1020006018`。
8. 删除有下级的父客户返回 `1020006011`。
9. 并发执行 `A→B` 与 `B→A`，最多一个成功，数据库递归检查无环。
10. 把父关系清空为 `null` 后数据库确实保存 NULL。
11. 解除直接下级后父客户可删除。
12. 清理后没有有效测试客户残留。

## 数据与权限边界

- 真实 API 使用租户 1 的 CRM 管理用户；跨租户测试只插入一条租户 118 的受控客户，结束后物理清理。
- 下级入口复用 `/crm/customer/page` 及既有场景数据权限，不新增绕过权限的裸列表接口。
- JaCoCo 原始报告位于 `Server/mitedtsm-module-crm/target/site/jacoco/`，属于构建产物，不提交。
