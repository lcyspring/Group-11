# 商机报价版本测试计划

## 自动化范围

- YAML 币种、汇率、税率、精度、舍入模式和版本上限校验；
- V1 草稿创建、金额计算、产品主数据快照和非法币种/税率拒绝；
- 锁定空报价拒绝、锁定后不可编辑、重开克隆及旧版本不可变；
- 输单/无效终止报价、结束商机禁止编辑和赢单必须锁定报价；
- 服务端覆盖客户端目录价，小数数量在请求、DO、数据库和合同间保持一致；
- 产品更新事务行锁和快照版本递增；
- 商机转合同继承锁定报价，忽略客户端伪造行编号、价格、数量、币种和税率；
- 转合同并发幂等、来源报价一致性和六位金额精度；
- 报价读写对象权限、功能权限和三语言管理端专项 ESLint。

## 真实 API 验收

`podman/tests/acceptance/verify-crm-quote.sh` 唯一参数为显式 KDL 路径。脚本通过真实 8080 API：

1. 登录并读取服务端 YAML 报价策略；
2. 创建 USD、小数数量和 13% 税率的 V1；
3. 验证未锁定不能赢单、锁定后不能编辑；
4. 重开 V2，验证 V1 快照不变并按 6% 税率重新核算；
5. 锁定 V2、赢单并提交带伪造产品行的合同请求；
6. 验证合同只采用 V2 锁定快照和 `decimal(24,6)` 金额；
7. 重复转换验证幂等；
8. 删除全部临时数据并数据库对账为零。
