# BPM 发起导航与通知可靠性测试计划

1. 校验发起流程只接受可解析的浏览器路由，并保留合法 query/hash。
2. 校验审批详情只接受明确的 `.vue/.tsx` 组件路径；无扩展名路径、相对路径、协议相对路径和
   catch-all 404 路由均被拒绝。
3. 校验 CRM 销售、CRM 财务、办公协作分类点击后只显示对应流程。
4. 校验受管模型 YAML 明确区分发起路由和审批详情组件，不猜测扩展名或列表页含义。
5. 校验流程表达式三语标题、字段和校验文本完整。
6. 校验短信关闭时审批不调用 Provider。
7. 校验短信启用时四种 BPM 事件使用稳定模板 code。
8. 校验模板缺失/Provider 异常在默认策略下不向审批接口传播，显式 fail-fast 时才传播。
9. 校验运行 YAML、公开管理端 URL、模板配置和部署脚本无状态预检。
10. 在 Ubuntu 26.04 工具链中完成 BPM 全量测试、JaCoCo、Server/Web 生产构建。
11. 校验可判定的存量错误路径由幂等 SQL 修复并逐条审计，未知值保持原样并由前端严格拒绝。
