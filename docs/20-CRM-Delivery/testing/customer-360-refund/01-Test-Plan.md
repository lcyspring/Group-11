# 客户 360 退款/冲销测试计划

## 后端自动化

- Mapper 必须从 `crm_receivable_refund` 分别统计记录数和审批通过金额；
- 金额子查询必须带客户、逻辑删除和 `audit_status = 20` 条件；
- Service 对空计数、空金额统一归零；
- 净回款等于回款毛额减审批通过退款/冲销额；
- 待回款按净回款计算；
- 历史退款超过回款时，净回款不得为负。

## 前端验证

- API 类型包含退款记录数、审批通过退款额和净回款额；
- 摘要同时展示毛回款、退款/冲销、净回款和待回款；
- 客户退款 Tab 只读展示三个对象权限场景和详情入口；
- 中、英、阿三语键完整；
- `lint:crm-customer-360` 零警告，Web production build 通过。

## 运行验收

- 记录插入审批通过退款前读取客户 360 基线；
- 插入审批通过退款后，退款计数增加、退款额增加、净回款等额减少、待回款等额增加；
- 再插入草稿/审批中记录，只增加数量，不改变退款财务金额；
- 删除临时数据后摘要恢复基线，数据库残留为 0。
