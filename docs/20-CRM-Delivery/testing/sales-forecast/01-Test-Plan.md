# 测试计划

## 后端

1. 两条日数据按月合并商机数、预计金额和概率金额；
2. 无数据月份返回 0、`0.00`、`0.00`；
3. SQL 必须按预计成交日期过滤；
4. SQL 必须排除已结束和逻辑删除商机；
5. 概率金额必须引用阶段 `percent` 和商机 `total_price`；
6. 明细分页按预计成交日期、负责人范围和活跃状态查询；
7. CRM 全量测试并生成 JaCoCo。

## 前端

1. 新页签可加载汇总和分页明细；
2. 图表与表格使用同一汇总响应；
3. 明细展示阶段名称、阶段赢单率、预计成交日期和金额；
4. Ubuntu 26.04 Vite production build。
