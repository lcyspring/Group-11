# 测试计划

## 后端

1. 两条日数据按月合并预测/实际商机数与预测/实际金额；
2. 无数据月份四项指标返回 0 或 `0.00`；
3. SQL 必须按预计成交日期过滤；
4. SQL 必须排除已结束和逻辑删除商机；
5. 实际金额必须引用 `end_status=1`、`end_time` 和商机 `total_price`；
6. 明细分页按预计成交日期、负责人范围和活跃状态查询；
7. CRM 全量测试并生成 JaCoCo。

## 前端

1. 新页签可加载汇总和分页明细；
2. 图表与表格使用同一汇总响应；
3. 明细展示阶段名称、预计成交日期和预测金额，不再展示概率金额；
4. 页面显式展示预测周期、分组粒度和两项指标口径；
5. Ubuntu 26.04 Vite production build。
