# 测试计划

## 后端

1. 两个北京区县必须合并成一个北京市城市项；
2. 区县必须分别归并到北京、上海、广东省份；
3. 国内区县必须在国家层汇总为中国；
4. 客户数与成交客户数均必须守恒；
5. 结果按客户数降序输出；
6. 请求使用选择的负责人范围；
7. 城市钻取必须展开到所选城市全部区县；
8. 客户明细必须保留时间和负责人范围，并稳定分页；
9. 请求区域层级与区域 ID 不匹配时返回空页且不查询 Mapper；
10. CRM 全量测试并生成 JaCoCo。

## 前端

1. 城市、省份、国家三个页签均可按公共筛选条件刷新；
2. 城市/国家图表显示全部客户和成交客户双系列；
3. 表格显示客户数、成交数和两个占比；
4. 城市、省份、国家区域名称均可打开客户明细弹窗；
5. 弹窗支持分页和客户详情跳转；
6. 中、英、阿三语 key 完整；
7. Ubuntu 26.04 Vite production build。
