# 垃圾池反馈与缓存刷新测试计划

1. 有进行中商机的公海客户转入垃圾池时，返回垃圾池专属业务码和“不能转入垃圾池”提示；
2. 有未完结销售单据的公海客户转入垃圾池时，返回垃圾池专属业务码和正确提示；
3. 两个拒绝分支均不得更新客户池状态；
4. 转入成功后刷新公海来源列表；
5. 已缓存的垃圾池页面每次重新激活时重新查询分页数据；
6. 成功转入立即发出垃圾池列表失效信号，相邻激活请求不得形成并发请求风暴；
7. 页面刷新不得依赖固定路由名称，避免动态菜单名称变化后失效；
8. 恢复到公海和永久删除始终在固定操作列内并排、不换行，且保留各自权限控制；
9. Ubuntu 26.04 容器执行 CRM 回归、前端专项、ESLint 及生产构建。
