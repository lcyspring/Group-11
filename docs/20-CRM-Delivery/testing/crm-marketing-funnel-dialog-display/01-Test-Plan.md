# CRM 群发与漏斗弹窗显示测试计划

## 目标

验证历史群发收件人稳定显示业务名称，漏斗明细明确区分阶段与结单结果，并保证两个弹窗在低高度
视口中仍能访问表格与分页操作。

## 用例

1. 历史收件人不依赖当前目标候选列表即可返回客户和联系人名称；
2. 仅选择客户、没有联系人的收件人不会因空联系人 ID 异常；
3. 前端优先展示响应中的 `customerName`、`contactName`；
4. 漏斗阶段列使用当前选中状态，终态使用独立结果列；
5. 两个弹窗均启用视口内滚动和明确最大高度；
6. 中、英、阿语言资源均包含结单结果字段；
7. CRM 后端全量回归不得失败。
8. 普通滚动弹窗只限制最大高度，内容较少时不得按最大高度留下空白；全屏模式仍使用固定滚动高度。

## 容器入口

- 前端：`podman/config/verify-crm-display-dialogs-ubuntu-26.04.yaml`
- 后端：`podman/config/test-crm-ubuntu-26.04.yaml`
