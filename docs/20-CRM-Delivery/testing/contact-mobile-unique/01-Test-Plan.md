# 联系人手机号唯一性测试计划

## 范围

- API 必填及格式契约。
- 同一客户创建、更新及并发创建的手机号唯一性。
- 不同客户间手机号复用。
- 逻辑删除后的手机号复用。
- SQL 幂等、生成列及唯一索引。
- 前端必填规则、多语言、Lint 和生产构建。
- CRM 回归及覆盖率记录。

## 用例

| ID | 场景 | 层级 | 预期 |
|---|---|---|---|
| MOB-001 | 新建联系人不传手机号 | API | 返回 `400` 和“手机号不能为空” |
| MOB-002 | 同一客户重复新建手机号 | 单元/API | 返回 `1020003008`，不插入重复数据 |
| MOB-003 | 更新为同客户另一联系人的手机号 | 单元/API | 返回 `1020003008`，排除当前 ID |
| MOB-004 | 不同客户使用同一手机号 | API | 均可创建 |
| MOB-005 | 两请求并发创建相同手机号 | API/数据库 | 一个成功、一个 `1020003008`，有效记录为 1 |
| MOB-006 | 逻辑删除后重新使用手机号 | API/数据库 | 新联系人创建成功 |
| MOB-007 | 手机号两端空格 | 单元 | 服务层规范化后再查重 |
| MOB-008 | 内部历史调用传空手机号 | 单元 | 唯一性辅助方法兼容跳过，API 仍负责拒绝 |
| MOB-009 | 迁移连续执行两次 | SQL | 生成列、唯一索引、查重查询索引各一项 |
| MOB-010 | CRM 全量回归 | 单元 | 所有 `Crm*Test` 通过 |
