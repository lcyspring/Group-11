# CRM 客户导入预检测试计划

## 服务端

- 标准表头自动映射和自定义表头显式映射；
- 模板隐藏字典 Sheet 不计入客户数据行；
- 预检阶段不写客户表；
- 逐行创建、更新和失败结论及聚合数量；
- 文件内/数据库重复、格式、字典、地区和最大行数边界；
- 当前租户与创建人隔离、过期拒绝；
- 确认导入结果持久化及重复确认幂等。

执行 CRM 模块全量测试和 JaCoCo，避免只验证新增类而破坏既有客户闭环。

## 管理端

- 页面必须先预检再确认，不能直接调用兼容导入接口；
- API、页面、三语言资源执行专项 ESLint；
- Ubuntu 26.04 容器执行契约测试及 production build。

## 数据库与运行

- compatibility manifest 重复执行成功；
- 使用重建后的 Server/Web 访问真实接口；
- 验证预检前后客户数量不变，确认后只产生一次业务写入；
- 验收结束清理临时客户和预检快照。
