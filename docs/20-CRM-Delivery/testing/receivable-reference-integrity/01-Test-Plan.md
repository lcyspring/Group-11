# 回款引用完整性测试计划

## 自动化

1. Mapper 必须计算四级引用状态，并支持按状态过滤分页。
2. 响应必须保留原始客户、合同 ID，不以关联对象缺失为由清空财务台账字段。
3. 前端必须显式展示异常状态，异常引用不得进入编辑或对象详情跳转。
4. 客户统计必须要求客户与合同均有效且合同属于该客户。
5. 员工业绩必须继续按审批通过回款台账汇总，不因业务对象软删除而丢失金额。

## 运行对账

1. YAML 指定的历史回款必须返回引用状态 `30` 和原始引用 ID。
2. 有效状态筛选不得包含该记录，双异常状态筛选必须精确返回该记录。
3. 客户统计金额必须与有效客户—合同 MySQL 汇总一致。
4. 员工业绩金额必须与审批通过回款 MySQL 台账一致。
5. 两类口径差额必须等于异常回款金额。
6. 验收前后历史数据库行快照必须完全相同。
