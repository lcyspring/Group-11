# 测试计划

## 后端

- 明细请求缺少状态组或阶段 ID 时校验失败；
- 普通阶段先校验阶段属于状态组，并使用该阶段排序查询；
- 服务端将负责人或部门用户范围写入分页请求；
- 普通阶段查询包含后续活跃商机和赢单；
- 赢单查询只包含同状态组赢单；
- 两类查询均按创建时间过滤并服务端分页；
- 空用户范围返回空页，不执行数据查询；
- CRM 全量测试和 JaCoCo。

## 前端

- 只有 `crm:business:query` 权限显示详情按钮；
- 阶段行调用阶段分页 API，并传递 `statusId/pageNo/pageSize`；
- 赢单行调用赢单分页 API；
- 翻页保留主查询条件和所选阶段；
- 主筛选变化关闭旧弹窗；
- 商机和客户名称可进入对应详情；
- Ubuntu 26.04 Prettier、ESLint 和 production build。
