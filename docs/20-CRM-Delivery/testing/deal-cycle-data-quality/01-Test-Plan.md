# 成交周期数据质量测试计划

## 自动化

1. 四类响应 VO 必须返回负样本数。
2. 按员工、地区、产品统计必须按客户去重计数。
3. 日期区间聚合和员工聚合不得丢失负样本数。
4. 产品名称归一化不得覆盖负周期或负样本字段。
5. 三张表都必须展示数据质量列，第一 Y 轴不得强制为 0。
6. 中文、英文和阿拉伯文文本纳入零警告 ESLint。

## 运行对账

1. 从 MySQL 按同一首个审批合同口径读取负样本数和周期。
2. 日期 API 的负样本合计必须与 MySQL 1:1，且仍包含负周期。
3. 员工和地区 API 负样本数必须与 MySQL 一致。
4. 产品 API 必须显式返回负样本；同一客户可以出现在多个产品分组，不用跨产品求和作为全局样本数。
5. 元数据 API 必须返回 YAML 中的 `customer.deal-cycle` 口径。
6. 验收只读现有数据，不修改或删除历史记录。
