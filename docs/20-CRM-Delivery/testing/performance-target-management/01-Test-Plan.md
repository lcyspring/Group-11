# 测试计划

## 后端契约

- 查询响应的 12 个月、四个季度和年度目标均为十进制字符串；
- 缺失月份补零，季度和年度仍由后端 `BigDecimal` 精确汇总；
- 公司、部门、个人范围继续使用已有服务端校验；
- 查询、维护、删除继续使用三个相互独立的权限。

## 前端逻辑

- 金额接受 0～18 位整数和最多两位小数，拒绝负数和三位小数；
- 计数只接受 0～18 位非负整数，拒绝小数；
- 大于 JavaScript 安全整数范围的金额仍能精确累计；
- 12 个月正确汇总为四个季度和年度目标；
- 任一输入无效时预览不展示错误合计，保存前给出错误提示。

## 页面与构建

- 公司、部门、个人范围和五类指标可切换；
- 未配置、保存、更新和删除状态可正确刷新；
- 保存/删除按钮分别受独立权限控制；
- 中、英、阿文案、Prettier、ESLint 和 production build 通过；
- 全部构建和测试在 Ubuntu 26.04 Podman builder 中执行。
