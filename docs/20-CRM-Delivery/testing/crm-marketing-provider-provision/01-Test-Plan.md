# 测试计划：CRM 营销 Provider provision

## 环境

从运行 MySQL 的 System 模块复制短信渠道、短信模板、邮件账号、邮件模板四张空结构到隔离测试库。
测试使用唯一模板编码和邮箱，结束时删除测试库，不改真实 Provider 配置。

## 用例

1. `create-only` 连续执行两次，四类记录各保持一条；
2. `managed` 使用相同稳定键更新短信签名/模板及 SMTP 账号/邮件模板；
3. `disabled` 执行后保留 managed 的配置值；
4. 短信模板关联指定渠道、邮件模板关联指定邮箱账号。

## 通过标准

- 四个场景全部成功；
- 无重复稳定键记录；
- 更新值与 KDL 一致；
- disabled 零覆盖；
- 测试库执行后自动删除，秘密不出现在脚本输出。
