# Podman 三阶段交付测试计划

日期：2026-07-18。分支：`develop`。

1. 三个阶段脚本均通过 Bash 语法检查，并只接受一个 YAML 路径；`compile.sh` 必须拒绝非法白/黑名单及空工作选择；
2. 镜像封装 `check` 校验所有目标产物但不改变镜像和容器；
3. 部署 `check` 不加载、拉取、构建或启动任何镜像/容器；
4. 静态门禁确认 `deploy.sh` 不含构建命令和 Host 产物路径；
5. 完整执行编译、封装、`replace`，确认 8080/8081/8082 健康；
6. 执行至少一个单组件封装和 `replace-*`，确认其他容器不被替换；
7. 记录配置覆盖和实际验证结果，不把只读预检冒充运行验收。
8. 所有公开编译 YAML 同时显式声明 include/exclude，已删除脚本名和 Host 工具链安装脚本不存在残留引用；
9. 根目录仅有四个日常脚本，验收与低频维护脚本按职责归档；部署使用 `pod create/run --replace`。
