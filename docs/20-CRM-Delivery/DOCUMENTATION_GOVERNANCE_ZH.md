# 文档治理规范

更新日期：2026-07-19。

## 文档分层

| 层级 | 目录 | 维护规则 |
|---|---|---|
| 输入基线 | `docs/01-*` 至 `docs/05-*`、`docs/Proj-Docs-v-6/` | 保留原始分析时点，不用后续结果反写历史结论 |
| 当前交付 | `docs/20-CRM-Delivery/` | 随 `develop` 源码维护并提交 |
| 操作真源 | `podman/*_ZH.md`、`podman/config/*_ZH.md` | 命令、字段、镜像和部署行为的唯一现行说明 |
| 本机证据 | ignored 的报告、诊断包、凭据和构建产物 | 不提交，不作为团队唯一证据 |

## `20-CRM-Delivery` 目录职责

- `features/`：一个业务能力一个目录，说明状态机、数据、权限和交付入口；
- `testing/`：一个功能或缺陷一个目录，保存计划、结果和覆盖率；
- `bugs/`：Bug 总表和独立日志，历史事实不因配置迁移而改写；
- `decisions/`：仍有效的业务与工程 ADR；失效决策应标记“已取代”或删除重复草案；
- `runtime/`、`evidence/`：只保留仍能解释当前实现的汇总证据，不堆积逐次临时观察；
- `user-guide/`：面向人工验收，使用当前菜单名、操作路径和预期结果；
- `planning/`：只记录未完成项、依赖和完成定义，已完成计划应更新状态而不是长期保留错误命令。

## 写作与链接规则

1. 中文为主；命令、字段、状态码和源代码标识保留原文。
2. 首段先写结论、状态、日期和分支；测试结果必须区分“自动化通过”和“外部账号/UAT 待办”。
3. 现行 Podman 命令只引用 `.kdl`。Spring 原生 `application.yaml` 可保留；前端只保留 Deno
   manifest/lock，不保留已退休的 pnpm lock/workspace。
4. 不复制大段字段清单；操作文档链接到 `KDL_FIELDS_ZH.md`，功能文档只说明本功能涉及的字段。
5. 不写本机绝对路径、真实账号、Token、密码、私有 IP 或诊断包内容。
6. 删除目录前先检查入链；删除后修复所有现行文档引用。历史 Bug 日志允许描述当时的旧路径。
7. 构建产物、JaCoCo HTML、OCI tar、数据库备份和 `*-local.kdl` 不进入文档目录。

## 变更检查

文档提交前至少执行：Markdown 相对链接检查、旧脚本/旧 Podman YAML 引用检查、Shell `bash -n`、
KDL 解析与 runtime-config 无状态门禁。若只改业务说明，可跳过耗时编译，但必须在提交说明中避免
宣称未执行的测试。
