# 销售漏斗指标契约完善

## 差异来源

对应 `GAP-RPT-002`：统计指标公式、业务时间、刷新和权限需要形成明确契约。
现有赢单转化率由 Web 使用通用工具临时计算，API 只返回分子与分母。

## 本次完成

- 漏斗按日期响应新增 `businessWinRate`；
- 后端统一口径为 `赢单商机数 / 商机总数 * 100`；
- 使用 `HALF_UP` 保留两位小数；
- 分母为零时固定返回 `0.00`；
- Web 图表直接消费后端指标，移除重复公式；
- 商机金额 SQL 使用 `COALESCE`，全空金额返回 0；
- 修复新增商机明细分页参数透传，并在重新筛选时回到第一页；
- 修复小时统计分桶继续落入日分支的问题。

## API 兼容性

新增字段为向后兼容变更，原 `businessCount`、`businessWinCount` 保留。前端已切换
到服务端指标，避免不同客户端出现不同舍入口径。

## 未扩展范围

本次不改变“赢单”的现有状态定义，也不改变商机统计使用 `create_time` 的既有
业务时间。两项如需调整，应进入 ADR-005/ADR-015，而不是在本次实现中猜测。
