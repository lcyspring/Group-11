# CRM-FEATURE-043：线索活动真源与转客户迁移闭环

- 分支：`develop`
- 状态：已实现
- 配置：`mitedtsm.crm.activity`

本功能为线索和客户建立任务、通话、短信三类 CRM 活动真源，并在线索转客户事务中迁移全部
活动。转换不再只创建客户和首联系人：跟进、任务、通话、短信、对象权限和唯一转换审计在同一
事务边界内完成，任一步失败均回滚，重复转换由线索行锁、转换终态和租户级唯一键共同拒绝。

## 活动模型

- 任务支持未开始、进行中、完成、未完成、取消和超时状态；每个命令使用乐观版本保护；
- 创建、修改、开始、完成、未完成、取消、超时和迁移均写入不可变动作轨迹；
- 超时任务仍允许负责人开始处理，避免定时标记把业务任务永久锁死；
- 通话保存方向、结果、号码快照、起止时间、受保护录音地址和 YAML 限制内的时长；
- 短信保存方向、状态、内容快照、系统日志编号、渠道编号和失败原因；
- 任务指派及终态处理复用系统通知模板，不建立第二套通知基础设施。

## 转换迁移不变量

1. 转换前锁定线索并验证未转换、对象写权限和首联系人输入；
2. 创建客户和首联系人，迁移既有跟进记录；
3. 任务、通话和短信统一改绑目标客户，填充 `sourceClueId`，联系人引用切换为新首联系人；
4. 写入唯一的转换审计，记录客户、首联系人及四类迁移数量；
5. 原线索活动分页返回空，目标客户分页可按来源线索追溯；
6. 转换完成后线索保持只读，重复请求明确拒绝，不重复创建客户或迁移活动。

## 权限和管理端

活动接口同时检查功能权限和关联 CRM 对象权限。对象等级遵循 `OWNER > WRITE > READ`：负责人
可写可读，协作写权限可读，纯读权限不能处理任务。只有显式公共且未转换的线索允许无团队权限
只读，客户和在管线索不会借公共分支越权。

线索和客户详情页均增加任务、通话、短信三个页签；创建和任务处理使用结构化表单，动作轨迹可
展开查看。已转换线索和公共对象只读，客户侧展示来源线索及本次迁移数量摘要。中文、英文和
阿拉伯文同步维护。

## 配置与运行

YAML 显式配置活动版本、受保护录音目录、最大通话秒数，以及超时调度 cron、时区、Redis 锁、
租约、批量大小和部署安全上限。命令行只接受 YAML 路径。调度器按租户游标分页，多实例通过
Redisson 锁单执行；单条失败记录错误，不阻断其余租户和批次。

数据库迁移创建五张表、三个动作权限和两个通知模板，并为历史已转换线索只回填可证明的客户、
首联系人和跟进数量，不虚构历史任务、通话或短信。迁移已在现有 MySQL 上重复执行成功。
