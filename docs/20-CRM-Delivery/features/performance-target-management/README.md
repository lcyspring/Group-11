# CRM-FEATURE-017：业绩目标维护前端

更新日期：2026-07-14

## 需求来源

- 原型 `PG-0380`～`PG-0382`：个人、部门、公司业绩目标维护；
- `US-MKT-001`：维护个人、部门和公司业绩目标；
- `US-RPT-005`：按月评估五类目标完成度；
- `GAP-MKT-001`、`GAP-RPT-001/002`；
- 前置能力 `CRM-FEATURE-015/016`。

## 本次实现

- 员工业绩页面新增“业绩目标维护”标签页；
- 支持公司、部门和个人三种范围，公司范围按后端契约固定使用 `scopeId=0`；
- 支持成交金额、回款金额、跟进次数、新增客户和新增商机五类指标；
- 一次录入 12 个月目标，并实时预览四个季度和年度合计；
- 未配置目标时显示明确提示，保存后自动刷新，已有目标可按独立权限删除；
- 保存按钮受 `crm:performance-target:update` 控制，删除按钮受
  `crm:performance-target:delete` 控制，查询继续复用统计查看权限；
- 金额目标最多两位小数，计数目标只接受非负整数；
- 前端使用 `BigInt` 定点缩放汇总，避免 JavaScript `Number` 对大额金额产生精度损失；
- 目标查询响应中的月度、季度和年度数值统一改为十进制字符串；
- 中、英、阿三语文案齐备。

## 精度契约

目标值可能超过 JavaScript 安全整数范围，因此前后端不通过 JSON Number 传递或累计
金额。后端使用 `BigDecimal.toPlainString()` 返回，前端金额按分缩放为 `BigInt` 后求和。
例如：

```text
9007199254740991.99 + 0.01 = 9007199254740992.00
```

计数目标使用零位小数的同一算法，金额始终保留两位小数用于预览。

## 阶段边界

公司、部门、个人目标之间是否必须满足逐级分解合计关系尚未获得业务签署，本批不硬编码
推测规则。当前页面沿用员工业绩页的年度筛选器；目标复制、批量导入和目标调整历史也不在
现有已签差异范围内。

本批未发现新的既有缺陷，因此没有新增 Bug 编号或虚构 Bug 日志；实现属于差异分析中的
新增能力。测试基础设施缺陷 `TEST-BUG-007` 已在前一批独立记录并关闭。
