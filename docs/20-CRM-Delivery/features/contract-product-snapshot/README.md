# CRM-FEATURE-026：合同产品不可变成交快照

## 来源与范围

- `GAP-OPP-002`、`GAP-PIM-001`；
- `ADR-003` 产品来源与交易快照；
- 商机赢单转合同和普通合同创建、修订链路。

本批将合同产品行从“产品 ID + 当前目录展示”完善为显式成交快照。现有 CRM 产品模型只有
名称、编码、单位、价格，没有规格字段，因此本批只保存真实存在的名称、编码、单位和目录价，
不虚构规格，也不引入多币种、税率或 SKU 版本。

## 实现

- `crm_contract_product` 增加 `product_name_snapshot`、`product_no_snapshot`、
  `product_unit_snapshot`；
- 新建合同和赢单商机转合同时由服务端读取产品目录生成快照，忽略前端行编号和目录价；
- 合同修订时，同一产品行保留原名称、编码、单位和目录价快照；切换产品才生成新快照；
- 产品改名、改编码、改单位、改价、下架或从可选目录消失后，合同详情仍优先显示成交快照；
- 历史空快照由幂等迁移回填，重复执行只补空值，不以当前目录覆盖已冻结值；
- 更新接口接收合同产品行 `id`，校验行必须属于当前合同，拒绝重复行编号。

## 兼容边界

- 迁移前已物理删除、无法关联产品目录的历史行保持空快照，接口仍尝试旧目录兼容展示；
- 合同成交价和数量仍允许在可编辑合同状态内修订；目录价及文本快照不可由请求伪造；
- 商机报价版本、锁定、重开、多币种、税率和 SKU 版本仍保留在后续决策包中。
