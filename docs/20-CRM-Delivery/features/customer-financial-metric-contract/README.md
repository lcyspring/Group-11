# 客户成交与回款指标契约修正

## 发现的问题

- 客户成交率由两个前端组件各自计算，零分母和舍入口径未成为 API 契约；
- “未回款金额”列实际调用百分比函数，显示内容与“回款完成率”完全相同；
- 客户成交率组件含非法 UTF-8 字节，注释长期显示乱码。

## 本次完成

- 日期维度、员工维度响应增加后端 `customerDealRate`；
- 员工维度响应增加 `unreceivablePrice`、`receivableRate`；
- 成交率和回款率统一保留两位小数，零分母返回 `0.00`；
- 未回款金额固定为合同金额减回款金额，并使用金额格式展示；
- 前端只消费服务端指标，不再重复公式；
- 清除客户转化组件的非法 UTF-8 字节，不改业务模板结构。

## 当前口径边界

当前既有 SQL 的合同金额按合同下单日期统计，回款金额按回款日期统计，因此二者差值
表达筛选期内的净金额差，不等同某批合同截至当前的真实应收余额。后续如要展示合同
应收余额，必须按合同集合关联回款并明确截止时间，不能沿用本字段做会计口径承诺。

测试见 `docs/20-CRM-Delivery/testing/customer-financial-metric-contract/`；缺陷见
`STAT-BUG-024`、`STAT-BUG-025`。
