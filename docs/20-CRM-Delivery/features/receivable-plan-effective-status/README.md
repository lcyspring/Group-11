# CRM-FEATURE-022：回款计划金额守恒与生效状态

日期：2026-07-14

## 目标

依据 `GAP-FIN-001/002`、`GAP-RPT-002`、原型 `US-FIN-001～003/012` 和
ADR-007/008，完成合同金额、计划、回款审批和计划待办之间的可解释闭环。

## 已完成

- 合同级行锁串行化计划期次和计划总额分配；
- 新建/修改后计划总额不得超过审批通过合同金额；
- 计划关联回款使用计划行锁，防止并发创建多个回款；
- 已关联回款的计划禁止修改和删除；
- 只有审批通过回款完成计划并计入已回款金额；
- 草稿、审批中、驳回、取消保持待回款或逾期；
- 待办角标与默认待回款列表统一按提醒日口径；
- API 显式返回 `status/receivedPrice/unreceivedPrice`，前端不再自行猜测；
- 列表、客户/合同子列表、待办和详情统一展示状态与金额。

## 边界

当前模型一个计划只关联一笔回款；本批保持兼容，不扩展多笔回款拆分。审批通过仍只表示
CRM 业务确认，不表示银行到账、会计入账或核销。
