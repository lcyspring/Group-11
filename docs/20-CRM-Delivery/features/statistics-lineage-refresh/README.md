# CRM-FEATURE-032：统计指标血缘目录与显式实时刷新

## 完成内容

- 在 YAML 中声明 CRM 统计刷新模式、权限模式、历史重算边界和对账方式；
- 为客户、漏斗、业绩、画像、排行和工单六个统计域登记 35 项已有指标；
- 每项指标显式包含编码、来源表、来源字段、业务时间、公式、过滤规则和数据权限；
- 新增只读 `/crm/statistics-metadata/catalog` 接口，按统计域返回指标目录；
- 接口按请求域检查对应的六类原有统计权限，不能用客户统计权限读取漏斗或排行目录；
- 六个统计页面统一展示刷新模式、本页最后刷新时间、手动刷新入口和可展开的血缘目录；
- 中文、英文和阿拉伯文界面文本同步。

## 刷新与口径边界

`REALTIME_QUERY` 表示每次页面查询直接使用当前业务表聚合，不承诺缓存、定时快照或离线数仓。
本次只登记已经存在且公式明确的指标，不添加未冻结的同比、环比或预测公式。

历史查询按当前台账状态重算；对账通过授权明细钻取或来源台账导出完成。配置记录口径但不承载
SQL，业务查询仍由受控 Mapper 执行，前端不能提交表名、字段或自定义 SQL。

## 权限与租户边界

- 元数据接口不返回业务行，但仍要求当前统计域的查询权限；
- 指标目录逐项说明租户隔离和 CRM 授权用户范围；
- 实际统计继续复用原 Service、数据权限和 MyBatis 租户拦截，不因展示血缘而扩大数据范围；
- 工单统计保留当前租户和创建人/处理人范围，没有 `query-all` 时不会展示他人工单数据。

## 构建入口

使用 `podman/config/verify-crm-statistics-lineage-ubuntu-26.04.kdl`，命令行只传 KDL 路径。
