# OA 请示审批

请示使用独立 `bpm_oa_work_request` 业务表，流程定义键为 `oa_work_request`。发起时先持久化业务单据，再创建 BPM 流程实例并保存实例编号；流程状态事件通过监听器回写业务状态。

请示包含标题、正文、紧急程度和发起人，支持流程节点的发起人选人。列表按 BPM 流程实例状态显示审批中、审批通过、审批不通过或已取消，并提供详情、审批进度和本人运行中流程取消操作。管理端入口位于协作工作区，权限为 `bpm:oa-work-request:query/create`。

请示发起后立即进入受管审批流程，因此业务内容保持只读；若内容有误，应取消运行中的流程后重新发起，避免审批上下文与业务单据不一致。审批通过时记录完成时间。

部署前需在 BPM 模型配置中发布 `oa_work_request` 流程；未发布时接口会明确返回流程定义不存在，不能伪造审批成功。
