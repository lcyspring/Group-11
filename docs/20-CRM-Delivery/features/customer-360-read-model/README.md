# 客户 360 只读聚合（GAP-CUS-003）

## 完成内容

- 客户详情新增统一只读摘要，聚合联系人、商机、销售单据、回款计划、回款、发票、工单和
  合同附件数量；
- 按 ADR-002 将原型“销售订单”显式映射为 CRM 合同，不建立第二套重复销售单据；
- 财务摘要统一采用审批通过合同额、审批通过回款额、有效蓝票减红票净额；
- 待回款额和待开票额由服务端计算，并对历史超额数据做非负展示保护；
- 客户详情新增只读发票 Tab，继续复用发票对象权限和详情生命周期接口；
- 工单计数沿用参与人范围，拥有 `crm:work-order:query-all` 时才统计客户全部工单；
- 客户 360 服务保留客户对象 READ 权限校验和租户拦截，不提供跨客户聚合后门；
- 三语言界面明确提示 OA 任务域尚未实现，避免将工单伪装为任务。

## 聚合口径

| 指标 | 口径 |
|---|---|
| 销售单据数 | 该客户未删除 CRM 合同数 |
| 合同额 | 审批状态为“通过”的合同金额合计 |
| 已确认回款 | 审批状态为“通过”的回款金额合计 |
| 有效净开票 | 未作废、非草稿发票的 `方向 × 金额` 合计 |
| 待回款/待开票 | 合同额减对应净额，小于零时展示零 |
| 合同附件 | 该客户合同下未删除附件数 |
| 可见工单 | 当前用户创建或处理的工单；全量权限例外 |

OA 任务、CRM 退款和费用对象没有可复用聚合真源，本功能不返回伪造的零记录；它们应在
各自业务聚合落地后通过新的只读适配接入。
