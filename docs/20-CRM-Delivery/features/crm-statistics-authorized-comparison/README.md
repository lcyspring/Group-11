# CRM 授权统计范围与精确同比环比

日期：2026-07-15

分支：`develop`

## 已实现

- 为客户分析、销售漏斗、员工业绩、客户画像和排行榜五组统计 Controller 增加统一范围门禁；
- 将浏览器提交的部门、人员视为不可信输入，在进入统计 Service 前校验；
- 显式人员必须属于所选部门及其下级部门，并落入当前用户的 CRM 组织可读负责人范围；
- 部门汇总必须保证部门树全部人员均被当前组织范围覆盖，防止父部门查询夹带未授权子部门；
- ALL/CRM 管理员可读取所选部门树，但仍校验人员与部门的一致性；
- 空部门允许返回空统计，组织、用户或授权 API 异常统一失败关闭；
- 员工业绩响应新增 `monthOnMonthRate`、`yearOnYearRate`；
- 同比、环比改由 Server 使用 `BigDecimal`、两位小数、`HALF_UP` 统一计算；
- 基期为零返回空值，前端图表保留断点、表格显示 `--`，不再用 JavaScript `Number` 重算。

## 授权算法

```text
登录用户 + deptId/userId
  → 解析 CRM 组织可读负责人集合
  → 展开所选部门及全部子部门
  → 校验 userId 属于部门树
  → 校验人员或完整部门树被组织范围覆盖
  → 通过后调用原统计 Service
```

通用统计接口只得到组织范围 READ，不产生对象 WRITE/OWNER、导出或转移权限。客服工单统计仍使用
创建人、处理人和 `query-all` 的专用授权语义，不套用负责人统计门禁。

2026-07-16 真实 SELF 用户验收发现组织 API 会继承当前数据权限并过滤部门读取，已通过
`STAT-SEC-003` 修复：组织解析阶段局部关闭环境数据权限，随后仍执行本文定义的完整 CRM 范围校验。
本人、同部门他人、整部门、跨统计域、跨租户和匿名矩阵均已通过运行验证。

## 计算口径

```text
(本期值 - 基期值) / 基期值 × 100
```

结果保留两位小数；基期为零时不可比较，返回 `null`。金额与计数均使用相同公式，但原始统计值和增长率
始终在后端十进制计算，避免客户端浮点误差及多个页面产生不同口径。
