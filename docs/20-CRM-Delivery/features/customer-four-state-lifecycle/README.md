# CRM-FEATURE-029：客户四态生命周期闭环

完成日期：2026-07-14。分支：`develop`。

## 交付内容

- 客户默认创建为潜在客户，支持潜在、意向、成交、流失四态；
- 独立状态命令执行合法值、重复状态、流失原因和对象写权限校验；
- 行锁和单事务同步客户状态、兼容成交标记及不可变状态历史；
- 客户分页按四态筛选，详情显示状态、变更时间和流失原因；
- 前端提供四态标签、变更表单、流失原因必填和独立状态轨迹；
- 客户画像由已成交/未成交二态升级为四态分布；
- 新库初始化和已有 Podman 数据卷都执行同一幂等迁移。

## 接口

- `PUT /crm/customer/update-lifecycle-status`：更新客户状态；
- `GET /crm/customer/lifecycle-record-list`：查询授权客户的状态历史；
- `GET /crm/customer/page?lifecycleStatus=20`：按状态筛选；
- 原客户状态画像 URL 保持不变，响应字段升级为 `lifecycleStatus`。

旧 `/crm/customer/update-deal-status` 作为兼容入口保留，内部映射到新命令，不再绕过历史。

## 未扩张边界

公海回收仍按兼容 `dealStatus` 判断，未擅自规定意向或流失客户的保护期；该规则需和公共池策略
一起冻结。状态历史只提供只读接口，不提供修改或删除入口。
