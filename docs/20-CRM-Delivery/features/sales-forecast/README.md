# 销售预测

## 差异来源

- 原型页面：`PG-0272 销售预测分析`、`PG-0273 销售预测报表`；
- Story：`US-RPT-001`、`OPP-009`；
- 差异：`GAP-RPT-001`、`GAP-RPT-002`。

## 本次完成

- 新增销售预测汇总 API 和活跃商机分页 API；
- 统计使用预计成交日期 `crm_business.deal_time`，不再误用创建时间；
- 只统计 `end_status IS NULL` 的活跃商机；
- 增加独立实际结单时间，预测与实际不再复用预计成交日期；
- 按请求时间粒度补齐无数据区间，并统一金额为两位小数；
- 前端销售漏斗页新增销售预测页签；
- 展示预测金额、实际赢单金额、两类商机数和可分页预测商机明细；
- 页面显式展示预测周期、分组粒度与指标来源；
- 补齐中、英、阿三语文案。

## 指标契约

| 指标 | 口径 |
|---|---|
| 预测商机数 | 预计成交日期在筛选范围、未结束、未删除的商机数 |
| 预测金额 | 按 `crm_business.deal_time` 分组的活跃商机 `SUM(total_price)` |
| 实际赢单数 | 实际结单时间在筛选范围且 `end_status=1` 的商机数 |
| 实际赢单金额 | 按 `crm_business.end_time` 分组的赢单商机 `SUM(total_price)` |
| 组织范围 | 请求负责人，或请求部门及子部门负责人 |
| 空金额 | 按 0 计算 |
| 输出精度 | 两位小数，`HALF_UP` |

## 边界

预测金额是当前活跃商机在预计成交日期上的滚动视图，不伪装成历史时点快照。实际金额使用终态时间；
若需要“当时预测值与后来实际值”的严格准确率回测，仍应增加按日版本快照。

测试见 `docs/20-CRM-Delivery/testing/sales-forecast/`。
