# 销售预测

## 差异来源

- 原型页面：`PG-0272 销售预测分析`、`PG-0273 销售预测报表`；
- Story：`US-RPT-001`、`OPP-009`；
- 差异：`GAP-RPT-001`、`GAP-RPT-002`。

## 本次完成

- 新增销售预测汇总 API 和活跃商机分页 API；
- 统计使用预计成交日期 `crm_business.deal_time`，不再误用创建时间；
- 只统计 `end_status IS NULL` 的活跃商机；
- 使用商机当前阶段配置的 `percent` 计算概率金额；
- 按请求时间粒度补齐无数据区间，并统一金额为两位小数；
- 前端销售漏斗页新增销售预测页签；
- 展示预计金额、概率金额、商机数和可分页商机明细；
- 明细返回并展示当前阶段赢单率，便于核对公式；
- 补齐中、英、阿三语文案。

## 指标契约

| 指标 | 口径 |
|---|---|
| 活跃商机数 | 预计成交日期在筛选范围、未结束、未删除的商机数 |
| 预计成交金额 | `SUM(crm_business.total_price)` |
| 概率金额 | `SUM(total_price * crm_business_status.percent / 100)` |
| 业务时间 | `crm_business.deal_time` |
| 组织范围 | 请求负责人，或请求部门及子部门负责人 |
| 空金额/概率 | 按 0 计算 |
| 输出精度 | 两位小数，`HALF_UP` |

## 边界

当前阶段赢单率是可配置的实时值，没有保存历史概率快照。因此回看历史区间会按
当前阶段配置重算，不宣称是当时的预测快照。若需要预测准确率回测，应新增按日/按
版本快照，而不是从现表推测历史状态。

测试见 `docs/20-CRM-Delivery/testing/sales-forecast/`。
