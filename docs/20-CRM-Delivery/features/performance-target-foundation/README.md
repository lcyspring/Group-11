# CRM-FEATURE-015：业绩目标数据基础

更新日期：2026-07-14

## 需求来源

- 原型 `PG-0380`～`PG-0382`：个人、部门、公司业绩目标；
- 原型 `PG-0287`、`PG-0289`～`PG-0292`：成交金额、回款金额、跟进次数、新增客户、新增商机目标完成度；
- `US-MKT-001`：维护个人、部门和公司业绩目标；
- `US-RPT-005`：按月评估五类目标完成度；
- `GAP-MKT-001`、`GAP-RPT-001/002`。

## 原型对照结论

原型目标表同时展示年度、季度和 12 个月数值。年度、季度和月度如果分别保存，
会产生三套可互相冲突的数据。本次以月度值作为唯一事实源：

- 一条数据库记录对应一个范围、年度、月份和指标；
- 季度目标固定汇总相应三个月；
- 年度目标固定汇总 12 个月；
- 后续目标完成率统计直接消费同一月度事实源。

目标范围为公司、部门、个人；目标指标为成交金额、回款金额、跟进次数、新增客户、
新增商机。金额允许两位小数，三类计数目标必须为非负整数。

## 本次实现

- 新增租户隔离表 `crm_performance_target`；
- 新增年度目标保存、删除和按范围/年度查询 API；
- 一次保存严格要求 12 个月，服务端新增缺失月份并更新已有月份；
- 公司范围固定使用 `scopeId=0`，部门和人员通过 System API 校验有效性；
- 数据库 CHECK 约束保护范围、年度、月份、指标和非负值；
- 生成式有效唯一键保护同一租户下目标不重复，并允许逻辑删除后重新设置；
- 并发首次设置冲突转成明确业务错误，整个年度事务回滚；
- 查看复用 `crm:statistics-performance:query`；维护和删除使用独立权限；
- 迁移只创建写权限，不自动向既有角色授权；
- 中、英、阿三语权限名称随迁移写入菜单国际化表；
- Podman MySQL 初始化脚本显式执行新迁移。
