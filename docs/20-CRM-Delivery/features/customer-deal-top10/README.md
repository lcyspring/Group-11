# 客户成交金额 TOP10

## 差异来源

- 原型页面：`PG-0328 客户成交TOP10`；
- Story：`US-RPT-009`；
- 客户需求点：`CUS-019`；
- 差异：`GAP-RPT-001`、`GAP-RPT-002`。

## 本次完成

- 新增客户成交金额 TOP10 后端统计接口；
- 以审批通过合同为成交依据，按合同下单时间筛选；
- 按当前客户负责人/部门范围实施组织数据过滤；
- 按成交金额降序、客户 ID 升序稳定排序并限制前 10 名；
- 返回客户、审批合同数和成交金额；
- 客户统计页新增横向柱状图和明细表；
- 客户名称支持钻取现有客户详情路由；
- 补齐中、英、阿三语文案。

## 指标契约

| 指标 | 口径 |
|---|---|
| 成交客户 | 筛选期内至少有一份审批通过合同的有效客户 |
| 成交金额 | `SUM(crm_contract.total_price)` |
| 合同数 | 审批通过且未删除的合同数量 |
| 业务时间 | `crm_contract.order_date` |
| 组织范围 | 客户当前负责人属于请求用户/部门及子部门负责人范围 |
| 排序 | 成交金额降序；同额时客户 ID 升序 |
| 输出数量 | 最多 10 个客户 |

## 边界

当前实现按客户当前负责人划分组织范围，没有合同签署时的负责人历史快照。因此历史
报表随客户转移可能改变组织归属；如需历史归属不变，应新增合同负责人快照或明确采用
合同负责人字段。

测试见 `docs/20-CRM-Delivery/testing/customer-deal-top10/`。
