# CRM-FEATURE-016：五类业绩目标完成度

更新日期：2026-07-14

## 需求来源

- 原型 `PG-0287`、`PG-0289`～`PG-0292`：成交金额、回款金额、跟进次数、新增客户、新增商机；
- `US-RPT-005`：按月评估目标完成度；
- `GAP-RPT-001/002`、`GAP-MKT-001`；
- 前置能力 `CRM-FEATURE-015`。

## 本次实现

- 新增统一目标完成度 API，一次返回年度精确汇总和 12 个月明细；
- 支持公司、部门和个人三种目标范围；
- 公司范围必须选择父部门为 0 的组织根，部门范围不能同时选择人员，个人范围必须选择人员；
- 只接受完整自然年，阻止用部分年度实际值对比全年目标；
- 目标值、实际值使用十进制字符串返回，避免浏览器 JSON Number 丢失大额精度；
- 月度与年度完成率由后端统一按 `实际值 / 目标值 × 100` 计算并保留两位小数；
- 目标为 0 时完成率返回空值，前端显示 `--`；
- 员工业绩页面新增“业绩目标完成度”标签页；
- 前端可切换五类指标，展示年度摘要、目标/实际柱状图、完成率折线和月度明细；
- 中、英、阿三语文案齐备。

## 实际值口径

| 指标 | 来源 | 业务时间 | 范围/状态 |
|---|---|---|---|
| 成交金额 | `crm_contract.total_price` | `order_date` | 审批通过、合同负责人 |
| 回款金额 | `crm_receivable.price` | `return_time` | 审批通过、回款负责人 |
| 跟进次数 | `crm_follow_up_record` | `create_time` | 客户类跟进、创建人 |
| 新增客户 | `crm_customer` | `create_time` | 当前客户负责人 |
| 新增商机 | `crm_business` | `create_time` | 当前商机负责人 |

全部查询排除逻辑删除记录。负责人发生转移后，新增客户/商机历史会按当前负责人归属；
这是沿用现有排行榜口径。是否需要“创建时负责人快照”属于后续指标治理决策，不在本批
硬编码另一套归属规则。

## 权限与边界

完成度查询复用 `crm:statistics-performance:query`。前端当前按现有筛选器展示部门或个人
目标；公司目标已由后端支持，后续目标维护页面应提供显式公司范围入口。目标分解合计
规则和历史负责人快照仍待业务签署。

本批发现并关闭测试基础设施缺陷 `TEST-BUG-007`，独立日志位于
该问题已作为历史构建兼容缺陷关闭，当前 Node 22 容器入口不再使用该实验参数。
