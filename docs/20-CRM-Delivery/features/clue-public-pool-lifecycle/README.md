# CRM-FEATURE-042：公共线索完整状态机

- 分支：`develop`
- 状态：已实现
- 配置：`mitedtsm.crm.pool-policy.clue`

公共线索使用显式 `OWNED/PUBLIC` 状态，不再用“负责人为空”隐式推断。创建时未指定负责人
的线索直接进入公共池；在管线索可以填写原因手工退回，超过 YAML 配置天数未跟进的线索由
租户感知、集群加锁的定时任务自动迁移。已转换线索保持终态，永不进入公共池。

## 状态与动作

| 当前状态 | 动作 | 结果 |
|---|---|---|
| 新建且负责人为空 | 创建 | PUBLIC，写入首次入池事件 |
| OWNED | 负责人填写原因退回 | PUBLIC，清空全部对象团队权限 |
| OWNED | 超期未跟进 | PUBLIC，记录自动迁移原因 |
| PUBLIC | 用户单条或批量领取 | OWNED，恢复 OWNER 权限并计入当日额度 |
| PUBLIC | 具备分配能力的人员批量分配 | OWNED，恢复 OWNER 权限但不占自助领取额度 |

公共状态禁止编辑、删除、转移、转客户、写跟进和维护团队。详情读取只对显式 PUBLIC、未转换
且负责人为空的线索开放；其他无对象权限线索保持拒绝。

## 配额与并发

- 每日自助领取额度、重复领取冷却、最大持有量和批量上限均来自 YAML；
- 领取和分配按线索编号排序并逐行加锁，同一线索并发只有一个请求成功；
- 同一负责人容量使用租户和用户唯一守卫行串行化；守卫由单条 UPSERT 直接取得排他锁；
- 所有增加持有量的命令使用 `READ_COMMITTED`，等待守卫后重新统计最新已提交数量；
- 创建、转移、自助领取和主管分配统一执行最大持有量检查。

真实 MySQL 验收在已有 99 条未转换线索时并发分配两条公共线索，结果为一条成功、一条返回
`1020001007`，最终持有量严格为 YAML 上限 100。修复前发现的快照穿透和锁升级死锁均有
独立 Bug 日志。

## 数据与管理端

迁移新增归属事件、每日领取计数和负责人容量守卫表，为线索补充负责人取得时间、入池时间、
前负责人、原因编码、当前入池原因详情和累计次数，并增加状态 CHECK、查询索引、三语言菜单和
四个动作权限。手工原因在公共分页和详情原样回显，领取或分配后清空本轮原因快照。

管理端提供公共线索筛选、分页、详情、单条/批量领取和主管分配；原线索列表及详情提供填写
原因后退回入口。创建表单允许负责人留空，中文、英文和阿拉伯文同步。
