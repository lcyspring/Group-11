# CRM 客服工单最小闭环

## 范围

- 对应：GAP-WO-001、ADR-010、US-WO-001/002/004/006；
- 来源：客户、商机或合同，客户必填且来源客户必须一致；
- 编号：`W-YYYYMM-NNNN`，Redis 按月递增；
- 状态：待处理 → 处理中 → 已完结；处理中 → 已退回 → 创建人重提；
- 权限：默认只查询本人创建或本人处理，`query-all` 可查看租户全部；
- 创建人仅可修改本人待处理/已退回工单；处理人仅可操作本人负责工单；
- 完结必须有解决方案，退回必须有原因；
- 每次创建、修改和状态操作写入不可混淆的轨迹；
- 分派/重提、退回、完结发送配置化站内信；
- Web 提供列表、筛选、创建/修改、处理动作和轨迹详情；
- SQL 提供两张业务表、字典、六项权限、三语菜单和三类通知模板。

## 不在本批范围

SLA、签到、自动升级、处理组、抄送和重新分派属于 ADR-011，不在最小闭环中。
