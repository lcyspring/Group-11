# CRM-FEATURE-020：合同审批修订与重提闭环

更新日期：2026-07-14

## 需求来源

- `GAP-APR-002`、`GAP-ORD-003`、`GAP-CTR-002`；
- 已批准的 `ADR-006`；
- 原型中的草稿、审批中、通过、驳回、取消及重提动作。

## 本次实现

- 合同只允许在草稿、审核不通过、已取消状态编辑，审批中和已通过禁止编辑；
- 修订审核不通过或已取消合同后显式回到草稿，再由原提交入口发起新流程实例；
- 列表和详情页为审核不通过/已取消合同提供“修订”入口；
- 提交审批增加合同 WRITE 对象权限，并用数据库行锁串行化同一合同的重复提交；
- BPM 取消值 `4` 正确映射为 CRM 取消值 `40`；
- 审批回调携带流程实例 ID，仅当前流程可回写；重复终态、旧流程和乱序事件幂等忽略；
- 终态回写使用“当前流程 + 审批中”的条件更新，避免并发终态相互覆盖；
- 仅从未提交审批的新草稿允许删除，已产生流程历史的合同必须保留；
- 商机转换合同在后续修订中继续强制保持来源商机、客户和负责人不变；
- 中、英、阿三语增加“修订”文案。

## 状态动作

| 状态 | 编辑 | 提交 | 查看流程 | 删除 |
|---|---|---|---|---|
| 草稿且无流程历史 | 是 | 是 | 否 | 是 |
| 修订后的草稿 | 是 | 是 | 是（上一流程） | 否 |
| 审批中 | 否 | 否 | 是 | 否 |
| 审批通过 | 否 | 否 | 是 | 否 |
| 审核不通过/已取消 | 修订后回草稿 | 先修订 | 是 | 否 |

## 边界

本批不新增“驳回到发起人”和“最终否决”两个独立业务状态，继续遵循 ADR-006 的 MVP
状态集。流程定义发布、审批候选人和撤回权限由 BPM 配置负责；合同应用服务只维护业务
状态、当前流程关联及幂等回写。
