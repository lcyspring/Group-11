# 客户公海完整规则闭环

- 分支：`develop`
- 状态：已实现
- 决策：ADR-014

本功能把客户从“负责人是否为空”的隐式二态升级为在管、公海、垃圾池三态数据模型中的
在管/公海闭环。客户入池、领取和主管分配均使用行锁与条件更新，联系人负责人、OWNER
权限、客户状态和不可变归属轨迹处于同一事务。

已实现规则：

- 未跟进、未成交回收期限由租户配置覆盖，缺省值来自显式 YAML；
- 金牌/VIP 客户使用双倍保护期；
- 活跃商机、未完结销售单据、成交客户和锁定客户阻止回收；
- 自助领取具有每日原子额度和同人同客户重复领取冷却；
- 主管分配不占自助额度且不受重复领取冷却影响；
- 自动任务使用 YAML cron、Asia/Shanghai 时区、游标分页和 Redis 分布式锁；
- 自动回收默认批量和最大安全批量均来自 YAML，租户覆盖、后端扫描和前端输入共享同一上限；
- 入池通知只在事务提交后发送；
- 公海列表按入池时间倒序展示前负责人、时间、原因和累计次数；
- 租户配置页可维护回收、保护、领取、批量和提醒规则，支持中文、英文和阿拉伯文；
- 旧 A/B/C 客户等级幂等迁移为原型 1～5 五级模型。

本闭环不把垃圾池和公共线索冒充已完成；两者继续作为 ADR-014 后续状态机实现。
