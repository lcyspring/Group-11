# 客户成交状态统计

## 差异来源

- 原型：`PG-0321 客户状态`；
- Story：`US-RPT-009`；
- 差异：`GAP-RPT-001`、`GAP-RPT-002`。

现有画像统计已有级别、来源、行业和地区分析，但没有客户成交状态分布入口。

## 本次完成

- 新增客户成交状态统计 API：
  `GET /crm/statistics-portrait/get-customer-deal-status-summary`；
- 按负责人范围、客户创建时间和逻辑删除状态聚合 `deal_status`；
- Service 复用画像统计的部门/人员范围解析；
- 前端画像页新增“客户状态分析”页签、环形图和明细表；
- 前端计算已成交/未成交客户占比；
- 补齐中、英、阿三语文案；
- 新增 Service 范围测试和 Mapper SQL 契约测试。

## 指标契约

| 项目 | 口径 |
|---|---|
| 数据源 | `crm_customer` |
| 业务时间 | 客户 `create_time` |
| 状态字段 | `deal_status` |
| 数据范围 | 请求人员或请求部门及子部门人员 |
| 删除规则 | `deleted = 0` |
| 维度 | 已成交、未成交 |
| 刷新方式 | 页面查询时实时聚合 |

## 原型边界

当前 CRM 客户聚合只有布尔成交状态，不能无依据推导原型中的“潜在、意向、成交、
流失”四态。本次完成可由现有数据直接证明的二态统计基线；四态生命周期仍需先为
客户聚合增加正式状态模型和迁移规则，不能在统计 SQL 中硬编码猜测。

测试证据见 `docs/20-CRM-Delivery/testing/customer-deal-status-statistics/`。
