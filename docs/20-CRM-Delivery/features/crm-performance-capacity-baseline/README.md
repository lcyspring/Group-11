# CRM 性能与容量基线

本功能为 CRM 建立可重复执行的本地单节点性能基线，而不是用单次 `curl` 延迟代替容量证据。
验收覆盖客户列表、商机列表、客户趋势、合同排行和销售漏斗五类只读请求：

- 命令行只接收一个 KDL 路径，账号、数据范围、并发和阈值均显式配置；
- 每个场景先预热，再以固定请求量和并发数正式采样；
- 同时判断 HTTP 状态和业务响应码，记录 p50、p95、p99、错误率与吞吐；
- 报告包含主机、容器和数据库表规模，便于结果解释；
- 任一场景超过 KDL 阈值时脚本以非零状态退出。

该基线用于发现相对回退和支撑本地验收，不承诺生产流量容量。生产部署仍需使用目标硬件、目标数据量
和目标网络重新压测。
