# CRM-FEATURE-013：商机阶段销售漏斗

更新日期：2026-07-14

## 差异来源

- 原型：`PG-0274 销售漏斗分析`、`PG-0275 销售漏斗报表`；
- Story：`OPP-008`、`US-RPT-002`；
- 差异：`GAP-RPT-001`、`GAP-RPT-002`。

## 原有问题

原页面的漏斗图只有“客户、商机、赢单”三段，所谓“客户视角”使用固定
`60/40/20` 作为图形值；下方“阶段”表实际按 `end_status` 展示赢单、输单、
无效，并不是配置的商机阶段。页面无法选择状态组，也没有阶段金额和相邻转化率。

## 本次实现

- 新增状态组必填的阶段漏斗 API；
- 前端从当前用户可用状态组中默认选择第一项，并支持切换；
- SQL 返回全部已配置阶段的最后阶段存量，包含活跃和三种终态商机；
- Service 从最后阶段向前累加，形成单调的“已到达本阶段”漏斗；
- 返回每阶段累计商机数、累计金额及相对上一阶段的转化率；
- 终态结果由独立分布图展示，不把赢单、输单、无效伪装成线性阶段；
- 首阶段有数据时为 `100.00%`，零分母统一为 `0.00%`；
- 时间口径沿用当前统计页：商机 `create_time` 落在筛选范围；
- 组织口径沿用统计服务：指定负责人，或指定部门及子部门负责人；
- 补齐中、英、阿三语文案，并将原非法 UTF-8 组件重建为规范 UTF-8。

## 指标示例

原始最后阶段存量为阶段一 10、阶段二 4、阶段三 1 时：

| 阶段 | 已到达数量 | 相邻转化率 |
|---|---:|---:|
| 阶段一 | 15 | 100.00% |
| 阶段二 | 5 | 33.33% |
| 阶段三 | 1 | 20.00% |

## 边界

现表只保存商机当前阶段，没有阶段变更快照。本实现表达“当前管道中已到达阶段的
累计视图”，不宣称可以重建任意历史时点的真实阶段迁移。历史转化趋势需要新增
不可变阶段事件或日快照后另行实现。
