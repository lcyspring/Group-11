# OA 工作报告（日/周/月报）

## 完成能力

- 日报、周报、月报统一业务模型，服务端按报告日期自动归一周期；
- 同一作者、报告类型和周期只能存在一份有效报告；
- 草稿支持创建、修改、删除，提交后形成不可修改、不可删除的报告事实；
- 报告包含已完成工作、未完成工作、下一步计划、问题与协助、接收人和附件；
- 作者可查看“我的报告”，接收人可查看“收到的报告”，其他用户不能读取详情；
- 接收人去空、去重并排除作者，提交前校验用户有效性；
- 管理端提供列表、筛选、草稿维护、提交和详情，操作项统一使用弹出菜单；
- 中、英、阿三语言及显式菜单权限已配置。

## 数据与接口

- 表：`crm_work_report`；
- API：`/admin-api/crm/work-report/*`；
- 菜单权限：`crm:work-report:query/create/update/delete`；
- 迁移：`database/migrations/new-oa-work-report.sql`。

工作报告不进入 BPM 审批。它是作者提交给指定接收人的不可变办公报告，不把“已读”或“认可”伪造成审批结论。
