# 客户城市、省份与国家分布

## 差异来源

- 原型页面：`PG-0324 所在城市`、`PG-0326 按省份分布`、`PG-0327 按国家分布`；
- Story：`US-RPT-009`；
- 差异：`GAP-RPT-001`、`GAP-RPT-002`。

## 既有问题

既有页面只有一个省份地图，接口说明却称为城市分析。SQL 按客户保存的最细
`area_id` 分组，Service 转换成省份后直接返回；同一省份的多个城市/区县会形成
重复省份项，地图和值均可能失真。

## 本次完成

- 区域统计统一保留原始 SQL，再按 `AreaTypeEnum` 转换为城市、省份或国家；
- 转换后按目标区域 ID 二次聚合客户数和成交客户数；
- 结果按客户数降序、区域 ID 升序稳定输出；
- 新增城市与国家统计 API，保留既有省份 API 兼容性；
- 客户画像页新增城市、省份、国家三个独立页签；
- 城市/国家展示全部客户与成交客户双系列图及占比表；
- 城市、省份、国家表格支持点击区域名称打开客户明细分页；
- 明细严格复用当前统计时间、负责人和区域下级范围，并可进入客户详情；
- 明细接口同时要求画像统计权限与客户查询权限；
- 补齐中、英、阿三语文案。

## 指标契约

| 指标 | 口径 |
|---|---|
| 客户数 | 创建时间在筛选范围、当前负责人在组织范围、未删除且有区域的客户数 |
| 成交客户数 | 上述客户中 `deal_status = true` 的数量 |
| 城市/省份/国家 | 从客户保存的 `area_id` 沿 `AreaUtils` 父链归并到目标层级 |
| 未识别区域 | 目标层级无法解析时统一聚合为“未知” |
| 排序 | 客户数降序；同量按区域 ID 升序 |

## 边界

- 当前成交口径沿用客户布尔成交字段，不是合同发生时间口径；
- 组织范围采用客户当前负责人，历史客户转移会改变过去区间的归属。

测试见 `docs/20-CRM-Delivery/testing/customer-region-distribution/`。
