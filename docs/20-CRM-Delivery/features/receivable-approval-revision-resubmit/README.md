# CRM-FEATURE-021：回款审批修订重提与生效口径

更新日期：2026-07-14

## 需求来源

- `GAP-FIN-001/002`、`GAP-APR-002`；
- 已接受的合同生效边界和 `ADR-006` 状态集；
- 原型中的回款审批、驳回/取消修订和重提动作。

## 本次实现

- 只有审批通过合同可创建回款计划或回款；
- 创建回款计划、回款均要求来源合同 WRITE 对象权限；
- 回款只允许草稿、审核不通过、已取消状态编辑，审批中和已通过禁止编辑；
- 审核不通过/已取消回款修订保存后显式回草稿，再发起新审批流程；
- 提交审批要求回款 WRITE 对象权限；锁定合同和回款，防止额度校验、编辑和提交并发串写；
- 回调校验当前流程实例 ID，重复、旧流程、乱序和并发终态不会覆盖当前状态；
- 只有从未提交且未关联计划的新草稿可删除，保留所有审批历史；
- 回款和回款计划金额最小为 `0.01`，后端拒绝零数和负数；
- “已回款金额”只累计审批通过回款，草稿和审批中不提前计入；
- 前端列表、合同详情子列表和回款详情同步限制动作并提供“修订”入口；
- 回款计划合同下拉禁用未审批通过合同，中、英、阿三语补充文案。

## 明确边界

本批的 `APPROVE` 只表示 CRM 回款业务审批通过，用于 CRM 销售回款展示；不声明已经发生
银行资金到账、会计入账、核销或 ERP/Pay 同步。ADR-007/008 仍保持 Open，退款、冲销、
币种、计划总额和会计对账规则不得由本实现反推。
