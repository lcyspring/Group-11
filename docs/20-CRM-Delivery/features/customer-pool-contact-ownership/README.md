# 客户公海联系人归属闭环

- 分支：`develop`
- 状态：子闭环完成

客户进入公海时，客户下的联系人同步清空负责人和 OWNER 权限；客户被领取或
分配时，联系人同步获得新负责人和唯一 OWNER 权限。团队成员的 READ/WRITE 权限保留，
不会因负责人变更被整体删除。

归属字段、对象权限、公海记录和客户负责人变更处于同一事务。入池时先同步
联系人，再删除客户 OWNER 权限，避免中途失去操作联系人的授权。

兼容迁移只修复同租户、关联有效客户且能由客户负责人唯一推导的历史联系人，
并在事务内统一负责人字段、唯一 OWNER 权限。孤立联系人不自动绑定或伪造归属。

该子闭环不定义公海超期天数、保护期、容量、垃圾池和定时任务频率，这些仍属
ADR-014 规则包。
