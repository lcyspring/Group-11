# CRM-FEATURE-014：销售阶段漏斗明细钻取

更新日期：2026-07-14

## 需求来源

- `OPP-008`：阶段数量、金额和转化率可核对；
- `US-RPT-002`：销售管理者能够定位转化瓶颈；
- `GAP-RPT-001/002`：统计口径、权限和明细一致。

## 本次实现

- 阶段漏斗表新增“详情”操作；
- 普通阶段明细包含：当前阶段排序不小于所选阶段的活跃商机，以及赢单商机；
- 赢单阶段明细只包含赢单；
- 状态组和阶段 ID 均由服务端校验，阶段必须属于请求状态组；
- 明细继续按负责人，或部门及子部门负责人范围过滤；
- 创建时间范围与阶段汇总完全一致；
- 明细服务端分页并稳定按预计成交日期、商机 ID 排序；
- 钻取 API 单独要求 `crm:business:query`，前端对无权限用户隐藏入口；
- 弹窗展示商机、客户、金额、预计成交日期、当前阶段和负责人，可进入商机/客户详情；
- 主筛选条件变化时关闭旧明细，防止展示上一次筛选结果。

## 口径一致性

阶段汇总的数量是“所选阶段及后续活跃商机 + 赢单”。普通阶段钻取使用相同的
状态排序、活跃/赢单条件、状态组、创建时间和用户范围，因此分页总数应与汇总数量
相同。赢单行使用独立查询，分页总数应与赢单数量相同。

## 边界

本功能不扩大现有统计用户范围，也不替代 CRM 对象级团队权限模型。统计查询沿用
销售统计的组织范围，详情入口额外要求商机查看权限。双角色、跨部门的真实数据集
负向验收仍属于 `STAT-SEC-001` 后续证据。
