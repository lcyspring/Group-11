# CRM-FEATURE-064：固定种子大规模演示数据

生成和部署保持独立：生成器只读取独立 KDL，将固定 seed、规模和日期范围渲染为 SQL、cleanup、
manifest 与 checksum，不连接数据库。`deploy.sh` 只消费已生成 manifest；默认保留已有数据，显式
`replace` 使用 `tenant-crm-demo` 清理范围先清空租户 CRM/OA 演示业务事实，再插入完整新数据集。

当前覆盖客户、多联系人、客户公海、线索/公共线索、客户归属轨迹、商机、产品、合同、回款计划、
回款、发票、报销、退款、营销、客户关怀、工单、OA 日程和 OA 任务。批次名、手机号、不可投递邮箱、
日期、金额、生命周期和状态分布均由 seed 确定；相同 KDL 可重新生成相同业务键和分布。

当前安全上限为客户 10000、商机 50000、工单 100000。生成产物位于被 Git 忽略的
`database/generated/`，仓库只提交生成器、模板、示例配置和测试证据。

基准管理员、四名销售及财务/客服/客户成功账号共同承担业务对象：基准管理员同时拥有负责人样本和
只读参与样本，演示销售形成下属负责人样本，客户与线索分别生成明确的公海样本。因此“我负责的”、
“我参与的”、“下属负责”和“公海”不会再依赖用户手工造数。审批历史只生成可解释的导入状态，
不伪造 Flowable 运行实例。

2026-07-18 本机按示例规模完成一次 `replace`：客户160（公海32）、联系人240、线索160（公共32）、
商机180、合同72、回款计划144、工单160；旧客户和旧商机残留均为0。替换后运行配置已恢复
`mysql.dataset_mode: preserve`。
