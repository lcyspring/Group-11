# CRM-FEATURE-064：固定种子大规模演示数据

生成和部署保持独立：生成器只读取独立 YAML，将固定 seed、规模和日期范围渲染为 SQL、cleanup、
manifest 与 checksum，不连接数据库。`deploy.sh` 只消费已生成 manifest；默认保留已有数据，显式
`replace` 还必须开启 cleanup 和持久数据确认。

第一版覆盖客户、首联系人、客户归属轨迹、商机、工单和工单创建轨迹。批次名、手机号、不可投递
邮箱、日期、金额、生命周期和状态分布均由 seed 确定；相同 YAML 可重新生成相同业务键和分布。

当前安全上限为客户 10000、商机 50000、工单 100000。生成产物位于被 Git 忽略的
`database/generated/`，仓库只提交生成器、模板、示例配置和测试证据。

2026-07-18 已由 `deploy.sh` 在本机执行一次显式替换：1000 客户、1000 联系人、1000 归属轨迹、
1600 商机、2000 工单和 2000 工单轨迹均已写入，六类计数和三类孤儿引用完成对账。替换后配置恢复
`preserve`，普通部署不会重复覆盖。

财务合同、回款计划、回款、发票、报销、退款和 BPM 实例属于下一扩展批，不在本版中伪造。
