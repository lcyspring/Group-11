# CRM 群发送达与打开分析

分支：`develop`。对应 `MKT-003`。

## 功能

- 短信：提供商处理中、送达、失败、送达时间和送达率；
- 邮件：提供商处理中、服务商接受、失败、接受时间、首次打开时间和打开率；
- 任务结果弹窗支持手工刷新，定时群发任务每轮也批量同步提供商日志；
- 收件人对象范围沿用群发创建人/审核人/发送人门禁；
- 邮件打开令牌为 128 位随机值，不在管理 API 中返回；
- 无效令牌不泄露是否存在，重复打开不改变首次打开时间。

## 指标口径

- 短信送达率 = 短信送达数 / 已创建短信提供商日志数；
- 邮件打开率 = 首次打开数 / 邮件服务商接受数；
- `record-only` 记录为未跟踪，不进入任何真实送达或打开分子；
- 邮件服务商接受不能表述为最终送达。

## 配置

运行 YAML 的 `crm_marketing` 提供：

- `provider_mode`：`record-only` 或 `system`；
- `tracking_enabled`：是否追加邮件像素；
- `public_base_url`：像素公开基地址；生产邮件必须为外部 HTTPS；
- `delivery_sync_batch_size`：每轮最大状态回收数。

命令行仍只接收 YAML 路径，运行脚本将字段转换为 Spring 环境配置。
