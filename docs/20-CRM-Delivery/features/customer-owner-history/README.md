# CRM 客户归属记录实现说明

完成日期：2026-07-13

## 需求与源码结论

- `US-CUS-006` 要求转移负责人时保留变更前后归属。
- `PR-CUS-003` 覆盖客户公海、归属和协作。
- 差异项 `GAP-CUS-001` 要求在复用现有客户与数据权限能力的基础上完善。
- 现有转移弹窗和客户详情页已经完整，本次没有重做。原实现仅对入公海、领取/分配写单一 `owner_user_id`，无法表达初始归属和转移前后负责人，也没有历史查询入口。

## 实现范围

| 事件 | 类型 | 原负责人 | 新负责人 | 兼容字段 `owner_user_id` |
|---|---:|---|---|---|
| 创建、线索转换、有负责人的导入 | 3 | 空 | 负责人 | 新负责人 |
| 转移 | 4 | 原负责人 | 新负责人 | 新负责人 |
| 进入公海 | 1 | 原负责人 | 空 | 原负责人 |
| 领取或分配 | 2 | 空 | 新负责人 | 新负责人 |

- 扩展 `crm_customer_owner_record`，新增可空的 `previous_owner_user_id` 和 `new_owner_user_id`。
- 保留 `owner_user_id` 及类型 1/2 的语义，原 CRM 公海统计 SQL 无需改动。
- 迁移脚本可重复执行，会回填旧类型 1/2 记录可确定的前/后负责人，不伪造旧数据的操作人或初始分配历史。
- 客户创建、转移、入公海、领取/分配的业务写操作和历史写入位于同一事务。
- 新增 `GET /crm/customer/owner-record-list?customerId=...`，Controller 要求 `crm:customer:query`，Service 额外执行客户 `READ` 数据权限校验。
- 操作人复用 `BaseDO.creator`，Controller 批量获取原负责人、新负责人和操作人名称，无 N+1 用户查询。

## 前端完善

- 客户详情增加懒加载“归属记录”页签，展示类型、原负责人、新负责人、操作人和时间。
- 用户被删除或名称无法回显时保留 `#用户 ID`；负责人为空时显示“无/未分配”。
- 领取、分配成功后，如页签已挂载则立即刷新，不会显示过期历史。
- 增加中文、英文、阿拉伯文文案。

## 边界

- 历史记录是客户归属审计数据，删除客户时不级联删除。
- 未给历史客户批量伪造“初始分配”；上线前已存在的非公海归属只会从后续变更开始记录。
