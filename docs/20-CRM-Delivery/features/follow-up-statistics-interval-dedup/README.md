# CRM 跟进客户跨周期去重

更新日期：2026-07-14

## 来源

`GAP-RPT-001` 要求统计指标逐项明确来源、业务时间和刷新口径。员工客户分析默认按周展示，因此“跟进客户数”必须在每个展示区间内按客户去重。

## 原实现问题

Mapper 每天执行 `COUNT(DISTINCT biz_id)`，Service 对周/月/季度/年的每日结果直接求和。同一客户在区间内跨多天被跟进时，每天都会贡献一次，导致跟进客户数膨胀；跟进次数求和则没有这个问题。

## 实现

- Mapper 返回按“日期 + 客户 ID”去重后的轻量明细，而不是不可再次去重的每日计数。
- 每条明细保留当日首次跟进时间，供 Service 匹配目标展示区间。
- Service 在每个时间区间内对客户 ID 执行 `distinct` 后计数。
- 跟进次数仍沿用每日记录数求和，保持原有含义。

## 性能边界

查询结果最多为请求范围内“活跃客户数 × 活跃天数”，少于逐条跟进明细；后续大数据量压测如仍超出指标，应引入受控统计读模型，而不是回退为错误的每日计数求和。

测试见 `docs/20-CRM-Delivery/testing/follow-up-statistics-interval-dedup/`，缺陷见 `STAT-BUG-021` 独立日志。
