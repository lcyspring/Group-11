# CRM-FEATURE-052：营销活动可维护闭环

## 能力

- 活动列表支持编码、名称、负责人和状态组合筛选；
- 负责人来自当前租户真实系统用户，新建默认当前登录用户，不使用固定账号；
- 草稿可维护编码、名称、负责人、时间、预算、目标线索/客户数、说明及既有关联关系；
- 生命周期为草稿、进行中、已锁定、已终止、已完成；
- 草稿支持编辑、启动和受控删除，进行中支持锁定、终止和完成，锁定后支持终止和完成；
- 终止和完成必须填写原因或总结，已启动活动保留审计记录而不允许物理删除。

## 权限与一致性

- 查询、维护和删除分别使用 `crm:marketing-campaign:query/update/delete`；
- 普通用户只能维护本人负责的活动，CRM 管理员可在显式权限下管理全量；
- 后端校验负责人是有效系统用户、结束时间晚于开始时间、编码唯一和状态迁移合法；
- 删除活动与关联对象在同一事务中完成，非草稿删除会被拒绝。

## 交付

- 后端草稿删除命令、权限和菜单迁移；
- 管理端完整列表、表单和生命周期操作；
- 中、英、阿三语文案；
- Ubuntu 26.04 容器专项测试、全量 CRM 测试、覆盖率和生产构建证据。

真实运行验收使用 `podman/verify-crm-campaign.sh`，命令行只接收 YAML 路径；账号、租户和服务地址
均来自显式配置。验收会创建、读取、编辑并删除一条临时草稿，不保留测试业务数据。
