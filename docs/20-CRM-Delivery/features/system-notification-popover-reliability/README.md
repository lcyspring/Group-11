# 顶部站内信弹层可靠性

日期：2026-07-15

分支：`develop`

## 已实现

- 首次进入管理端后异步预取未读消息列表，不阻塞其他页面内容；
- 点击通知按钮立即打开弹层，未完成时展示加载骨架；
- 最近一次成功列表作为即时缓存，弹层每次打开再后台刷新；
- 快速点击、悬停和聚焦共享同一个进行中的请求，不产生并发风暴；
- 请求失败保留已有缓存，无缓存时显示可重试状态；
- 明确区分“查看列表”和“标记已读”，打开弹层不再错误清除未读红点；
- 通知按钮覆盖头部完整命中区域并支持键盘焦点；
- 跳转全部消息时主动关闭弹层；
- 轮询定时器随组件生命周期释放，避免布局切换后重复轮询。

## 状态流

```text
布局挂载 → 后台预取 → 最近成功缓存
                  ↓
点击/键盘打开 → 立即显示缓存或骨架 → 单请求后台刷新 → 更新列表
                                      ↘ 失败时保留缓存或显示重试
```

未读数量继续由 `/system/notify-message/get-unread-count` 决定；只有消息详情或“标记已读”业务命令可以改变
服务端已读状态。
