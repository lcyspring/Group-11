# CRM 客户名称长度校验实现说明

完成日期：2026-07-14

## 依据

- `GAP-CUS-001` 要求完成客户字段映射和边界规则。
- 原型 UAT `TC-CUS-003` 明确要求客户名称最多 100 个字符，101 个字符应提示错误。
- 现有数据库列容量不是业务长度契约，不能替代 API 参数校验。

## 实现

- `CrmCustomerSaveReqVO.name` 增加 `@Size(max = 100)`，同时约束创建和更新 API。
- 客户表单输入框增加 `maxlength=100` 和字数显示。
- 前端表单规则增加 100 字符上限，提供中文、英文、阿拉伯文提示。
- 自动化测试覆盖 100 字符边界成功和 101 字符失败。

## 设计说明

前端约束用于即时反馈，后端 Bean Validation 是不可绕过的最终规则。二者使用同一数值 100，避免通过原始 HTTP 请求绕过页面后写入超长业务数据。
