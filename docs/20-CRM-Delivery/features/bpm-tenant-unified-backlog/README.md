# BPM 租户治理与 CRM 统一待办

## 完成内容

- CRM 待办中心新增“流程审批待办”，直接复用 BPM `todo-page`，不复制流程引擎和任务表；
- 只有具备 `bpm:task:query` 权限的用户才显示入口、统计数量并请求数据；
- 待办表格支持流程名称、摘要、发起人、当前任务、创建时间、分页和审批详情跳转；
- 待办、已办、管理分页以及任务按 ID、流程实例和 ID 集合查询统一增加当前 Flowable 租户；
- 流程定义、部署和模型按 ID、集合及 Deployment 查询统一增加当前租户；
- BPMN 模型、模型 XML 原文读写和模型清理先完成租户校验，清理范围限制在当前租户；
- 模型列表和排序接口补齐显式查询、更新权限；
- 中文、英文和阿拉伯文文案已同步。

## 运行边界

统一待办只聚合当前登录人的 Flowable 审批任务。它不创建示例审批、不复制已办数据，也不把
CRM 工单伪装成流程任务。当前租户没有任务时返回空列表是合法结果。

构建、测试和覆盖率由 `podman/config/verify-bpm-governance-ubuntu-26.04.yaml` 显式控制，
命令行只传配置文件路径。
