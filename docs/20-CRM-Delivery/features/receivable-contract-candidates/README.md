# 可回款合同候选

日期：2026-07-16。分支：`develop`。

## 能力

- 新建回款时无需先选择客户，默认加载当前用户可写且审批通过的合同；
- 普通用户仅返回本人负责或直接拥有 WRITE 的合同，CRM 管理员返回全部审批通过合同；
- 候选展示客户、合同编号、合同名称和剩余可回款金额；
- 选择合同自动带出客户，并继续由服务端强制客户—合同一致；
- 剩余金额扣除审批中和已审批回款，已无额度的合同保留展示但禁止选择；
- 客户仍可作为可选过滤条件，选择客户后只显示该客户候选。

## API

`GET /crm/contract/receivable-candidates?customerId={可选}`

接口需要 `crm:receivable:create` 功能权限，对象范围不会因为组织只读权限而扩大。
