# CRM-FEATURE-035：客户 360 退款/冲销净回款闭环

## 完成内容

- 客户 360 接入 CRM 退款/冲销真实业务表，不再把审批通过回款毛额当成最终净回款；
- 新增退款/冲销记录数、审批通过退款/冲销额和净回款额；
- 待回款改为“审批通过合同额－净回款额”，退款通过后恢复对应待回款；
- 草稿、审批中、驳回和取消记录只进入记录数，不改变财务净额；
- 历史异常数据出现退款大于回款时，净回款按零保护，待回款不显示负数；
- 客户详情新增只读退款/冲销 Tab，按退款对象权限展示我负责、我参与和下属负责三类明细；
- 三语界面显式区分毛回款、退款/冲销和净回款，避免同一个“回款额”标签承载多个口径。

## 金额口径

```text
审批通过回款毛额 = SUM(审批状态 = 20 的回款金额)
审批通过退款/冲销额 = SUM(审批状态 = 20 的退款/冲销金额)
净回款额 = MAX(回款毛额 - 退款/冲销额, 0)
待回款额 = MAX(审批通过合同额 - 净回款额, 0)
```

两种反向业务类型都减少已确认回款：客户退款表示资金退回，业务冲销表示原回款确认被撤销。审批中的金额
仍由退款业务自身占用额度，但在审批通过前不改变客户 360 已确认财务事实。

## 权限边界

摘要继续以客户 READ 权限为入口，提供该客户的财务聚合。退款明细入口额外要求
`crm:receivable-refund:query`，并继续执行退款对象场景权限；客户 READ 不自动扩大为全部退款对象明细权限。

## 当前边界

本功能关闭退款真源到客户 360 的聚合断点。报销/费用仍没有冻结的 CRM 真源、分类和会计归属，不能用退款
记录或固定零值代替；该部分继续等待 ADR-017。
