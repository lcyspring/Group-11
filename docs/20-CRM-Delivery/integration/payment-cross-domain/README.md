# 支付与 CRM/OA/营销跨域集成规范

状态：规范已完成，业务自动集成尚未实现。分支：`develop`。日期：2026-07-18。

本目录定义支付流水与 CRM 财务、OA 借款/报销、营销付费场景之间的所有权、映射、回调、权限和
审计边界。它是后续实现门禁，不把现有 `pay_refund` 误认成 CRM 退款，也不声称当前已自动生成
CRM 回款或执行 OA 付款。

| 文件 | 内容 |
|---|---|
| `01-Boundary-and-Flows.md` | 聚合所有权、允许流程、禁止依赖和当前不集成项 |
| `02-API-and-Errors.md` | 现有接口、目标回调契约、错误码和幂等响应 |
| `03-Data-Dictionary.md` | 映射键、状态、金额、审计字段和建议关联表 |
| `04-Security-and-Compensation.md` | 签名、重放、租户隔离、重试、补偿和对账 |
| `05-KDL-and-Review.md` | 显式 KDL 草案、分阶段开关和评审清单 |

实现原则：业务域决定“为什么收/退/付”，Pay 域决定“通过哪个渠道怎样结算”。支付成功只能成为业务
事实的结算证据，不能绕过 CRM/BPM 审批；业务审批通过也不能伪造渠道已经成功。
