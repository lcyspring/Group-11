# ADR-020：CRM 合同到 ERP 履约订单边界

状态：Accepted。日期：2026-07-16。

## 决策

1. CRM 合同是唯一销售协议；不新增重复的 CRM 销售订单。ERP 销售订单是下游履约聚合。
2. 仅审批通过且有效签署的合同可由授权用户显式创建 ERP 订单，不在审批回调内自动创建。
3. CRM 通过 ERP 应用 API 单向依赖 ERP；ERP 不依赖 CRM，Controller 不跨域访问 Mapper。
4. CRM 与 ERP 主数据保持独立，通过租户内一对一客户/产品映射连接。
5. 首次创建冻结完整请求和哈希。CRM 合同唯一键、请求唯一键与 ERP 来源唯一键、请求唯一键共同处理重复点击、超时和回写中断。
6. ERP 外部来源订单的销售事实不可编辑或删除；仓储履约操作不受影响。
7. ERP 状态通过事件回传，显式刷新是补偿机制，不以轮询代替事件。
8. 币种、精度、舍入、容差和来源标识全部由 YAML 冻结版本管理。

## 结果

该边界避免 CRM 与 ERP 双向耦合、重复销售事实和跨库事务。代价是需要维护映射、补偿状态和冻结快照，但这些对象都有明确唯一键、状态机和可观测字段。
