# ADR-004：商机赢单转合同

## 决策

- 只有 `endStatus=WIN` 的商机允许创建来源合同；
- 转换由有合同创建权限的用户显式触发，赢单动作本身不自动建合同；
- 自动预填客户、商机、负责人、商机产品、折扣和金额，用户补充合同日期与签约信息；
- 一个商机只有一个有效的“转换来源合同”；重复点击和并发请求返回同一合同 ID；
- 转换只创建 CRM 合同草稿，不提交审批、不创建 ERP 订单；
- 已有历史关联合同时视为已转换，优先复用最早的有效合同，不擅自合并历史重复数据。

## 失败规则

| 场景 | 结果 |
|---|---|
| 商机未赢单 | 拒绝创建 |
| 请求客户/负责人被篡改 | 服务端覆盖为商机客户和负责人 |
| 重复点击 | 返回已有合同 ID |
| 并发首次转换 | 数据库唯一键保留一份，另一请求返回胜出合同 ID |
| 合同被逻辑删除 | 允许重新显式转换 |

合同被驳回、取消或作废后的重新转换规则，先复用同一合同修订/重提，不创建第二份来源合同。
