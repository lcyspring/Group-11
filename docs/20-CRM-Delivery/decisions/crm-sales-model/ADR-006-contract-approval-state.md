# ADR-006：合同审批状态语义

## MVP 决策

合同业务域拥有 `auditStatus`，Flowable 拥有流程实例和任务状态。合同复用
`crm-contract-audit`，监听器只能通过合同应用服务幂等回写，页面不把单个任务状态当合同终态。

| 合同状态 | 含义 | MVP 动作 |
|---|---|---|
| `DRAFT(0)` | 未提交草稿 | 可编辑、删除、提交 |
| `PROCESS(10)` | 流程审批中，尚未业务生效 | 审批任务处理；合同不进入回款生效口径 |
| `APPROVE(20)` | 流程通过，合同生效 | 允许建立回款计划和回款，不直接创建 ERP 订单 |
| `REJECT(30)` | 审核不通过 | 保留流程历史；修订保存后回草稿并可重提 |
| `CANCEL(40)` | 流程取消/撤回 | 保留流程历史；修订保存后回草稿并可重提 |

## 明确不合并的语义

“驳回到发起人”和“最终否决”在现有 CRM 枚举中都只能落到 `REJECT`，MVP 不伪造两个
业务终态。需要区分时扩展业务状态和流程回写契约，并迁移历史数据。

`CRM-FEATURE-020` 已完成修订/重提、重复回调、旧流程回调和乱序终态保护。流程发起人的
主动撤回入口、真实 Flowable 并发压力验证，以及“驳回到发起人/最终否决”的状态拆分仍
属于后续批次。
