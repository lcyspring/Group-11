# ADR-012：默认角色—动作矩阵

状态：Accepted（MVP）

日期：2026-07-15

实施分支：`develop`

## 决策

角色仅作为权限模板。Server 只校验 `crm:<对象>:<动作>` 功能权限字符和统一数据范围，不按“销售主管”、
“财务”等角色名称硬编码业务分支。一个用户持有多个角色时，功能权限和组织数据范围分别取并集，之后仍
必须通过对象权限、状态机和字段校验。

## 默认职责矩阵

`Q/C/U/D/E` 分别表示查询、创建、修改、删除、导出；实际配置使用完整权限字符。

| 角色模板 | 主要功能许可 | 默认组织范围 | 对象动作约束 |
|---|---|---|---|
| 销售人员 | 线索、客户、联系人、商机、合同、回款计划 `Q/C/U/D`；按职责配置 `E` | SELF | WRITE/OWNER 才能修改；OWNER 才能转移和维护团队 |
| 销售主管 | 销售域 `Q/C/U/D`，可按需配置 `E/distribute` | DEPT 或 DEPT_AND_CHILD | 组织范围只读；修改、删除、转移、导出仍需直接对象授权 |
| 审批人 | BPM 待办/处理，以及相应业务对象 `query` | SELF 或业务指定部门 | 流程候选人身份本身不授予 CRM 对象权限 |
| 财务 | 合同、回款计划、回款、退款、发票 `query`，按职责配置创建、处理、开票、红冲、作废、导出 | DEPT_CUSTOM 或 ALL | 金额动作同时受对象 WRITE/OWNER 和财务状态机限制 |
| 客服 | 客户、联系人、合同只读；工单 `query/create/process`，主管另配 `assign/query-all` | SELF、DEPT 或 DEPT_AND_CHILD | `query-all` 是工单显式能力，不外溢为其他 CRM 对象权限 |
| CRM 管理员 | 全部 CRM 功能权限 | ALL | 仅配置的管理员角色可绕过对象/组织范围；租户隔离和状态约束仍有效 |

## 敏感动作

| 动作 | 功能门 | 数据门 | 说明 |
|---|---|---|---|
| 查看列表/详情/附件 | 对象 `query` | 直接 READ/WRITE/OWNER，或负责人落入组织读范围 | 附件继承父对象权限 |
| 修改/业务命令 | 对象 `update` 或专用动作 | 当前用户直接 WRITE/OWNER | 组织范围不授写 |
| 删除 | 对象 `delete` | 当前用户直接 WRITE/OWNER，并满足状态约束 | 已生效财务事实不得以删除冲销 |
| 转移/分配 | `update/distribute/assign` | 当前用户直接 OWNER 或明确管理员例外 | 目标用户仍需有效且同租户 |
| 导出 | 对象 `export` | 每个对象均为当前用户直接 WRITE/OWNER | READ、部门范围、下属 OWNER 均不足以导出 |
| 审批处理 | BPM 任务权限 | 当前任务候选/受理人 + 对象可读 | 流程身份不升级对象写权限 |
| 金额/签署/开票 | 专用功能字符 | 直接 WRITE/OWNER + 状态机 | Server 必须重算和校验，不依赖前端隐藏按钮 |

## 兼容与演进

现有数据库角色无需改名。管理员代码从 Java 枚举引用迁到显式配置；其他角色继续在系统角色菜单中配置。
新增业务动作时必须先增加独立权限字符或复用已有动作并记录理由，禁止新增角色名判断。
