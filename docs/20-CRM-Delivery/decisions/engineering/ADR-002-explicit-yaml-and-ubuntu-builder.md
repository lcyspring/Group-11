# ADR-002：显式 YAML 配置与 Ubuntu 26.04 构建环境

状态：已接受，待实施

日期：2026-07-13

## 决策

1. 构建、部署、停止和归档脚本统一读取显式 YAML 配置。
2. 命令行参数只允许指定 YAML 配置文件路径；操作类型由所调用脚本决定，不再用命令行参数或环境变量维护另一套配置。
3. 端口、Pod/容器/卷名称、镜像及版本、超时、代理策略、构建范围和资源限制全部进入 YAML。
4. 敏感值放在不提交的本地 YAML 或 Secret 挂载文件中；仓库只提供字段完整的示例，不提交真实凭据。
5. Server、InitService 和管理 Web 后续统一在专用 Ubuntu 26.04 builder 镜像中构建。
6. builder 镜像固定 JDK、Maven、Node 和 pnpm 工具链，并输出可审计版本信息；宿主机只运行 rootless Podman、挂载源码/缓存并接收产物。
7. 不恢复无软链接文件系统的源码暂存兼容路径。

## 验收条件

- 所有入口在参数数量不是 1 时失败，并提示唯一的 YAML 路径用法。
- YAML 缺字段、类型错误、端口冲突、明文敏感值策略违规时，在任何状态变更前失败。
- 同一提交在干净环境中通过 Ubuntu 26.04 builder 产出 Server JAR、InitService JAR 和 Web `dist-prod`。
- 构建产物记录源码提交、builder 镜像摘要和工具链版本。
- 现有 rootless Podman 健康检查、持久卷复用和 CRM 统计回归继续通过。

## 实施边界

当前兼容债务清理先独立提交。YAML 加载器和 builder 属于后续专门批次，避免与 CRM 业务修复及脚本删减混合。
