# ADR-021：客服工单处理组、派单与协作权限

状态：Accepted。日期：2026-07-16。

## 决策

1. 处理组是租户内独立聚合，不用部门或角色名称替代。组保存负责人、成员、支持类型、状态和稳定排序。
2. 自动派单仅使用启用且支持工单类型的处理组；候选人按待处理/处理中工单数升序，再按组排序、
   组 ID 和用户 ID 稳定决胜，不使用随机数或内存轮询游标。
3. 无候选人的行为由 YAML 决定：进入处理组未分配池，或要求显式处理人。默认进入未分配池。
4. 领取使用“待处理 + 指定组 + 处理人为空”的数据库原子更新；失败表示状态或归属已经变化。
5. 手工派单分为三层：无组工单创建人仅能在自己和下属范围内选择；处理组负责人可在本组派单；
   跨组或全组候选必须具有 `crm:work-order:assign-all`。
6. 读取权由创建人、处理人、处理组负责人、未分配池所属组成员、抄送人和 `query-all` 合并；
   抄送人和普通组成员不会因此获得开始、退回、完结或改派权限。
7. 抄送人保存为独立关系并去重。新抄送人收到一次通知，完结时创建人和抄送人按用户去重通知。
8. 描述、解决方案和抄送上限使用 YAML，不在 Controller、页面或脚本中形成新的隐式业务阈值。

## 结果

该模型把组织结构、处理组、对象参与关系和敏感动作权限分开，支持稳定自动派单和并发领取，
同时避免把部门、角色或前端可见性误当成服务端授权。移动签到、地理围栏、SLA、节假日、暂停和
自动升级继续由独立时空规则 ADR 决定，不混入本决策。
