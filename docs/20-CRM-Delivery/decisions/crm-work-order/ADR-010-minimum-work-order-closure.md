# ADR-010：客服工单最小闭环

- 状态：Accepted (MVP)
- 范围：CRM P0 最小客服工单

## 决策

1. 工单必须关联 CRM 客户，可选关联商机或合同作为来源；来源只保存业务类型和
   编号，不复制销售单据金额或状态。
2. 工单由 CRM 用户创建并指定一个处理人。MVP 不引入处理组、自动分派或区域路由。
3. 生命周期固定为：待处理 → 处理中 → 已完结；处理人可以退回，创建人可以修改后
   重新提交：`待处理/处理中 → 已退回 → 待处理`。
4. 创建人只可编辑自己创建且处于待处理或已退回的工单；处理人只可开始、退回和
   完结自己负责的工单；完结必须填写解决方案。
5. 工单保存描述、解决方案、附件 URL、客户、来源、处理人和每次状态操作轨迹。
6. 创建、重新提交、退回和完结通过现有站内信底座通知相关 CRM 用户。通知模板是
   数据库配置，不在 Java 代码中硬编码消息正文。
7. 查询默认只返回当前用户创建或负责的工单；具有 `crm:work-order:query-all` 的
   用户可以查看租户内全部工单。所有查询继续受租户过滤和功能权限保护。

## 不在本 ADR 范围

- SLA 起止、暂停、超时阈值和节假日；
- 移动签到、地理围栏、补签；
- 处理组、自动升级、跨租户工单；
- 工单独立 BPM 审批。MVP 状态流转记录在 CRM 操作轨迹中。

## 验收证据

- 后端状态机负向测试覆盖非法跳转、越权操作和完结缺少解决方案；
- 列表数据范围测试覆盖本人/处理人和 `query-all`；
- 前端可创建、处理、退回、重提、完结并查看轨迹；
- 数据库迁移可在空库和已有租户库幂等执行。
