# ADR-011：工单移动签到与 SLA 服务治理规则

- 状态：已关闭
- 分支：`develop`
- 日期：2026-07-16

## 决策

1. 工单可保存服务地点纬度、经度、围栏半径和“完结前必须签到”标识；未配置完整坐标时不允许发起移动签到。
2. 移动签到只接受当前处理人操作，服务端按 Haversine 距离计算，且定位精度不能超过 YAML 的安全上限；不采集后台轨迹，成功签到写入不可变记录并在待处理工单上原子开始处理。
3. SLA 策略按优先级从数据库选择，YAML 的 `default-policy-code` 只作为回退；首次响应、自动升级和解决截止时间均按工作分钟计算。
4. YAML 定义时区、工作日上下班时间、周末策略和调度锁；数据库工作日历可覆盖周末（调休）或普通工作日（节假日）。
5. 暂停 SLA 只允许当前处理人操作；恢复时将暂停秒数平移到三个截止时间，并写入工单轨迹。调度器使用 Redisson 锁，自动升级和逾期均写入轨迹，不改变工单业务状态。

## 安全与隐私边界

- 坐标只在工单参与人或 `query-all` 权限范围内返回。
- 服务器不信任前端距离值，距离、精度和处理人权限全部服务端复核。
- 浏览器拒绝定位、精度超限或越过围栏时不落成功签到记录。

## 配置

配置前缀为 `mitedtsm.crm.work-order-governance`，构建和运行环境必须通过 YAML 提供，不在 Java 或前端硬编码阈值。
