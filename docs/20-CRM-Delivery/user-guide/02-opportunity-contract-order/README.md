# 商机、合同与履约验收

## 产品与商机

1. 在 `产品管理`（`/crm/product`）准备有效产品，在 `系统配置 → 产品分类配置`核对分类；
2. 在 `商机管理`（`/crm/business`）为测试客户新建商机，添加产品、数量、折扣、税率和币种；
3. 推进阶段时必须填写说明；非法回退、越权推进和并发旧版本提交应被拒绝；
4. 创建报价版本，锁定后价格行不可修改；重开生成可编辑版本，旧版本保持可读；
5. 标记输单时填写原因；标记赢单后执行转合同，重复点击只返回同一合同。

## 合同与 ERP 履约

1. 打开 `合同管理`（`/crm/contract`），核对客户、签约联系人、产品成交快照、金额和来源商机；
2. 提交合同审批，在 `待办事项`（`/crm/backlog`）查看、通过或驳回；驳回后修订重提；
3. 上传受保护附件并执行签署，已签署版本不可覆盖；验证签署作废和动作轨迹；
4. 创建补充协议，核对基线/目标快照、金额下限、依据附件锁定和独立审批；通过后原合同投影更新；
5. 在 `ERP 履约映射`（`/crm/erp-fulfillment-mapping`）配置客户/产品映射；对已签署合同创建履约单，
   重试保持幂等，刷新后可看到 ERP 状态回传。

完整的前端点击步骤、失败路径和验收记录见 [ERP 订单域前端人工验收](02-ERP订单人工验收.md)。

## 预期证据

- 报价版本号、锁定时间、币种/税率和不可变产品快照；
- 商机到合同、合同到 ERP 单据的来源 ID 与幂等键；
- 审批业务状态和流程状态均可解释，不出现“无流程实例”；
- 下架或改价后的产品不改变历史合同成交内容。
