# ERP 订单域前端人工验收

## 前置数据

1. `CRM → 产品管理`：准备至少一个启用产品；
2. `ERP → 产品 → 产品信息`：准备对应 ERP 产品；
3. `CRM → 客户管理`与`ERP → 销售 → 客户信息`：各准备一个客户；
4. `CRM → ERP 履约映射`：分别在“客户映射”“产品映射”页签建立一对一映射；
5. CRM 合同必须包含产品行、审批状态为“通过”，并在合同详情“合同生命周期”上传签署件后完成签署。

## 主闭环

1. 打开 `CRM → 合同管理`，进入上述合同详情；
2. 切换“ERP 履约”页签；
3. 核对客户映射、全部产品映射均显示“已映射”，页面没有阻塞原因；
4. 点击“创建履约订单”并确认；
5. 页面应显示 ERP 订单号、请求 ID、尝试次数、源币种→ERP 币种、汇率和金额；
6. 刷新浏览器或重新进入页签，不得生成第二张订单；创建按钮应不可重复创建；
7. 进入 `ERP → 销售 → 销售订单`，按刚才显示的 ERP 订单号筛选并打开详情；
8. 核对客户、产品、数量、六位金额和 CRM 合同快照一致；该外部来源订单的客户、产品和价格行不得
   任意编辑，也不得删除；
9. 在销售订单操作菜单点击“审核”；
10. 回到 CRM 合同“ERP 履约”页签点击“刷新履约状态”，ERP 审核状态应更新为已审核；
11. 可继续在 `ERP → 销售 → 销售出库`选择该订单创建出库单，审核后再次回 CRM 刷新，出库进度应增加；
12. 如需退货，使用 `ERP → 销售 → 销售退货`选择该订单，审核后 CRM 退货进度应增加。

## 必测失败路径

| 操作 | 预期 |
|---|---|
| 未审批合同进入 ERP 履约 | 显示合同未审批阻塞，创建按钮不可用 |
| 已审批但未签署 | 显示未签署阻塞 |
| 删除客户映射 | 显示客户未映射，不创建 ERP 订单 |
| 删除任一产品映射 | 明确指出未映射产品 |
| 对同一合同重复进入/刷新 | 始终关联同一 ERP 订单，不重复创建 |
| 尝试编辑或删除 CRM 来源 ERP 订单 | 前端禁用或后端拒绝 |
| ERP 审核、出库、退货后不主动刷新 | CRM 可通过“刷新履约状态”补偿同步 |

## 验收记录

记录 CRM 合同编号、ERP 订单号、请求 ID、映射关系、源金额/ERP 金额、审核状态、出库/退货数量以及
失败路径截图。不要用 ERP 手工新增订单代替 CRM 合同创建履约订单的主闭环。
