# 线索、客户与联系人验收

## 主链

1. 打开 `CRM 系统 → 线索管理`（`/crm/clue`），新建 `UAT-日期-线索转换`，填写联系人姓名和手机；
2. 编辑该线索并新增任务/通话/短信活动，确认列表和详情即时刷新；
3. 将另一条线索转入 `公共线索`（`/crm/clue/public`），分别验证领取和管理员分配；
4. 将主线索转换为客户，确认客户、首联系人和活动轨迹一次生成，重复转换不会产生第二份数据；
5. 在 `客户管理`（`/crm/customer`）打开详情，核对联系人、商机、合同/销售单据、回款计划、回款、
   退款、发票、附件和工单关系入口；
6. 新建同名或同手机号客户，预期出现疑似重复确认，取消后不写入；
7. 在 `联系人管理`（`/crm/contact`）新增联系人，手机号为空或同客户重复时应被服务端拒绝；设置首
   联系人后，客户列表和详情一致回显。

## 公海与垃圾池

- `客户公海`（`/crm/customer/pool`）：验证转入反馈、领取额度、保护期、冷却期、分配和联系人归属同步；
- `客户垃圾池`（`/crm/customer/garbage`）：转入后列表应立即出现；恢复公海后可再次转入；永久删除
  仅在无受保护引用且有权限时显示并成功；
- `系统配置 → 客户公海配置/客户限制配置`：修改规则后再验证，不允许用代码常量代替配置。

## 预期证据

- 转换前后线索 ID、客户 ID、首联系人 ID 和三类活动数量；
- 公共池/公海/垃圾池操作后无需手工刷新即可看到结果；
- 无权限用户看不到管理动作，直接调用接口也返回拒绝；
- 转换后的线索业务字段只读，跟进轨迹仍可审计。
