# 营销与客户关怀验收

1. `营销活动`（`/crm/marketing-campaign`）：验证草稿、进行中、暂停、完成、取消五态，负责人来自
   真实用户选择；草稿可删，进行中先终止；
2. `竞争对手`（`/crm/competitor`）：维护优势、劣势、网站和说明，非法网址应拒绝；
3. `短信邮件群发`（`/crm/marketing-outreach`）：选择客群、模板、计划时间和渠道，保存草稿、提交
   独立审核、通过后发送，失败收件人可重试；
4. 验证客户同意、退订、频控和审核拒绝，未同意或已退订客户不得进入发送名单；
5. `客户关怀`（`/crm/customer-care`）：配置生日、节假日、成交后回访规则，手工触发和调度触发
   均生成记录；重复调度不得重复发送；
6. 本地 `record-only/system` Provider 只证明任务、审核、结果和重试闭环，不宣称外部短信/邮件送达。

预期：每个收件人保留不可变结果快照；活动、群发和关怀的删除/终止按钮符合状态和权限，操作列
在常用宽度下可点击。
