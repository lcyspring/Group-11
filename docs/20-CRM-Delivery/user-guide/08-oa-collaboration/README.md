# OA 协作功能测试指南

## 客户拜访

入口：`审批管理 → 客户拜访`。

1. 点击“发起客户拜访”，选择当前有权维护的客户。
2. 联系人下拉框只显示该客户联系人；填写未来的计划起止时间、地点和目的。
3. 提交后在列表点击“审批进度”，确认流程实例和当前审批人可见。
4. 审批人在“我的待办”通过申请。
5. 返回客户拜访列表，已通过记录显示“登记拜访结果”。
6. 填写实际起止时间、拜访结果、下次联系时间及可选附件并提交。
7. 打开客户详情的跟进记录，确认新增一条“上门拜访”，内容和下次联系时间与拜访结果一致。

异常检查：

- 选择其他客户的联系人会被服务端拒绝；
- 未审批通过不能登记结果；
- 实际结束时间不能晚于当前时间或早于实际开始时间；
- 已登记结果的记录不会再次显示操作入口，重复请求也不会生成第二条跟进。

## 请假、出差与借款

- 请假：`审批管理 → 请假管理`，验证工作日、余额预占和审批结果。
- 出差：`审批管理 → 出差管理`，验证小时级起止时间、附件和结束后关联报销。
- 借款：`审批管理 → 借款管理`，验证岗位额度、超额升级审批、未还阻断和分次还款。

## 工作报告

进入 `办公协作 → 工作报告`：

1. 在“我的报告”中新建日报、周报或月报，选择报告日期；系统自动计算对应周期；
2. 填写已完成工作和下一步计划，可补充未完成工作、问题与协助及附件；
3. 至少选择一名接收人，作者本人不会作为接收人重复保存；
4. 草稿可以编辑或删除；点击提交并确认后，报告不可再修改或删除；
5. 接收人在“收到的报告”中查看详情，非作者和非接收人无权读取。

同一报告类型和周期只能创建一份。例如同一周内选择不同日期，仍属于同一份周报。

如果提交时提示“流程定义不存在”，管理员应检查部署 KDL 是否开启 BPM 自动恢复，并运行对应
`bpm-provision-*-local.kdl`，不要在数据库中手工伪造流程记录。
