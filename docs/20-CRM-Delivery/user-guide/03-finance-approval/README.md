# 财务与审批验收

## 前置数据

准备一份审批通过或已签署合同，金额建议 `10000.00`；准备两期回款计划各 `5000.00`。所有日期
使用页面日期组件，不输入自由文本。

## 回款计划与回款

1. `回款计划`（`/crm/receivable-plan`）：新增两期计划，合计超过合同金额应失败；
2. `回款管理`（`/crm/receivable`）：合同候选只能显示当前用户有权且可回款的真实合同，不应只有
   演示项“我是合同”；
3. 新增正金额回款并提交审批，在 `待办事项` 查看；通过后客户 360 和统计只按生效口径汇总；
4. 驳回或取消后允许修订重提，审批中不可直接修改/删除，重复回调不重复入账；
5. 过期未收计划在待办和逾期口径中出现，处理后无需手工刷新。

## 发票、退款与报销

- `发票管理`（`/crm/invoice`）：草稿 CRUD、正式开具、部分/全部红冲、蓝/红票作废、税号唯一和
  合同金额守恒；本地结果是 CRM 开票台账，不代表税务平台成功；
- `退款/冲销管理`（`/crm/receivable-refund`）：选择已生效回款，新建退款或业务冲销，超可退金额
  应失败；验证提交、驳回修订、取消重提、通过和不可变轨迹；
- `报销管理`（`/crm/reimbursement`）：费用分类、明细、附件、申请人/部门快照、可选客户/合同关系，
  后端重新汇总总额；验证草稿 CRUD 与审批全路径。

## 审批预期

- 表单按钮和审批标签为中文，确认/取消不出现英文默认文本；
- 查看审批必须进入对应流程实例，缺失历史实例显示可恢复状态，不抛未处理 Promise；
- 操作按钮不应被不可点击的省略号截断；窄屏时进入明确下拉菜单；
- 每次审批后业务列表、待办数量和详情立即刷新。
