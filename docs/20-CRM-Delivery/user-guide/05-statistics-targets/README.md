# 经营统计与目标验收

## 页面

- 客户分析 `/crm/statistics/customer`
- 排行榜 `/crm/statistics/ranking`
- 员工业绩 `/crm/statistics/performance`
- 客户画像 `/crm/statistics/portrait`
- 销售漏斗 `/crm/statistics/funnel`
- 工单统计 `/crm/statistics/work-order`

## 验收步骤

1. 先用同一筛选条件记录列表事实数，再打开统计页核对金额、数量和时间范围；
2. 切换本人、部门、部门及下级、公司范围，验证数据随授权范围变化；
3. 验证同比/环比无基期、零基期、正负增长和两位小数；
4. 点击 TOP、地区、漏斗和工单图表钻取，明细合计必须等于图表值；
5. 在员工业绩维护月度/年度五类目标，验证个人、部门、公司三层及精确完成率；
6. 新增/修改业务数据后使用显式刷新，统计应基于实时真源而非旧前端缓存；
7. 另一租户和无权限角色不得通过页面、导出或直接 API 获得数据。

保存证据时记录指标名、来源页面、筛选条件、业务时间、统计值、明细值和用户范围；这七项缺一
不能判定统计正确。
