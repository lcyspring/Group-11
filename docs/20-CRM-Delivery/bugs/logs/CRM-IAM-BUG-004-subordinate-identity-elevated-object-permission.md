# CRM-IAM-BUG-004：下属身份错误提升对象写入和导出权限

状态：Fixed

日期：2026-07-15

分支：`develop`

## 现象

旧详情切面会遍历当前用户全部下属：只要下属对对象持有 WRITE/OWNER，上级即通过同等级校验；导出也将
下属用户 ID 加入可导出主体。该逻辑绕开角色部门范围，使“是组织上下级”隐式等价于对象写入/导出授权。

## 风险

- 上级即使角色数据范围为 SELF，也可能修改下属对象；
- 导出功能权限叠加下属 OWNER 后可批量带出金额和客户资料；
- 权限来源无法从当前用户对象团队记录审计还原。

## 修复

- 下属列表与平台组织可读负责人集合取交集；
- 组织和上下级关系只授予 READ；
- WRITE/OWNER 只认当前用户直接对象授权；
- 导出逐对象只认当前用户直接 WRITE/OWNER，管理员例外由 YAML 配置。

专项测试验证下属 OWNER 不再满足当前用户导出条件。
