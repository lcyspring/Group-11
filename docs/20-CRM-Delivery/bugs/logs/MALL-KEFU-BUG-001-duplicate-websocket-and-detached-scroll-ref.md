# MALL-KEFU-BUG-001：客服重复 WebSocket 与消息列表空引用

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

进入商城客服后，Firefox 报告无法建立 `/infra/ws` 连接；快速切换会话时同时出现
`clientHeight` 的空引用异常，消息列表交互被中断。

## 根因

- `useWebSocket` 默认立即连接，页面挂载后又显式调用 `open()`，同一页面会创建重复连接；Nginx
  访问日志记录到同一令牌在数秒内连续多次返回 `101`；
- WebSocket 地址用字符串替换拼接，未明确处理同源 HTTP/HTTPS、令牌编码和无令牌状态；
- 消息请求返回、`nextTick` 和会话切换可能交错，原代码使用非空断言读取已经卸载的滚动容器；
- 防抖滚动回调在组件销毁后仍可能执行。

## 修复

- WebSocket 改为 `immediate: false`，只在挂载完成后建立一条连接；
- 新增显式 URL 构造器，根据当前页面或配置域名生成 `ws/wss` 地址并编码令牌；
- 为异步消息请求增加会话版本号，旧请求不能再写入新会话；
- 消息列表和会员信息侧栏的滚动 DOM 引用都改为生命周期安全检查，历史消息加载后等待 DOM 更新；
- 两个滚动组件销毁时均取消防抖回调。

## 验证

Ubuntu 26.04 容器专项测试 3/3 通过，行、分支、函数覆盖率均为 100%，客服目录 ESLint
零警告。部署前主机握手验证 `/infra/ws` 返回 `101 Switching Protocols`。
