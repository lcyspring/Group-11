# NFR-OBS-BUG-001：阈值失败后诊断摘要路径失效

## 现象

诊断脚本正确返回 `FAIL` 并打印摘要路径，但随后错误清理钩子删除了摘要目录。压缩包仍在，控制台
给出的 `summary=` 路径却无法直接读取，增加故障处置时间。

## 根因

脚本把所有非零退出都视为未完成异常；SLI 越界也是预期的非零结果，因此触发了同一 `ERR` 清理钩子。

## 修复

移除失败时删除已采集目录的钩子。无论诊断结论为 PASS 还是 FAIL，都保留摘要目录和压缩包；本机
输出目录仍由 Git ignore 隔离。

## 验证

- 人为把磁盘可用阈值设为 100%，当前值 72% 时脚本返回非零；
- `SUMMARY.md` 与 `.tar.gz` 均保留且可直接读取；
- 失败项明确为 `disk_free=72%`；
- 演练前后 7/7 服务容器持续运行，没有重启。
