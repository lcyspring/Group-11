# CRM-FILE-BUG-001：合同附件公共 URL 绕过对象权限

- 发现/关闭日期：2026-07-15
- 级别：P0 / 数据安全
- 关联：GAP-IAM-003、GAP-SEC-003

## 现象与根因

合同生命周期直接返回 `fileUrl`，Infra 下载路由又是匿名访问。知道或获得 URL 的用户无需合同 READ 权限
即可读取附件；创建附件还信任客户端 URL、类型和大小，能绑定任意外部或公开文件。

## 修复关键

- YAML 定义公共路由禁止读取的目录与 CRM 合同上传目录；
- 新增合同专用上传/下载端点，分别要求合同 WRITE/READ；
- 创建时核对受管文件元数据和合同专属目录，禁止跨合同绑定，响应移除原始 URL；
- 公共路由在访问存储前返回 404，并规范化斜线和路径片段防止前缀绕过；
- 前端统一使用认证 API 上传和下载。

## 验证与遗留

合同生命周期 11/11、Infra 文件专项 28/28、CRM 231/231、专项 ESLint 与 production build 均通过。
历史公开附件需要物理迁移；仅隐藏 URL 不能让外部已保存的旧链接失效。
