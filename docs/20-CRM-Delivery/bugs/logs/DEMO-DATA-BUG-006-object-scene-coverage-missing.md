# DEMO-DATA-BUG-006：演示对象场景分类不足

## 现象

超级管理员能查询大量客户，但“我负责的”几乎为空，数据主要集中在“我参与的/全部”；客户、联系人、
线索与后续财务对象无法同时验证不同场景筛选。

## 根因

生成器把核心对象全部分配给四个演示销售，没有给 YAML 中的基准管理员生成负责人对象，也没有明确
构造其只读参与关系。总量断言通过，但业务分类覆盖不足。

## 修复

- 基准管理员与四名演示销售共同承担客户、联系人、线索、商机、产品、合同、计划和回款；
- 对非管理员负责的在管对象增加管理员只读参与关系；
- 独立生成客户公海和公共线索，公海客户不关联活动商机、合同、跟进或工单；
- 增加“管理员负责、管理员参与、演示销售负责、公海”四类数据断言。

## 验证

生成 SQL 的可执行断言同时核对四类场景。2026-07-18 替换后，基准管理员分页 API 返回：在管客户
128、“我负责的”25、“下属负责”103、客户公海32；数据库只读参与关系103条。
