# BPM-APPROVAL-BUG-004：审批成功后通过按钮残留

- 发现日期：2026-07-19
- 分支：`develop`
- 状态：Closed

## 现象

在多个业务审批详情中完成“通过”后，页面未立即移除当前任务的操作按钮；部分情况下需要再次点击“通过”或手动刷新，才能看到最新审批状态。

## 根因

公共审批组件在接口成功后仅通知父页面异步重新查询，当前运行任务仍保留在本地状态。刷新响应较慢或短暂返回旧任务快照时，已完成任务会继续显示为可审批状态。

## 修复

- 审批通过或拒绝成功后，立即将当前任务标记为已处理并移除操作按钮；
- 对当前页面生命周期内已处理的任务编号建立防回挂保护，忽略迟到刷新返回的同一旧任务；
- 后续节点产生的新任务使用不同任务编号，仍会正常显示操作按钮。

## 回归结果

- `deno task verify:bpm-approval-refresh`：通过；
- 覆盖通过、拒绝后的即时失效，以及旧刷新响应不得恢复已处理按钮。
