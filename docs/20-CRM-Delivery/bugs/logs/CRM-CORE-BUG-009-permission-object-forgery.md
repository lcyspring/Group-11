# CRM-CORE-BUG-009：权限更新未绑定请求业务对象

发现/关闭日期：2026-07-13
级别：P0 / 安全
关联差异：`GAP-IAM-003`、`GAP-LEAD-002`

## 现象与风险

权限更新接口通过请求中的 `bizType/bizId` 做对象 OWNER 校验，但 Service 只确认 `ids` 对应权限记录数量存在，没有核对这些权限 ID 是否属于该业务对象。攻击者可提交一个自己拥有的对象通过切面，再把 `ids` 指向另一个对象的权限记录。

## 修复

- Service 查询全部权限 ID，并逐条校验实际 `bizType/bizId` 与请求完全一致。
- 不匹配统一返回 `1020007000`，不泄露目标权限细节。
- 权限 ID 列表改用 `@NotEmpty`，空数组不再形成成功的空操作。
- 已转换线索团队权限写操作额外通过线索行锁和只读状态校验。

## 验证

- 单元测试用线索权限 ID 配合伪造客户对象，确认在写 Mapper 前返回 `1020007000`。
- 真实 API 使用当前用户拥有的转换目标客户 ID 包装线索协作者权限 ID，同样返回 `1020007000`，权限级别保持 2。
- 正常对象参数在已转换线索上返回 `1020001002`；添加、编辑、移除团队成员三项均无数据变化。

## 状态

已关闭。
