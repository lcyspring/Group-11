# WEB-OBS-BUG-001：管理端默认加载百度统计外部脚本

## 现象

浏览器控制台报告无法加载：

`https://hm.baidu.com/hm.js?a1ff8825baa73c3a78eb96aa40325abc`

## 根因

公共 `Web/.env` 硬编码了演示用百度统计代码，导致所有本地、测试和生产构建都默认
向第三方域名发起请求。离线环境、内容拦截器或 CSP 会拒绝该请求。

## 修复

- 公共环境默认不配置统计代码；
- Ubuntu 26.04 构建 YAML 增加 `web.baidu_analytics_code`；
- `disabled` 明确表示关闭，只有显式配置合法代码时才注入 Vite 构建环境；
- 命令行仍只接收 YAML 配置文件路径。

## 验证

- Ubuntu 26.04 Web 生产构建成功；
- `Web/dist-prod` 和实际 Web 容器均不包含百度统计域名及旧统计代码；
- Web 热替换后 `http://127.0.0.1:8081/` 返回 200；
- Server 与 Mall 未中断。

## 说明

这条脚本加载失败本身不会导致回款接口抛出“系统异常”；同一时间出现的业务异常
由 `CRM-REC-BUG-001` 的数据库字段缺失引起，两个问题已分别修复。
