# CRM-RECEIVABLE-WRITEOFF-BUG-003：核销成功后列表状态不变且重复显示核销入口

- 发现日期：2026-07-19
- 分支：`develop`
- 状态：Closed

## 现象

回款核销成功后，回款管理列表没有展示已核销金额和剩余可核销金额；全额核销的记录仍显示“回款核销”。
再次点击只能打开空白的新建表单，无法确认先前核销结果，也无法执行冲销。

## 根因

- 回款分页响应不包含核销汇总，前端只能用“审批通过”判断是否显示核销入口；
- 核销成功后弹窗直接关闭，没有刷新主列表和核销记录；
- 页面没有核销台账及冲销交互，已实现的查询和冲销接口没有可用入口。

## 修复

- 回款分页批量聚合有效核销金额，返回已核销和剩余可核销金额，避免逐行查询；
- 主列表展示两项核销金额，全额核销后隐藏“回款核销”，改为“核销台账”；
- 新增独立核销台账弹窗，支持新增、余额提示、历史记录和冲销；
- 新增或冲销成功后立即刷新台账和主列表；
- 补齐金额、时间、来源和长度校验，并将并发重复流水号转换为明确业务错误。

## 回归结果

- CRM 全量测试 543/543 通过；
- 回款核销服务专项测试 5/5 通过；
- Web 核销与空操作菜单专项测试 2/2 通过。
