# STAT-DATA-002：合同下单日期早于客户创建日期

## 基本信息

- 发现日期：2026-07-13
- 级别：P1 数据质量/口径
- 模块：CRM / 合同 / 客户成交周期
- 状态：开放

## 现象

既有客户 16 创建时间为 2024-02-23，首次审批合同下单日期为 2024-02-07，成交周期因此为 `-16` 天。统计 SQL 如实暴露负值，但负周期不符合持续时间直觉。

## 可能原因

- 客户在合同实际下单后才补录 CRM，历史回填本身合法。
- 合同表单允许选择早于客户创建时间的下单日期，缺少业务校验。
- “成交周期”产品定义可能应使用合同录入时间或审批完成时间，而不是可回填的下单日期。

## 当前处理

- 不修改历史业务数据。
- 不对统计结果取绝对值、静默归零或排除记录，以免掩盖真实数据和口径冲突。
- 保持接口返回 `-16`，作为待决策证据。

## 待确认与修复方向

1. 产品确认成交时间取下单日期、合同创建时间还是审批通过时间。
2. 若仍取下单日期，确认是否允许补录历史合同；若不允许，在合同创建/更新增加日期下限校验。
3. 若允许补录，明确负值展示、标记异常或按 0 天处理的规则，并通过显式 YAML 业务口径配置，而不是硬编码。
4. 确认口径后补迁移/校验、单元测试和 Podman 数据闭环，再关闭本项。
