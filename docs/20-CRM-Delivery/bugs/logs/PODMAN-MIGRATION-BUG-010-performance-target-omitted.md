# PODMAN-MIGRATION-BUG-010：业绩目标迁移未进入运行清单

- 日期：2026-07-16
- 分支：`develop`
- 级别：P0 / 部署一致性
- 状态：已关闭

## 现象

已有环境可使用业绩目标表和权限，但从基础数据库启动的新环境没有执行
`new-crm-performance-target.sql`，目标列表请求可能返回 5xx，前端又把失败误显示成“未配置目标”。

## 根因

兼容迁移使用显式 manifest，但五个较早新增的 CRM 幂等迁移没有登记；运行配置测试也只检查清单内文件，
未反向检查 CRM 增量迁移是否全部进入 manifest。

## 修复关键

- 把联系人首联系人、联系人手机唯一、客户层级、客户归属历史和业绩目标五个迁移补入 manifest；
- 运行配置门禁反向扫描全部 `new-crm-*.sql`，任一遗漏立即失败；
- 业绩目标管理和完成度页面增加显式失败态与重新加载，不再把接口错误伪装成空数据；
- 增加只接受 YAML 配置路径的真实运行验收。

## 验证

- 目标表：1；维护/删除权限：2；三语权限记录：6；
- 目标列表 API：通过；完成度 API：12 个月；
- 迁移重复执行：通过；
- Ubuntu 26.04 构建与前端目标契约：通过。
