# MP-FE-BUG-003：无公众号账号时多个子页面永久转圈

## 现象

取消公众号子菜单强制跳转后，用户、标签、草稿、发布、模板和自动回复等页面一直显示加载遮罩；
其他页面还可能保留上一次账号的数据或允许对无效账号执行操作。

## 根因

十个公众号页面只监听账号选择器的 `change` 事件，并把它作为首次请求和结束加载的唯一入口。
账号为空或账号接口失败时不会产生 `change`，各页面初始化为 `loading=true` 的状态永久无法释放。

## 修复

- 共享选择器显式区分并发出 `unavailable: empty|error`；
- 十个消费页面统一监听不可用状态，复位账号、列表、分页、图表和加载遮罩；
- 账号接口失败保留重试入口，空账号保留主动配置入口；
- 修复公众号统计对 Ref 本身而非 `accountId.value` 的无效判断；
- 修复自定义菜单请求开始时错误地把加载状态设为 `false`。

## 验证

Ubuntu 26.04 专项 5/5，十个消费页面扩大 ESLint 零警告，production build 通过。最终运行
环境账号数为 0，可直接覆盖空账号事件路径。
