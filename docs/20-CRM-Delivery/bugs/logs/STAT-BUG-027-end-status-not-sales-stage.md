# STAT-BUG-027：结束状态被误当作销售阶段

## 现象

销售漏斗下方表格列名为“阶段”，数据却来自按 `crm_business.end_status` 分组的
接口，只显示赢单、输单、无效。图表所谓“客户视角”使用固定 `60/40/20` 作为
ECharts value，真实数量只拼在名称中。

## 影响

- 无法看到配置的初步接洽、需求确认、方案报价等商机阶段；
- 固定图形宽度会掩盖真实阶段比例；
- 没有阶段金额和阶段间转化率，无法满足 `OPP-008 / US-RPT-002`；
- 不同部门使用不同状态组时数据被混为一张图。

## 根因

原实现复用了结束状态统计，并用固定值维持图形顺序，没有建立状态组、状态排序和
漏斗累计口径。

## 修复关键

- 新增状态组必填的阶段漏斗接口；
- SQL 以 `crm_business_status` 为基表，合并活跃商机当前存量与赢单；
- Service 从赢单向前累计，计算已到达各阶段的数量、金额和相邻转化率；
- 输单、无效作为流失排除；
- 前端删除固定值视角，图表和表格统一使用服务端指标。

## 验证证据

- 累计算法、SQL 契约、请求校验新增 3 项测试；
- Ubuntu 26.04 CRM 88/88；
- Ubuntu 26.04 Web production build 成功。

## 清理与遗留风险

测试使用 Mapper/Service stub，没有写入数据库。当前没有不可变阶段历史，因此只
表示当前管道累计漏斗；历史阶段转化必须另建事件或快照。
