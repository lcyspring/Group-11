# CRM-WO-BUG-005：处理组更新因逻辑删除占用成员唯一键而失败

- 发现/关闭日期：2026-07-16
- 级别：P1 / 处理组维护
- 分支：`develop`

## 现象与根因

真实 API 更新处理组时返回 500。更新流程先对旧成员执行逻辑删除，再插入新的成员集合；
成员表唯一键为 `(tenant_id, group_id, user_id)`，不包含 `deleted`，因此同一成员重新加入时
命中唯一键 `uk_crm_work_order_group_member`。

## 修复

处理组成员关系不承载业务轨迹，更新时使用带租户过滤的物理删除清理旧关系，再按排序写入
新成员集合；工单轨迹仍保存在 `crm_work_order_record`，不受影响。

## 验证

新增/更新处理组测试和真实 API 验收覆盖更新、成员回显及零残留；Ubuntu 26.04 CRM 全量回归通过。
