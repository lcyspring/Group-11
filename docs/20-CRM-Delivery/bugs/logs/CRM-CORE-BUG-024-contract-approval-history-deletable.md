# CRM-CORE-BUG-024：已提交合同审批历史可被删除

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-CTR-002、GAP-APR-002、ADR-006
- 状态：已关闭

## 现象与根因

旧删除逻辑只检查是否存在回款，没有检查审批状态或流程历史。无回款的审批中、已通过、
驳回、取消合同都可被删除，造成 BPM 流程 businessKey 指向不存在的业务对象，并破坏审计
留痕；前端也对全部状态展示删除按钮。

## 修复关键

后端只允许 `DRAFT` 且从未产生 `processInstanceId` 的新草稿删除；前端使用相同条件展示
删除入口。修订后的草稿保留上一流程 ID，因此也不可删除。

## 验证

专项断言驳回合同删除返回 `1020000009` 且 Mapper 不执行删除，同时新草稿仍可正常删除；
CRM 全量 124/124，Web production build 通过。
