# CRM-REFUND-FE-BUG-001：退款表单固定截断可选回款

- 发现/关闭日期：2026-07-15
- 级别：P1 / 可用性与规模边界

## 现象与风险

退款表单初始化时只读取“我负责”和“我参与”各第一页 100 条审批通过回款。账号历史回款超过该数量后，
较早的合法回款不会出现在下拉框中，用户无法发起退款/冲销；固定 100 条也没有任何业务依据。

## 修复关键

- 下拉框改为服务端远程检索，输入回款编号时分别查询负责人和参与人场景；
- 两个授权来源按回款 ID 合并去重；
- 编辑已有退款或从指定回款上下文打开时，单独读取当前回款并补入候选，确保分页截断不影响回显；
- 使用请求序号丢弃晚到的旧搜索结果，避免快速输入时旧响应覆盖新响应。

## 验证

- 退款专项 ESLint 零警告；
- Ubuntu 26.04 Web production build 通过；
- CRM 后端全量 224/224，负责人/参与人场景和缺省权限回归通过。
