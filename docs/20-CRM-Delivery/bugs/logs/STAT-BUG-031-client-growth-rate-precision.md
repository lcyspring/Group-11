# STAT-BUG-031：同比环比由浏览器浮点重算导致口径和精度漂移

状态：Fixed

日期：2026-07-15

分支：`develop`

## 现象

合同数量、合同金额和回款金额三个业绩页面收到本期、上期数值后，各自在浏览器把值转换为 JavaScript
`Number` 并计算增长率。大额十进制数可能在计算前丢失精度，且图表、表格与后续客户端容易形成多套舍入口径。

## 根因与修复

后端响应只有原始统计值，没有提供权威同比、环比字段。Server 现统一用 `BigDecimal` 按
`(本期 - 基期) / 基期 × 100` 计算，保留两位小数并使用 `HALF_UP`；基期为零返回 `null`。前端直接消费
`monthOnMonthRate/yearOnYearRate`，只保留空值格式化，不再二次计算。

## 回归

覆盖正增长、负增长、循环小数舍入、零基期和超过 JavaScript 安全整数的大额十进制；三个页面的图表与
表格均使用同一服务端字段。
