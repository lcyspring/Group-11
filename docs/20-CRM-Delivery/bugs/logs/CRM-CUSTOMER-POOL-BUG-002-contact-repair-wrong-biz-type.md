# CRM-CUSTOMER-POOL-BUG-002：联系人历史修复脚本误用客户业务类型

- 发现日期：2026-07-15
- 分支：`develop`
- 状态：Closed

## 现象

联系人归属历史修复脚本最初将 OWNER 权限条件写为 `biz_type = 2`。CRM 枚举中
`2` 表示客户，联系人应为 `3`。首次评审修正了删除和插入值，却遗漏了插入查询的
`LEFT JOIN` 条件；第一次运行因此跳过 1 条应补权限，并为另 1 条联系人插入了重复 OWNER。

## 根因

迁移脚本直接书写数字常量时未逐项对照 `CrmBizTypeEnum`，代码评审只关注了租户、OWNER
级别和幂等逻辑，没有首先验证业务类型映射。

## 修复

- 使用单一会话常量 `@crm_contact_biz_type := 3` 和 `@crm_owner_permission_level := 1`，
  删除、去重、插入及存在性检查不再重复散落数字常量；
- 迁移使用事务包裹，任一 SQL 失败时不提交半成品；
- 执行前以枚举和基础表数据双重核对业务类型；
- 将迁移前后客户权限计数纳入运行验收，证明客户权限不受影响。

## 影响

最初会删除客户权限的版本没有执行。遗漏 `LEFT JOIN` 的中间版本执行一次后只造成 1 条联系人
OWNER 重复，并漏补 1 条联系人 OWNER；客户权限指纹保持不变。最终版本在第二次迁移中清理该
重复项并补齐缺失项。修正版仍只处理同租户、关联有效客户且可从客户负责人唯一推导的联系人；
孤立联系人不自动处理。
