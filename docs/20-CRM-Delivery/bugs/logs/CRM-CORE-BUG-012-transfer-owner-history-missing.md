# CRM-CORE-BUG-012：客户转移未保留前后归属

- 发现日期：2026-07-13
- 严重程度：中
- 状态：已修复
- 关联：`US-CUS-006`、`GAP-CUS-001`

## 现象

原 `transferCustomer` 只转移 CRM 权限、更新 `crm_customer.owner_user_id` 并可选转移关联对象。`crm_customer_owner_record` 只写入进入公海和领取/分配两类事件，转移后无法追溯原负责人、新负责人、操作人和发生时间。

## 影响

- 客户归属争议无法通过业务数据回溯。
- 原型要求的“记录前后归属”验收条件不成立。
- 如果单独在转移后非事务地补写历史，还可能产生客户已转移但历史缺失的不一致状态。

## 根因

原归属记录表是为公海统计增加的事件表，模型只有一个 `owner_user_id` 和类型 1/2，未扩展为完整的客户归属历史；转移服务也未接入该表。

## 修复

1. 新增 `previous_owner_user_id` 和 `new_owner_user_id`，增加初始分配、转移类型。
2. 保留 `owner_user_id` 兼容原公海统计。
3. 在转移数据权限、客户负责人、关联对象全部处理后写入 `oldOwner -> newOwner`，整个方法保持同一事务。
4. 增加受 CRM READ 数据权限保护的历史 API 和客户详情页展示。

## 验证

- 真实 API 转移返回 `type=4, 1 -> 100, operator=1`。
- 强制历史 INSERT 失败后，客户 owner 和 CRM Owner 权限均仍属于原用户，历史数未增加。
- 无数据权限访问返回 `1020007001`，跨租户访问返回 `403`。
- 原公海按日期/按用户统计接口回归通过。
