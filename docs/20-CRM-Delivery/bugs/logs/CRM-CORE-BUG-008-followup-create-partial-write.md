# CRM-CORE-BUG-008：跟进创建失败会遗留已插入记录

发现/关闭日期：2026-07-13
级别：P0
关联差异：`GAP-LEAD-002`

## 现象与根因

跟进创建先 INSERT 跟进记录，再调用对应客户、商机、线索等服务更新最后跟进信息，但方法没有事务。后续业务对象更新失败时，接口返回失败，前面的跟进记录却已经提交，形成部分写入。已转换线索增加只读校验后会稳定暴露此问题。

## 修复

- `createFollowUpRecord` 增加 `@Transactional(rollbackFor = Exception.class)`。
- 跟进删除也加入事务，使线索可写校验与删除处于同一事务。
- 已转换线索新增/删除跟进统一返回 `1020001002`。

## 验证

真实 API 在已转换线索上创建跟进：线索校验失败返回 `1020001002`，随后 MySQL 按业务对象和唯一内容查询均确认新增记录为 0；既有历史跟进保持一条并已复制到客户。CRM 全量 42/42。

## 状态

已关闭。
