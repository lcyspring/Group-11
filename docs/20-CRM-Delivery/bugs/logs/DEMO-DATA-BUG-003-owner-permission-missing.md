# DEMO-DATA-BUG-003：生成客户缺失 OWNER 权限关系

日期：2026-07-18。分支：`develop`。优先级：P0/可用性。状态：已关闭。

## 现象

生成客户的 `pool_status=0`、`owner_user_id=1`，但客户详情仍不显示转移、锁定和“放入公海”。

## 根因

详情页通过 `crm_permission` 团队成员列表判断当前用户是否具有 OWNER 权限，不直接信任业务表的
`owner_user_id`。生成器只写负责人字段，没有为客户、联系人和商机生成 `level=1` 的 OWNER 关系，
形成表面在管、实际不可操作的数据。

## 修复与验证

- 客户、联系人和商机创建后同步写入 `crm_permission`；
- cleanup 在删除业务对象前按 biz_type 删除对应权限；
- 客户/联系人/商机权限分别为 1000/1000/1600，且对象去重计数完全一致；
- 抽查客户为 `pool_status=0`、负责人 1、OWNER 用户 1、权限级别 1；Server `UP`。
