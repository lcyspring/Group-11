# CRM-CUSTOMER-GARBAGE-BUG-002：永久删除引用校验遗漏下游业务

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

原客户删除保护只覆盖少量销售对象，不能支撑物理删除。已转换线索、财务、报销和客服
工单仍引用客户时继续删除，会产生孤立业务记录或在不同数据库约束下运行结果不一致。

## 修复

- 新增统一引用查询，按固定顺序检查十一类有效下游数据；
- 查询显式携带租户并忽略自动租户拼接，避免 UNION 子查询作用域不一致；
- 任一引用存在即返回引用名称，永久删除不执行权限、跟进或客户的部分清理；
- 最终删除同时比较租户、客户 ID、GARBAGE 状态和空负责人。

## 验证

专项测试覆盖客服工单引用拒绝及无引用状态条件物理删除；Ubuntu 26.04 CRM 305/305 通过。
