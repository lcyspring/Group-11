# WEB-I18N-BUG-002：审批详情页签标题显示英文

- 日期：2026-07-16
- 分支：`develop`
- 级别：P1
- 状态：已关闭

## 现象

中文环境从待办、已办或 CRM 审批入口进入审批详情后，顶部页签标题可能仍显示 `Process Detail`，
语言切换后已经打开的隐藏路由页签也不能同步更新。

## 根因

隐藏路由在模块加载时调用翻译函数，把启动时语言对应的文本固化进 `meta.title`。页签渲染阶段收到的
已经是 `Process Detail`，而不是可按当前语言解析的翻译键，因此只能把英文原样显示。

## 修复关键

- 隐藏路由的标题统一保存 `common.router.*` 翻译键，不在路由模块加载阶段提前翻译；
- 页签继续在渲染阶段按当前语言解析标题，审批详情中文标题固定为“流程详情”；
- 对全部 45 个隐藏路由标题增加中、英、阿三语完整性检查；
- 增加审批详情页签的三语精确值断言，并禁止整个静态路由模块残留任何模块加载期 `t()` 调用；
- 修复首次改造遗漏的个人中心标题，避免移除翻译函数后触发 `ReferenceError: t is not defined`。

- 审批前端专项自动化：6/6；

- Ubuntu 26.04 Podman Web production build：通过。
