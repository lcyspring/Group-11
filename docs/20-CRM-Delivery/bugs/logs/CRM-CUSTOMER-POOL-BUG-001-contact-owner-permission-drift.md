# CRM-CUSTOMER-POOL-BUG-001：客户领取后联系人归属与权限漂移

- 发现日期：2026-07-15
- 分支：`develop`
- 状态：Closed

## 现象

客户进入公海时联系人 `owner_user_id` 被清空，但联系人对象的旧 OWNER 权限仍保留。
客户被新用户领取或分配后，客户归属已更新，联系人负责人却没有恢复。这会造成
客户详情、联系人列表、对象权限和后续跟进的负责人不一致。

## 根因

`receiveCustomer` 中留有“是否处理关联联系人” TODO，只更新客户与客户权限。
既有联系人批量负责人方法只更新表字段，没有同步 `crm_permission`。

## 修复

- 公海领取/分配后批量恢复联系人负责人；
- 新增幂等 `replaceOwnerPermission`，只替换唯一 OWNER，保留 READ/WRITE；
- 进入公海时清空联系人 OWNER 权限；
- 将联系人同步移到客户 OWNER 权限删除之前，避免授权顺序反转；
- 全部变更保持在原客户公海事务中，任一失败整体回滚。
- 新增幂等兼容迁移，以同租户客户当前负责人修复可唯一推导的历史联系人；
- 无有效客户的孤立联系人不自动修复，避免伪造归属。

## 回归结果

- Ubuntu 26.04 CRM 全量回归 268/268 通过；
- CRM 行覆盖率 41.55%；
- Ubuntu 26.04 Server 生产构建通过；
- 历史数据迁移后负责人不一致、失效/缺失/重复 OWNER 均为 0，最终脚本连续两次幂等；
- 通过显式 YAML 热替换，8080 恢复 `UP`，8081/8082 保持可用。
