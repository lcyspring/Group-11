# CRM-PRODUCT-BUG-003：产品分类 CRUD 类型与异常边界失效

状态：Fixed

日期：2026-07-19

分支：`develop`

## 现象

- 产品分类表单使用 `any` 和 `as unknown as` 绕过请求类型检查；
- 表单提交前检查的是 Ref 对象本身，空表单实例判断永远不会命中；
- 删除确认、删除请求和列表刷新共用空 `catch`，接口失败也会被静默吞掉；
- 页面仍使用 CRLF，产品分类说明标题缺字，源码目录混有不参与编译的单行外链教程。

## 根因

产品分类页面沿用早期脚手架生成代码，创建、更新和响应对象共用一个宽泛类型；确认框取消与业务请求失败
没有明确异常边界。后续国际化修复只覆盖了文案，没有同步收口这些旧实现。

## 修复关键

- 分离产品分类创建、更新、列表查询和响应类型；
- 表单、表格和组件 Ref 使用明确类型，删除双重类型断言；
- 只在用户取消删除确认时返回，删除接口和刷新失败继续进入统一请求错误处理；
- 更新表单与分类列表并行加载，提交期间统一禁用确认和取消按钮；
- 统一两个组件为 LF，修复说明标题，并删除七个 Java 源码目录中的单行外链教程文件；
- 专项测试增加类型载荷、Ref 和空 `catch` 回归门禁，ESLint 同时覆盖产品分类 API。
