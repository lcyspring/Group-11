# CRM-FIN-BUG-012：已结束回款暴露失效审批入口

- 发现日期：2026-07-17
- 分支：`develop`
- 状态：Closed

## 现象

回款管理中部分已审批记录显示“查看审批”，点击后 BPM 页面提示无流程实例。

## 根因

项目原始 CRM 演示数据保存了 2024 年的 Flowable 流程实例编号，但当前数据库没有配套的历史流程实例。既有孤立审批修复只处理“审批中”状态，没有处理审批通过、驳回和取消等已结束状态；前端只判断编号非空，因此继续显示无效入口。

## 修复

- 保留已结束单据的审批业务结果，不伪造或重建历史流程；
- 在修复审计表保存原实例编号、原状态和原因；
- 清空不存在于 `ACT_HI_PROCINST` 的失效编号，使前端不再显示不可用的“查看审批”；
- 同一治理覆盖合同、回款、退款、报销和合同补充协议，避免其他审批域重复出现相同问题；
- 审批中的孤儿记录仍按既有规则转为已取消，以允许用户修订并重新提交。

## 数据来源

问题记录来自 `database/bootstrap/crm-2024-09-30.sql` 的项目原始演示数据，不是本轮功能代码创建。当前修复通过兼容迁移应用于已有数据，也会在新空卷 bootstrap 时执行。

## 回归要求

迁移连续执行两次无错误；所有终态单据的非空流程编号都必须能在 `ACT_HI_PROCINST` 找到；修复审计记录可追溯；有效的新流程仍可正常打开。

## 回归结果

兼容迁移重复执行通过；回款终态孤立流程编号数量为 0。三条 2024 演示回款保留审批通过状态，
失效编号已清空；2026 年真实验收生成的有效审批实例编号全部保留并可查询。
