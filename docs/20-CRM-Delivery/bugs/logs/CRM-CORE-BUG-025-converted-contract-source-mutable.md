# CRM-CORE-BUG-025：商机转换合同可绕过页面改写来源归属

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-OPP-003、ADR-004/005
- 状态：已关闭

## 现象与根因

转换表单虽锁定客户、商机和负责人，但通用合同更新 Service 会接收请求中的客户和商机。
调用者绕过页面即可把带 `source_business_id` 的合同改挂其他客户或商机，破坏“客户和
负责人继承来源商机”的已接受决策。

## 修复关键

更新时读取锁定后的旧合同；只要存在 `sourceBusinessId`，服务端强制回填旧 `businessId`
和 `customerId`，负责人继续沿用既有不可更新规则。

## 验证

专项以伪造客户/商机请求修订驳回合同，捕获落库对象仍为原商机 `10`、客户 `20` 且负责
人不更新；合同专项 15/15、CRM 全量 124/124。
