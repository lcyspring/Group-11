# DEMO-DATA-BUG-007：演示账号用户名包含下划线无法登录

## 现象

演示账号已启用、密码和角色均正确，但登录返回 400：`账号格式为数字以及字母`。

## 根因

初版用户名为 `d20260718_user_02`，包含系统登录请求校验不允许的下划线。数据库字段约束比接口约束宽，
仅靠 SQL 插入无法发现该问题。

## 修复与验证

用户名统一改为纯字母数字形式 `d20260718user01` 至 `d20260718user08`；加载后断言增加正则检查。
重新 replace 后，`d20260718user02` 使用基准管理员密码登录成功，返回 `code=0`。
