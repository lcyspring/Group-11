# WEB-I18N-BUG-004：生产语言分包被裁空导致界面大面积英文

- 日期：2026-07-18
- 分支：`develop`
- 级别：P0 / 构建回归
- 状态：已关闭

## 现象

替换 Web 容器后，页面语言标记仍为 `zh-CN`，但大部分界面显示英文、原始文案或翻译键。

## 根因

Vite ESM 迁移把 Vue i18n 插件的 `include` 修正为真正匹配 `src/locales/**`。项目语言入口是
TypeScript 聚合模块，插件却把聚合模块当作纯消息资源编译，删除了它们导入的 `common`、`crm`、
`bpm` 等模块。最终中文、英文和阿拉伯文入口分包都只剩 94 字节和一个 `OAuth 2.0` 条目。

此前配置使用了一个实际不匹配语言目录的路径，因此问题被偶然掩盖；不能恢复这种无效路径。

## 修复关键

- i18n 插件只匹配其支持的 JSON、JSON5、YAML 消息资源，不再处理 TypeScript 聚合模块；
- 增加配置契约测试，禁止重新包含整个 TypeScript 语言目录；
- Ubuntu 构建在产物生成后强制执行三语分包完整性门禁；
- 入口 HTML 默认语言同步修正为 `zh-CN`。

## 验证

- 源码与客户详情本地化专项：4/4，通过；
- 三语生产分包完整性：通过，中文 450285 字节、英文 313416 字节、阿拉伯文 860304 字节；
- Web production build：Ubuntu 26.04 容器通过。
