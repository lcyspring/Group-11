# WEB-TYPE-BUG-001：Web 全仓 Vue 类型检查存在大规模历史错误

- 发现日期：2026-07-14
- 状态：已关闭
- 级别：P1 / 质量基础设施

## 现象

修复 Node 堆限制后，Ubuntu 26.04 的 `vue-tsc --noEmit` 能够完整执行，初次基线在脚本与 Vue 模板中
记录 653 条诊断，包括缺失声明、错误字段、不可达逻辑、旧 Element Plus 参数、空值类型及被乱码
注释吞掉的声明。生产 Vite 构建不会执行全量类型检查，因此过去构建成功没有暴露这些问题。

本批类型检查同时发现发票三语言对象中“模块标题”和“购方抬头”重复使用 `title` 键，已改为
`managementTitle` 与 `title`，避免后定义值静默覆盖前定义值。

## 根因与修复

- 将 TypeScript 固定为 `5.7.3`、`vue-tsc` 固定为 `2.2.10`，消除检查器与编译器版本漂移；
- 保留脚本和 Vue 模板代码生成，不使用 `@ts-nocheck`、错误白名单或跳过模板检查；
- 按实际 API、响应模型与组件契约修复 653 条诊断，同时修复由类型检查暴露的库存派生账误操作、
  角色表单 `.value` 误用、盘点校验字段、订单详情时间格式化和 CRM 跟进字段等运行缺陷；
- 对历史代码采用兼容基线 `strict: false`，关闭未使用变量噪声；严格模式迁移属于独立增强，不影响
  当前全仓类型门禁的真实性。

## 验证结果

2026-07-18 使用标准 Ubuntu 26.04 YAML 入口执行：

```bash
bash podman/compile.sh podman/config/check-web-types-ubuntu-26.04.yaml
```

`vue-tsc --noEmit` 退出码为 0，脚本与模板诊断均为 0；同一工具链的 CRM 导出专项 Vite production
build 同时通过。该入口继续作为后续 Web 变更的全仓回归门禁。
