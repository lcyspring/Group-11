# WEB-TYPE-BUG-001：Web 全仓 Vue 类型检查存在大规模历史错误

- 发现日期：2026-07-14
- 状态：Open / 全仓技术债；本批新增目录单独设门禁
- 级别：P1 / 质量基础设施

## 现象

修复 Node 堆限制后，Ubuntu 26.04 的 `vue-tsc --noEmit` 能够完整执行，但在 BPM、Mall、Pay、
System 等既有目录报告上千行历史类型错误，包括缺失声明、`never` 推导、旧 Element Plus 参数、
空值类型和未使用变量。生产 Vite 构建不会执行全量类型检查，因此过去构建成功没有暴露这些问题。

本批类型检查同时发现发票三语言对象中“模块标题”和“购方抬头”重复使用 `title` 键，已改为
`managementTitle` 与 `title`，避免后定义值静默覆盖前定义值。

## 当前处理

- 保留 `podman/config/check-web-types-ubuntu-26.04.yaml` 作为全仓债务测量入口；
- 生成的全局组件声明会把所谓“局部 vue-tsc”重新扩展为整个 Web 依赖图，因此不建立虚假的局部
  类型通过门禁；
- CRM 发票交付通过目录级 ESLint、前端测试和生产构建三道验证；
- 全仓历史错误不能在发票提交中批量掩盖或关闭，应按模块逐步建立基线并修复。
