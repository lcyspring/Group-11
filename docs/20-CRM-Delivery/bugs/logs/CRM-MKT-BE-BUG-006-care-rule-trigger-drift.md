# CRM-MKT-BE-BUG-006：客户关怀规则触发口径漂移

## 现象

旧实现只支持生日和节假日，日期仅用正则校验，`99-99` 也能保存；节日关怀读取所有客户，
没有按成交客户筛选；成交后第 N 天回访无法配置和触发。

## 根因

计划表和服务只实现了早期生日/节假日骨架，`event_month_day` 被所有规则共用，规则字段没有
按类型归一化，节日任务通过全表客户列表在内存中查找首联系人。

## 修复

- 新增成交后定期回访类型和 `follow_up_days` 字段；
- 生日按联系人生日当天触发，节假日按有效 `MM-dd` 及成交客户触发；
- 定期回访以客户进入成交生命周期的日期为事实源；
- 使用数据库联表直接选择成交客户首联系人；
- 显式使用 YAML 时区计算业务日期，并保留事件日幂等唯一键。

## 验证

真实 API 验证三类规则详情契约；CRM 全量 470/470 通过。
