# CRM-MKT-SEC-BUG-001：群发查询权限可读取全租户收件人

## 现象

只有 `crm:marketing-outreach:query` 的普通用户可分页读取同租户全部群发任务，并通过收件人结果接口
获得其他创建人的手机号和邮箱。活动与竞品已按负责人范围收敛，群发对象遗漏同类限制。

## 根因

群发列表、详情和收件人接口只校验功能权限，Service 和 Mapper 没有合并创建人/特权审核角色范围。

## 修复

- 普通查询用户只可读取本人创建的群发任务；
- CRM 管理员、群发审核人和发送操作人可读取全租户任务，支撑审核与执行职责；
- 详情、目标 ID 和收件人手机号/邮箱统一复用同一对象范围门禁；
- Mapper 在 SQL 层加入创建人过滤，避免先读全量再内存裁剪。

## 验证

- 非创建人查询详情和收件人均拒绝；
- 创建人可读本人任务；
- 审核/发送角色可读全量；
- 普通分页 SQL 带创建人范围，特权分页不添加该限制；
- Ubuntu 26.04 CRM 测试、真实双用户 API 负向通过后关闭。
