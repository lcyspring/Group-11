# CRM-FIN-BUG-011：报销后端可用但管理端入口缺少业务页面

- 发现日期：2026-07-15
- 分支：`develop`
- 状态：Closed

## 现象

数据库菜单已指向 `crm/reimbursement/index`，后端也已提供报销与费用分类接口，但 Web 中没有对应页面。用户无法在管理端创建、修订、查看或管理费用分类，附件专用上传入口也无法从界面使用。

## 根因

上一批只完成财务域表结构、后端服务和真实 API 验收，前端交付尚未进入实现。若直接复用通用上传组件，还会绕过报销 ID 绑定和受保护目录约束。

## 修复

- 新增报销列表、表单、详情和费用分类管理页面；
- 创建时采用两阶段保存，附件只能上传到已创建报销单的受保护目录；
- 请求显式挑选可写字段，不发送页面合计作为 `totalAmount`；
- 详情展示服务端总额、动作轨迹和金额快照；
- 同步中英阿三语文案；
- 增加状态、删除条件、金额精度和日期范围专项测试与覆盖率门槛。
