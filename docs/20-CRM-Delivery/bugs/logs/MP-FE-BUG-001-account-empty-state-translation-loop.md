# MP-FE-BUG-001：公众号未配置通知显示键名并重复弹出

## 现象

公众号管理没有配置账号时持续弹出 `common.noAccountConfig`，而不是对应语言的说明。

## 根因

共享国际化钩子把 `common.*` 识别为顶级命名空间，因此 `useI18n('mp')` 不会为它自动补上
`mp.`。账号选择器每次挂载又都会重复发送通知并跳转。

## 修复

- 公众号选择器显式引用 `mp.common.selectAccount`；
- 未配置账号时直接进入账号管理页，不再产生跨子模块反复出现的瞬时通知；
- 捕获初始化查询和路由异常，避免形成未处理 Promise；
- 保留自动跳转账号管理的引导行为。

## 验证

2026-07-16 运行复验发现 8081 曾继续提供修复前的 Web 产物，产物内仍包含
`message.error("common.noAccountConfig")`。重新使用 Ubuntu 26.04 构建并部署 Web 后，
公众号专项 3/3 通过，且从运行容器资源中确认该通知调用为 0 处；服务端健康状态为 `UP`。
