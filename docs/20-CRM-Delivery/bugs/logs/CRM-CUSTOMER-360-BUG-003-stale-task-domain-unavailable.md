# CRM-CUSTOMER-360-BUG-003：客户详情错误显示任务域未接入

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

客户 360 摘要长期显示“任务域未接入”，但 OA/CRM 任务能力已经存在。该说明与运行能力不一致，
会误导测试人员把已实现功能判断为缺失。

## 根因

客户 360 响应保留了早期硬编码字段 `taskSupported=false`，前端据此显示固定占位文本；摘要查询没有
统计 `crm_task` 中关联当前客户的任务。

## 修复

- 客户 360 聚合查询按 `biz_type=2` 和当前客户 `biz_id` 统计有效任务；
- API 响应改为真实 `taskCount`，删除 `taskSupported`；
- 前端卡片和中、英、阿三套语言资源统一显示“客户任务”及数量；
- 不保留旧占位字段兼容，避免错误能力状态继续传播。

## 验证

CRM 全量测试 537/537 通过。运行态客户摘要接口返回 `taskCount` 且不再返回 `taskSupported`；
Server 健康状态为 `UP`。
