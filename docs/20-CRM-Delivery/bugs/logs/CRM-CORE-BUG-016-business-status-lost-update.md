# CRM-CORE-BUG-016：并发商机状态更新互相覆盖

更新日期：2026-07-14

## 现象

两个请求可同时读取相同活动状态，再先后执行按 ID 更新；后提交者覆盖先提交者，
调用方仍收到成功。

## 根因

旧实现使用无旧值条件的 `updateById`，没有状态版本或 compare-and-set 条件。

## 修复关键

Mapper 更新同时匹配商机 ID、旧 `status_id` 和旧 `end_status`。影响行数为 0 时
返回 `1020002004`，要求刷新后重试；结束原因与状态在同一 SQL 中保存。

## 验证

Service 模拟条件更新 0 行，确认抛出指定业务错误；Ubuntu CRM 76/76。

## 遗留风险

真实 MySQL 双请求竞争尚未补运行时证据，后续应加入 Podman API 并发场景。

## 状态

代码已关闭，运行时并发证据待补。
