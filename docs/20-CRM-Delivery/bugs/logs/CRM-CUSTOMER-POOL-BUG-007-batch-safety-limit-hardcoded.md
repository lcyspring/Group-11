# CRM-CUSTOMER-POOL-BUG-007：自动回收批量安全上限多处硬编码

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

客户公海自动回收默认批量已经来自 YAML，但租户请求校验、Mapper 扫描保护和管理端输入框仍分别
写死 5000。部署人员无法通过配置收紧安全边界，前后端也可能在后续修改时产生不同上限。

## 修复

- 新增 `mitedtsm.crm.pool-policy.customer.auto-pool-max-batch-size`，作为唯一部署安全上限；
- 保存租户覆盖时在 Service 层动态校验，超限返回业务码 `1020006028`；
- 历史租户值超过当前 YAML 上限时，有效配置收敛到安全上限；
- Mapper 接收显式最大扫描量，不再包含业务性 5000 常量；
- 配置查询返回安全上限，管理端输入框和三语言提示使用该返回值。

## 验证

- Ubuntu 26.04 CRM 自动化 292/292 通过；
- 客户公海前端专项 ESLint 0 warning、0 error；
- Server 37 模块和 Web 生产构建通过；
- 真实 API 查询得到默认批量 500、上限 5000；保存 5001 被拒绝，随后查询仍为 500；
- 最新 Server、Web、Mall 已在 rootless Podman 中运行，8080 健康状态为 UP，8081/8082 为 200。
