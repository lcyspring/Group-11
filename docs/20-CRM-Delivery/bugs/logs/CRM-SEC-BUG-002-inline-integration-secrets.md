# CRM-SEC-BUG-002：应用配置包含硬编码第三方凭据

## 现象

公共与 local 应用配置包含 AI、公众号、小程序、社交登录、地图、快递、数据库和消息队列凭据，
构建后的 JAR 可直接提取这些值，且不同成员无法只修改一份显式运行配置完成部署。

## 根因

上游示例配置和历史测试账号被长期当作运行配置使用；Podman YAML 没有应用集成层字段，导致服务
只能依赖源码内 profile 值。

## 修复

- 公共与 Podman 使用的 local profile 移除活动行及注释中的真实集成凭据；
- 所有值改为环境占位符，未配置的可选 Provider 默认关闭或使用不可调用占位值；
- Podman YAML 增加公众号、小程序、JustAuth、地图和支付回调显式字段；
- `deploy.sh` 从唯一 YAML 路径读取并注入容器，不增加第二组命令行参数；
- 静态门禁拒绝公共/local profile 再出现内联密钥。

## 验证

公共/local profile 静态凭据门禁通过，Server 容器显式环境核对通过。未配置 AI Provider 时不再
隐式创建默认模型，TDengine 凭据由 YAML 注入；8080/8081/8082 均完成真实启动验收。
