# OA-EVENT-BUG-004：创建日程入口无响应且编辑区混入筛选栏

状态：Fixed

日期：2026-07-19

分支：`develop`

## 现象

日程管理把新增/编辑字段直接塞在页面顶部筛选区，“创建一条日程”只尝试聚焦第一个输入框，没有打开
可见的创建界面，用户感知为按钮无响应。新增、筛选和编辑状态相互混合，也不符合系统其他 CRUD 页面的
交互方式。

## 根因

早期页面以单文件压缩写法快速接入 CRUD，没有独立表单组件；空状态按钮通过 DOM 查询聚焦输入框，
既没有稳定组件引用，也没有明确打开动作。日程 API 的创建、更新与响应对象也共用一个宽泛类型。

## 修复关键

- 新增标准 `OaEventForm` 弹窗组件；
- 顶部“新增日程”、空状态“创建一条日程”和列表“编辑”统一调用 `openForm`；
- 弹窗提供标题、时间范围、地点、提前提醒、全天日程和说明字段；
- 增加必填与结束时间晚于开始时间校验，提交期间禁用按钮；
- 创建和更新成功后关闭弹窗并立即刷新当前日/周/月视图；
- API 分离创建、更新和响应类型，保留已验证的 Spring `LocalDateTime` 时间转换；
- 删除确认只忽略用户取消，真实删除错误不再被吞掉。
