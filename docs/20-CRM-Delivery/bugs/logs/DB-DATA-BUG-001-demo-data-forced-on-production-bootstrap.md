# DB-DATA-BUG-001：空库初始化强制携带上游演示数据

- 发现日期：2026-07-17
- 分支：`develop`
- 状态：Closed

## 现象

上游数据库基线同时包含表结构、基础配置和 CRM 演示业务数据。新建生产数据卷时无法明确选择是否保留演示数据，也容易误以为切换配置会覆盖已有持久数据。

## 根因

初始化流程只有固定 bootstrap 清单，没有独立的数据集选择层；初始化、清理和已有库替换的生命周期边界未显式表达。

## 修复

增加 `legacy-demo-v1` 与 `none` 数据集，由运行 YAML 的 `mysql.dataset` 在空卷初始化时显式选择。已有卷不重新应用数据集。另提供独立数据集替换入口，要求 YAML 同时声明是否清理旧数据以及是否确认修改持久数据；普通部署永不自动替换。

## 回归结果

独立空卷验证 `none` 的 CRM 核心业务记录为 0、CRM 表为 67、保留 1 个管理员和必要部门链；`legacy-demo-v1` 保留 20 个用户和 74 条 CRM 核心演示记录。配置门禁验证未授权清理及未确认持久数据修改均被拒绝。
