# CRM-FIN-BUG-013：回款提交提示负数剩余额度

- 发现日期：2026-07-17
- 分支：`develop`
- 状态：Closed

## 现象

部分草稿点击“提交审批”时提示当前剩余可回款为负数，例如 `-10`。

## 根因

历史演示数据可能已经使审批中及审批通过回款合计超过合同金额。提交时重新校验额度是正确的，但错误消息直接输出“合同金额减已占用金额”，没有将面向用户的可用额度限定为零，因而暴露负数。

## 修复

保留服务端提交前的合同行锁和最新额度校验；将剩余可回款统一限定为不小于零。历史超额时继续拒绝新提交，但明确显示剩余额度为 `0`，不再显示负数，也不会创建无效 BPM 实例。

## 回归要求

增加历史已占用金额超过合同金额的服务测试，验证返回额度为零、审批实例未创建；同时保留并发提交和正常额度测试。

## 回归结果

新增历史超额测试通过；CRM Ubuntu 26.04 全量 481/481。提交前仍锁定合同并重新读取草稿，历史
超额时返回剩余额度 `0`，不会创建 BPM 实例。
