# CRM-FILE-BUG-002：受保护临时文件可跨合同绑定

- 发现/关闭日期：2026-07-15
- 级别：P0 / 数据隔离
- 关联：GAP-IAM-003、GAP-SEC-003

## 现象与根因

第一版保护目录统一为 `crm-protected/contract`。创建附件只校验该公共前缀，因此同时拥有两份合同写权限的
用户若取得合同 A 上传返回的临时 URL，可以把同一文件绑定到合同 B；对象权限正确但文件归属边界不完整。

## 修复关键

- YAML 继续配置稳定基目录，运行时按合同 ID 派生 `crm-protected/contract/{contractId}`；
- 上传只写入目标合同目录；
- 创建附件同时校验受管文件路径属于请求合同专属目录；
- 客户端不能通过修改 `contractId` 重用另一合同的临时文件。

## 验证

合同生命周期 11/11、CRM 231/231 通过。运行态在合同 7 上传后改绑合同 11，返回 `1020000024`；
原合同绑定、授权下载和清理均成功。
