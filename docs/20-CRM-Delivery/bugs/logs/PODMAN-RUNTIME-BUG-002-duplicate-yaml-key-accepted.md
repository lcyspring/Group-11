# PODMAN-RUNTIME-BUG-002：YAML 重复键计数错误导致重复配置未被拒绝

## 发现

新增结构化测试使用同一路径的两个 `sample.value`，预期解析失败，但首次测试
返回成功。

## 根因

AWK 解析器匹配键时写成 `found = 1`，每次匹配都把计数重置为 1；结束阶段的
`found > 1` 永远不成立。

## 影响

若保留该缺陷，同名配置值可能输出多行并进入 Shell 变量，既不符合 YAML 单值
预期，也可能令部署行为不可预测。

## 修复

将匹配逻辑改为 `found++`，重复键时返回退出码 2。

## 回归

`parser-duplicate.yaml` 已稳定被拒绝；随后完整运行配置测试 18/18 通过。
