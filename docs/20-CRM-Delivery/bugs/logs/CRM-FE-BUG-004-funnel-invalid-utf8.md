# CRM-FE-BUG-004：销售漏斗组件含非法 UTF-8

## 现象

`FunnelBusiness.vue` 的中文注释出现 `分�?`、`加载�?` 等乱码，标准补丁工具读取
文件时报 `invalid utf-8 sequence`。

## 影响

- 标准化补丁和部分静态分析工具无法处理该文件；
- 不同编辑器可能继续以错误编码保存并扩大乱码范围；
- 增加构建环境和团队协作的不确定性。

## 根因

历史文件混入非 UTF-8 字节，且此前构建只转译有效代码，没有执行严格源码编码门禁。

## 修复关键

以规范 UTF-8 重建组件，清除乱码注释，同时保留并升级为真实阶段漏斗功能。

## 验证证据

- `iconv -f UTF-8 -t UTF-8` 严格读取通过；
- Ubuntu 26.04 Prettier 通过；
- Ubuntu 26.04 Vite production build 成功。

## 清理与遗留风险

未产生测试数据。仓库其他历史前端文件仍可能存在编码债务，后续应将全量严格 UTF-8
检查加入独立质量门禁。
