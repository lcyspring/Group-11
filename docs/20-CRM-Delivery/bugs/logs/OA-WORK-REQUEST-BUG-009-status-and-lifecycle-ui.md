# OA 请示审批状态与生命周期信息不可用

## 现象

请示列表把审批状态显示为裸数字，只展示标题、正文和状态，无法查看详情、审批进度或取消仍在运行的本人流程；紧急程度也是难以解释的数字。

## 根因

- 列表状态列没有接入 BPM 流程实例状态字典；
- 前端已保存 `processInstanceId`，但列表没有提供流程生命周期操作；
- 业务表包含发起时间和审批完成时间，页面未展示；审批通过时也没有写入 `approved_time`。

## 修复

- 使用 `bpm_process_instance_status` 字典渲染真实状态；
- 紧急程度改为普通、紧急、特急三个明确标签；
- 增加详情、审批进度和运行中流程取消操作，并在缓存页面激活时刷新状态；
- 列表和详情补充编号、发起时间、审批完成时间与完整正文；
- 状态监听收到审批通过事件时同步记录审批完成时间；
- 已提交请示保持不可编辑，避免审批中的流程变量与业务单据发生分叉。

## 回归

Web 契约覆盖字典状态、详情、进度、取消、完成时间和缓存刷新；后端单测覆盖审批通过状态及完成时间回写。
