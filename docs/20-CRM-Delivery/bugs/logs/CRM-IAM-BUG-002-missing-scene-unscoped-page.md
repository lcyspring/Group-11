# CRM-IAM-BUG-002：缺省场景导致普通用户分页无对象范围

- 发现/关闭日期：2026-07-15
- 级别：P0 / 数据越权

## 现象与风险

通用 `CrmPermissionUtils.appendPermissionCondition` 只处理负责人、参与人和下属三个显式场景。
调用方省略 `sceneType` 时三个分支均不进入，最终 SQL 没有负责人或参与人条件。具备功能菜单权限的
普通用户可能因此读取当前租户内不属于自己的 CRM 对象分页。

## 根因

实现把“未选择页面场景”误当成“无需对象数据权限”。场景是用户视图选择，不是授权开关；缺省值
不能扩大到全量。CRM 管理员与普通用户也未在缺省分支显式区分。

## 修复关键

- 普通用户缺省场景统一解析为 OWNER，只返回本人负责对象；
- CRM 管理员缺省场景保留原全量能力；
- 显式负责人、参与人、下属场景不被覆盖；
- 退款表单需要可选来源时，显式分别请求负责人和参与人列表，再按回款 ID 去重，不依赖缺省全量。

## 验证

- 新增 3 个纯函数回归：普通用户缺省、管理员缺省、显式场景；
- Ubuntu 26.04 CRM 全量 224/224，失败 0、错误 0、跳过 0；
- 退款专项 8/8、Web 专项 ESLint 和 production build 全部通过。

## 遗留验证

自动化已经关闭代码缺陷。后续建立双普通角色运行数据集时，还应对客户、联系人、商机、合同、回款、
发票和退款分页逐项执行“省略 sceneType”负向验收，形成接口级越权证据。
