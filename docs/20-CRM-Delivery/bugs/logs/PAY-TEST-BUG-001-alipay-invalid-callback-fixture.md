# PAY-TEST-BUG-001：支付宝测试夹具生成非法回调 URL

日期：2026-07-18。分支：`develop`。优先级：P1/测试门禁。状态：已关闭。

## 现象

Ubuntu 26.04 容器执行 Pay 全量时，支付宝条码、PC、扫码和 H5 的 29 个用例在进入各自渠道断言前，
统一被 `notifyUrl 必须是 URL 格式` 拦截。初次结果为 167 个测试、11 个失败、18 个错误、35 个跳过。

## 根因

公共测试工具的 `randomURL()` 实际生成相对资源路径，适用于头像/附件类字段，但支付宝测试把它用于
带 Hibernate `@URL` 约束的绝对 Provider 回调地址。随机 POJO 还可能生成非法 `returnUrl`，导致夹具
语义与生产 DTO 约束漂移。

## 修复关键

- 支付宝测试基类提供固定 HTTPS origin 加随机路径的合法回调 URL；
- 四类支付宝测试统一复用该生成器；
- 公共下单夹具显式把非当前用例关注的 `returnUrl` 设为空；
- 不放宽生产 URL 校验，也不修改全局 `randomURL()`，避免影响附件类既有用例。

## 验证

- 支付宝四类 + 回调日志专项：43/43；
- Pay 模块全量：167 个，132 通过、35 个外部集成测试跳过，失败 0、错误 0；
- 关联 reactor 模块构建全部成功。
