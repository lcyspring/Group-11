# CRM-DATA-BUG-003：历史审批回款引用已删除客户或合同

- 发现日期：2026-07-15
- 级别：P1 / 历史数据一致性
- 状态：已关闭（保留历史台账，不改写业务归属）

## 现象

运行验收筛选审批通过回款时，发现 1 条有效回款记录无法同时关联到有效客户和有效合同。直接按回款选择
客户会使客户 360 返回“客户不存在”，说明历史财务记录与业务对象软删除状态不一致。

## 影响

- 以回款为入口跳转客户详情可能失败；
- 跨客户财务报表仍可能统计该回款，但客户 360 无法提供对应视图；
- 若直接删除回款或重绑客户，会改变已审批财务历史。

## 治理结果

现有回款创建、更新和审批链已校验合同、客户与对象权限，新数据不应再产生同类引用。历史记录缺少可证明
的新归属，因此不自动删除、不恢复业务对象、不重绑到名称相似对象。回款列表和详情保留原始客户、合同 ID，
并使用引用状态显式标记完整、客户缺失、合同缺失/错配或双异常；异常记录禁止编辑和错误跳转。

客户经营统计只统计客户与合同关系都有效的回款，避免把孤儿财务历史归到不存在的客户；员工业绩按财务台账
口径保留全部审批通过回款，避免静默丢失已审批金额。两类指标的差额作为历史异常金额保留并可对账。

## 验证证据

- 真实历史记录引用状态：`30`（客户、合同双异常），原始 ID 为 `2/1`；
- 员工业绩财务台账：`17002.000000`；客户统计有效关系：`16914.000000`；保留差额：`88.000000`；
- 验收前后历史行快照完全一致，未修改、未删除、未重绑；
- 分页状态筛选、详情读取、客户统计和员工业绩均与 MySQL 1:1。
