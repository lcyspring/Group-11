# CRM-HOME-BUG-002：首页公告跳转 404

状态：Fixed

日期：2026-07-19

分支：`develop`

## 现象

点击首页“公告总数”或通知公告卡片右上角“更多”后进入 404 页面，无法从首页继续查看公告。

## 根因

通知公告菜单由后端菜单数据动态注册，真实路由名称为 `SystemNotice`，完整 URL 还取决于父菜单；首页若直接
硬编码 `/system/notice`，会绕过动态路由的名称解析。当当前账号没有公告管理路由时，也缺少可用的用户通知
页兜底目标。

## 修复关键

- 首页公告入口统一交给 `handleNoticeMoreClick` 处理，不再硬编码菜单 URL；
- 当前账号拥有 `system:notice:query` 权限且运行时已注册 `SystemNotice` 时，按路由名称打开公告管理；
- 未注册公告管理路由时，回退到静态且始终可用的 `MyNotifyMessage` 路由；
- 公告总数改用公告分页接口返回的真实 `total`，首页重新激活时同步刷新公告列表和总数；
- 单个首页接口失败不再让整个页面永久停留在骨架屏。

## 回归

- 首页交互测试 2/2 通过，覆盖待办、公告入口、真实数据刷新和加载结束条件；
- 数据库菜单确认公告组件名为 `SystemNotice`；
- 当前 8081 部署产物已包含按名称跳转和 `MyNotifyMessage` 兜底逻辑；
- `/system/notice` 与 `/user/notify-message` 经 Web 容器访问均返回 HTTP 200。

浏览器若仍持有旧入口 chunk，需要执行一次强制刷新后再验证。
