# BPM-TASK-FE-BUG-001：审批后我的待办必须手动刷新

- 日期：2026-07-16
- 分支：`develop`
- 级别：P1
- 状态：已关闭

## 现象

用户从“我的待办”进入审批详情并完成处理，返回列表后已完成任务仍然显示；CRM 待办事项中的 BPM
任务列表也存在同样问题，必须刷新浏览器才能看到最新结果。

## 根因

待办、已办和 CRM BPM 待办子列表只在首次挂载时请求数据。管理端标签页开启 KeepAlive 后，返回页面只会
重新激活组件，不会再次执行挂载生命周期。

## 修复关键

- 待办、已办和 CRM BPM 待办子列表在缓存页重新激活时刷新；
- 使用初始化标记跳过首次挂载后的重复请求，避免修复刷新问题时引入双请求；
- 列表筛选和页码保留，只更新当前查询结果。

## 验证

- 缓存页重新激活契约测试：1/1；
- 目标目录 ESLint：零错误；
- Ubuntu 26.04 Podman Web production build：通过。
