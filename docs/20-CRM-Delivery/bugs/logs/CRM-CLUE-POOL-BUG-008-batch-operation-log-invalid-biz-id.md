# CRM-CLUE-POOL-BUG-008：批量领取和分配写入非法审计业务编号

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

批量领取和主管分配把整个线索编号列表作为操作日志 `bizNo`。日志落库层要求该字段为单个
`Long`，因此成功分配 `[410]` 后仍抛出 `NumberFormatException`。业务事务已经提交，但操作日志
记录失败并污染运行错误日志；批量领取使用相同表达式，也存在同一问题。

## 修复

批量命令先规范化编号集合，并把第一条线索编号作为标量审计锚点写入日志上下文。日志动作仍
保留完整批量数量，归属事件继续逐条记录，因此不会丢失各线索的状态迁移证据。领取和分配两条
入口统一使用该标量业务编号。

## 验证

增加注解契约回归，Ubuntu 26.04 CRM 全量自动化 324/324。最新 rootless Podman 环境重新执行
主管分配后，API 返回成功，操作日志为 `biz_id=411`、“分配了 1 条公共线索”；当前 Server 日志
没有新增 `NumberFormatException` 或操作日志记录异常，测试数据和对应测试日志已清理。
