# CRM-WO-BUG-004：无派单权限仍返回候选人与处理组完整名册

- 发现/关闭日期：2026-07-16
- 级别：P1 / 最小披露
- 分支：`develop`

## 现象与根因

派单上下文虽然返回 `manualAssignmentAllowed=false`，仍计算并返回自己、下属或指定处理组成员；
处理组管理列表也允许仅具有工单查询权限的用户读取负责人和完整成员姓名。页面禁用选择框不能消除
接口层的人员信息披露。

## 修复

Controller 在没有普通派单权限时返回空候选列表；服务层对非负责人选择指定组返回空候选；处理组
完整管理列表只允许 `crm:work-order-group:query`。自动派单仍在服务端使用组成员，不依赖把名册返回前端。

## 验证

Ubuntu 26.04 后端全量、Web 专项 ESLint 和真实派单上下文验收通过。
