# CRM-CORE-BUG-022：合同编辑状态门禁反向

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-ORD-003、GAP-APR-002、ADR-006
- 状态：已关闭

## 现象与根因

旧 Service 允许 `DRAFT` 和 `PROCESS` 编辑，导致审批人正在审核时合同正文和产品金额仍可
变化；同时 `REJECT`、`CANCEL` 被禁止编辑，使原型要求的修订/重提无法完成。代码注释和
状态条件共同固化了错误门禁。

## 修复关键

仅允许草稿、审核不通过、已取消编辑；编辑时锁定合同记录。审核不通过/已取消的保存动作
显式把业务状态恢复为草稿，再由提交动作发起新流程。前端列表和详情同步展示“修订”。

## 验证

专项断言审批中编辑返回 `1020000001`，驳回和取消合同修订后均写入草稿；CRM 合同
15/15、全量 124/124，Ubuntu Web ESLint 和 production build 通过。
