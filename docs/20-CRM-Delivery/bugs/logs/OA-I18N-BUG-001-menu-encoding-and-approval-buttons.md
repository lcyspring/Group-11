# OA-I18N-BUG-001 工作流程菜单乱码与审批按钮语言不一致

## 问题现象

- 工作流程下的办公协作、请假管理、审批中心以及 CRM 逾期入口显示乱码。
- 审批通过弹出区域可能同时出现中文、英文按钮名称。

## 原因分析

菜单迁移被手工执行时，MySQL 客户端未显式声明 UTF-8，中文和阿拉伯文字符串以错误字符集写入，形成双重编码。审批流程模型还会把内置按钮名称固化为中文或英文，运行页面此前直接展示该历史值，没有按当前语言重新映射。

## 修复

- OA 菜单迁移显式执行 `SET NAMES utf8mb4`。
- 重新写入四组菜单及三语言数据，并用十六进制结果核验 UTF-8 字节。
- 审批页面把内置操作名统一映射到当前语言；真正由管理员自定义的按钮名称保持不变。
- 增加前端契约测试，防止内置按钮再次直接显示模型中的固化语言。

## 验证

- 数据库菜单中文值均为正确 UTF-8 字节。
- 中文、英文、阿拉伯文菜单记录均可正常读取。
- Ubuntu 26.04 前端门禁覆盖审批按钮语言映射。
