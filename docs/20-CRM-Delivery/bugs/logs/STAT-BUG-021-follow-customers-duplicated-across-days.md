# STAT-BUG-021：跟进客户数在周/月内跨天重复

更新日期：2026-07-14

## 现象

员工客户分析默认按周展示。旧实现先按天统计去重客户数，再由 Service 将每天数字相加。同一客户周一、周二各被跟进一次时，会被当作两个跟进客户。

## 根因

`COUNT(DISTINCT biz_id)` 的去重边界停留在 SQL 的每日分组，Service 聚合到更大区间时已经失去客户 ID，无法再次去重。

## 修复关键

- SQL 改为按日期、客户 ID 分组，返回当日首次跟进时间和客户 ID。
- Service 在每个请求区间内按客户 ID `distinct` 计数。
- 记录数仍按日求和，不改变“跟进次数”口径。

## 验证

- 周一客户 1、周二客户 1/2：跟进次数 3、跟进客户数 2。
- Ubuntu 26.04 容器 CRM 全量 62/62。

## 遗留风险

大范围、高活跃客户数据会增加投影行数；需在性能基线阶段压测并决定是否建立统计读模型。

## 状态

已关闭。
