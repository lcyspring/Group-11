# CRM-ACTIVITY-BUG-001：任务终态轨迹丢失原状态

- 模块：CRM / 活动 / 任务状态机
- 状态：已关闭
- 分支：`develop`

## 现象与根因

任务执行完成、未完成或取消时，实现先修改内存对象的状态，再使用同一对象生成动作轨迹，导致
`fromStatus` 被覆盖为目标状态。数据库中的最终状态正确，但审计会错误显示为“完成 → 完成”等
自迁移，无法还原真实业务动作。

## 修复与验证

状态命令在更新前保存旧状态，乐观更新成功后使用旧值和目标值写不可变轨迹；并发更新失败不写
轨迹。活动 Service 专项和 CRM 354 项 Ubuntu 26.04 回归通过；真实任务轨迹为
`CREATE → START → COMPLETE`。
