# CRM-INV-BUG-001：红冲外部幂等键依赖回滚后不稳定的自增 ID

- 发现/关闭日期：2026-07-14
- 级别：P0 / 外部一致性

## 现象与风险

早期红冲实现先插入红票，再使用 `原蓝票 ID + 红票自增 ID` 作为外部适配器请求号。若外部平台
已成功但本地事务随后回滚，MySQL 自增值通常不会回退；再次提交会得到新红票 ID 和新请求号，
外部平台可能把同一次红冲执行两次。

## 根因

把数据库持久化技术 ID 错当成跨事务重试稳定的业务身份。该问题在 local-ledger 下不产生外部
副作用，但会污染未来税务平台适配契约。

## 修复与验证

请求号改为 `invoice:red:{原蓝票 ID}:{税务红票号}`。税务红票号在同租户唯一且由同次命令携带，
回滚重试时保持不变。Service 测试验证请求号，Provider 返回不同请求号时事务直接回滚。
