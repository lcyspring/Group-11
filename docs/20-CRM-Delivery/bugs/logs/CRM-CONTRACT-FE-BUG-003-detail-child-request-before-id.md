# CRM-CONTRACT-FE-BUG-003：合同详情子页在合同编号就绪前请求

- 日期：2026-07-16
- 分支：`develop`
- 级别：P1 / 前端可靠性
- 状态：已关闭

## 现象

进入合同详情时并发出现 `contractId=undefined`、`must not be null`、参数类型转换失败和参数缺失提示，详情页无法稳定展示。

## 根因

父页面以空对象初始化合同，却立即挂载全部业务页签；生命周期面板的立即监听在详情接口返回前调用生命周期和补充协议接口。TypeScript 非空断言只影响编译，不会阻止运行时传出 `undefined`。

## 修复关键

- 仅在 `contract.id` 有效后挂载依赖合同编号的全部页签；
- 生命周期加载函数和监听器分别增加空编号防线；
- 路由编号统一转为正安全整数，非法值不进入接口；
- 增加源码契约测试，防止父级挂载门禁或子级防御被移除。

- 合同初始化与生命周期防御测试：2/2；
