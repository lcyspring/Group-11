# CRM-PRODUCT-BUG-002：产品分类编辑弹窗语言域重复

- 发现日期：2026-07-17
- 分支：`develop`
- 状态：Closed

## 现象

产品分类配置的新增、编辑弹窗中，标题、父级分类、占位提示或校验信息出现英文或原始国际化键，与当前界面语言不一致。

## 根因

组件已经使用 `crm.product` 命名空间，却再次请求 `product.category.*`，最终组成不存在的重复路径；弹窗标题还使用通用 `Create/Update`，没有产品分类语义。

## 修复

分类字段统一使用相对键 `category.*`，并为新增、编辑标题分别补充中文、英文和阿拉伯文翻译。确认、取消和成功提示继续使用全局公共语言键。

## 回归要求

专项测试验证命名空间只拼接一次，六个分类字段键和两个标题键均存在，三种支持语言都提供显式标题。
