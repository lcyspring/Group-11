# STAT-SEC-002：统计接口可伪造部门或人员越权查询

状态：Fixed

日期：2026-07-15

分支：`develop`

## 现象

客户、漏斗、业绩、画像和排行榜接口只校验统计菜单权限。调用方可修改查询字符串中的 `deptId/userId`，
原 Service 会直接展开该部门的全部下级人员，未与登录用户的平台部门范围和 CRM 组织可读范围求交。

## 根因与修复

对象详情读取已经有 CRM 授权切面，但聚合统计只有功能权限门，没有针对筛选范围的 Server 端数据门。
新增统一统计范围注解、切面和校验服务：验证部门存在、人员属于部门树，并要求人员或完整部门树被当前
组织可读负责人范围覆盖。组织相关 API 异常、缺少登录身份或缺少范围参数均失败关闭，返回错误码
`1020014006`。

## 回归

覆盖 ALL、SELF 本人、SELF 他人、本部门全覆盖、父部门夹带未授权子部门、人员与部门不一致、空部门、
组织 API 异常和缺失登录身份；同时验证五组通用统计 Controller 均挂载统一门禁。
