# PODMAN-CONFIG-BUG-014：手写配置解析器与 KDL 迁移产生双重语义

## 发现

配置文件已批量改为 KDL，但公共读取层仍由 AWK 手工识别引号、注释、布尔和两层节点；数据库、营销
Provider、演示数据测试还在使用 YAML 格式的 `sed/awk` 文本替换。

## 根因

早期 YAML 读取器只覆盖项目所需的极小子集。迁移时若仅更换扩展名和函数名，会把同一套手写语法判断
继续带入 KDL，并使测试与实际配置格式产生分叉。

## 影响

- KDL 语法是否有效取决于自写 AWK，而不是标准解析器；
- 重复节点、转义字符串和结构深度可能在不同脚本中得到不同结果；
- 测试生成的临时配置仍可能是 YAML 文本，无法真实覆盖生产读取链路；
- 后续扩展配置会同时修改解析器、业务脚本和测试，维护成本持续增加。

## 修复

固定并本地编译 dasel `v3.11.2`，公共读取层改为由 dasel 原生解析 KDL；统一拒绝重复节点、数组、
空值和第三层结构。测试配置改写统一调用 dasel，不再保留 YAML 文本解析或替换逻辑。

## 回归

- 157 份正常 KDL 全量解析通过；
- 125 份 Git 旧 YAML 与新 KDL 的 dasel 结构化结果等价；
- 重复节点和非法深度两个负向 fixture 均返回退出码 2；
- 全部 Podman Shell 脚本通过 `bash -n`；
- 演示数据生成器契约测试通过；
- deploy/stop/build-images/build-image-archives 四个无状态 check 通过且未改变容器、镜像和数据。
