# CRM-CLUE-POOL-BUG-002：无团队权限的线索可能被误当作公共对象

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

历史权限切面对没有团队权限的 READ 请求只校验客户公海，非客户对象会直接返回。线索团队
权限缺失或异常清理后，普通用户可能按已知编号读取并非公共池的在管线索。

## 修复

线索无团队权限时必须读取显式状态，只有 `PUBLIC`、未转换且负责人为空的对象继承公共只读；
在管、已转换、不存在或状态不一致均返回对象权限不足。

## 验证

权限切面专项覆盖公共线索允许和非公共线索拒绝。真实 SELF 数据范围临时角色读取公共详情
返回 `code=0`，读取他人拥有且无直接授权的线索返回 `1020007001`。
