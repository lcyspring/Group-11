# CRM-CORE-BUG-035：已关联回款计划可删除且并发绑定非原子

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-FIN-001
- 状态：代码关闭/待真实并发补证

## 现象与根因

计划删除不检查 `receivable_id`，可留下回款指向已删除计划；回款创建又采用普通读取后更新，
两个并发请求都可能在插入回款后绑定同一计划，后写覆盖先写。

## 修复关键

删除、修改和回款绑定统一锁计划行；已关联计划禁止修改/删除，绑定在锁内复核空值，第二个
并发事务抛错并随外层事务回滚其回款。

## 验证与遗留

专项覆盖关联删除拒绝和重复绑定拒绝；CRM 149/149。真实 MySQL 双事务绑定待补压力证据。
