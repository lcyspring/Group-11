# OA 日程与任务页面显示为空

## 现象

日程管理和任务工作台进入后组件整体空白。生产 chunk 在 `setup()` 中调用裸 `dayjs()`，但没有生成模块导入，浏览器在首次 API 请求前即抛 `ReferenceError: dayjs is not defined`。此外日程查询还使用带空格的时间格式，后端 `LocalDateTime` 参数转换会返回 400。

## 修复

- 日程 API 将展示格式转换为 ISO `yyyy-MM-ddTHH:mm:ss`；
- 两个页面显式导入 `dayjs`，不依赖未生效的自动导入；
- 日程和任务页面为无数据场景增加明确空状态；
- 空状态直接提供“创建第一条”操作，页面始终可操作。

## 验证

专项测试覆盖 ISO 转换和两个页面的空状态入口。
