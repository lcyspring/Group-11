# CRM-CORE-BUG-031：回款和计划接受零数或负金额

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-FIN-001、金额守恒
- 状态：已关闭

## 现象与根因

请求 VO 只校验金额非空。绕过前端 `min=0.01` 后可提交 `0` 或负数；负回款会增加合同
剩余额度并污染回款、业绩和统计口径。回款计划存在相同后端缺口。

## 修复关键

回款和回款计划请求统一增加 `@DecimalMin("0.01")`，将页面约束落实到服务端输入边界。

## 验证

验证测试覆盖回款/计划的零数、负数和最小 `0.01`，3/3 通过；CRM 全量 143/143。
