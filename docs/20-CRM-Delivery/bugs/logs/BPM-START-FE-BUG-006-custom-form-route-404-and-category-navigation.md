# BPM-START-FE-BUG-006：发起流程卡片 404 与分类导航无反馈

## 现象

审批中心“发起流程”中的 CRM 销售、CRM 财务、办公协作卡片点击后进入 404；左侧分类点击后右侧内容没有明确变化。

## 根因

- 流程模型把 Vue 组件源码路径（例如 `/crm/contract/index`）误写为浏览器路由；
- 请示审批实际菜单路由是 `/bpm/collaboration/work-request`，模型却使用 `/bpm/oa/work-request/index`；
- 分类交互依赖动态 `$refs + offsetTop` 控制 Element Plus 内部滚动容器，坐标和引用结构不稳定。

## 修复

- 受管 BPM 模型的发起路径改为真实浏览器路由，审批详情路径改为明确的 Vue 详情组件；
- 跳转前按配置原值解析并拒绝 Vue Router 的 catch-all 404 匹配，不再裁剪 `/index` 或映射旧路径；
- 无效配置显示明确错误并返回流程列表；
- 左侧分类改为确定性筛选，点击后右侧只展示所选分类，不再依赖滚动坐标。

## 验证

Ubuntu 26.04 中 8 组前端专项用例通过，显式路径校验、404 拦截、分类筛选、详情组件严格解析及
YAML 路由契约行/分支/函数覆盖率均为 100%。
