# OA-MENU-BUG-003：左侧出现无效重复日程管理

## 现象

左侧菜单同时出现两个“日程管理”，其中一个不能正常进入业务页面。

## 根因

数据库已通过 `system_menu` 提供动态日程路由，前端 `remaining.ts` 又注册了同名、同路径的静态顶层
路由。静态路由没有隐藏属性，因而与动态菜单重复进入侧栏并发生路由名称冲突。

## 修复

删除静态 `OaEvent` 路由，只保留权限系统下发的 `bpm/oa/event/index` 动态菜单。日程 CRUD 权限和页面
组件均保持不变。

## 验证

Ubuntu 26.04 Web 编译通过。部署后权限接口只返回一个“日程管理”：菜单编号6236、路径 `event`、
组件 `bpm/oa/event/index`；源码不再注册第二个 `OaEvent`。
