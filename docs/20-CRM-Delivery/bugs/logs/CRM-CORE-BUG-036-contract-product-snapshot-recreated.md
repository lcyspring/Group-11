# CRM-CORE-BUG-036：合同修订重建产品行导致成交快照漂移

- 发现/关闭日期：2026-07-14
- 级别：P0 / 历史交易数据完整性
- 关联：GAP-OPP-002、ADR-003

## 现象与根因

合同详情返回产品行 `id`，但保存请求的产品模型没有 `id`。编辑合同时所有旧产品行都会被
判断为删除、所有请求行都会重新插入。若产品目录已经改名、改单位、改价或下架，合同修订会
丢失原行身份，并把交易时信息刷新成当前目录信息。

## 修复

- 保存请求显式接收产品行 `id`；
- 更新时校验行编号属于当前合同且不重复；
- 产品未变化时保留旧名称、编码、单位和目录价快照；
- 新增或切换产品时才读取当前目录生成新快照；
- 已下架产品通过表单中的历史快照选项继续可见。

## 验证

- 合同服务修订快照场景通过；
- 外部产品行编号负向场景通过；
- Ubuntu 26.04 CRM 全量 173/173。
