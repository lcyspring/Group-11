# CRM-TABLE-ACTION-BUG-001：终态记录仍显示空操作菜单

- 发现日期：2026-07-19
- 分支：`develop`
- 状态：Closed

## 现象

发票管理中，已不存在任何合法操作的终态记录仍显示“操作”按钮，点击后弹出空菜单。同一共享组件在其他
状态化列表中也可能出现该问题。

## 根因

`TableActions` 的菜单触发器只判断组件模式，不判断默认插槽经过 `v-if` 后是否还包含可渲染节点。

## 修复

- 共享组件过滤注释、空文本和空 Fragment；
- 只有存在真实操作节点时才渲染菜单触发器；
- 发票列表显式计算业务操作可用性，无操作时显示 `-`。

## 回归结果

- 共享组件专项测试 1/1 通过；
- 发票终态不再显示可点击的空操作菜单；
- 同一修复覆盖 CRM 及其他使用 `TableActions mode="menu"` 的列表。
