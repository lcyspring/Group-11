# CRM-FIN-BUG-008：财务域缺少报销业务真源

- 发现日期：2026-07-15
- 分支：`develop`
- 状态：后端与数据库已修复；前端和流程部署跟踪中

## 现象

财务域已有回款、发票和退款/冲销表及接口，但没有 CRM 报销、费用明细或费用分类表。“财务模块数据库表结构（回款/发票/报销/退款）”实际只完成三项，无法通过验收。

## 根因

早期边界决策把报销延后到 OA 与财务边界确认，后续没有冻结业务对象归属，导致数据库、接口、审批和统计均无可实现的真源。

## 修复

接受 ADR-017：报销真源归 CRM 财务域、BPM 只负责编排。新增分类、主单、明细和动作轨迹四表，并实现服务端金额汇总、合同客户一致性、对象权限和幂等审批状态迁移。

## 回归要求

SQL 两次幂等执行、服务测试、权限逻辑与真实 Podman CRUD 已通过。管理端页面和可执行流程定义作为后续独立工作项跟踪，覆盖率记录在 `testing/crm-reimbursement/`。
