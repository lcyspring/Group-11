# CRM-WORKORDER-DB-BUG-007：工单导出权限误挂到工单统计页面

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

部署工单导出权限时生成了两个同名权限节点，其中一个错误挂在“工单统计”页面下。

## 根因

历史菜单中客服工单和工单统计都使用 `work-order` 路径。初版迁移只按路径选择父节点，不能唯一
区分两个业务页面。

## 修复

- 父节点改用唯一组件路径 `crm/workorder/index` 识别；
- 幂等清理误挂节点及对应语言、角色关联；
- 新增迁移契约测试，禁止重新使用有歧义的路径作为父节点条件。

## 验证

兼容迁移重复执行成功。运行库仅保留一条 `crm:work-order:export`，父节点为“客服工单”，CRM
管理员角色关联为一条。
