# CRM-CORE-BUG-029：回款审批回调非幂等且旧流程可串写

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-FIN-002、GAP-APR-002
- 状态：代码关闭/待真实流程并发补证

## 现象与根因

旧监听器只传业务 ID 和终态；重复回调在非审批中状态抛错，重提后也无法区分上一流程的
迟到回调。普通读取再更新还可能让并发终态互相覆盖。

## 修复关键

监听器传递流程实例 ID；同流程同终态幂等成功，旧流程和乱序事件告警后忽略；有效事件按
“回款 ID + 当前流程 + 审批中”条件更新，只有首个并发终态生效。

## 验证

专项覆盖取消映射、重复驳回和旧流程通过；CRM 143/143。真实 Flowable 双终态压力测试
尚未执行，因此保留待运行补证状态。
