# CRM-FE-BUG-007：确认框与业务请求共用空 catch

状态：Fixed

日期：2026-07-19

分支：`develop`

## 现象

客户、线索、联系人、商机、合同、产品、回款等核心页面把确认框、API 请求、文件下载和列表刷新放在
同一个 `try` 中，再用空 `catch` 收尾。用户取消操作时没有提示是合理的，但服务端拒绝、网络失败或刷新
失败也被静默吞掉，表现为按钮“没反应”或数据没有变化。

## 根因

脚手架将 Element Plus 确认框的 `cancel` 拒绝值与业务异常混为一类。后续页面复制了同一结构，形成跨
CRM 模块的重复异常处理缺陷。

## 修复关键

- 新增纯函数 `resolveDialogAction`，只识别 Element Plus 的 `cancel` 和 `close`；
- 其他拒绝原因保持原异常并交给统一请求错误处理；
- 删除、导出、领取公共线索、转公共池和转垃圾池均先结束确认阶段，再执行业务请求；
- 导出 loading 只包围真实下载请求，取消确认不会进入加载状态；
- 清除 CRM Vue 目录全部空 `catch`，同时删除回款列表的一处 `any + as unknown as`；
- 增加递归源码门禁，后续新增空 `catch` 将直接导致测试失败。
