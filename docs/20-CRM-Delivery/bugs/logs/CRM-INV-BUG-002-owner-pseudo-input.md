# CRM-INV-BUG-002：发票草稿请求暴露可传但不会生效的负责人字段

- 发现/关闭日期：2026-07-14
- 级别：P1 / API 契约与越权风险

## 现象与根因

早期草稿 Save VO 同时暴露 `contractId` 和 `ownerUserId`，Service 又用合同负责人覆盖客户端值。
这形成“接口文档允许修改、实际静默丢弃”的伪契约，也容易在后续重构时误把客户端负责人写入，
造成跨归属越权。

## 修复与验证

- 拆分 Create、Update 和公共草稿字段 VO；
- 创建请求只允许选择合同，客户和负责人从已审批合同继承；
- 修改请求不再包含合同、客户或负责人；
- Service 使用显式字段白名单构造更新对象，不依赖 Bean Copy 后置空字段；
- 单元测试捕获更新 DO，确认合同、客户、负责人、状态和方向均不会被写入。
