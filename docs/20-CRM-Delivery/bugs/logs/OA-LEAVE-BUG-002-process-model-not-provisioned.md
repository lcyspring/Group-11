# OA-LEAVE-BUG-002 请假流程模型未随部署配置

## 现象

进入请假新建页面时提示“OA 请假的流程模型未配置，请检查”，无法发起审批。

## 根因

请假业务代码使用流程定义 Key `oa_leave`，但 BPM 聚合配置只包含出差、借款、客户拜访和财务流程，遗漏了请假模型。新建或重建数据库卷后不会自动创建 `oa_leave`，导致前端无法查询到可用流程定义。

## 修复

- 新增可跟踪的 `bpm-provision-leave.example.yaml`，显式定义审批角色、权限、分类、表单路由和模型 Key。
- 将请假模型加入 `bpm-provision-all.example.yaml`，确保标准部署自动配置。
- 将本机 ignored 聚合配置同步加入请假模型，并对当前运行环境执行幂等配置。
- 将示例配置纳入运行配置门禁和中文配置说明。

## 验证要求

- `oa_leave` 存在已部署且可用的流程定义；
- 请假创建页能加载模型，不再显示未配置提示；
- 审批角色包含请假查询与 BPM 待办处理权限；
- 重复执行配置不产生重复模型或角色。
