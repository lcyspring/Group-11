# CRM-FUNNEL-UI-BUG-004：阶段明细混用结单结果且弹窗溢出

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

“吃饭 - 阶段商机明细”中，“商机阶段”一列显示“赢单”“输单”，与弹窗标题矛盾；较低视口下
分页器同时被窗口底部裁切。

## 根因

通用商机响应为了列表展示，会在商机已经结单时把 `statusName` 转换为终态名称。漏斗明细直接
复用该字段，把互相独立的“配置阶段”和“结单结果”混成了同一列。

弹窗没有视口级高度限制，内容高度由表格和分页器共同撑开。

## 修复

- “商机阶段”固定显示当前精确查询的配置阶段；
- 新增“结单结果”列，独立显示赢单、输单、无效，未结单显示 `-`；
- 中、英、阿三套语言资源补齐新字段；
- 弹窗内容限制在视口内并提供内部滚动。

## 验证

Ubuntu 26.04 容器前端专项 17/17 通过，专项测试覆盖率 100%；CRM 后端全量 537/537 通过。
