# STAT-BUG-025：客户转化组件包含非法 UTF-8 字节

更新日期：2026-07-14

## 现象

`CustomerConversionStat.vue` 被识别为 Non-ISO 扩展字符文件，严格 UTF-8 转换在
第 23 字节即失败，文件内多处注释显示为 `图�?` 等乱码。

## 根因

历史文件混入损坏的多字节字符；此前构建工具容忍并替换，掩盖了源文件编码问题。

## 修复关键

仅删除 22 个非法字节并保留有效 UTF-8 内容，随后正常应用业务补丁；没有通过
重新生成组件掩盖差异。

## 验证

`file` 识别为 Unicode UTF-8；Ubuntu 26.04 Vite production build 成功。

## 状态

已关闭。
