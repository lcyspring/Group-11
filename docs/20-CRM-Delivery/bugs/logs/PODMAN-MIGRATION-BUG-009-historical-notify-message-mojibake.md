# PODMAN-MIGRATION-BUG-009：历史工单站内信保留乱码正文

- 发现/关闭日期：2026-07-15
- 级别：P1 / 数据一致性

## 现象与根因

通知模板修复后，新消息恢复中文，但已发送的 `system_notify_message.template_content` 保存的是旧模板渲染结果，
页面仍显示乱码。正文由乱码静态模板和正确 UTF-8 JSON 参数混合组成，整串反向字符集转换会破坏工单标题。

## 修复关键

- 按稳定 `template_code` 处理分派、退回、完结三类历史消息；
- 从 `template_params` 提取工单号、标题和原因，重新拼装中文正文；
- 只处理 JSON 有效且必需参数存在的乱码记录，迁移可重复执行；
- 不对整串正文做 charset 反转。

## 验证

历史 ID 11～15 均恢复，示例为：`客服工单 W-202607-0003「工单分派与筛选运行验收」已分派给你，请及时处理。`
站内信双重编码特征计数为 0。
