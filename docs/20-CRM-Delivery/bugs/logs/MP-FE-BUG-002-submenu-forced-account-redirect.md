# MP-FE-BUG-002：公众号子菜单被强制跳回账号管理

## 现象

未配置公众号账号时，进入用户、标签、消息、素材、草稿、发布、菜单、自动回复或统计等子页面，
页面会立即跳转到公众号管理的第一项“账号管理”，用户无法停留并确认当前功能状态。

## 根因

共享账号选择器把“账号列表为空”与“必须离开当前路由”等同处理，在每个子页面挂载后直接调用
路由跳转。此前删除重复通知时只移除了 Toast，没有同时解除强制导航，形成了交互回归。

## 修复

- 无账号时保留用户主动选择的公众号子页面；
- 页面内显示说明和“配置公众号”操作，仅用户点击后进入账号管理；
- 账号查询失败显示独立失败态和重试操作，不把网络异常伪装成未配置；
- 账号选择器补齐 `v-model` 初值和变更事件，避免父子选择状态漂移；
- 继续禁止空账号 Toast 和自动关闭当前页签。

## 验证

Ubuntu 26.04 组件专项 4/4、ESLint 零警告、production build 通过。运行库有效公众号账号为
0，最终部署资源的空账号分支直接返回并渲染当前页说明；账号管理路由只绑定用户主动点击按钮。
