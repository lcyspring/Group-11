# CRM-WORKORDER-BUG-005：工单列表缺少筛选与勾选导出

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

客服工单列表没有离线导出入口，无法按当前筛选条件归档，也不能只导出人工勾选的工单。

## 根因

工单域只实现分页查询，没有导出响应模型、导出接口、独立权限和前端下载交互。

## 修复

- 增加 `crm:work-order:export` 权限和幂等数据库迁移；
- 增加 Excel 导出接口，复用现有数据权限并强制执行授权范围；
- 未勾选记录时导出全部筛选结果，勾选后只导出选中记录；
- 支持跨页保留选择，变更筛选条件时清理旧选择；
- 导出文件使用可读的类型、优先级、状态、来源、客户、处理组、处理人和协作人名称，避免输出内部状态编码；
- 补齐中、英、阿三套界面文本。

## 验证

CRM 全量 540/540 通过；前端工单专项 9/9 通过，ESLint 0 warning。勾选编号仍通过工单授权查询，
不能借助导出越过个人、处理组或全量查询范围。
