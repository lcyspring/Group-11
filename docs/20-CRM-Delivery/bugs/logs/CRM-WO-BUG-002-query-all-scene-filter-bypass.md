# CRM-WO-BUG-002：全量权限忽略显式个人视图筛选

- 发现/关闭日期：2026-07-14
- 级别：P1 / 权限与数据范围
- 关联：US-WO-002、GAP-WO-001、GAP-IAM-003

## 现象与根因

工单分页把 `sceneType=1/2` 的“我创建/我处理”判断放在 `!queryAll` 分支中。拥有
`crm:work-order:query-all` 的用户显式选择个人视图时，条件被完全忽略，结果仍为租户全量。
这不会跨租户，但违反用户选择并可能在调度页面暴露非预期数据。

## 修复

先处理 backlog，再无条件应用显式 `sceneType`；只有未选择场景且没有 `query-all` 时，才应用
创建人或处理人的默认并集范围。增加 Mapper 查询包装器测试，覆盖全量用户的两个显式场景及
未选择场景的租户全量行为。

## 验证

- `CrmWorkOrderMapperTest`：3/3；
- Ubuntu 26.04 CRM 全量：170/170。
