# CRM-CLUE-POOL-BUG-005：历史空转换状态会被公共池迁移遗漏

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

历史 `crm_clue.transform_status` 和 `follow_up_status` 允许为空。公共池分页与自动迁移使用明确
的未转换条件，SQL 三值逻辑会遗漏 NULL，造成一部分历史未转换线索既不在公共池，也不会被
自动迁移。

## 修复

幂等迁移先把空转换和跟进状态归一为 `false`，再回填 OWNED/PUBLIC 状态、负责人时间、入池
快照和归属事件。已转换记录继续保持终态。

## 验证

完整启动与三次 `rebuild-server` 均重复执行迁移成功；NULL 归一、CHECK、索引、菜单和角色
授权没有产生重复记录或状态漂移。
