# CRM-CORE-BUG-027：回款编辑状态门禁反向

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-FIN-002、GAP-APR-002
- 状态：已关闭

## 现象与根因

旧 Service 允许草稿和审批中回款编辑，却禁止驳回/取消回款修订。审批过程中可改变日期、
方式和金额，审批人看到的内容可能与最终落库不一致；原型要求的修订/重提同时被阻断。

## 修复关键

编辑时锁定回款，只允许草稿、审核不通过、已取消状态；驳回/取消保存后显式回草稿。
提交时先锁合同再锁回款，按锁后最新金额复核额度并创建新流程。前端同步提供“修订”。

## 验证

专项覆盖审批中拒绝、驳回和取消修订、新流程重提 4 条核心路径；回款专项 20/20、CRM
143/143、Web production build 通过。
