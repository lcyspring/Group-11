# CRM-FE-BUG-005：计划页面自行计算草稿金额且列名错误

- 日期：2026-07-14
- 级别：P1
- 关联：GAP-FIN-001、GAP-RPT-002
- 状态：已关闭

## 现象与根因

列表、详情和待办直接用嵌套回款金额计算已收/未收，不区分审批状态；待办还把实际回款日
误标成成交周期，并将未回款金额列重复标成已回款金额。

## 修复关键

所有页面改用 API 的 `status/receivedPrice/unreceivedPrice`；只有已回款状态显示实际日期，
统一状态标签和三语文案，关联后同步禁用计划编辑/删除入口。

## 验证

目标文件 Prettier、ESLint 和 Ubuntu 26.04 production build 通过。
