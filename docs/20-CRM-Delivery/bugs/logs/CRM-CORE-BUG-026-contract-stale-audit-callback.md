# CRM-CORE-BUG-026：合同审批回调非幂等且旧流程可串写

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-APR-002、ADR-006
- 状态：代码关闭/待真实流程并发补证

## 现象与根因

旧监听器只传合同 businessKey 和结果，不传流程实例 ID；Service 对非审批中状态抛错。
因此重复终态事件会失败，合同重提后旧流程的迟到回调也无法与当前流程区分。读取状态后
普通更新还可能被两个并发终态相互覆盖。

## 修复关键

监听器同时传递流程实例 ID。Service 对同流程同终态幂等成功，对旧流程和乱序状态告警后
忽略；有效事件按“合同 ID + 当前流程 ID + 审批中”执行条件更新，只有首个终态生效。

## 验证

单元测试覆盖重复驳回、旧流程通过、取消映射及 CAS 更新分支；CRM 全量 124/124。真实
Flowable 双回调压力场景尚未执行，因此状态保留“代码关闭/待运行补证”。
