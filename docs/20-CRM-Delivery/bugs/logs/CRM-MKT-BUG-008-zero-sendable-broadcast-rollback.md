# CRM-MKT-BUG-008 群发无有效收件人时无法诊断

## 问题现象

新建短信群发时提示“没有可用的 CRM 联系人或客户名单”，群发草稿和收件人解析结果同时回滚，用户无法判断是手机号缺失、邮箱缺失、未授权还是已退订。

## 原因分析

保存草稿和校验是否允许提交审核被放在同一个事务阶段。解析结果全部被抑制时，服务在保存末尾抛出异常，导致本来已经生成的逐收件人原因无法保留。

## 修复

- 草稿保存始终保留完整收件人解析结果，包括被抑制的记录和原因。
- 仅在提交审核时阻止有效收件人为零的任务。
- 错误信息明确提示查看收件人结果并补充联系方式或渠道授权。
- 前端保存后检测有效收件人数；为零时自动打开收件人结果，不再只显示不可操作的红色异常。
- 收件人结果为草稿或已驳回任务提供“授权”和“退订”操作，渠道授权必须由有权限的用户显式确认。
- 授权状态变更后按原客户/联系人目标集合重新解析联系方式、授权状态及抑制原因，并同步刷新有效人数和任务列表。
- 重新解析不再从客户编号、联系人编号的并集反推目标，避免额外加入客户主联系人或改变原目标集合。
- 已提交审核、待发送或已发送任务禁止重建收件人，避免审批中途改变发送范围。

## 验证要求

- 缺手机号的短信目标保存为“已抑制 / 手机号为空”。
- 零有效收件人的草稿可以再次编辑。
- 零有效收件人的草稿不能提交审核。
- 补齐联系方式并在收件人结果中完成渠道授权后，可直接重新解析并正常进入审核流程。
- CRM 后端测试 486 项全部通过。
- 群发前端专项测试 12 项全部通过，行、分支、函数覆盖率均为 100%。
- Ubuntu 26.04 环境下 Server 与 Web 生产构建通过。
