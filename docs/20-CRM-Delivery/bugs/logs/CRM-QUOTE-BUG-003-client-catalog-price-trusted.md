# CRM-QUOTE-BUG-003：客户端可伪造商机目录价

- 模块：CRM / 商机产品
- 状态：已关闭
- 严重性：P0
- 分支：`develop`

## 现象与根因

商机保存曾直接接受请求中的 `productPrice`。攻击或旧客户端可提交任意目录价，使折扣、报价历史
和合同成交快照失去可信产品目录来源。

## 修复关键

- 服务端批量读取有效 CRM 产品并覆盖商机目录价；
- 报价明细再次从产品主数据生成名称、编码、单位、分类、版本和目录价快照；
- 商机转合同只读取锁定报价，完全忽略客户端产品行和标准价。

## 验证

真实 API 故意提交目录价 `999999`，V1 和合同均返回服务端目录价 `100.000000`；合同单测同时
验证伪造行编号不会被复用。
