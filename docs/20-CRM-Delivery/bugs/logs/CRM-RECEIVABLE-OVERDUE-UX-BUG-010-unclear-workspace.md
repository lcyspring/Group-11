# CRM 回款逾期管理入口和处置不清晰

## 现象

待回款与逾期回款共用“待回款提醒”入口，只能通过下拉框判断；页面未说明逾期含义、逾期时长和建议处置，操作名称“创建回款”也不符合登记到账的业务语义。

## 修复

- 左侧使用单一“回款计划与逾期”入口，避免与页面内状态下拉重复；
- 入口数量复用服务端统一提醒统计，页面内按待回款、已逾期、已完成切换；
- 逾期页面明确说明判定口径与处理步骤；
- 显示当前结果总数和每条计划的逾期天数；
- 操作改名为“登记回款”；
- 待回款、逾期、已完成切换时标题和说明同步变化。

## 验证

专项测试覆盖单一入口、内部三状态筛选、逾期说明、逾期天数与登记回款动作；管理端生产资产由
Ubuntu 26.04 工具链容器构建通过。
