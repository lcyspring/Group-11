# OA-EVENT-BUG-007：日程列表直接显示毫秒时间戳

状态：Fixed

日期：2026-07-19

分支：`develop`

## 现象

日程列表的开始时间和结束时间直接显示 `1784908800000` 一类毫秒值，编辑日程时同一响应值也无法稳定
回填到使用格式化字符串的日期范围控件。

## 根因

项目的 Jackson 时间配置把 `LocalDateTime` 响应序列化成毫秒时间戳，日程 API 却把响应类型声明为字符串，
列表和编辑弹窗因此直接消费了未经标准化的运行时值。

## 修复关键

- 在 OA 日程 API 边界增加唯一的响应时间标准化函数；
- 数字时间戳、13 位数字字符串和已有日期字符串统一转换为 `YYYY-MM-DD HH:mm:ss`；
- 无效或空时间不再产生 `Invalid Date`；
- 列表显示和编辑回填共享标准化后的同一数据，不在组件内重复修补；
- 自动化测试验证时间戳往返、数字字符串、非法时间和 API 映射调用。
