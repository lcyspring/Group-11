# OA-EVENT-BUG-006：清空日程日期后发送非法时间参数

状态：Fixed

日期：2026-07-19

分支：`develop`

## 现象

清空日程管理的日期条件后，前端请求携带 `from=InvalidTDate`，Spring 无法把它转换成
`LocalDateTime`，页面无法展示全部日程。

## 根因

页面无条件使用被清空的锚点计算日/周/月范围，并无条件把两个范围参数传给 API；后端又把 `from/to`
声明为必填，数据访问层也无条件拼接时间重叠条件，因此整个契约不支持“无日期条件”。

## 修复关键

- 日期锚点为空或非法时不再计算时间范围，API 不发送 `from/to`；
- 保留日期控件的清空能力，清空语义明确为查询当前用户全部有效日程；
- 后端将 `from/to` 改为可选参数，Mapper 仅在参数存在时追加时间条件；
- 上一周期/下一周期在无日期条件下以今天为基准恢复范围查询；
- 回归门禁同时校验前端空范围、Controller 可选参数和 Mapper 条件拼接。

## 回归

- OA 日程源码契约测试通过；
- BPM 后端测试、ESLint `--max-warnings=0`、Vite 8 production build 和三语言包校验通过。
