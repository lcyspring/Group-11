# STAT-BUG-029：未审批回款提前完成回款计划

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-FIN-001/002、GAP-RPT-002、ADR-007
- 状态：已关闭

## 现象与根因

回款草稿创建时立即写入计划 `receivable_id`。计划查询、待办和前端只判断该 ID 是否存在，
因此草稿、审批中、驳回和取消都会被标为已回款，并提前冲减未回款金额。

## 修复关键

统一以关联回款 `audit_status=APPROVE(20)` 作为计划完成条件；API 显式返回执行状态、已生效
金额和未回款金额，未审批回款仍保留关联历史但不进入生效口径。

## 验证

Mapper 2/2，Controller 同时覆盖待回款、逾期、已回款三态及金额；CRM 149/149，Web
Prettier、ESLint 和 production build 通过。
