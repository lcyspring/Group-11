# CRM-CLUE-POOL-BUG-006：手工退回原因未保留在线索池快照

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

手工退回公共线索时，用户填写的具体原因只写入归属历史记录。线索主表仅保存
`MANUAL_PUT_POOL` 原因编码，因此公共列表和详情只能显示“手动退回”，无法向领取人说明本次
退回的实际原因；历史轨迹与当前公共池快照也不一致。

## 修复

在线索主表新增 `pool_reason_detail` 当前入池原因快照。手工退回原样保存用户填写的原因，自动
入池保存可解释的策略原因；领取或主管分配后清空本轮池快照。公共列表和详情优先显示原因详情，
缺失时再回退为稳定原因编码的本地化文案。

## 验证

Ubuntu 26.04 CRM 全量自动化 324/324、公共线索专项 ESLint 和 production build 通过。最新迁移
首次执行及 `rebuild-server` 重复执行成功；真实手工退回后，公共分页和详情均原样返回“客户明确
要求两周后再联系，退回公共池供团队重新分配”。
