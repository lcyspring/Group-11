# CRM-CLUE-POOL-BUG-007：公共线索重复出现在普通线索列表

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

公共线索拥有独立列表后，普通线索分页仍只依赖对象权限范围，没有显式限定 `OWNED`。管理员或
保留历史对象权限的用户可能在普通列表和公共池同时看到同一线索；普通列表操作按钮也只判断
转换状态，可能向公共线索暴露编辑、删除等不符合池状态机的入口。

## 修复

普通线索分页在服务端强制限定 `pool_status=OWNED`，公共线索只能通过公共池分页进入。管理端
普通列表的编辑、删除和手工退回入口同时要求线索处于 `OWNED`，后端状态机继续作为最终门禁。

## 验证

Ubuntu 26.04 CRM 全量自动化 324/324、普通线索与公共线索专项 ESLint 和 production build
通过。同一测试线索在退回前由普通分页返回 1 条，退回后公共分页返回 1 条且普通分页返回 0 条；
公共详情继续可读，测试数据已清理。
