# CRM-BPM-BUG-001：合同、回款和退款服务已接 BPM 但流程定义缺失

- 发现日期：2026-07-15
- 分支：`develop`
- 状态：Closed

## 现象

合同、回款和退款/冲销服务分别使用 `crm-contract-audit`、`crm-receivable-audit` 和
`crm-receivable-refund-audit` 发起流程，但 Flowable 运行库没有这些定义。接口和监听器存在并不等于用户可以提交审批。

## 根因

项目关闭资源目录自动部署，每个业务流程必须经过 BPM 模型治理创建和发布。此前运行验收只覆盖 Java 状态逻辑，缺少租户内的可执行定义和明确审批角色。

## 修复

- 将治理脚本泛化为一份 YAML 管理一个流程模型；
- 新增合同、回款、退款三份无凭据示例配置；
- 角色、审批账号、权限码、分类、模型键、表单路径和节点名称全部来自 YAML；
- 审批角色权限按权限码解析菜单及祖先菜单后合并；
- 四类 CRM 模型重复运行均比较规范化模型内容，配置未变化时跳过发布。

## 回归结果

- 合同、回款、退款和报销四个 key 均有且仅有一个有效定义；
- 四份本机 YAML 再次执行均跳过部署；
- `TK202607-0002` 完成提交、待办、驳回、修订、新流程实例重提和通过；
- 动作轨迹为 `1/3/5/2/3/4`，退款金额 `321.450000`；
- 原回款已通过金额占用更新为 `321.450000`，剩余可退 `15694.550000`；
- 当前四类 CRM 审批没有遗留运行实例。
