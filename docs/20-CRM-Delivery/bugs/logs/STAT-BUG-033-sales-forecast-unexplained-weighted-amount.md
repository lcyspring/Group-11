# STAT-BUG-033：销售预测概率金额不可解释且缺少实际对照

日期：2026-07-18。分支：`develop`。状态：已修复。

## 现象

销售预测展示未解释的“概率金额”，预测周期入口不明确，响应不能直接回答预测金额与实际金额的差异。

## 原因

旧实现把当前阶段赢单率乘以商机金额生成概率金额，但没有历史概率快照；同时商机没有独立实际结单
时间，无法按真实赢单日期聚合实际金额。

## 修复

- 增加 `crm_business.end_time`，历史终态按更新时间回填，新终态在状态事务中显式写入；
- 按预测周期与分组粒度返回预测商机数、实际赢单数、预测金额、实际赢单金额；
- 预测金额使用活跃商机预计成交日期，实际金额使用赢单商机实际结单时间；
- 删除页面概率金额，并在页面显式说明两项金额的数据来源。

## 回归

后端聚合、SQL 契约、结单时间写入、前端字段与三语说明均有自动化覆盖；迁移进入 bootstrap 和
compatibility 清单并执行幂等部署测试。
