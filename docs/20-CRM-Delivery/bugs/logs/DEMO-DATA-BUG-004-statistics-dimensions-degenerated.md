# DEMO-DATA-BUG-004：演示数据关系退化导致 CRM 统计缺图或缺阶段

## 现象

旧数据集中合同与商机均为 50 条，合同加载会把全部商机更新为赢单；客户地区、行业、来源和等级为空，
开放商机缺少预计成交日期。扩大数据后首次运行还发现阶段和结单结果使用同一个模数，五个阶段中有两个
阶段始终为零。数据行数存在，但漏斗、预测和画像仍然不能形成完整图表。

## 根因

- 核心对象按“每类同数量”生成，没有维护商机、合同、计划和回款之间的业务基数；
- 生成器只校验总行数，没有校验统计维度是否实际有值；
- 阶段序号与结单结果采用相同周期，两个独立维度发生确定性相关；
- 大量 `NOW()` 和空画像字段使时间窗口与分组统计缺少真源。

## 修复

- 调整为 180 个商机、72 份合同、144 个回款计划和 90 笔回款，保留开放、输单、无效和未回款事实；
- 创建批次独立的五阶段销售流程，阶段分布与结单分布改用互质周期；
- 补充预计成交、实际结单、地区、行业、来源、等级及稳定时间分布；
- 财务、营销、OA、工单生成多状态数据，并让工单状态与操作轨迹一致；
- manifest 增加 `04-validate.sql`，任何空阶段、缺画像、缺预测真源、缺逾期/未来计划或孤儿引用都会终止部署。

## 验证

- 开放 / 赢单 / 输单 / 无效：76 / 72 / 16 / 16；
- 五个开放阶段：15 / 15 / 16 / 15 / 15；
- 预测 / 实际金额均大于零；客户画像维度为 8 / 8 / 10 / 5 类；
- 未回款计划中逾期和未来各 27 条；
- `deploy.sh` replace、SQL 内置断言、Server `UP`、Web `200` 均通过，运行配置已恢复 `preserve`。
