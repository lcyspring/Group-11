# OA-TRIP-BUG-002 出差新增后列表未自动刷新

## 现象

新增并提交出差申请后返回出差管理，刚创建的记录不会立即显示，必须手动刷新页面。

## 根因

出差列表只在组件首次挂载时查询。系统使用页签缓存，创建页返回列表时复用已有列表组件，
不会再次触发 `onMounted`，因此页面保留创建前的数据。

## 修复

- 首次挂载完成后记录初始化状态；
- 缓存页签重新激活时自动查询最新列表；
- 保留分页和筛选条件，不通过整页刷新绕过状态同步；
- 增加工作流契约测试，防止重新退化为仅首次加载。

## 验收预期

创建出差申请并返回列表后，新记录无需手动刷新即可显示；从详情或审批进度返回时，状态变化也会自动同步。

## 关联修复：审批详情请求 `id=NaN`

审批详情以内嵌组件方式加载出差业务表单时，业务编号通过 `id` 属性传入；原出差详情却只读取 URL
查询参数，因此把缺失值转成 `NaN` 并请求 Long 类型接口。现统一优先读取组件属性、兼容独立详情页查询参数，
并在请求前校验为正安全整数，非法编号不会再发送到后端。
