# DEMO-DATA-BUG-007：replace 后残留旧业务数据

## 现象

执行演示数据替换后仍保留旧客户和旧商机，页面结果混入无法由当前 seed 重现的记录。

## 根因

原 cleanup 只按当前 `DEMO2-<seed>` 批次标识删除。上游演示数据、其他 seed 批次和未带批次标识的
旧业务事实均不在清理条件内，因此实现的是“重放当前批次”，不是数据集替换。

## 修复

- 生成配置显式声明 `replacement_cleanup_scope: tenant-crm-demo`；
- cleanup 完整删除目标租户 CRM 业务/主数据及演示 OA 日程任务；
- 保留管理员、客户公海策略、合同策略和工单策略等可复用配置；
- 删除所有历史生成批次的用户、角色、角色菜单和登录令牌，不再只匹配当前 seed。

## 验证

生成后断言租户客户、联系人、线索和商机总数必须与 YAML 完全相等，任何旧事实残留都会使替换失败。
本机实际替换后为客户160、联系人240、线索160、商机180；不匹配当前批次业务键的客户和商机均为0。
