# STAT-BUG-019：跟进次数与跟进客户数字段互换

更新日期：2026-07-14

## 现象

员工客户分析中，Service 分别得到跟进记录数与去重客户数，却把记录数写入 `followUpCustomerCount`，把客户数写入 `followUpRecordCount`。按日期和按员工接口都存在同样问题。

在正常数据中，一位客户可能有多条跟进，所以页面会把较大的跟进次数显示为客户数、较小的去重客户数显示为次数。

## 根因

链式 setter 的两个参数位置手工写反；既有测试没有使用两个不同数值验证响应字段语义。

## 修复关键

- `followUpRecordCount` 只接收记录数聚合。
- `followUpCustomerCount` 只接收 `COUNT(DISTINCT biz_id)` 聚合。
- 按日期、按员工各增加 7/3 数值哨兵测试，避免同值输入产生假阳性。

## 验证

- 两个专项用例 2/2。
- Ubuntu 26.04 容器 CRM 全量 61/61。

## 状态

已关闭。
