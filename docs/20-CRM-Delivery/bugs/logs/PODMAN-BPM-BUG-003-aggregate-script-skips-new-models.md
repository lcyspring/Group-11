# PODMAN-BPM-BUG-003 聚合脚本跳过新增流程模型

## 现象

`bpm-provision-all-local.yaml` 已配置请假和客户拜访，但 `deploy.sh full` 输出只恢复回款、报销、合同、退款、
出差和借款，新增模型被静默跳过。

## 根因

`provision-bpm-models.sh` 使用硬编码的旧六项循环，没有同步聚合 YAML 后续加入的 `leave` 和
`customer_visit`。

## 修复

- 将聚合发布顺序扩展为八项完整模型；
- 运行配置门禁逐项检查脚本是否包含聚合清单要求的模型；
- 对当前运行环境重新执行幂等发布。

## 验证

标准完整部署及手工聚合发布当前输出九个模型组；重复执行全部复用现有角色、分类和模型。
