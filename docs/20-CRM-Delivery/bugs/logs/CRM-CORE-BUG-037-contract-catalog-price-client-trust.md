# CRM-CORE-BUG-037：合同产品目录价由客户端提供

- 发现/关闭日期：2026-07-14
- 级别：P1 / 数据完整性
- 关联：GAP-OPP-002、ADR-003

## 现象与根因

合同创建与更新直接把请求中的 `productPrice` 保存为产品目录价。调用者可以提交与产品目录
不一致的值，导致合同展示的“产品单价”并非成交时目录价，且无法审计其来源。

## 修复

新产品行的目录价只从服务端 CRM 产品目录读取；已有且产品未变化的行保留原目录价快照。
请求仍保留该字段以兼容旧前端，但服务端不再信任其值。合同成交价 `contractPrice` 仍是允许
协商和修订的业务字段。

## 验证

自动化提交伪造目录价 `999.00`，持久化结果为目录真实价格 `88.00`；Ubuntu 26.04 CRM
全量 173/173。
