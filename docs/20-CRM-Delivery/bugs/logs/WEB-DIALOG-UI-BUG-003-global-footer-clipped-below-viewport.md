# WEB-DIALOG-UI-BUG-003：多模块弹窗底部操作区被视口裁切

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

多个模块的表单或明细弹窗在低高度窗口或内容较多时超出浏览器底部，确认、取消和分页区域不可见，
用户只能缩放页面或无法继续操作。

## 根因

共享 `Dialog` 组件只在调用方显式设置 `scroll` 时限制正文高度。大多数业务弹窗使用默认配置，
Element Plus 弹窗随正文自然增高，页头、正文和页脚也没有形成受视口约束的纵向布局。

## 修复

- 非全屏弹窗统一限制为 `calc(100vh - 32px)`；
- 弹窗容器改为纵向 Flex，头部和底部操作区固定参与布局；
- 正文设置 `min-height: 0` 和内部滚动，内容增长不再把页脚推出视口；
- 全屏模式保持 `100vh`，不影响原有全屏切换。

## 验证

增加共享弹窗视口契约测试，并与群发结果、销售漏斗弹窗专项一起在 Ubuntu 26.04 容器执行。
