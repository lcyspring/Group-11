# STAT-BUG-028：“已回款金额”提前计入未生效回款

- 日期：2026-07-14
- 级别：P1
- 关联：GAP-FIN-001、GAP-RPT-002
- 状态：已关闭

## 现象与根因

合同精简列表和详情使用 `selectReceivablePriceMapByContractId` 汇总已回款金额，但查询把
草稿、审批中、审批通过全部求和。草稿在提交额度规则中明确不占用，审批中也尚未成为业务
确认终态，却会提前减少页面未回款金额和新建回款默认金额。

## 修复关键

汇总条件收敛为仅 `CrmAuditStatusEnum.APPROVE(20)`；审批中的金额仍只用于提交额度预占，
不冒充已回款。

## 验证

Mapper 契约测试断言查询只含状态 `20`，明确排除 `0`、`10`；CRM 143/143，前端统计
7/7，production build 通过。

## 语义边界

这里的“已回款”仍只表示 CRM 业务审批通过，不宣称银行到账或会计入账；后者等待
ADR-007/008。
