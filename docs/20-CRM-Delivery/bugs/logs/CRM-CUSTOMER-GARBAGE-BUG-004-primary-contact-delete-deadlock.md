# CRM-CUSTOMER-GARBAGE-BUG-004：首联系人保护阻塞垃圾客户永久删除

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

永久删除要求客户不存在联系人，而联系人模块禁止删除客户当前唯一首联系人。客户一旦
创建过联系人，就无法仅通过业务接口解除引用保护，形成规则死锁。

## 修复

- 联系人删除命令显式接收当前操作人；
- 只有 YAML 配置的 CRM 管理员且客户处于 GARBAGE 时，允许删除唯一首联系人；
- 在管、公海、普通用户和非首联系人逻辑保持原规则；
- 状态判断在客户锁和联系人当前读之后执行，避免并发状态漂移。

## 验证

新增“垃圾池管理员可删除”和“管理员在公海仍不可删除”两项测试；真实 API 在收到
`1020006032` 后删除首联系人并成功永久删除客户。
