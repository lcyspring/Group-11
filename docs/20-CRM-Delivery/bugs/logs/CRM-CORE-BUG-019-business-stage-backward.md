# CRM-CORE-BUG-019：商机阶段可通过状态接口倒退

更新日期：2026-07-14

## 级别与关联

- 级别：P0（销售漏斗数据正确性）；
- 关联：`GAP-OPP-001`、`US-OPP-004`、`BC-03`；
- 状态：已关闭。

## 现象

处于较高排序阶段的活动商机，可以在“变更商机状态”下拉框选择较低排序阶段；直接调用
`PUT /crm/business/update-status` 也能成功写入回退阶段。漏斗因此不再反映真实的前向
销售进展。

## 根因

前端加载并展示同一状态组的全部阶段；后端只调用
`validateBusinessStatus(statusTypeId, statusId)` 校验归属，没有比较当前与目标阶段的
`sort`。页面和服务端都缺少前向状态机约束。

## 修复关键

- 前端候选阶段过滤为 `target.sort > current.sort`；
- 服务端验证当前、目标阶段并强制同一比较，防止绕过页面；
- 回退使用独立错误码 `1020002005`；
- 普通推进增加必填说明，并追加到操作日志；
- 重复阶段、结束状态、并发冲突的既有错误语义保持不变。

## 验证

- 请求校验 7/7；商机 Service 5/5；
- CRM 105/105，Failures/Errors/Skipped 均为 0；
- Ubuntu 26.04 Web Prettier、ESLint、production build 通过；
- JaCoCo 分支覆盖率 25.20%。

## 剩余边界

向前跨级是“禁止”还是“二次确认”在既有文档中互相冲突，本修复不擅自选择；当前只关闭
明确的阶段倒退缺陷。
