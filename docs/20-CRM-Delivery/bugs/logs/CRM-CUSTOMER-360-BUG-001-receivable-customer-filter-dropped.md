# CRM-CUSTOMER-360-BUG-001：回款权限分页丢失客户过滤

日期：2026-07-14

分支：`develop`

状态：已修复

## 现象与影响

`CrmReceivablePageReqVO` 已声明 `customerId`，但带 CRM 对象权限的通用分页 Mapper 没有把该
字段拼入查询。调用方按客户加载回款时若复用通用分页，会返回当前用户有权查看的其他客户
回款，造成客户 360 关系串数据；虽然不是跨权限泄漏，仍违反筛选契约和财务上下文隔离。

## 修复与验证

- 通用权限分页增加 `customer_id = customerId` 条件；
- 将业务筛选构造拆成可单测函数，权限条件仍在原调用链执行；
- 增加回归测试确认 SQL 片段和绑定参数包含客户 88；
- Ubuntu 26.04 CRM 自动化 208/208 通过。
