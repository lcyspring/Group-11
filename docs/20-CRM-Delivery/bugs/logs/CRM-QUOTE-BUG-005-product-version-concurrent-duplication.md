# CRM-QUOTE-BUG-005：并发产品更新可能生成重复快照版本

- 模块：CRM / 产品目录
- 状态：已关闭
- 严重性：P1
- 分支：`develop`

## 现象与根因

新增产品版本字段后，更新逻辑按“先读旧版本、再写旧版本加一”执行，但方法没有事务行锁。两个
并发修改可能都从 V7 写成 V8，使不同主数据内容共享同一快照版本。

## 修复关键

- 产品更新加入事务；
- 读取当前产品改为租户隔离的 `SELECT ... FOR UPDATE`；
- 行锁内完成唯一编码、分类校验、版本递增和更新。

## 验证

新增专项单测验证锁定读取并从 V7 递增至 V8；CRM 全量 364/364 通过。
