# CRM-CORE-BUG-014：客户名称未执行 100 字符业务上限

- 发现日期：2026-07-14
- 修复日期：2026-07-14
- 模块：CRM / 客户管理
- 严重级别：中
- 状态：已修复并验证
- 关联：`GAP-CUS-001`、`TC-CUS-003`

## 现象与影响

客户创建/编辑前端没有 `maxlength` 或长度规则，后端 `CrmCustomerSaveReqVO.name` 也只有非空校验。101 字符及更长名称不会按 UAT 契约返回友好校验错误，可能进入 Service 并形成超出业务规范的数据。

## 根因

字段长度只隐含在存储结构中，没有在 API DTO 与前端表单显式表达，导致业务契约依赖数据库容量。

## 修复

- 后端增加 `@Size(max = 100)`。
- 前端增加 100 字符输入上限、字数显示和表单规则。
- 补充中、英、阿三语提示。
- 新增 100/101 字符边界单测。

## 验证

- 定向测试 2/2。
- CRM 全量 58/58。
- ESLint 和 Web 生产构建通过。
- Podman 真实 API 对 101 字符返回 code 400，数据库有效记录数为 0。

## 遗留风险

CRM 其他文本字段仍应按原型和数据库逐项建立显式业务长度，不能由本修复推断为已全部完成。
