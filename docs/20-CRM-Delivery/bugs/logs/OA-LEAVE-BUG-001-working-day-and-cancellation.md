# OA-LEAVE-BUG-001 请假天数与取消流程标识错误

## 问题现象

- 同一工作日请假显示为 0 天。
- 跨周末请假把周六、周日计入工作日。
- 结束时间等于开始时间仍可提交。
- 取消请假时把业务单编号传给 BPM，导致流程取消失败或找不到流程实例。

## 原因分析

前后端没有共享明确的请假日期口径；原页面也未展示服务端最终计算结果。取消操作混淆了请假业务编号与 BPM `processInstanceId`。

## 修复

- 服务端按起止日期闭区间计算工作日并排除周末。
- 起止时间增加严格先后关系校验。
- 请假原因限制为 10～200 字。
- 响应返回服务端计算的请假天数，页面同步展示。
- 取消流程显式传递 `processInstanceId`。
- 中文、英文、阿拉伯文文案保持一致。

## 验证结果

- BPM 模块：56 项测试，0 失败，6 项项目原有测试跳过。
- OA 请假前端专项：5/5 通过。
- Ubuntu 26.04 容器 Web 生产构建通过。

## 后续风险

当前规则只排除周末，法定节假日、调休工作日、假期余额、附件和审批通过后的幂等扣减仍需继续实现。
