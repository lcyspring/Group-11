# CRM-CLUE-POOL-BUG-003：负责人容量统计使用旧快照导致并发穿透

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

容量守卫虽然把同一用户的事务串行化，但 MySQL 默认 `REPEATABLE_READ` 会复用事务早期用户
校验建立的快照。等待守卫锁的第二个事务仍看到等待前的 99 条，两次分配都成功，真实并发
验收最终得到 101，突破 YAML 上限 100。

## 修复

创建、转移、自助领取和主管分配等增加线索持有量的事务统一使用 `READ_COMMITTED`。事务在
取得负责人守卫后重新统计当前已提交持有量，再执行上限判断。

## 验证

增加事务隔离级别契约测试。真实 MySQL 在 99 条基线下并发分配两条，最终一条成功、一条返回
`1020001007`，持有总量严格为 100。
