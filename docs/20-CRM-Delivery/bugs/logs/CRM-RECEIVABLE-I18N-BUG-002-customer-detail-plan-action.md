# CRM-RECEIVABLE-I18N-BUG-002：客户详情回款计划按钮显示翻译键

- 日期：2026-07-18
- 分支：`develop`
- 级别：P1 / 可用性
- 状态：已关闭

## 现象

客户详情“回款”页签顶部按钮显示 `crm.receivablePlan.createReceivablePlan`，回款金额列和期数选项还混有中文格式文本。

## 根因

客户详情组件调用了未在任何语言中定义的 `createReceivablePlan` 键。回款与回款计划页面还通过字符串拼接写死
`（元）` 和“第 N 期”，切换到英文或阿拉伯文时仍保留中文格式。

## 修复关键

- 为“新建回款计划”补齐中文、英文和阿拉伯文资源；
- 金额列统一使用可插值的 CNY 文案模板；
- 回款期数选项改用三语 `periodOption` 模板；
- 扩展回款计划本地化专项，覆盖客户详情按钮和同类硬编码清零。

## 验证

- 回款计划本地化专项：6/6，通过；
- 三语生产分包完整性：通过；
- Web production build：Ubuntu 26.04 容器通过。
