# PAY-SEC-BUG-001：Provider 回调完整报文写入 INFO 日志

日期：2026-07-18。分支：`develop`。优先级：P0/安全。状态：已关闭。

## 现象与风险

支付、退款和转账匿名回调在进入签名解析前，把完整查询参数和正文写入 INFO 日志。Provider 报文可能
包含交易标识、用户标识、签名和渠道扩展信息；普通运行日志的访问面通常大于支付原始事实，形成不必要泄漏。

## 根因

控制器为早期联调直接记录 `params` 和 `body`，没有区分受控支付审计存储与普通诊断日志。三个入口复制了
相同行为。

## 修复关键

- 支付、退款、转账回调统一只记录 channelId、参数数量和正文长度；
- 不记录参数值、正文、Headers、签名或用户标识；
- 抽出同一摘要函数，避免三个入口再次漂移；
- 增加包含模拟签名、用户和交易号的单测，断言摘要中不存在原始内容。

## 验证与遗留风险

专项单测 2/2，Pay 模块全量 167 个测试中 132 通过、35 个外部集成测试跳过，失败 0、错误 0。
Pay 受控表仍会按现有设计保存渠道原始通知用于审计，应继续通过
Pay 管理权限和数据库访问控制保护；本修复不删除支付事实，也不改变 Provider 签名解析和回调响应。
