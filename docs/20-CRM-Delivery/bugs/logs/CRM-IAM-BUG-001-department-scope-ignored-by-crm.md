# CRM-IAM-BUG-001：CRM 忽略角色部门数据范围

状态：Fixed

日期：2026-07-15

分支：`develop`

## 现象

系统角色已经配置本部门、自定义部门、本部门及下级或全部数据，但 CRM 主表使用 `owner_user_id`，通用
部门权限 SQL 只识别 `dept_id/user_id`。CRM 列表因此仍只有“本人/参与/下属”，详情切面也不读取平台
部门范围，角色配置与实际可见数据不一致。

## 根因与修复

CRM 对象权限和平台部门权限是两套未合并的机制。新增统一授权服务，将平台部门范围转换为可读负责人
用户集合；新增组织范围场景，并在详情 READ 校验中合并。组织范围 API 缺失时收敛为 SELF，防止异常时
开放全量。

## 回归

部门范围转换、ALL、SELF/空返回均有单测；CRM 239/239 通过。
