# CRM-MARKETING-UI-BUG-004：收件人名称退化为编号且弹窗溢出

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

群发收件人结果中的客户、联系人显示为 `#14`、`#17` 等编号；内容较多时弹窗超过可视高度，
分页器贴住或越过窗口底部。

## 根因

前端使用“当前用户此刻可选的群发目标”反查历史收件人名称。管理员查看其他负责人创建的群发、
客户负责人变化或目标不再可选时，候选列表不包含历史编号，只能退化显示 `#ID`。

弹窗内部同时包含筛选、送达统计、表格和分页，但没有视口级高度约束。

## 修复

- 收件人分页响应新增 `customerName`、`contactName`；
- 新增独立响应组装器，按当前页编号批量查询客户和联系人，避免逐行查询；
- 客户级收件人没有联系人时显式处理空 ID；
- 前端优先使用接口名称，仅将编号反查保留为兼容兜底；
- 弹窗内容限制在 `calc(100vh - 150px)` 内并提供内部滚动。

## 验证

Ubuntu 26.04 容器前端专项 17/17、CRM 后端全量 537/537 通过；收件人响应组装器行和方法
覆盖率均为 100%。
