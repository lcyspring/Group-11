# BPM-DETAIL-FE-BUG-007：审批详情错误嵌入业务列表页

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

CRM 回款审批的“审批详情”嵌入了完整的回款管理列表，出现搜索、重置、新增、导出、场景页签和分页，
无法识别本次流程对应的单条回款。合同、报销和退款模型存在同类配置风险。

## 根因

- BPM 的 `formCustomCreatePath` 是浏览器发起路由，`formCustomViewPath` 则是审批页加载的 Vue 详情组件，
  旧配置把两个字段都写成了业务列表路由；
- 前端通过 `includes` 模糊匹配组件路径，`/crm/receivable` 会优先命中
  `views/crm/receivable/index.vue`；
- `businessKey` 从流程接口返回字符串，而部分 CRM 详情组件只声明数字 ID，精确切换到详情组件后仍可能拒绝合法 ID；
- 报销和退款原有详情仅提供弹窗形态，不能直接嵌入审批页。

## 修复

- 自定义详情组件改为完整模块路径精确解析，不再使用子串匹配；
- 前端拒绝浏览器路由和无扩展名路径，不再提供旧列表路径兼容映射；
- 新增幂等、带审计记录的数据修复，将存量流程定义的错误路径一次性改成明确详情组件；
- 新模型 YAML 明确区分发起路由与详情组件路径；
- 回款、合同详情接受流程接口约定的数字或字符串业务 ID；
- 报销、退款拆出可复用详情内容，并新增审批页嵌入组件。

## 验证

Ubuntu 26.04 容器中审批导航与组件专项测试 8/8 通过，路径解析相关行、分支、函数覆盖率均为
100%，相关 Vue/TS/ESLint 门禁通过。运行库 43 条可判定历史路径已完成修复和定义级审计；CRM/OA
受管流程遗留错误路径为 0。未知测试模型值 `2` 保持原样并被严格拒绝，不增加前端兼容分支。
