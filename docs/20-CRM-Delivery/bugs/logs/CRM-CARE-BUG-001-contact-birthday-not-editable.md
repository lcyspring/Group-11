# CRM-CARE-BUG-001：生日查询没有可维护的数据入口

日期：2026-07-18。分支：`develop`。状态：已修复。

## 现象

客户关怀页面提供“客户生日查询”，但 `crm_customer` 没有生日字段；真实查询读取
`crm_contact.birthday`，联系人新增/编辑、响应 VO 和前端类型又没有暴露该字段，用户无法从页面录入
生日，查询长期缺少可靠数据源。

## 原因

营销迁移和关怀服务已经把生日定义为联系人事实，但联系人 CRUD 契约未同步扩展；页面标题同时把
“联系人生日”误写成“客户生日”。

## 修复

- 联系人保存/响应 VO 增加 `LocalDate birthday`；
- 联系人新增、编辑、列表和详情增加生日；
- 根据人工验收要求增加 `crm_customer.birthday` 幂等迁移及客户新增、编辑、列表、详情入口；
- 生日查询增加“客户生日/联系人生日”显式对象选择，分别执行数据权限分页；
- 客户、联系人 API 类型与中、英、阿三语字段补齐；
- 关怀页签与表格明确对象类型，不再把两类生日混为同一字段。
- 客户、联系人和关怀生日响应显式序列化为 `yyyy-MM-dd`，不向前端返回 LocalDate 数组。

## 回归

新增前端源码契约测试，验证日期控件使用 `YYYY-MM-DD`、表单状态/API 类型和三语文案。后端客户关怀
测试分别覆盖客户生日分页、联系人生日分页、当天触达和闰日换算。生日自动触达仍以联系人为收件人
事实来源；客户自身生日用于查询，避免在没有明确联系人授权时推断收件人。
真实 API 创建、读取、清理临时客户，验证生日持久化和字符串响应格式。
