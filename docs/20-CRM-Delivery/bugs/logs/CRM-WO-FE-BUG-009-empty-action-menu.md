# CRM-WO-FE-BUG-009：客服工单空操作菜单

- 日期：2026-07-19
- 分支：`develop`
- 级别：P1
- 状态：已修复

## 现象

客服工单列表的“操作”入口始终显示。对于已完成工单、非负责人且无派单权限的工单，点击入口后弹出空菜单，用户无法判断当前记录是否可以处理。

## 根因

页面只在菜单内部按状态、负责人和 `v-hasPermi` 隐藏按钮，却没有同步判断菜单是否至少包含一项可用操作。权限指令在子节点上生效后，父级菜单仍然保留。

## 修复关键

新增 `hasWorkOrderActions(row)`，复用编辑、派单、领取、开始、退回、完成、重提和删除按钮的状态、归属及权限条件；只有至少存在一项可用操作时才渲染 `TableActions`。这样无操作记录不再显示空菜单，已有按钮行为保持不变。

## 验证

新增工单页面契约测试，确认菜单使用统一可用性守卫，并覆盖处理权限判断；运行 `deno task verify:crm-work-order-collaboration`。
