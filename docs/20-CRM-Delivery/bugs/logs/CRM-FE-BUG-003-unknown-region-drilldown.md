# CRM-FE-BUG-003：未知区域被渲染为可钻取链接

## 现象

区域层级无法解析时，后端按契约返回 `areaId = null`、`areaName = 未知`。初版区域钻取
仍把所有区域名称渲染为链接，点击“未知”会发送缺少区域 ID 的请求。

## 影响

请求无法通过 `@NotNull` 校验并返回 400，用户看到可点击入口但无法获得结果。

## 根因

前端区域响应类型错误地把 `areaId` 声明为必填 number，模板也没有区分可解析区域与
未知汇总项。

## 修复关键

- 将前端 `areaId` 类型改为 `number | null`；
- 只有非 null 区域 ID 渲染为钻取链接；
- “未知”区域保留统计值，但名称使用不可点击文本。

## 验证证据

- TypeScript 模板分支只把 number 传入钻取方法；
- Ubuntu 26.04 Web production build 成功。

## 清理与遗留风险

没有产生测试数据。真实页面仍需用一条无法解析区域的客户数据做浏览器交互复测。
