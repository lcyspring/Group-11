# BPM-TASK-FE-BUG-002：待办查看审批进入不存在的流程并产生未捕获异常

## 现象

CRM 待办事项中的合同、回款“查看审批”可以跳入审批详情，但页面提示查询不到流程信息，浏览器同时
出现 `Uncaught (in promise) error 2`。

## 根因

运行库有一条合同和一条回款仍标记为审批中，但其 `process_instance_id` 在 Flowable 运行表和历史表
中均不存在。这两条 2024 年遗留记录不是可办理待办。审批详情页还会并发启动详情和模型请求，模型
请求未等待详情成功，任一请求失败都可能形成未处理的 Promise 拒绝。

## 修复

- 增加 CRM 孤立审批修复记录表，保留业务类型、业务编号、租户、原流程 ID 和状态变化；
- 幂等检查合同、回款、退款、报销和合同补充协议五类审批聚合；
- 仅将“审批中且 Flowable 历史实例不存在”的记录转为已取消并清空失效流程 ID，使其可修订重提；
- 审批详情改为先加载审批详情，成功后再加载模型视图；
- 空流程 ID、详情失败、模型失败和用户列表失败均在组件内收敛，不再产生未捕获 Promise；
- 刷新路径显式丢弃已处理 Promise，避免 Vue 生命周期报警。

## 运行验收

- 修复前：孤立合同 1、孤立回款 1；
- 修复后：孤立合同 0、孤立回款 0；
- 审计记录 2 条，分别保留原流程实例 ID；
- 第二次执行迁移不新增记录，证明幂等；
- 页面专项、ESLint 和 Ubuntu 26.04 production build 结果记录于测试目录。

运行验收入口为 `podman/tests/acceptance/verify-crm-orphan-approvals.sh`，命令行只接收 YAML 路径，服务账号、租户和
MySQL 连接全部来自显式配置。
