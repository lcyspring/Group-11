# CRM-CLUE-POOL-BUG-001：普通更新可绕过转移命令修改负责人

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

线索更新请求仍携带 `ownerUserId`，服务直接把请求转换为数据对象后更新。持有编辑权限的用户
可以绕过转移接口，修改负责人但不迁移 OWNER 权限，也不写归属事件，造成负责人、权限和
审计轨迹不一致。

## 修复

普通更新强制保留锁内读取的原负责人。负责人只能通过转移、公共池领取或主管分配命令改变，
这些命令统一执行权限、容量和归属事件处理。

## 验证

单元测试提交不同负责人后断言更新对象仍保留原负责人；真实 API 更新名称并伪造负责人 104，
响应成功但详情负责人仍为 1。最新 CRM 324/324 通过。
