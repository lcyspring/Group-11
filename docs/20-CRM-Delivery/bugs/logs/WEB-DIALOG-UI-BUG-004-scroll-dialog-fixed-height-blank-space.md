# WEB-DIALOG-UI-BUG-004：滚动弹窗内容较少时保留大片空白

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

CRM 短信群发的“收件人结果”弹窗在结果数量较少时仍接近占满整个浏览器高度，分页下方和内容区
出现大片无意义空白。该问题也会影响其他启用了 `scroll` 的共享弹窗。

## 根因

共享 `Dialog` 组件对外提供的参数名和注释均为 `maxHeight`，实现时却把该值作为
`ElScrollbar` 的固定 `height`。因此调用方配置的最大高度被错误解释为固定高度，内容无法按实际
尺寸收缩。

## 修复

- 普通模式把 `maxHeight` 传给滚动容器的 `max-height`，少量内容按实际高度展示；
- 只有全屏模式使用固定 `height`，保证全屏滚动区域仍占满可用空间；
- 修复位于共享 `Dialog` 层，群发结果及其他滚动弹窗统一生效；
- 增加自动化契约，防止后续重新把最大高度绑定为固定高度。

## 验证

Ubuntu 26.04 容器内执行群发、漏斗和共享弹窗专项，18/18 通过，ESLint 0 warning；专项统计的
行、分支和函数覆盖率均为 100%。
