# CRM-QUOTE-BUG-001：结束商机的报价仍可被改写

- 模块：CRM / 商机报价
- 状态：已关闭
- 严重性：P0
- 分支：`develop`

## 现象与根因

原商机编辑接口只校验对象存在，没有把商机终态与报价状态合并为同一事务门禁。输单或无效后仍
可能修改产品和金额，导致历史输单原因与报价快照不一致。

## 修复关键

- 商机编辑先锁商机行，再要求当前报价为草稿；
- 输单/无效在同一事务将当前草稿或锁定报价冻结为 `TERMINATED`；
- 已结束商机、锁定报价和终止报价统一拒绝通用编辑；
- 前端按状态隐藏编辑、锁定和重开入口。

## 验证

终止专项单测、赢单/输单回归及真实 API 锁定后编辑负向用例通过。
