# CRM-CORE-BUG-034：回款计划总额和期次可并发突破合同约束

- 日期：2026-07-14
- 级别：P0
- 关联：GAP-FIN-001、金额守恒
- 状态：代码关闭/待真实并发补证

## 现象与根因

旧 Service 创建计划时只取最大期次加一，不校验同合同计划总额，也没有并发锁。两个请求可
得到相同期次，计划合计也可超过合同金额；修改计划同样没有总额校验。

## 修复关键

创建和修改统一先锁合同，按锁后计划集合计算剩余可计划金额；超额返回显式错误。创建期次
也在同一合同锁内分配，使同合同写入串行，不影响不同合同。
