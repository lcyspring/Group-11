# CRM-CONTRACT-AMENDMENT-BUG-001：候选补充协议提前推进有效版本

发现日期：2026-07-16。状态：Fixed。

## 现象

已签署合同创建补充协议成功，但立即提交时返回“合同版本已经变化”。数据库中原合同并没有其他
已生效修改。

## 根因

补充协议创建轨迹使用目标版本号，`currentVersion()` 却直接读取最新轨迹。创建、修改或提交中的
候选版本因此被错误解释为当前有效版本，基线版本校验必然失败。

## 修复关键

新增 `CrmContractChangeRecordMapper.selectLatestEffective()`。当前有效版本只接受原合同生命周期
动作 1～8，以及补充协议审批通过生效动作 12；补充协议创建、编辑、提交、驳回和取消均不推进
有效版本。

## 验证

- 生命周期 Service 增加候选版本不推进、动作 12 推进的回归测试；
- CRM 全量 426/426；
- 第二轮真实 API 验收成功创建、提交并审批生效到下一版本。

未产生需要清理的错误合同投影；失败发生在提交前。
