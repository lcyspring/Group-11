# WEB-TABLE-BUG-001：多模块表格操作项被省略号截断

- 日期：2026-07-16
- 分支：`develop`
- 级别：P1
- 状态：已关闭

## 现象

CRM、ERP、BPM、商城、IoT、系统管理等页面把多个操作按钮塞入宽度不足的固定列。按钮文字被单元格
省略号截断，部分点击区域也被裁切；同一业务在不同语言下表现不一致。

## 根因

- 页面各自维护操作列宽度，没有统一的按钮布局和折叠规则；
- 三到八个动作仍使用普通横排，固定列宽度却只有 150～340 像素；
- 缺少覆盖整个管理端的静态检查，新页面可以继续引入相同问题。

## 修复关键

- 新增通用 `TableActions` 组件；少量操作使用不收缩、不换行的稳定横排；
- 高密度操作统一使用“操作”弹出菜单，菜单项保持原权限、状态条件、禁用条件和事件处理；
- 扫描 788 个 Vue 文件中的 320 个操作列，将 52 个高密度操作列迁移到弹出菜单；
- 86 个双操作列统一为 220 像素稳定横排，约 145 个多操作列统一使用 `TableActions`；
- 新增 `analyze-table-actions.mjs --check`，按按钮数量、列宽和菜单模式阻止高风险操作列回归；
- 新增 SFC 解析门禁，逐个编译检查 144 个使用 `TableActions` 的页面；
- 操作列的固定宽度收敛为菜单模式 140 像素或经过计算的横排宽度。
- 垃圾池旧测试从手工 320 像素契约同步到 `TableActions + 220px` 双操作契约，避免布局治理后测试误报。

## 验证

- 全量操作列静态门禁：320 个操作列，高风险 0；
- `TableActions` 页面解析：144/144；
- 操作、审批刷新、本地化及审批页签契约：6/6；
- 目标目录 ESLint：零错误；
- Ubuntu 26.04 Podman Web production build：通过。
