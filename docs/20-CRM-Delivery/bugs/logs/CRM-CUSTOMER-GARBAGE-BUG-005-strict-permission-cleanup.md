# CRM-CUSTOMER-GARBAGE-BUG-005：隔离后的联系人删除再次要求权限记录

- 日期：2026-07-15
- 分支：`develop`
- 状态：已修复

## 问题

客户转入垃圾池时会主动清空客户及联系人团队权限。管理员解除联系人引用时，删除命令在
对象授权和业务删除都通过后仍调用“权限必须存在”的清理方法，返回 `1020007000` 并回滚。

## 修复

联系人删除后的权限清理改为 `deletePermissionIfPresent`。删除前权限仍由功能权限、对象
权限切面、CRM 管理员角色和垃圾池状态共同校验；这里只让删除后的清理对“已经隔离”幂等。

## 验证

Ubuntu 26.04 CRM 307/307 通过；真实 API 成功删除已无团队权限的垃圾池首联系人，并完成
客户永久删除。
