# DEMO-DATA-BUG-005：全部业务归属密讯导致排行榜只有一人

## 现象

合同金额、回款金额、客户数、联系人、跟进等排行榜均只有“密讯”一行；工单处理人统计也只有一个人。

## 根因

生成器只配置一个 `owner_user_id`，并把客户、联系人、商机、合同、回款、跟进和工单处理人全部写成该
用户。运行租户又只有一个有效系统用户，因此聚合 SQL 即使正确，也不可能产生多人排行榜。

## 修复

- 创建 8 个可登录账号、5 类角色和差异化菜单权限；
- 四名销售稳定分担客户、联系人、线索、商机、合同、回款和跟进；
- 两名财务人员关联发票、报销和退款；两名客服/客户成功人员各处理 80 张工单；
- 用户密码继承显式配置的基准用户密码哈希，账号保持启用；
- cleanup 同步清理用户角色、角色菜单、演示用户和演示角色。

## 验证

- 合同和回款排行榜均形成 4 名销售；
- 工单处理人形成 2 人，各 80 张；
- `d20260718user02` 使用本机演示密码登录返回 `code=0`；
- 8/8 用户有角色，5/5 角色均有菜单，权限差异断言通过。
