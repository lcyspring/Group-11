# CRM-CUSTOMER-POOL-BUG-004：旧三级字典使金牌/VIP 保护业务不可达

- 模块：CRM / 客户等级 / 公海保护期
- 状态：已修复

公海规则按等级大于等于 4 使用双倍保护期，但旧系统字典和后端枚举只有 A/B/C 的 1～3，
页面无法选择 4/5；若只测试 Service 构造的 level=4，便会形成“测试通过、业务不可达”。

幂等迁移将旧 A/B/C 数据保序映射为 5/3/1，并补齐铜牌 2、金牌 4；后端枚举同步升级为
普通、铜牌、银牌、金牌、VIP 五级。重复迁移后字典和客户等级指纹不变，API 已真实创建
level=4 客户并完成入池/领取。
