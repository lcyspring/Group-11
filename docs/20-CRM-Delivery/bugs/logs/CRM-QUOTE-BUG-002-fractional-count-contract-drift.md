# CRM-QUOTE-BUG-002：报价数量请求与数据库精度不一致

- 模块：CRM / 商机、合同产品
- 状态：已关闭
- 严重性：P0
- 分支：`develop`

## 现象与根因

数据库数量列为 `decimal(24,6)`，但部分请求与响应模型仍按整数表达，`1.25` 等计量型数量无法在
商机、报价和合同间无损传递。

## 修复关键

- 商机和合同产品请求、DO、响应统一为 `BigDecimal`；
- 报价明细、合同明细和金额运算统一六位精度；
- 前端数量输入支持小数；
- 真实验收使用 V1 数量 2.5、V2 数量 1.25 覆盖全链路。

## 验证

真实合同返回数量 `1.250000`、行金额 `11.875000`、含税金额 `11.958125`，临时数据清理为 0。
