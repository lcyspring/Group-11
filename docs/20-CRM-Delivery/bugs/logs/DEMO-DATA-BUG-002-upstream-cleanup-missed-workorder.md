# DEMO-DATA-BUG-002：旧演示数据清理遗漏工单交易表

日期：2026-07-18。分支：`develop`。优先级：P1/数据。状态：已关闭。

## 现象

运行 YAML 已明确选择 `existing_dataset_policy: replace`、允许 cleanup 并确认持久数据变更，
`deploy.sh` 也按顺序执行旧数据 cleanup 和新数据 insert；但首次对账仍残留 1 条旧工单。

## 根因

`cleanup-upstream-crm-demo.sql` 建立时间早于工单模块，只清理原 CRM 客户、商机、合同和回款等表，
没有覆盖后来新增的工单、抄送、签到、动作轨迹和 SLA 实例表。部署配置与生成器均不是根因。

## 修复关键

- 在现有旧演示数据 cleanup 中补齐五张工单交易表；
- 保留工单分组、成员、节假日和 SLA 策略等可编辑基础配置；
- 继续使用原 `deploy.sh + replace 双确认`，不增加替换脚本或生成器职责。

## 验证

- 客户总数/新批次：1000/1000，旧客户 0；
- 商机总数/新批次：1600/1600，旧商机 0；
- 工单总数/新批次：2000/2000，旧工单 0；
- 全部兼容迁移成功，Server 健康状态 `UP`。
