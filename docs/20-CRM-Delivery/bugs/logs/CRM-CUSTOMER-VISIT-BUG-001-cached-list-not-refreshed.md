# CRM 客户拜访返回列表不刷新

## 现象

新建客户拜访成功并返回列表后，新记录不会立即出现，必须手动刷新浏览器。

## 根因

客户拜访列表由 `keep-alive` 缓存，只在首次 `onMounted` 查询。创建页返回时列表组件不会重新挂载，因而没有再次请求分页接口。

## 修复

- 首次挂载完成后记录初始化状态；
- 缓存页面每次 `onActivated` 时重新加载当前筛选条件和分页；
- 保留记录结果成功后的即时刷新，不依赖浏览器刷新。

## 回归

客户拜访 Web 契约增加“创建页返回缓存列表”场景；Ubuntu 26.04 门禁结果记录在 `testing/crm-customer-visit/README.md`。
