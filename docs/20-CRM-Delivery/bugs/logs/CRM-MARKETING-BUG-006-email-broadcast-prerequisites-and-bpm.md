# CRM-MARKETING-BUG-006：邮件群发未校验发送前置条件且审核可被绕过

## 现象

邮件群发点击发送时，模板、SMTP 账号、模板参数和收件人状态没有统一检查，配置缺失只返回笼统的系统异常；待审核任务页面还可以直接点击通过或驳回，审批结果与 BPM 流程实例可能不一致。

## 原因

群发服务只在保存草稿时校验模板 code 是否为空，没有复用系统邮件模板和 SMTP 账号的实际配置；审核按钮调用 CRM 兼容接口，没有强制通过 BPM 状态事件回写。

## 修复

- 增加邮件模板 readiness 契约，检查模板存在/启用、SMTP 账号完整性和缺失参数。
- 群发保存 BPM 流程实例编号，状态监听器只接受匹配的 BPM 终态事件。
- 发送接口和前端发送按钮统一执行 readiness；record-only 明确提示仅留痕，不伪装为真实投递。
- 待审核页面改为“查看审批”，直接通过/驳回入口不再展示。
- 修复 BPM 全量 provisioning 漏掉营销群发模型、Server 单独替换流程未执行 provisioning，以及营销模型 KDL 示例块格式错误导致流程定义未发布的问题。

## 验证

- CRM 前端营销契约测试：2 个文件全部通过。
- CRM 后端全量测试覆盖发送、审批、收件人和提供商结果路径。
- 本地 Podman 部署后确认 `crm-marketing-outreach-audit` 已发布且处于激活状态，`crm_marketing_broadcast.process_instance_id` 字段存在。
