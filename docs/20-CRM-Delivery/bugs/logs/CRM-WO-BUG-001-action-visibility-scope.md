# CRM-WO-BUG-001 工单动作按钮数据权限可见性

## 问题

功能权限只说明用户是否具备“处理工单”能力，不能说明当前工单是否分派给该用户。
若前端只按状态和权限字符显示按钮，非处理人也会看到开始、退回和完结操作，
点击后才被后端拒绝，造成越权能力暴露和误导。

## 修复

- 后端始终校验当前用户等于 `handler_user_id`，并在参与者范围校验后再返回动作错误；
- 前端同时校验权限字符、状态和当前用户是否为处理人；
- 创建人修改、重提和删除也同时校验当前用户与 `creator`。

## 验证

`nonHandlerCannotStartOrder` 及工单状态测试通过；非参与者返回统一无权查看，
不会通过错误差异泄露工单存在或处理人信息。
