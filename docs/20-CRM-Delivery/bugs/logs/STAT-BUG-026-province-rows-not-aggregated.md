# STAT-BUG-026：区域转换后未聚合导致省份重复

## 现象

画像 SQL 按客户保存的区县/城市 `area_id` 分组，Service 将每一行的 ID 替换成省份
ID 后直接返回。同省份存在多个原始区域时，响应会出现多个相同省份。

## 影响

- 省份地图接收重复名称，显示值取决于图表处理方式；
- 省份客户数和成交客户数无法作为单一汇总值核对；
- 原型城市、国家页面没有独立数据维度。

## 根因

区域层级转换只完成了“维度映射”，遗漏映射后的第二次 `GROUP BY`。

## 修复关键

- 抽取按 `AreaTypeEnum` 聚合的统一服务方法；
- 以目标城市/省份/国家 ID 为 key 累加客户数与成交数；
- 无法解析的区域统一归入“未知”；
- 按客户数降序和区域 ID 升序稳定返回；
- 新增城市、国家 API 和前端页签。

## 验证证据

- 北京东城区 2 人、西城区 3 人合并为北京市 5 人；
- 北京、上海、广东样本合计 15 人、成交 7 人，在国家层汇总为中国且总量守恒；
- Ubuntu 26.04 CRM 83/83、reactor 20/20 SUCCESS；
- Ubuntu 26.04 Web production build 成功。

## 清理与遗留风险

测试只使用内存 Area 元数据和 Mapper stub，没有产生数据库数据。尚缺真实跨国家数据
对账和区域钻取；这两项保留为后续工作。
