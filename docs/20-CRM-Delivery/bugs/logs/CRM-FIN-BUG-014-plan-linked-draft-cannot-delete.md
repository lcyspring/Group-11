# CRM-FIN-BUG-014：关联回款计划的未提交回款无法删除

- 发现日期：2026-07-17
- 分支：`develop`
- 状态：Closed

## 现象

回款管理中部分状态为“未提交”的记录不显示删除按钮。这些记录没有审批流程，但由回款计划创建并保留了计划编号。

## 根因

前端把“关联回款计划”作为隐藏删除按钮的条件，后端也直接拒绝删除带计划编号的草稿。该规则混淆了业务关联与审批状态，导致用户无法清理从未提交的草稿。

## 修复

未提交且不存在审批流程的回款草稿允许删除。若草稿关联回款计划，服务端在同一事务内锁定计划，以计划编号和回款编号进行条件更新，先释放计划占用，再逻辑删除回款及其数据权限。关系已被并发修改时整笔事务失败，避免误解除其他回款。

## 回归结果

两个回款列表均显示符合条件的删除操作；已提交记录仍不可删除。Ubuntu 26.04 中 CRM 483/483，Web production build 和专项契约测试通过。
