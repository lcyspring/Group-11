# PODMAN-WO-PERF-BUG-001：工单并发验收脚本未稳定进入全部场景

日期：2026-07-18。分支：`develop`。状态：已修复。

## 现象

首轮专项在创建场景后因 Bash `local` 同行引用尚未赋值变量退出；修正后，统计场景 50/50 返回应用
错误，因为工单统计接口要求 `interval` 和两个 `times` 参数，而脚本只请求了空参数 summary。

## 原因

- `local stage="$1" raw="...${stage}..."` 在 `set -u` 下展开顺序不可靠；
- 验收脚本没有复用统计接口的必填 VO 契约。

## 修复

- 局部变量声明与依赖赋值分行；
- 统计 URL 显式传递间隔以及 URL 编码的起止时间；
- 失败运行仍由退出 trap 清理，数据库确认临时工单残留为 0；
- 删除失败轮次证据，仅保留完整 250/250 通过的正式运行。

## 回归

正式运行五个场景各 50 次、并发 50，错误 0；最高 p95 355 ms、p99 363 ms；清理残留 0。
