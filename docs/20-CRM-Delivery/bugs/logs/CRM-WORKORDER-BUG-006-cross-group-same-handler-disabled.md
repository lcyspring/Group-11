# CRM-WORKORDER-BUG-006：跨处理组改派时同一处理人被错误禁用

- 日期：2026-07-18
- 分支：`develop`
- 状态：已修复

## 问题

同一客服同时属于 A1、A2 两个处理组。A2 停用后，将原 A2 工单转入仍启用的 A1 时，该客服在
A1 候选人中被置灰，后端也拒绝改派。

## 根因

前后端都只比较了新旧处理人，没有同时比较新旧处理组，把“同一处理人跨组迁移”错误识别成
“处理人未变化”。停用的是 A2 组关系，不应成为 A1 组内成员的全局禁用状态。

## 修复

- 前端仅在处理组和处理人均未变化时禁用候选项；
- 后端仅在处理组和处理人均未变化时拒绝请求；
- 目标组仍必须启用、支持该工单类型，处理人仍必须是目标组成员；
- 保留原工单组、处理人和状态的原子更新条件，避免并发改派覆盖。

## 验证

新增“停用 A2 后同一人员转入启用 A1”的前后端回归断言。CRM 全量 540/540、前端工单专项
9/9 通过，0 失败、0 错误、0 跳过。
