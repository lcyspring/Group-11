# CRM-MKT-FE-BUG-002：营销活动负责人硬编码且维护闭环不完整

## 现象

新建营销活动时，负责人字段默认并固定展示数字用户 `1`。页面没有真实用户选择、编辑、草稿删除、
终止入口和完整业务字段，状态列直接显示数字。若当前租户不存在用户 `1`，保存会失败；即使存在，
也可能把活动错误归属给无关用户。

## 根因

营销活动前端仍是早期验证页面：表单用字面量初始化 `ownerUserId: 1`，没有接入系统用户 API 和当前
登录用户；后端虽已校验负责人和活动状态，但页面未完整暴露既有生命周期，草稿也没有受控删除命令。

## 修复

- 负责人改为可检索的真实系统用户选择，新建时默认当前登录用户；
- 增加编码、名称、负责人和状态筛选，并显示负责人姓名及完整业务字段；
- 增加草稿编辑、启动、锁定、终止和完成入口，终止原因与活动总结显式录入；
- 编辑时读取服务端完整对象，保留既有关联关系；
- 新增草稿删除接口，仅负责人或 CRM 管理员可删除，活动启动后禁止物理删除；
- 新增独立删除权限及 CRM 管理员菜单授权。

## 验证

- 前端专项 6/6，纯函数行、分支、函数覆盖率均为 100%；
- Ubuntu 26.04 容器内 ESLint 和 Web production build 通过；
- 后端草稿删除、状态拒绝、越权拒绝和并发启动保护单元测试通过；
- CRM 全量测试与 JaCoCo 结果记录于对应测试目录。
