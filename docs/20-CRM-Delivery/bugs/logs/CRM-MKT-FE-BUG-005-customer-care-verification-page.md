# CRM-MKT-FE-BUG-005：客户关怀页面停留在验证页

## 现象

客户关怀页面只有压缩在一行的创建表单和只读列表，无法筛选、查看详情、编辑、启停、删除，
也没有祝福记录和客户生日查询入口。

## 根因

前端仅接入了计划保存与分页接口，没有建立客户关怀的维护模型，也未覆盖原型要求的计划、
记录与生日查询三个视角。

## 修复

- 页面重构为关怀计划、祝福记录、客户生日查询三个结构化页签；
- 增加筛选、编辑回显、启停和停用后删除；
- 操作项统一使用菜单，避免列宽变化导致按钮被省略号遮挡；
- 补齐生日、节假日、成交后定期回访及中英阿三语表单校验。

## 验证

Ubuntu 26.04 Web 专项 8/8，行、分支、函数覆盖率均为 100%；ESLint 和生产构建通过。
