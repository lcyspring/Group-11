# CRM-FIN-FE-BUG-002：回款合同选择被客户前置过滤

- 日期：2026-07-16
- 分支：`develop`
- 级别：P1 / 财务体验与权限
- 状态：已关闭

## 现象

新建回款必须先选客户，合同接口随后只返回该客户合同。选择“测试公海逻辑”时下拉只有“我是合同”，用户无法从合同维度识别可回款对象，也看不到合同编号、客户和剩余额度。

## 根因

旧表单复用按客户查询的合同精简接口，并在未选客户时禁用合同选择器；接口只返回名称和金额，且以客户 READ 作为入口，无法准确表达创建回款所要求的合同 WRITE 范围。

## 修复关键

- 新增专用可回款合同候选接口，客户参数改为可选；
- 只返回审批通过、当前用户直接 OWNER/WRITE 可操作的合同，管理员保持全量；
- 占用金额统一包含审批中和已审批回款，与创建金额上限一致；
- 合同选项展示客户、编号、名称和剩余额度，选择后自动带出客户；
- 无剩余额度合同可见但不可选，客户筛选仍然保留；
- API 继续以合同为真实来源覆盖请求客户编号，不能由前端绕过一致性。
