# CRM-CUSTOMER-360-BUG-002：已审批退款未冲减客户净回款

- 发现/关闭日期：2026-07-15
- 级别：P0 / 财务口径
- 关联：GAP-CUS-003、GAP-FIN-001/002、GAP-RPT-002

## 现象与影响

退款/冲销闭环落地后，客户 360 仍直接用审批通过回款总额计算待回款。退款已经审批通过并实际反向确认时，
页面仍显示原回款全部有效，可能把存在退款的客户错误判断为已足额回款。

## 根因

客户 360 SQL 建立时退款真源尚不存在，后续新增退款表后没有更新聚合血缘；响应模型也只有一个
`approvedReceivableAmount`，无法同时表达毛额和净额。

## 修复关键

- Mapper 增加退款记录数与审批通过金额；
- 响应拆分毛回款、退款/冲销和净回款；
- 待回款改为使用净回款；
- 非审批通过状态不进入金额，异常负净额做零保护；
- 客户详情补退款对象权限下的只读明细入口。

## 验证

客户 360 Service 3/3、Mapper 契约 1/1、CRM 232/232、专项 ESLint 和 Ubuntu production build 通过。
