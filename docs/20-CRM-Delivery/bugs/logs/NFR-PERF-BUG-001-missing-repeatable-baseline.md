# NFR-PERF-BUG-001：CRM 缺少可重复的性能与容量基线

## 现象

CRM 已有功能测试和零散请求观察，但无法回答固定并发下各核心接口的 p95、p99、错误率和吞吐，
也无法判断后续变更是否造成性能回退。

## 根因

原验收没有固定只读场景、预热策略、请求量、并发量和显式阈值；单次请求耗时不能构成性能基线，
构建耗时也不能代表 API 容量。

## 修复

- 新增单 YAML 驱动的性能基线脚本；
- 固定客户、商机和三类统计聚合共五个只读场景；
- 增加预热、并发正式采样、业务码校验和分位数统计；
- 记录主机、容器和数据库规模；
- 以 YAML 中错误率、p95、p99 和吞吐阈值决定退出状态。

## 验证

- 500/500 正式请求成功，五个场景错误率均为 0%；
- 最大 p95 为 101 ms，最大 p99 为 122 ms；
- 最低吞吐为 93.21 req/s；
- 五个场景均通过显式阈值。

## 边界

本修复建立本地单节点回退基线，不把当前开发机结果包装成生产容量承诺。
