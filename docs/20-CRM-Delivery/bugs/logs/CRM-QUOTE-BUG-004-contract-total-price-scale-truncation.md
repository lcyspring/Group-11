# CRM-QUOTE-BUG-004：合同金额列截断锁定报价精度

- 模块：CRM / 报价转合同
- 状态：已关闭
- 严重性：P0
- 分支：`develop`

## 现象与根因

真实验收中报价净额为 `11.281250`，合同 API 却返回 `11.28`，其余税额和产品行均正确。根因是
原始 `crm_contract.total_price` 仍为 `decimal(10,2)`，数据库在插入时静默舍入，破坏来源报价
不可变快照。

## 修复关键

- 幂等迁移将合同总金额扩为 `decimal(24,6)`；
- 验收脚本在业务场景前读取 information_schema，强制要求精度为 `24:6`；
- 失败时输出精简合同快照，避免只看到笼统断言错误。

## 验证

迁移后合同完整返回 `11.281250`，真实报价到合同验收通过；兼容迁移重复执行成功。
