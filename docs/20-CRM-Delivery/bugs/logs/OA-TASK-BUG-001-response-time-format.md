# OA 任务工作台截止时间显示原始时间戳

## 现象

任务工作台的截止时间可能直接显示 13 位毫秒时间戳，编辑任务时同一字段也无法稳定回填为日期时间。

## 根因

任务保存请求会把日期字符串转换为 epoch 毫秒，但列表响应未经任何标准化直接交给表格。后端序列化形态变化后，原始数值泄漏到了视图层。

## 修复

- 提供共享的 API 日期时间值标准化函数，支持数值、13 位数字字符串和日期字符串；
- 任务 API 在响应边界将 `dueTime` 统一为 `YYYY-MM-DD HH:mm:ss`；
- 日程和任务复用同一实现，非法值返回空字符串，不在组件中散落兼容分支。

## 回归

专项测试执行真实的时间戳、数字字符串、日期回算和非法输入断言；门禁结果记录在 `testing/oa-task-workspace/README.md`。
