## 4. Bug发现与修复记录

### BUG-001: 消息通知unread-count接口需要手动传receiverUserId

**严重级别**: 中  
**发现阶段**: API测试

**问题**: `GET /notification/unread-count` 要求传 `receiverUserId` 参数，但应从当前登录用户自动获取。

**修复**: Controller中使用 `SecurityFrameworkUtils.getLoginUserId()` 自动获取当前用户ID，移除 `@RequestParam`。

### BUG-002: 通知分页查询500错误 - 数据库表不存在

**严重级别**: 高  
**发现阶段**: API测试

**问题**: `infra_notification` 表在MySQL中不存在，导致查询报500。

**修复**: 在MySQL中创建 `infra_notification` 表，包含所有必要字段。

### BUG-003: 通知查询500错误 - 缺少tenant_id列

**严重级别**: 高  
**发现阶段**: API测试

**问题**: 多租户插件自动在SQL中添加 `WHERE tenant_id = 1`，但 `infra_notification` 表没有 `tenant_id` 列。

**修复**: `ALTER TABLE infra_notification ADD COLUMN tenant_id BIGINT DEFAULT 0 AFTER id;`
