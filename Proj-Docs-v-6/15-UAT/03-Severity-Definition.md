## 1. 严重级别总览

| 级别 | 名称 | 标识 | 说明 |
|------|------|------|------|
| P0 | 致命 | 🔴 Critical | 系统崩溃、数据丢失、核心流程阻断 |
| P1 | 严重 | 🟠 Major | 核心功能不可用、数据错误、安全漏洞 |
| P2 | 一般 | 🟡 Normal | 功能部分异常、边界处理不当、体验问题 |
| P3 | 轻微 | 🟢 Minor | 界面瑕疵、提示语不友好、非关键功能异常 |
| P4 | 建议 | 🔵 Suggestion | 优化建议、体验增强、非功能需求 |

## 2. P0 - 致命（Critical）

### 2.1 定义

导致系统无法正常运行、核心业务流程完全阻断、数据丢失或严重损坏的缺陷。

### 2.2 判定标准

- 系统崩溃、死机、无法启动
- 核心业务流程（招聘、入职、审批、薪资核算、合同签订）完全阻断
- 数据丢失或数据库损坏
- 薪资计算错误导致财务损失
- 安全漏洞导致敏感数据泄露
- 生产环境无法访问

### 2.3 典型示例

| 示例 | 模块 |
|------|------|
| 薪资核算功能计算出错，导致全员薪资错误 | 薪酬管理 |
| 审批流程驳回后无法重新提交，业务完全中断 | 审批中心 |
| 数据库连接池耗尽导致所有请求500错误 | 基础设施 |
| 未授权用户可以访问并导出所有员工薪资数据 | 系统管理 |
| 员工数据批量删除无确认机制且无法恢复 | 员工管理 |
| 考勤统计逻辑错误导致全员出勤天数计算为0 | 考勤管理 |

### 2.4 响应要求

| 要求 | 时限 |
|------|------|
| 确认时限 | 30分钟内 |
| 修复时限 | 2小时内 |
| 验证时限 | 30分钟内 |
| 升级机制 | 立即通知项目经理和技术负责人 |

---

## 3. P1 - 严重（Major）

### 3.1 定义

核心功能模块出现严重问题，影响主要业务流程但存在替代方案，或导致数据不准确的缺陷。

### 3.2 判定标准

- 核心功能不可用但有临时替代方案
- 计算结果错误（非薪资类）
- 权限控制失效导致越权访问
- 关键接口性能严重不达标
- 数据完整性问题（非数据丢失）
- 核心流程中某环节阻塞但可绕过

### 3.3 典型示例

| 示例 | 模块 |
|------|------|
| 批量导入员工功能报错，但单个录入正常 | 员工管理 |
| 组织架构中删除含子部门节点时未校验成功删除 | 组织架构管理 |
| 客户名称重复时后端返回500错误而非友好提示 | 客户管理 |
| 商机阶段推进跳过中间阶段直接到终态 | 商机管理 |
| 角色权限修改后缓存未刷新导致权限不生效 | 系统管理 |
| 合同关联已输单商机未做校验 | 合同管理 |
| 驾驶舱统计数据包含已离职员工 | 数据报表 |

### 3.4 响应要求

| 要求 | 时限 |
|------|------|
| 确认时限 | 1小时内 |
| 修复时限 | 4小时内 |
| 验证时限 | 1小时内 |
| 升级机制 | 2小时内未修复升级到技术负责人 |

---

## 4. P2 - 一般（Normal）

### 4.1 定义

功能存在缺陷但不影响核心业务流程，边界条件处理不当，或非关键功能异常。

### 4.2 判定标准

- 非核心功能异常
- 输入校验不严格（不影响数据安全）
- 边界条件处理不当
- 提示信息不准确
- 页面显示异常但不影响操作
- 性能未达预期但未影响使用

### 4.3 典型示例

| 示例 | 模块 |
|------|------|
| 特殊字符输入未做校验导致页面布局异常 | 招聘管理 |
| 身份证号校验规则不严格，允许输入18位字母 | 员工管理 |
| 销售漏斗金额精度丢失导致合计与明细有0.01差异 | 商机管理 |
| 大数据量导出Excel时超时 | 数据报表 |
| 邮件通知模板变量替换失败 | 消息通知 |
| 合同金额超1000万时前端显示科学计数法 | 合同管理 |
| 收款确认后应收账款余额未实时刷新 | 回款管理 |

### 4.4 响应要求

| 要求 | 时限 |
|------|------|
| 确认时限 | 4小时内 |
| 修复时限 | 24小时内 |
| 验证时限 | 4小时内 |
| 升级机制 | 每日Triage会议追踪 |

---

## 5. P3 - 轻微（Minor）

### 5.1 定义

不影响功能使用，但影响用户体验的界面瑕疵、文案错误、非关键功能小问题。

### 5.2 判定标准

- 界面排版小瑕疵（对齐、间距）
- 提示文案不通顺或存在错别字
- 非关键功能（如头像上传）缺少限制
- 操作不便但有明确替代方案
- 移动端适配小问题

### 5.3 典型示例

| 示例 | 模块 |
|------|------|
| 分页查询超过总页数时返回空数组而非空页提示 | 客户管理 |
| 站内信未读数量角标已读全部后未清零 | 消息通知 |
| 头像上传超大文件无限制 | 员工管理 |
| 移动端GPS定位精度问题 | 考勤管理 |
| 跟进记录时间线排序未按时间倒序 | 客户管理 |
| 修改密码时新旧密码相同未校验 | 系统管理 |

### 5.4 响应要求

| 要求 | 时限 |
|------|------|
| 确认时限 | 24小时内 |
| 修复时限 | 72小时内 |
| 验证时限 | 24小时内 |
| 升级机制 | 纳入迭代计划，不紧急升级 |

---

## 6. P4 - 建议（Suggestion）

### 6.1 定义

不影响功能使用，属于优化建议或体验增强需求，可纳入后续版本迭代。

### 6.2 判定标准

- 用户体验优化建议
- 性能优化建议（非紧急）
- 代码规范改进
- 文档完善建议
- 新增功能建议

### 6.3 典型示例

| 示例 | 模块 |
|------|------|
| 建议列表页增加批量操作功能 | 员工管理 |
| 建议增加暗色模式主题 | 全局 |
| 建议导出报表支持PDF格式 | 数据报表 |
| 建议增加键盘快捷键操作 | 全局 |
| 建议优化大数据量列表的虚拟滚动 | 客户管理 |

### 6.4 响应要求

| 要求 | 时限 |
|------|------|
| 确认时限 | 48小时内 |
| 修复时限 | 纳入下版本迭代 |
| 验证时限 | 48小时内 |
| 升级机制 | 产品经理评估后纳入需求池 |

## 7. 优先级定义

优先级与严重级别的关系：

| 严重级别 | 默认优先级 | 说明 |
|---------|-----------|------|
| P0致命 | 紧急 | 必须立即修复，阻塞发布 |
| P1严重 | 高 | 版本发布前必须修复 |
| P2一般 | 中 | 本版本尽量修复，可遗留≤3个 |
| P3轻微 | 低 | 本版本有资源和时间则修复 |
| P4建议 | 最低 | 纳入后续版本迭代 |

### 7.1 优先级调整规则

| 调整场景 | 调整方式 |
|---------|---------|
| 核心客户反馈的问题 | 优先级可提升一级 |
| 影响演示（Day10答辩） | 优先级可提升一级 |
| 修复成本极高但有替代方案 | 优先级可降低一级 |
| 重新打开的缺陷 | 优先级可提升一级 |

## 9. 升级规则

### 9.1 升级路径

```
测试工程师 → 测试经理 → 技术负责人 → 项目经理
```

### 9.2 升级触发条件

| 条件 | 处理方式 |
|------|---------|
| P0缺陷30分钟未确认 | 升级到测试经理 |
| P0缺陷2小时未修复 | 升级到技术负责人 |
| P0缺陷4小时未修复 | 升级到项目经理，启动应急会议 |
| P1缺陷4小时未修复 | 升级到技术负责人 |
| P1缺陷8小时未修复 | 升级到项目经理 |
| 同一缺陷重新打开2次 | 升级到技术负责人评审 |
| 缺陷验证超时 | 升级到测试经理 |

### 9.3 升级通知方式

| 时段 | 通知方式 |
|------|---------|
| 工作时间（09:00-18:00） | 企业微信群 + 站内信 |
| 非工作时间（紧急） | 电话 + 企业微信 |

## 10. 缺陷度量标准

### 10.1 发布标准

| 指标 | 标准 |
|------|------|
| P0致命缺陷 | 0个遗留 |
| P1严重缺陷 | 0个遗留 |
| P2一般缺陷 | ≤3个遗留 |
| P3轻微缺陷 | ≤10个遗留 |
| P4建议缺陷 | 不做限制 |

### 10.2 缺陷密度

| 指标 | 公式 | 合格标准 |
|------|------|---------|
| 缺陷密度 | 缺陷总数 / 千行代码 | <5 |
| 缺陷移除率 | 已关闭缺陷 / 缺陷总数 | ≥90% |
| 缺陷修复率 | 已修复缺陷 / 待修复缺陷 | ≥85% |
