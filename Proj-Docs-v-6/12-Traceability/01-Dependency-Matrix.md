## 4. 数据依赖关系

### 4.1 数据流向

```
基础数据（G01）
  ├── 用户表(sys_user) ──────→ G02, G03, G05, G06, G07, G08, G09
  ├── 部门表(sys_dept) ──────→ G02, G06, G07, G08, G09
  ├── 角色表(sys_role) ──────→ G02
  ├── 字典表(sys_dict) ──────→ G05, G06, G07, G08, G09
  └── 菜单表(sys_menu) ──────→ 仅G01内部使用

业务数据（G05-G09）
  ├── 招聘数据(resume, interview) ──→ G04（报表）, G06（入职）
  ├── 员工数据(employee) ──────────→ G04（报表）, G07（考勤）, G08（薪酬）
  ├── 考勤数据(attendance) ────────→ G04（报表）, G08（薪酬）
  ├── 绩效数据(performance) ────────→ G04（报表）, G08（薪酬联动）
  ├── 薪酬数据(salary) ────────────→ G04（报表）
  └── CRM数据(customer, order) ────→ G04（报表）

流程数据（G02）
  └── 审批记录(act_hi_*) ──────────→ G05-G09（业务状态更新）

消息数据（G03）
  └── 消息记录(message, sms, mail) ─→ G05-G09（通知送达）
```

## 6. 依赖风险管理

### 6.1 高风险依赖

| 依赖关系 | 风险等级 | 风险描述 | 缓解措施 |
|----------|----------|----------|----------|
| G02→G05-G09 | 高 | 审批流程未就绪，业务模块无法联调 | 提前定义审批接口规范，Mock先开发 |
| G04→G05-G09 | 高 | 报表数据源未就绪 | 报表先使用Mock数据开发，数据对接后切换 |
| G01→G02-G09 | 高 | 基础平台延迟，所有组无法开发 | Day1-Day2优先完成基础API |
| G06→G08 | 中 | 员工数据未就绪，薪酬无法计算 | 薪酬计算引擎独立开发，使用测试数据验证 |
| G07→G08 | 中 | 考勤数据未就绪，薪酬缺考勤数据 | 薪酬支持手动录入考勤数据 |

### 6.2 依赖管理策略

1. **接口先行**：各组在Day1-Day2完成接口定义，先定义接口契约再开发
2. **Mock开发**：所有外部依赖接口提供Mock实现，开发不阻塞
3. **每日对齐**：每日16:00接口对齐会，确保依赖双方进度一致
4. **风险升级**：依赖阻塞超过半日，立即升级至项目经理协调
