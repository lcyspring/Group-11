# Gap Analysis: 原型需求 vs MITEDTSM现有能力

## 1. 分析方法

将 CRM 原型需求与 MITEDTSM 现有模块逐项对比，标记为：

- **✅ 复用**: 现有能力直接满足，无需开发
- **🔧 扩展**: 现有能力可扩展后满足，少量开发
- **🆕 新建**: 无对应能力，需全新开发
- **❌ 超出**: 原型有此需求但MITEDTSM不适用或需重新评估

## 10. 关键发现

1. **最大复用点在基础设施层** (认证/权限/租户/BPM/日志)，而非业务层
2. **订单域扩展度最高** (50%)，可能可复用 module-erp 或现有 module-crm 的订单模型
3. **工单域最独立** (75%新建)，但实体简单
4. **BPM审批流是所有域的公共需求**，需优先完成审批流模板
5. **现有 module-crm 需对照检查**，可能减少客户/回款域的重复开发
