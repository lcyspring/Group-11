## 0. 文档范围说明

本文档为 MITEDTSM CRM + OA 子系统的详细用户故事规格说明，涵盖 **7个Epic、80条用户故事**，覆盖客户域(CUS)、商机域(OPP)、订单域(ORD)、财务域(FIN)、工单域(WO)、营销域(MKT)、办公自动化域(OA)。

**HR子系统（M01-M10）为P2增值功能**，不纳入CRM核心交付范围。其详细用户故事参见独立的SRS模块文档（见第14节追溯矩阵），本文档第2A节仅作模块摘要供规划参考。

本文档与 `01-Requirements-Summary.md` 配合使用 — Summary提供业务全景和优先级，本文档提供每条用户故事的详细验收标准和业务规则。

## 1. 文档约定

每条 User Story 包含以下部分:
- **ID**: 唯一编号 (域缩写-序号)
- **User Story**: As a... I want... So that... 格式
- **原型页面**: 对应的 Axure HTML 原型文件名
- **界面字段**: 从原型中提取的具体表单字段和表格列
- **验收标准**: Given/When/Then 格式, 含具体数据约束
- **业务规则**: 详细的业务逻辑和约束条件
- **边界条件**: 异常情况、空状态、错误处理
- **依赖**: 前置条件和其他模块依赖
- **估算**: T-shirt sizing (XS=0.5d, S=1d, M=2-3d, L=4-5d, XL=6-10d)

## 3. Epic CUS: 客户域 (Customer Domain) — 20 Stories

### CUS-001: 查看客户列表 (全部)

- **As a** 销售员, **I want** 查看所有我负责的客户列表, **So that** 我能快速浏览和定位客户
- **原型页面**: `客户列表.html` (包含 "全部/储备/淘汰" 三个Tab子页)
- **界面元素**:
  - 搜索栏: 搜索关键词(输入框), 客户名称, 客户状态(下拉), 联系人, 所属行业(下拉), 客户来源(下拉), 客户归属(下拉), 首联系人, 创建时间(日期范围)
  - 操作按钮: [查询] [重置] [删除]
  - 表格列: 客户名称, 客户状态, 首联系人, 手机号码, 所属行业, 客户来源, 创建时间, 客户归属
  - 分页: 底部页码+每页条数(10/20/50/100)
- **验收标准**:
  - Given 已登录且有客户数据, When 进入客户列表页, Then 默认显示全部客户(分页), 按创建时间倒序
  - Given 输入"密讯"到客户名称, When 点击查询, Then 返回名称包含"密讯"的客户
  - Given 选择客户状态="意向客户", When 点击查询, Then 只显示状态为"意向客户"的记录
  - Given 选择创建时间范围, When 点击查询, Then 返回该时间段内的客户
  - Given 点击重置, When 查询条件清空, Then 恢复默认全部数据
  - Given 客户归属≠当前用户且当前用户非销售经理, When 查看列表, Then 不显示该客户(数据权限)
- **业务规则**: 默认排序=创建时间 DESC; 模糊搜索匹配客户名称/手机号; 列表支持多选+批量删除
- **边界条件**: 无数据时显示空状态提示 "暂无客户数据，点击[创建客户]开始"; 搜索无结果时显示 "未找到匹配的客户"
- **依赖**: Auth Context (用户登录/权限); system_dict (行业/来源/状态下拉选项)
- **估算**: M (后端 1d + 前端 1.5d + 联调 0.5d)

### CUS-002: 创建客户

- **As a** 销售员, **I want** 创建一个新客户记录, **So that** 我能将潜在客户录入系统进行管理
- **原型页面**: `创建客户.html`
- **界面元素**:
  - 表单区1-基础信息: *客户名称(必填, input), *客户类型(必填, 下拉:企业/个人), *所属行业(必填, 下拉), 备注信息(textarea), 详细地址(input), 客户来源(下拉:广告/介绍/自拓/网销/展会/其他), 负责人员(人员选择器, 默认当前用户)
  - 表单区2-附加信息: 所在城市(下拉), 客户星级(1-5星), 客户状态(下拉, 默认"潜在客户"), 网址(input)
  - 底部按钮: [保存] [取消]
- **验收标准**:
  - Given 填写所有必填项, When 点击保存, Then 客户创建成功, 跳转到客户详情页, 显示成功提示
  - Given 未填客户名称, When 点击保存, Then 显示红色校验错误 "请输入客户名称"
  - Given 未填所属行业, When 点击保存, Then 显示校验错误 "请选择所属行业"
  - Given 客户名称+手机号与已有客户重复, When 点击保存, Then 弹出查重提示 "已存在相似客户[名称+手机], 是否继续创建?"; 确认后仍然创建
  - Given 点击取消, When 表单已填写内容, Then 弹出确认框"确认放弃已填写的内容吗?"
- **业务规则**: 创建成功后自动将当前用户设为客户归属人; 客户星级默认为1星; 客户来源可从字典配置
- **边界条件**: 客户名称最大100字符; 手机号格式校验(支持国内/国际); 网址格式校验
- **依赖**: Auth Context (负责人员选择器); system_dict (类型/行业/来源/城市/状态下拉)
- **估算**: M (后端 1d + 前端 1.5d + 校验 0.5d)

### CUS-003: 查看客户详情

- **As a** 销售员, **I want** 查看客户的完整信息, **So that** 我能全面了解客户情况以制定销售策略
- **原型页面**: `联系人.html` (客户详情+联系人Tab), `客户属性分析.html`
- **界面元素**:
  - 头部信息: 客户名称, 客户归属(如"Jessie"), 客户星级(1-5)
  - 操作按钮: [移入公海] [转移客户] [编辑] [删除]
  - Tab1-基本信息: 客户类型, 所属行业, 客户来源, 客户状态, 所在城市, 详细地址, 网址, 备注, 创建时间, 更新时间, 创建人
  - Tab2-联系人: 联系人子表(姓名/手机/座机/部门职务/QQ/微信/邮箱/角色/最后联系时间), [添加联系人]按钮
  - Tab3-关联商机: 该客户下所有商机列表(商机标题/阶段/金额/时间)
  - Tab4-关联订单: 该客户下所有订单列表(订单编号/标题/金额/状态/时间)
  - Tab5-跟进记录: 跟进时间线(时间/方式/内容/跟进人)
  - Tab6-操作日志: 变更记录(时间/操作人/变更内容)
- **验收标准**:
  - Given 点击客户名称链接, When 进入详情页, Then 所有Tab数据加载完成
  - Given 切换到联系人Tab, When Tab激活, Then 懒加载联系人列表
  - Given 客户归属≠当前用户且非经理, When 访问详情页, Then 返回403无权限
- **业务规则**: Tab内容懒加载(点击Tab时才请求API); 操作日志只读; 客户归属人总是可见全部Tab
- **边界条件**: 无联系人时Tab显示"暂无联系人"; 无商机时显示"暂无关联商机"
- **依赖**: CUS-001 (客户列表); Contact模块; Opportunity模块; Order模块
- **估算**: M (后端 0.5d + 前端 2d 多Tab)

### CUS-004: 编辑客户信息

- **As a** 销售员, **I want** 修改客户信息, **So that** 客户资料保持最新准确
- **原型页面**: `创建客户.html` (编辑模式复用创建表单)
- **界面元素**: 同CUS-002创建表单, 预填充已有数据; 客户归属只读(不可自行修改)
- **验收标准**:
  - Given 点击编辑按钮, When 进入编辑页, Then 表单预填充当前客户所有字段值
  - Given 修改客户名称+行业, When 保存, Then 更新成功, 操作日志记录变更
  - Given 客户归属≠当前用户, When 尝试编辑, Then [编辑]按钮隐藏/禁用
  - Given 销售经理, When 编辑下属客户, Then 允许编辑但归属人不可修改
- **业务规则**: 编辑后记录操作日志(@BizLog); 客户名称变更需再次查重; 客户归属只能通过"转移客户"功能变更
- **边界条件**: 并发编辑: 后保存者覆盖前保存者(乐观锁或最后写入胜出)
- **依赖**: CUS-003 (客户详情); BizLog SDK
- **估算**: S (复用创建表单)

### CUS-005: 删除客户

- **As a** 销售经理, **I want** 删除无效/重复的客户, **So that** 保持客户数据清洁
- **原型页面**: `客户列表.html` (列表页[删除]按钮), `1_1_5_删除简历.html`
- **界面元素**: 确认弹窗: "确认删除客户 [客户名称]? 删除后数据不可恢复。" [确认删除] [取消]
- **验收标准**:
  - Given 选中一个无关联业务的客户, When 点击删除并确认, Then 客户被逻辑删除(deleted=1), 提示"删除成功"
  - Given 客户有关联商机/订单/工单, When 点击删除, Then 提示"该客户存在关联业务(商机X个/订单Y个), 无法删除"
  - Given 选中多个客户, When 批量删除, Then 逐一校验关联; 部分成功部分失败时显示详细结果
  - Given 普通销售员, When 查看客户列表, Then [删除]按钮不可见
- **业务规则**: 逻辑删除(deleted=1), 不物理删除; 有关联业务时禁止删除; 批量删除限制最多50条

### OA-009: 工作报告

- **As a** 员工, **I want** 提交日报/周报/月报, **So that** 汇报工作进展
- **原型页面**: `工作报告.html`
- **界面元素**:
  - 表单: *报告类型(日报/周报/月报), *报告日期(日报=今天/周报=本周一/月报=本月1日), *报告内容(富文本编辑器, 最少50字符)
  - 列表: 按类型+时间筛选; 报告列表(类型/日期/内容摘要/提交时间)
- **验收标准**:
  - Given 日报, When 创建, Then 日期默认今天; 每日只能提交一份日报
  - Given 周报, When 创建, Then 日期默认本周一; 每周只能一份
  - Given 提交, When 保存成功, Then 直属上级可见
- **业务规则**: 每日/每周/每月限制一份; 提交后不可修改(可补充); 上级可见下属报告
- **估算**: S

### OA-010: 任务管理

- **As a** 员工, **I want** 创建和跟踪任务, **So that** 管理工作事项
- **原型页面**: `任务管理.html`, `普通任务详情_未开始_.html`, `普通任务详情_进行中_.html`, `普通任务详情_已完成_.html`, `普通任务详情_已超时_.html`, `普通任务详情_已取消_.html`
- **界面元素**:
  - 任务列表(看板视图): 未开始/进行中/已完成/已超时 四列
  - 创建任务: *标题, 描述, *负责人(用户选择), 优先级(高/中/低), *截止日期, 关联业务(选填)
- **验收标准**:
  - Given 创建任务, When 保存, Then 通知负责人; 状态="未开始"
  - Given 负责人点击开始, When 确认, Then 状态→"进行中"; 记录开始时间
  - Given 负责人点击完成, When 确认, Then 状态→"已完成"; 通知创建人; 记录完成时间
  - Given 截止日期到期且未完成, When 定时检查(每小时), Then 自动标记"已超时"
  - Given 创建人/负责人, When 点击取消, Then 状态→"已取消"(不可逆)
- **业务规则**: 超时检测(每小时定时); 创建人和负责人均可取消; 已完成/已取消不可修改
- **依赖**: module-job
- **估算**: M

### OA-011: 日程管理

  - 创建事件: *标题, *时间(开始-结束), 地点, 提醒(15分钟前/30分钟/1小时前/不提醒), 描述, 颜色标记
- **验收标准**:
  - Given 日视图, When 切换, Then 显示当日事件(按时间排列)
  - Given 创建事件并设置提醒, When 提醒时间到, Then 系统通知
  - Given 事件冲突(时间重叠), When 创建, Then 警告但允许
- **估算**: L (前端日历组件 3d)

  - 混合审批列表(类型标签:订单/回款/报销/请假/出差/借款/拜访/群发)

- **验收标准**:

  - Given 点击某行, When 跳转, Then 根据审批类型跳转到对应详情页

- **业务规则**: 统一通过BPM API查询所有待办; 按提交时间倒序; 支持批量审批(同类型)

- **估算**: L (BPM集成 2d + 前端 2d)
