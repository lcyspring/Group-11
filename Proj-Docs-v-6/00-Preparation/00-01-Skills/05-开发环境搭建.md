### 5.2 基础环境安装

**JDK 17**：

```bash
sudo apt install -y openjdk-17-jdk
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
source ~/.bashrc
java -version
```

**Maven 3.9**：

```bash
cd /opt
sudo wget https://dlcdn.apache.org/maven/maven-3/3.9.9/binaries/apache-maven-3.9.9-bin.tar.gz
sudo tar -xzf apache-maven-3.9.9-bin.tar.gz
sudo mv apache-maven-3.9.9 maven
```

在 `~/.m2/settings.xml` 中配置阿里云镜像：

```xml
<mirror>
    <id>aliyun-maven</id>
    <url>https://maven.aliyun.com/repository/public</url>
    <mirrorOf>central</mirrorOf>
</mirror>
```

**Node.js + pnpm**（前端 Web 需 pnpm >= 8.6，AdminMobile 需 pnpm >= 9, Node >= 20）：

```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo bash -
sudo apt install -y nodejs
npm config set registry https://registry.npmmirror.com
npm install -g pnpm
pnpm config set registry https://registry.npmmirror.com
```

### 5.3 Docker 环境（推荐）

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun

# 配置镜像加速
sudo tee /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": ["https://docker.m.daocloud.io"]
}
EOF

sudo systemctl restart docker
sudo usermod -aG docker $USER
```

### 5.4 密讯ETM本地开发启动

#### 仅启动中间件（推荐开发方式）

```bash
cd InstallPackage/dev
docker compose up -d --build
# 启动 MySQL + Redis + RabbitMQ + TDengine + InitService
```

#### 启动后端（IDEA）

在 IDEA 中打开 `Code/Server/`，运行 `MitedtsmServerApplication`，使用 `profile=local`。

#### 启动前端

```bash
# Web管理端
cd Code/Web
pnpm install
npm run dev

# Admin移动端
cd Code/AdminMobileFrontend
pnpm install
pnpm dev:h5

# Mall商城
cd Code/MallFrontend
pnpm install
# 使用 HBuilderX: 运行 → 运行到浏览器
```

#### 全栈部署

```bash
cd InstallPackage/docker
docker compose up -d --build
# 包含前后端 + 所有中间件 + InitService
```

## 常见问题

**Q：Docker 提示 permission denied？**

需要将当前用户加入 docker 组：`sudo usermod -aG docker $USER`，然后重新登录。

**Q：Maven 下载依赖很慢？**

检查 `~/.m2/settings.xml` 中的阿里云镜像配置是否正确。

**Q：前端 Vite 代理请求后端 404？**

检查 `Code/Web/.env` 中的 `VITE_PROXY_TARGET` 配置是否指向正确的后端地址。开发环境默认为 `http://localhost:8080`。

**Q：数据库初始化失败？**

检查 `InstallPackage/database/base/` 目录下的 SQL 脚本是否完整，以及 InitService 的数据库连接配置是否正确。
