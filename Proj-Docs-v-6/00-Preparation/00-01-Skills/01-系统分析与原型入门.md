## 核心内容

### 1.1 什么是系统分析

系统分析是将用户需求转化为系统结构的过程。它回答三个核心问题：

```
我们需要做什么？ → 系统分析
我们应该怎么做？ → 系统设计
我们做得对吗？   → 系统测试
```

### 1.2 原型设计方法

原型是系统的早期可运行版本，用于：
- 与用户确认需求
- 降低沟通成本
- 提前发现设计问题

**低保真原型**：手绘草图、线框图
**高保真原型**：交互式界面、可点击原型

### 1.3 密讯ETM系统业务概述

MITEDTSM（密讯ETM）是一个多模块企业级管理平台，覆盖 CRM、ERP、MES、WMS、Mall、Payment、BPM、AI 及海外服务，支持多租户和国际化（8种语言）。

**核心业务模块**：

| 模块 | 功能 | 定位 |
|------|------|------|
| 系统管理 | 用户/角色/菜单/租户/日志 | 基础支撑 |
| CRM | 客户/商机/报价/跟进 | 销售管理 |
| ERP | 产品/采购/销售/库存/财务 | 进销存 |
| BPM | 流程定义/任务审批/OA办公 | 工作流 |
| Mall | 商品/订单/营销/会员 | 电商商城 |
| Pay | 统一支付/多渠道路由 | 支付网关 |
| WMS | 仓库/库存/出入库 | 仓储管理 |
| MES | 工单/工序/质量/排产 | 制造执行 |
| AI | 聊天/绘图/写作/知识库 | AI智能 |
| 海外服务 | 跨境商城/多语言 | 国际化 |

### 1.5 参与者分析

| 参与者 | 说明 | 典型用例 |
|--------|------|----------|
| 平台管理员 | 系统最高权限者 | 管理租户、配置基础设施、监控系统 |
| 租户管理员 | 企业管理者 | 管理员工、审批流程、配置权限 |
| 普通员工 | 企业员工 | 打卡签到、发起请假、查看通知 |
| 商城会员 | 电商消费者 | 浏览商品、下单、支付 |
| 海外用户 | 跨境业务客户 | 多语言浏览、海外下单 |

## 常见问题

**Q：原型和正式系统有什么区别？**

原型重在"看起来像"（验证需求），正式系统重在"能运行"（生产可用）。原型通常不需要完整的后端，可以用 Mock 数据替代。

**Q：用例图应该画多细？**

用例图应该保持在"业务目标"层面，不要陷入"点击按钮"这种 UI 层面细节。一个好的用例回答"用户想达成什么目标？"
