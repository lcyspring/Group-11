### 9.3 后端单元测试示例

#### Service 层测试

```java
package com.meession.etm.module.system.service.user;

import com.meession.etm.framework.test.core.ut.BaseDbUnitTest;
import com.meession.etm.module.system.dal.dataobject.user.AdminUserDO;
import com.meession.etm.module.system.dal.mapper.user.AdminUserMapper;
import com.meession.etm.module.system.service.user.impl.AdminUserServiceImpl;
import org.junit.jupiter.api.*;
import org.springframework.boot.test.mock.mockito.MockBean;
import org.springframework.security.crypto.password.PasswordEncoder;

import javax.annotation.Resource;

import static org.junit.jupiter.api.Assertions.*;
import static org.mockito.ArgumentMatchers.any;
import static org.mockito.Mockito.when;

class AdminUserServiceImplTest extends BaseDbUnitTest {

    @Resource
    private AdminUserMapper userMapper;

    @MockBean
    private PasswordEncoder passwordEncoder;

    @Resource
    private AdminUserServiceImpl userService;

    @BeforeEach
    void setUp() {
        // 插入测试数据
        AdminUserDO user = new AdminUserDO();
        user.setUsername("testuser");
        user.setPassword("encoded_password");
        user.setNickname("测试用户");
        user.setStatus(0);
        userMapper.insert(user);
    }

    @Test
    void testGetUser() {
        AdminUserDO user = userService.getUser(1L);
        assertNotNull(user);
        assertEquals("testuser", user.getUsername());
    }

    @Test
    void testCreateUser_Success() {
        when(passwordEncoder.encode(any())).thenReturn("encoded_new");

        UserCreateReqVO reqVO = new UserCreateReqVO();
        reqVO.setUsername("newuser");
        reqVO.setPassword("raw_password");
        reqVO.setNickname("新用户");

        Long id = userService.createUser(reqVO);
        assertNotNull(id);
    }

    @Test
    void testCreateUser_DuplicateUsername() {
        UserCreateReqVO reqVO = new UserCreateReqVO();
        reqVO.setUsername("testuser");  // 已存在
        reqVO.setPassword("raw_password");

        assertThrows(RuntimeException.class, () -> userService.createUser(reqVO));
    }

    @Test
    void testDeleteUser() {
        userService.deleteUser(1L);
        // 逻辑删除后，selectById 默认过滤 deleted=1
        AdminUserDO deleted = userService.getUser(1L);
        assertNull(deleted);
    }
}
```

### 9.6 常见调试技巧

#### 后端调试

**1. SQL 日志输出**：

```yaml
logging:
  level:
    com.meession.etm.module.system.dal.mapper: DEBUG
    com.baomidou.mybatisplus: DEBUG
```

**2. IDEA 断点调试**：

- 条件断点：右键断点 → Condition → 输入表达式
- 异常断点：Run → View Breakpoints → Java Exception Breakpoints

**3. 租户上下文调试**：

```java
// 在可疑位置打日志查看当前租户
Long tenantId = TenantContextHolder.getTenantId();
log.debug("当前租户ID: {}", tenantId);
```

#### 前端调试

**1. Vue DevTools**：

安装浏览器插件，查看组件树、Pinia Store 状态、路由信息。

**2. 网络请求调试**：

Chrome DevTools → Network → 筛选 `/admin-api`，查看请求头（`tenant-id`、`Authorization`）和响应体。

**3. 常见错误排查表**：

| 现象 | 可能原因 | 检查方法 |
|------|----------|----------|
| 跨域错误 (CORS) | 后端未放行或 Vite Proxy 未配置 | 检查 `vite.config.ts` 的 proxy 配置 |
| 401 未授权 | Token 过期或缺失 | 检查 localStorage token，请求头 Authorization |
| 403 无权限 | 菜单/按钮权限未分配 | 检查 `system_menu` 表的 `permission` 字段是否与 `@PreAuthorize` 一致 |
| 租户数据看不到 | tenant-id 未传递或租户过滤异常 | 检查请求头 `tenant-id`，确认数据 `tenant_id` 正确 |
| 500 服务器错误 | 后端异常 | 查看 IDEA Console / 后端日志文件 |

## 常见问题

**Q：单元测试用 H2 内存数据库和 MySQL 有差异怎么办？**

基础 CRUD 和 MyBatis-Plus 操作通常兼容，但使用 MySQL 特有的函数（如 `GROUP_CONCAT`）时需要加 `@ActiveProfiles("local")` 切换到 MySQL。密讯ETM的 `BaseDbUnitTest` 默认使用 H2。

**Q：测试数据管理最佳实践？**

使用 `@BeforeEach` 在每个测试前准备数据，`@Transactional` 自动回滚。框架测试基类已内置 `@Transactional`。

**Q：多租户测试如何模拟？**

在测试方法中手动设置 `TenantContextHolder.setTenantId(1L)`，并在 `@AfterEach` 中清理。或使用 `@MockBean` mock `TenantContextHolder`。
