## 核心内容

### 8.1 工作流基本概念

```
┌──────────────────────────────────────┐
│          工作流核心概念               │
├──────────────────────────────────────┤
│                                      │
│  流程定义  →  流程实例  →  任务      │
│  (Process)   (Instance)   (Task)     │
│                                      │
│  模型 → 部署 → 定义 → 实例 → 任务    │
│                                      │
│  节点类型:                            │
│  - 用户任务 (UserTask)               │
│  - 排他网关 (ExclusiveGateway)       │
│  - 并行网关 (ParallelGateway)        │
│  - 服务任务 (ServiceTask)            │
│  - 会签/或签 (MultiInstance)        │
│                                      │
└──────────────────────────────────────┘
```

### 8.3 请假审批流程设计

**BPMN 流程定义**：

```
开始 → 提交申请 → [直属领导审批]
                        │
                  ┌─────┴─────┐
                  │            │
                  ▼ 通过       ▼ 驳回
              [HR审批]     提交申请（重新填写）
                  │
            ┌─────┴─────┐
            │            │
            ▼ 通过       ▼ 驳回
          结束(通过)   结束(驳回)
```

**流程变量**：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| applicantId | Long | 申请人ID |
| startTime | String | 请假开始时间 |
| endTime | String | 请假结束时间 |
| leaveType | String | 请假类型 (ANNUAL/SICK/PERSONAL) |
| reason | String | 请假原因 |
| leaveDays | Integer | 请假天数 |
| approved | Boolean | 审批结果 |

### 8.4 Flowable 集成要点

密讯ETM中 Flowable 已由框架层预集成，业务开发直接使用即可：

```yaml
# application.yaml 关键配置
flowable:
  database-schema-update: true        # 自动更新表结构
  async-executor-activate: false      # 开发环境关闭异步
```

业务模块依赖 `mitedtsm-spring-boot-starter-bpm` 即可获得 Flowable 的全部能力。

### 8.5 请假流程核心代码

#### 请假申请 Service

```java
package com.meession.etm.module.bpm.service.oa;

import com.meession.etm.module.bpm.controller.admin.oa.vo.BpmOALeaveCreateReqVO;
import com.meession.etm.module.bpm.dal.dataobject.oa.BpmOALeaveDO;
import com.meession.etm.module.bpm.dal.mapper.oa.BpmOALeaveMapper;
import com.meession.etm.module.bpm.framework.flowable.core.candidate.BpmTaskCandidateInvoker;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.flowable.engine.RuntimeService;
import org.flowable.engine.runtime.ProcessInstance;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.HashMap;
import java.util.Map;

@Slf4j
@Service
@RequiredArgsConstructor
public class BpmOALeaveServiceImpl implements BpmOALeaveService {

    private final BpmOALeaveMapper leaveMapper;
    private final RuntimeService runtimeService;

    @Override
    @Transactional
    public Long createLeave(Long userId, BpmOALeaveCreateReqVO reqVO) {
        // 1. 保存请假业务数据
        BpmOALeaveDO leave = new BpmOALeaveDO();
        leave.setUserId(userId);
        leave.setLeaveType(reqVO.getLeaveType());
        leave.setStartTime(reqVO.getStartTime());
        leave.setEndTime(reqVO.getEndTime());
        leave.setReason(reqVO.getReason());
        leave.setStatus(0);  // 0: 审批中
        leaveMapper.insert(leave);

        // 2. 构建流程变量
        Map<String, Object> variables = new HashMap<>();
        variables.put("leaveDays", reqVO.getLeaveDays());
        variables.put("startUserId", userId);

        // 3. 启动流程实例
        // 候选人由框架的 BpmTaskCandidateInvoker 自动计算
        ProcessInstance pi = runtimeService.startProcessInstanceByKey(
                "bpm_oa_leave",               // 流程定义Key
                "LEAVE-" + leave.getId(),     // 业务Key
                variables
        );

        // 4. 关联流程实例
        leave.setProcessInstanceId(pi.getId());
        leaveMapper.updateById(leave);

        log.info("请假流程启动: leaveId={}, processInstanceId={}", leave.getId(), pi.getId());
        return leave.getId();
    }
}
```

#### 候选人策略（密讯ETM核心特性）

密讯ETM对 Flowable 进行了深度定制，候选人计算由策略引擎驱动：

```java
// 框架层 BpmTaskCandidateInvoker 工作流：
//
// 1. 解析 BPMN 节点上的 flowable:candidateStrategy 扩展属性
// 2. 匹配对应的策略 Bean
// 3. 策略实现类计算候选人列表
//
// 内置策略：
//   BpmTaskCandidateUserStrategy          - 指定用户
//   BpmTaskCandidateRoleStrategy          - 按角色
//   BpmTaskCandidatePostStrategy          - 按岗位
//   BpmTaskCandidateDeptLeaderStrategy    - 部门负责人
//   BpmTaskCandidateStartUserStrategy     - 流程发起人
//   BpmTaskCandidateGroupStrategy         - 用户组
//   BpmTaskCandidateExpressionStrategy    - 表达式计算
```

#### 任务审批 Service

```java
@Service
@RequiredArgsConstructor
public class BpmTaskServiceImpl implements BpmTaskService {

    private final TaskService taskService;

    @Override
    @Transactional
    public void approveTask(String taskId, String comment, Boolean approved) {
        // 1. 添加审批意见
        if (comment != null) {
            taskService.addComment(taskId, null, comment);
        }


        // 2. 设置流程变量
        Map<String, Object> variables = new HashMap<>();
        variables.put("approved", approved);

        // 3. 完成任务
        taskService.complete(taskId, variables);

        log.info("任务审批完成: taskId={}, approved={}", taskId, approved);
    }
}
```

#### 请假审批 Controller

```java
@Tag(name = "管理后台 - OA请假")
@RestController
@RequestMapping("/bpm/oa/leave")
@RequiredArgsConstructor
public class BpmOALeaveController {

    private final BpmOALeaveService leaveService;

    @PostMapping("/create")
    @Operation(summary = "发起请假申请")
    @PreAuthorize("@ss.hasPermission('bpm:oa-leave:create')")
    public CommonResult<Long> createLeave(@Valid @RequestBody BpmOALeaveCreateReqVO reqVO) {
        Long loginUserId = SecurityFrameworkUtils.getLoginUserId();
        Long id = leaveService.createLeave(loginUserId, reqVO);
        return CommonResult.success(id);
    }

    @GetMapping("/page")
    @Operation(summary = "我的请假列表")
    public CommonResult<PageResult<BpmOALeaveRespVO>> getMyLeavePage(@Valid BpmOALeavePageReqVO reqVO) {
        PageResult<BpmOALeaveDO> pageResult = leaveService.getLeavePage(reqVO);
        return CommonResult.success(BpmOALeaveConvert.INSTANCE.convertPage(pageResult));
    }
}
```

## 常见问题

**Q：Flowable 表很多，哪些是核心表？**

核心运行表：`act_ru_task`（运行中任务）、`act_ru_execution`（执行实例）、`act_ru_variable`（流程变量）。核心历史表：`act_hi_taskinst`、`act_hi_procinst`。核心定义表：`act_re_procdef`、`act_re_deployment`。

**Q：密讯ETM的候选人策略和原生 Flowable 有什么区别？**

原生 Flowable 通过 `assignee`、`candidateGroups`、`candidateUsers` 指定审批人。密讯ETM扩展了 `candidateStrategy` 扩展属性，支持按角色、部门、岗位、表达式等动态计算审批人，解耦流程定义和审批人配置。

**Q：流程图文件放在哪里？**

传统方式放在 `resources/processes/` 目录，Flowable 会自动扫描部署。密讯ETM通过管理端在线设计，流程定义直接存储到数据库，随系统启动自动部署。
