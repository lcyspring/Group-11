**建表SQL**:

```sql
CREATE TABLE `trade_order` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `no` varchar(32) NOT NULL COMMENT '订单号',
  `type` int NOT NULL DEFAULT '0' COMMENT '订单类型',
  `terminal` int NOT NULL COMMENT '终端类型',
  `user_id` bigint NOT NULL COMMENT '用户ID',
  `user_ip` varchar(50) NOT NULL COMMENT '用户IP',
  `user_remark` varchar(500) DEFAULT NULL COMMENT '用户备注',
  `remark` varchar(500) DEFAULT NULL COMMENT '管理员备注',
  `status` int NOT NULL DEFAULT '10' COMMENT '订单状态',
  `product_count` int NOT NULL DEFAULT '0' COMMENT '商品数量',
  `total_price` int NOT NULL DEFAULT '0' COMMENT '订单总价（分）',
  `discount_price` int NOT NULL DEFAULT '0' COMMENT '优惠金额（分）',
  `delivery_price` int NOT NULL DEFAULT '0' COMMENT '运费（分）',
  `adjust_price` int NOT NULL DEFAULT '0' COMMENT '调整金额（分）',
  `pay_price` int NOT NULL DEFAULT '0' COMMENT '支付金额（分）',
  `pay_status` bit(1) NOT NULL DEFAULT b'0' COMMENT '支付状态',
  `pay_type` int DEFAULT NULL COMMENT '支付方式',
  `pay_channel` varchar(50) DEFAULT NULL COMMENT '支付渠道',
  `pay_time` datetime DEFAULT NULL COMMENT '支付时间',
  `pay_transaction_id` varchar(64) DEFAULT NULL COMMENT '支付流水号',
  `delivery_type` int NOT NULL COMMENT '配送方式',
  `delivery_status` bit(1) NOT NULL DEFAULT b'0' COMMENT '发货状态',
  `delivery_time` datetime DEFAULT NULL COMMENT '发货时间',
  `receive_time` datetime DEFAULT NULL COMMENT '收货时间',
  `receiver_name` varchar(50) NOT NULL COMMENT '收货人姓名',
  `receiver_mobile` varchar(20) NOT NULL COMMENT '收货人手机号',
  `receiver_area_id` int NOT NULL COMMENT '收货地区ID',
  `receiver_detail_address` varchar(200) NOT NULL COMMENT '收货详细地址',
  `refund_status` int NOT NULL DEFAULT '0' COMMENT '退款状态',
  `refund_price` int NOT NULL DEFAULT '0' COMMENT '退款金额（分）',
  `refund_time` datetime DEFAULT NULL COMMENT '退款时间',
  `coupon_id` bigint NOT NULL DEFAULT '0' COMMENT '优惠券ID',
  `coupon_price` int NOT NULL DEFAULT '0' COMMENT '优惠券金额（分）',
  `point_price` int NOT NULL DEFAULT '0' COMMENT '积分抵扣金额（分）',
  `approval_status` int DEFAULT NULL COMMENT '审批状态',
  `process_instance_id` varchar(64) DEFAULT NULL COMMENT '流程实例ID',
  `approval_comment` varchar(500) DEFAULT NULL COMMENT '审批意见',
  `approval_time` datetime DEFAULT NULL COMMENT '审批时间',
  `creator` varchar(64) DEFAULT NULL COMMENT '创建人',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updater` varchar(64) DEFAULT NULL COMMENT '更新人',
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` bit(1) NOT NULL DEFAULT b'0' COMMENT '是否删除',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_no` (`no`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_status` (`status`),
  KEY `idx_create_time` (`create_time`),
  KEY `idx_pay_time` (`pay_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单主表';
```

## 四、性能优化建议

### 4.1 索引优化

1. **联合索引**: 根据查询场景创建联合索引，如 `(user_id, status, create_time)`
2. **覆盖索引**: 查询只需要索引字段时，使用覆盖索引避免回表
3. **索引顺序**: 区分度高的字段放在索引前面

### 4.2 分库分表

当单表数据量超过2000万时，建议进行分库分表：

- **分表键**: 使用 `user_id` 作为分表键
- **分表策略**: 按 `user_id % 16` 分表
- **分库策略**: 按 `user_id / 16 % 4` 分库

### 4.3 冷热数据分离

- **热数据**: 最近6个月的订单数据，存储在主库
- **冷数据**: 6个月前的订单数据，归档到历史库
